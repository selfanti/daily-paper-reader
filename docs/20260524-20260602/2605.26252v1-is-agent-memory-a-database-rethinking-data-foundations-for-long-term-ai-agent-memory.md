## 摘要
长时间运行的AI智能体需要持久记忆。记忆支持跨会话学习，减少重复上下文注入，并允许审计过去决策。当前的智能体记忆系统和数据库范式将记忆视为存储。它们将正确性定位于记录、嵌入或边。每种仅提供长期记忆所需的部分能力。结果导致四种反复出现的失败模式：无监管增长、缺失语义修正、容量驱动的遗忘以及只读检索。在我们的愿景中，长期智能体记忆是一种新的数据管理工作负载。其正确性是状态轨迹的属性，而非单个记录。我们将其形式化为受控演化记忆（GEM）。GEM用四个状态级操作取代记录级数据库操作：摄取、修正、遗忘和检索。六个正确性条件控制状态的演化方式。三个结构性观察表明，无论存储模型如何，没有任何记录级系统能够满足这些条件。我们在属性图后端原型MemState中实现了该抽象。MemState验证了可行性并揭示了与原生引擎之间的差距。我们概述了三个研究方向，将记忆为中心的数据管理定义为一个工作负载。

## Abstract
Long-running AI agents need persistent memory. Memory supports learning across sessions, reduces repeated context injection, and enables auditing of past decisions. Current agent memory systems and database paradigms treat memory as storage. They localize correctness at records, embeddings, or edges. Each supplies only some of the capabilities that long-term memory requires. The result is four recurring failure modes: unregulated growth, missing semantic revision, capacity-driven forgetting, and read-only retrieval. In our vision, long-term agent memory is a new data-management workload. Its correctness is a property of the state trajectory, not of individual records. We formalize this as Governed Evolving Memory (GEM). GEM replaces record-level database operations with four state-level operators: ingestion, revision, forgetting, and retrieval. Six correctness conditions govern how the state evolves. Three structural observations establish that no record-level system can satisfy these conditions, regardless of the storage model. We realize the abstraction in MemState, a prototype on a property-graph backend. MemState validates feasibility and exposes the gap to a native engine. We outline three research directions that define memory-centric data management as a workload.

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想：Governed Evolving Memory (GEM)
- **记忆状态定义**：记忆状态 \(M_t = (D_t, S_t, P_t)\)，其中 \(D_t\) 为内容（语义单元，包含值历史与显著性信号），\(S_t\) 为结构组织（带类型边），\(P_t\) 为演化策略（声明式规则）。
- **四个状态级操作**（替代CRUD）：
  - 摄取（Ingestion）：将外部输入集成到现有状态，新值追加到语义单元的字段历史中，保留旧值作为证据。
  - 修正（Revision）：基于内部证据合并冲突、沿依赖边传播更新，保留被取代值的来源链。
  - 遗忘（Forgetting）：按显著性信号（访问上升、废弃下降）进行分级衰减（压缩→隐藏→归档），可恢复。
  - 检索（Retrieval）：返回当前值，同时增加被访问单元的显著性（状态修改）。
- **六个正确性条件**：
  - C1 查询正确性：返回最近非归档值。
  - C2 转换正确性：每次状态转换遵循策略，已废弃值不会作为当前值返回。
  - C3 依赖一致性：沿扩展边（entailment）传播更新。
  - C4 来源保留：遗忘和修正保留可达单元的来源链。
  - C5 有界活跃状态：活跃状态大小受策略定义的界限约束，归档内容可恢复。
  - C6 检索诱导适应：每次检索增加单元显著性，重复检索减少被遗忘风险。
- **三个结构性观察**（证明记录级系统无法满足GEM）：
  1. 纯函数检索无法满足C6，因为需要修改状态。
  2. C5（相关性驱动遗忘）与C6在CRUD引擎上无法联合满足，因为检索只读则相关性必须近似。
  3. 追加式存储无法满足C2（多个值无优先级），无类型传播无法满足C3（依赖对引擎不可见）。

### 关键技术细节（MemState 原型）
- **后端**：Kuzu嵌入式属性图引擎。
- **数据模型**：每个“主题”（Topic）作为自包含语义单元，包含标题、摘要、密集嵌入、字段值历史（分层存储）。主题间通过“扩展边”（entailment）和“关联边”（relatedness）连接。
- **策略实现**：\(P_t\) 以声明式规则（事件-条件-动作）存储在状态中，提交时检查后置条件，违反则回滚（C2升为数据模型保证）。

## 6. 论文的主要结论与发现
- 长期智能体记忆不是一个数据库问题，而是一个新型数据管理工作负载，其正确性定义在状态轨迹上。
- 记录级CRUD操作必然导致四种失败模式，任何现有数据库或记忆系统均无法同时满足GEM的四个能力要求。

- 需要三个研究方向：原生引擎设计、轨迹级基准与评估、隐私与多租户下的记忆遗忘。

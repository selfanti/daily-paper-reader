## 摘要
大型语言模型（LLM）智能体在推理、工具使用和多步交互方面展现出强大能力，但它们通常从零开始解决问题，未能复用先前经验中的成功策略或失败教训。在收集到的经验上进行微调可以提升复用效果，但当出现更强或更合适的执行器时，这种方法缺乏灵活性。我们提出ExpGraph，一种模型无关的经验学习框架，使得冻结且可替换的LLM执行器能够通过外部经验复用来提升性能，而无需更新参数。ExpGraph将历史轨迹总结为可复用的技能和失败教训，将其组织为自演进经验图中的节点，并通过图扩散和效用感知排序检索有用经验。一个轻量级的检索协作者通过强化学习进行训练，利用对比有无检索经验时执行器性能的反馈，同时图根据下游任务结果在线更新。我们在ExpSuite上评估ExpGraph，涵盖问答、数学推理、代码生成以及包括ALFWorld和AppWorld在内的多步智能体环境。ExpGraph在静态任务上，针对较小和较大执行器分别比最强基线提升12.2%和4.7%；在智能体环境中提升21.4%和12.7%，同时将平均交互步骤减少12.7%和21.6%。消融实验表明，图结构经验、效用感知排序和自适应检索共同实现了在多种任务和执行器模型上的有效经验复用。

## Abstract
Large language model (LLM) agents have shown strong capabilities in reasoning, tool use, and multi-step interaction, but they often solve tasks from scratch and fail to reuse successful strategies or failure lessons from prior experience. Fine-tuning on collected experience can improve reuse, but it is inflexible when stronger or more suitable executors emerge. We propose ExpGraph, a model-agnostic experience learning framework that enables frozen and replaceable LLM executors to improve through external experience reuse without parameter updates. ExpGraph summarizes historical trajectories into reusable skills and failure lessons, organizes them as nodes in a self-evolving experience graph, and retrieves useful experiences through graph diffusion and utility-aware ranking. A lightweight retrieval copilot is trained with reinforcement learning using feedback that compares executor performance with and without retrieved experiences, while the graph is updated online from downstream task outcomes. We evaluate ExpGraph on ExpSuite, covering question answering, mathematical reasoning, code generation, and multi-step agentic environments including ALFWorld and AppWorld. ExpGraph improves over the strongest baseline by 12.2% and 4.7% on static tasks with smaller and larger executors, and by 21.4% and 12.7% in agentic environments, while reducing average interaction steps by 12.7% and 21.6%. Ablations show that graph-structured experience, utility-aware ranking, and adaptive retrieval jointly enable effective experience reuse across diverse tasks and executor models.

## 1. 核心问题与整体含义

**研究动机**：现有 LLM 智能体在推理、工具使用和多步交互中表现出色，但通常每个任务都从零开始求解，未能系统性地复用历史成功策略或失败教训。一种常见的解决方案是对执行器进行微调，但 LLM 进化迅速，每当更强的执行器出现时，微调过程需要重复，导致经验学习与特定模型实例紧密耦合，缺乏灵活性。

**整体含义**：ExpGraph 提出“模型无关的经验学习”概念，允许冻结且可替换的 LLM 执行器通过外部经验复用来提升性能，而无需修改参数。这实现了经验学习与执行器训练的解耦，使得相同的外部经验系统可以服务于不同执行器。

## 2. 方法论

### 核心思想
- 将历史轨迹总结为可复用的**技能**（成功策略）和**失败教训**，组织成**自演进经验图**。
- 使用轻量级**检索协作者**（retrieval copilot）根据当前任务自适应地控制图扩散范围和效用-语义权衡，选取最有用的经验。
- 通过**强化学习（PPO）**优化协作者，奖励基于“有经验 vs 无经验”的下游任务性能差异；同时在线更新图节点效用和结构。

### 关键技术细节
- **经验图构造**：
  - 轨迹 → 自然语言经验单元（技能/教训）作为节点。
  - 节点包含文本、嵌入、估计效用、检索次数。
  - 新节点插入时，与 top-K 余弦相似度 ≥ θ 的节点建立无向边（保持稀疏性）。
- **检索过程**：
  - **语义种子**：根据任务嵌入选取 top-m 最相似节点。
  - **图扩散**：使用 personalized PageRank 从种子出发扩散，由参数 ρ 控制扩散广度（ρ 越大，越集中在种子附近）。
  - **效用感知排序**：对扩散后候选节点计算 UCB 分数（结合估计效用和探索项），并与语义相似度加权（权重 λ 可调）。最终选取 top-K。
- **经验学习**：
  - 奖励：`r = (s_with - s_without) + η * s_with`，其中 η=1 鼓励绝对任务质量。
  - 协作者（基于 Qwen2.5-3B）输出离散控制变量 R（扩散广度）、W（similarity-utility 权衡），使用 PPO 优化。
  - 图在线更新：检索到的节点效用通过指数移动平均更新；新轨迹总结后作为新节点插入（含近重复过滤和容量管理）。

## 3. 实验设计

### 数据集 / 场景

  - 问答：ARC-C、CommonsenseQA、GPQA、MMLU、OBQA（准确率）

## 6. 主要结论与发现

- **ExpGraph 在所有设置中全面最优**：
  - 静态任务：较小执行器提升 12.2%、较大执行器提升 4.7%。
  - Agentic 环境：提升 21.4%（小）和 12.7%（大）；同时减少交互步骤 12.7% 和 21.6%。
- **经验复用对较弱执行器和交互任务更有效**：增益幅度随执行器能力下降和任务长度增加而增大。
- **零样本迁移有效**：
  - 小→大迁移几乎无性能损失；大→小迁移仍有收益。
  - 非推理→推理执行器的迁移表现优良。
- **各组件均贡献**：Flat Memory（无图结构）降幅最大，图扩散和效用排序在 agentic 任务中关键。
- **模型无关设计成功**：冻结执行器仅通过上下文注入经验即可提升，无需参数更新。

## 7. 优点

- **模型无关性**：不修改执行器参数，兼容闭源/无法微调的 LLM，支持执行器即插即用。
- **图结构记忆**：超越平面最近邻检索，通过图扩散连接语义相关或策略相关的经验，实现更丰富的知识传递。
- **自适应检索协作者**：使用 RL 训练，根据任务和执行器动态调整扩散和效用权重，奖励来自真实下游性能（而非相似度启发式）。
- **闭环演化**：经验图随任务反馈在线更新，自动积累有用知识、淘汰低效节点。
- **实验全面**：覆盖静态和 agentic 环境、多种执行器、大量消融与迁移设置，结论可靠。

## 8. 不足与局限

- **基准覆盖有限**：论文承认实验集中在特定推理和交互环境（ALFWorld、AppWorld），尚未在更长周期的真实应用（如网页浏览、科学发现、多智能体协作）中验证。
- **图构造超参数固定**：邻居数量、相似度阈值、图容量、效用更新率等均为固定值，可能在不同领域的记忆分布下需要更自适应的调整。
- **奖励信号粗糙**：当前仅使用任务最终标量分数，没有利用中间步骤的过程级信用分配（如过程奖励模型），可能学习效率有限。
- **经验利用方式**：当前仅通过提示注入经验，而提示级别的指导可能无法稳定转化为执行器行为（存在与解释不一致的风险）。更紧密的结合（如基于经验轨迹微调或知识蒸馏）可能进一步提升效果。
- **训练成本**：尽管执行器不更新，但协作者和图维护仍需要额外计算（4×A6000 GPU 训练数十小时），在资源受限场景可能需要权衡。

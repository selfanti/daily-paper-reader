## MemCog: From Memory-as-Tool to Memory-as-Cognition in Conversational Agents 

Zihan Li 1, Xingyu Fan 1, Feifei Li 1, Wenhui Que 1

1WeChat, Tencent Inc., Beijing, China 

{muselli, fanxfan, niyali, victorque}@tencent.com 

Abstract 

Existing agent memory systems universally fol-low what we term a Memory-as-Tool paradigm where a single query triggers one-shot retrieval of flat passage lists, suffering from passive in-vocation, reasoning-retrieval decoupling, and structural mismatch between retrieved frag-ments and the agent’s navigational needs. We propose MemCog , a Memory-as-Cognition 

system that makes memory access an integral part of the reasoning process. MemCog or-ganizes user knowledge as Navigable Mem-ory Store with associative link graphs, exposes Cross-Dimensional Navigation Interface for multi-step reasoning-driven traversal, and em-ploys Proactive Reasoning Protocol that drives agents to spontaneously initiate memory explo-ration from conversational context. We addi-tionally construct ProactiveMemBench 1, the first benchmark for evaluating proactive mem-ory triggering. Experiments show that Mem-Cog achieves state-of-the-art on passive QA benchmarks (92.98 on LoCoMo, 95.8 on Long-MemEval) while substantially outperforming baselines on ProactiveMemBench, demonstrat-ing the advantage of Memory-as-Cognition. 

1 Introduction 

Long-term memory (Packer et al., 2024; Chhikara et al., 2025; Rasmussen et al., 2025; Xu et al., 2026; Yue et al., 2026) is fundamental to maintaining co-herent, personalized interactions across extended dialogues, enabling agents to capture evolving user knowledge from fragmented conversations and gen-erate historically informed responses beyond fixed context windows. From MemGPT(Packer et al., 2024) to HyperMem(Yue et al., 2026), agent mem-ory systems have grown increasingly sophisticated in what they retrieve. We argue that this pattern, which we term Memory-as-Tool , imposes fundamental limitations on how agents can utilize long-term knowledge. This Memory-as-Tool paradigm imposes three fundamental limitations. First, Invocation Bottle-neck : the memory system is activated only when the agent explicitly issues a tool call to retrieve user memories relevant to a question, leaving memory as a passive responder to query demands. The mem-ory system lacks the ability to proactively trigger retrieval. Existing benchmarks evaluate exclusively around factual questions, diverging from daily con-versational scenarios and the proactive triggering patterns of human memory. Second, Reasoning-Retrieval Decoupling : the Memory-as-Tool system retrieval remains a one-shot call, preventing the agent from dynamically planning further exploration when initial results are insufficient. The agent’s reasoning capabilities cannot intervene in the memory system’s isolated retrieval pipeline, which constrains response qual-ity and forces the two modules to optimize inde-pendently without co-evolution. Third, Structural Mismatch : Even when mem-ory is organized hierarchically(Xu et al., 2026; Ras-mussen et al., 2025; Yue et al., 2026), the top-k

Existing methods cannot extend their structural capabilities to guide the agent’s directed iterative retrieval within the memory store. To address the above three limitations, we propose MemCog, the first memory system that transitions from Memory-as-Tool to Memory-as-Cognition . First, we design a Proactive Reason-ing Protocol that drives agents from passively wait-ing for queries to actively reasoning within mem-ory, addressing the invocation bottleneck. We ad-vance toward a Memory-as-Cognition paradigm in which retrieval and reasoning become insepa-rable within a ReAct loop augmented by proac-tive memory triggering. This design serves as a principled engineering stepping stone, anticipat-ing the eventual maturation of fully architectural solutions for deployment in personalized agent set-tings. Second, we instantiate a Cross-Dimensional Navigation Interface that replaces the single re-trieval tool call with multi-granularity navigation actions, where the agent autonomously determines the next action at each step based on current obser-vations, achieving tight coupling between retrieval and reasoning. Third, we propose Navigable Mem-ory Store , which provides agents with structured navigational paths from hierarchical dimensions to specific memory sections, enabling agents to plan subsequent navigation based on the structural context returned at each step. Additionally, we con-struct ProactiveMemBench , the first benchmark for evaluating proactive memory triggering capa-bilities, filling the gap left by existing factual QA benchmarks limited to passive retrieval scenarios. To the best of our knowledge, MemCog is the first system that unifies proactive reasoning with structured navigational memory, transforming memory access from a passive preprocessing step into an integral part of the agent’s reasoning pro-cess. Our main contributions are summarized as follows: • We propose Memory-as-Cognition , aparadigm that transforms passive memory access into an integral part of the agent’s reasoning process, thereby eliminating the invocation bottleneck inherent in Memory-as-Tool systems. • We construct ProactiveMemBench , the first benchmark for evaluating proactive mem-ory triggering, built through a six-step LLM-driven pipeline spanning 500 instances across five topic domains with 98.4% human valida-tion acceptance. • Experiments on three benchmarks validate the effectiveness of MemCog, achieving state-of-the-art on passive QA tasks and substantially outperforming baselines on proactive memory triggering. Ablation studies further confirm the protocol–structure synergy in MemCog, with all components proving indispensable and mutually reinforcing. 

2 Related Work 

2.1 Single-Shot Memory Retrieval 

The most prevalent paradigm treats retrieval as a single-round tool call. Mem0 (Chhikara et al., 2025) retrieves atomic facts via semantic similar-ity; MemGPT (Packer et al., 2024) swaps fixed-size pages through function calls inspired by OS virtual memory; LangMem (LangChain, 2026) provides an SDK-level search interface. Despite different storage backends, these systems share a common access pattern: one query in, one flat list out, with no opportunity for iterative refinement. 

2.2 Structured Memory Organization 

Recent work enriches memory with structural pri-ors while retaining the single-shot interface. Hy-perMem (Yue et al., 2026) employs a hypergraph-based three-level funnel capturing high-order as-sociations, yet executes as a single pass. A-Mem (Xu et al., 2026) adopts Zettelkasten-inspired notes with activation-based triggering but returns results in one batch. HippoRAG (Gutiérrez et al., 2024) builds a knowledge graph with Personalized PageR-ank retrieval in a single step. These innovations improve what is retrieved but leave the query-in, passages-out access pattern unchanged. 

2.3 Multi-Step Reasoning over Knowledge

2.4 Proactive Systems 

Existing proactive systems operate at dialogue strat-egy level rather than memory triggering: proactive dialogue systems (Deng et al., 2023) steers conver-sations toward predefined goals; Generative Agents (Park et al., 2023) synthesize memories via sched-uled reflection; Agentic Retrieval-Augmented Gen-eration (Jeong et al., 2024; Yan et al., 2024; Wang et al., 2024) improve retrieval strategies but involve no agent-driven navigation. MemCog’s proactivity instead operates at the memory level: agents spon-taneously initiate multi-step navigation when con-textual cues suggest relevant stored associations. 

3 Method 

3.1 Overview 

As illustrated in Figure 2, MemCog comprises three tightly integrated components: (1) Navigable Memory Store (§3.2) that organizes user knowl-edge into a hierarchical, link-rich database; (2) 

Cross-Dimensional Navigation Interface (§3.3) that replaces single-shot retrieval with multi-step, reasoning-driven memory traversal; (3) Proactive Reasoning Protocol (§3.4) that drives agents to spontaneously initiate memory exploration during real-time conversation. Together, these compo-nents transform memory access from the conven-tional Memory-as-Tool pattern (one query in, flat passages out) into a Memory-as-Cognition pro-cess where retrieval and reasoning are interleaved across multiple steps, with each navigation action informed by prior observations. 

3.2 Navigable Memory Store 3.2.1 Hierarchical Organization 

The core design principle of Navigable Memory Store is to support agent multi-step navigation rather than optimizing for single-shot retrieval. We organize user knowledge as a hierarchical memory store with three levels of granularity: • Dimension : top-level categories that parti-tion the user’s knowledge space (e.g., personal preferences, work, social relationships, impor-tant events). • Page : individual knowledge units within each dimension, each covering a coherent topic. • Section : fine-grained information blocks within a page, containing title, summary, struc-tured facts, related pages, temporal context, and confidence scores. Dimensions provide entry points, Pages pro-vide depth, and cross-dimensional associative links within Sections provide lateral connectivity. This design uniquely combines hierarchical browsability with cross-branch linkage, ensuring that every step returned during navigation carries sufficient struc-tural context for the agent to reason about what to explore next. 

3.2.2 Cross-dimensional associative Link 

Cross-dimensional associative links establish an 

overlay graph that facilitates seamless traversal across dimensional boundaries. We define four link types to support different reasoning patterns: • related_to : semantic associations across di-mensions (e.g., “Dietary Precautions” ↔

“Business Trip to Beijing”), supporting cross-3dimensional association. • temporal_next : directed temporal succession (e.g., “project v1” → “project v2”), support-ing timeline tracing. • caused_by : directed causal relations (e.g., “peanut allergy” ← “restaurant incident”), sup-porting causal reasoning. • contrasts_with : contrastive relations (e.g., “like spicy” ↔ “dislike sweet”), supporting preference comparison. These typed links serve as explicit navigational affordances: when an agent reads a page, the vis-ible links suggest further exploration directions, naturally inducing multi-step traversal without re-quiring additional prompting. 

3.2.3 Construction and Evolution Initial Construction. Given a user’s conversa-tion history, Page construction proceeds in three stages: (1) an LLM extracts user-relevant informa-tion from conversations as structured facts; (2) facts are clustered into dimensions and pages via seman-tic grouping; (3) pages are organized into sections, which are interconnected via cross-dimensional associative links established based on entity co-occurrence, temporal proximity, and causal indica-tors. 

Incremental Update. After each conversation turn, Page evolves through: (1) extracting new in-formation from the latest dialogue; (2) matching extracted facts to existing pages for section updates, or creating new pages when no match is found; (3) establishing cross-links for newly created or sub-stantially updated pages; (4) performing contradic-tion detection and confidence score updates when new information conflicts with existing entries. 

3.3 Cross-Dimensional Navigation 

Rather than exposing a single search tool that re-turns top-k fragments for a given query, MemCog provides agents with a set of navigation actions at multiple granularities: • list_dimensions() : returns all top-level di-mensions with brief descriptions, providing a global overview of the memory store. • browse_dimension(dim) : returns page titles and summaries within a specified dimension, enabling targeted exploration. • read_page(page_id) : returns the full content of a specific page including all sections and outgoing links. • follow_link(link) : navigates an inter-page hy-perlink and returns the target page content, enabling cross-dimensional traversal without requiring the agent to know the top-level struc-ture of other dimensions. This design offers agents multiple granularities of access. The agent autonomously decides which action to invoke at each step based on its current reasoning state. Critically, each action returns not only the requested content but also structural con-text (available links, sibling pages, dimensional position) that informs subsequent navigation deci-sions. 

3.4 Proactive Reasoning Protocol Protocol Design. The Proactive Reasoning Pro-tocol is implemented as a structured system prompt defining the agent’s memory-usage behavioral norms (Appendix A). It specifies two modes of memory engagement: (1) Proactive association :the agent spontaneously initiates memory naviga-tion when conversational context suggests poten-tial connections to stored knowledge, specifically when: (i) mentions of people, places, or topics link to past memories; (ii) the conversation would bene-fit from historical context; (iii) user statements con-tradict stored information. (2) Reactive response :the agent navigates memory in response to explicit user questions about past information. The protocol further specifies cross-dimensional navigation rather than stopping at first results, fol-lowing links to build complete pictures, integrating information naturally, adopting autonomous rout-ing strategies, and reasoning about which dimen-sions are most relevant before acting. 

Selective Surfacing. Proactive retrieval does not imply indiscriminate information sharing. The agent evaluates recalled information along four di-mensions before surfacing: relevance to current context, temporal validity, sensitivity, and redun-dancy. These judgments are performed by the LLM’s own reasoning during the Think phase, requiring no additional gating module. Detailed strategies and illustrative examples are provided in Appendix D and Table 8.

We adopt a bottom-up construction approach: rather than extracting memory cues post-hoc from existing dialogues, we first define atomic memory units and then weave them into naturalistic conver-sations through a six-step LLM-driven pipeline. (1) We generate a detailed user persona covering de-mographics, expertise, routines, and social relation-ships to serve as a global consistency constraint. (2) Conditioned on the persona, we produce 60 mem-ory units across five trigger type (i.e. temporal, entity, emotional, behavioral pattern, and multi-hop) and each represents a piece of information the user might naturally reveal. (3) We score pair-wise associations among memory units along four dimensions (entity overlap, semantic relevance, as-sociation reasonability, and conversational coher-ence) to form a weighted association graph. (4) Guided by the graph, we distribute memory units across 20 sessions (3–4 pairs each), deliberately placing highly associated pairs in different sessions to create cross-session memory dependencies. (5) For each session, we generate a 20-turn dialogue where memory unit information is planted naturally in user utterances. (6) Finally, we produce 100 eval-uation questions per topic, each specifying a trigger utterance and exactly three ground-truth candidate memory units that the model should proactively recall. We repeat this pipeline for five topic domains, yielding a total of 500 test instances spanning 100 sessions and 2000 dialogue turns. 4.2 Main Results 4.2.1 Passive QA Performance 

Table 1 presents results on the LoCoMo bench-mark. MemCog achieves 92.98 overall accu-racy with GPT-4.1-mini, surpassing all baselines under the Memory-as-Tool paradigm. This re-sult demonstrates that the Memory-as-Cognition, where multi-step navigation is interleaved with reasoning, does not sacrifice passive QA perfor-mance compared to single-shot retrieval pipelines. Notably, MemCog consistently outperforms all baselines across all three QA backbone configu-rations (GLM: 89.83, GPT-4o-mini: 92.66, GPT-4.1-mini: 92.98), demonstrating robustness to back-bones with varying reasoning capabilities and con-text lengths. Specifically, GLM-5.1 offers a longer context window but relatively weaker reasoning, GPT-4o-mini exhibits stronger reasoning but a shorter context window, and GPT-4.1-mini per-forms well in both dimensions. To illustrate the lim-itations of the Memory-as-Tool paradigm, we com-pare against its current SOTA method, HyperMem, which demands an extremely large context window during retrieval-augmented QA (nearly 30K tokens per query). Consequently, HyperMem performs only slightly below MemCog on long-context mod-els, achieving 89.77 (-0.06) with GLM-5.1 and 92.73 (-0.25) with GPT-4.1-mini, but degrades sub-stantially under low context budgets, dropping to 83.69 (-8.97) with GPT-4o-mini. This indicates that the QA model’s reasoning capability fails to benefit the memory system, exposing a fundamen-tal limitation of the Memory-as-Tool paradigm. In contrast, MemCog achieves consistently strong per-formance across all backbone configurations, con-firming that its memory system design serves as ef-fective scaffolding that compensates for both weak reasoning and limited context budgets. 

4.2.2 Long-Term Memory Retrieval 

Table 2 reports results on LongMemEval. Mem-Cog achieves 95.8, the highest score among all evaluated systems, outperforming baselines under Memory-as-Tool paradigm. This confirms that MemCog’s multi-step navigation, which integrates the model’s reasoning capability into the retrieval process, effectively handles long-term memory retrieval where information is distributed across extended interaction histories. In particular, the score on multi-session category (92.48, the high-est among all methods) further substantiates this conclusion, as it requires retrieving and associating information across disparate conversation sessions. Case studies for multi-session and other categories are further provided in Appendix D.

Removing cross-dimensional associative links (w/o Graph Over-lay) degrades both LoCoMo (-6.79) and Proactive-MemBench (Recall@5 -4.19, LLM-judged Preci-sion -4.00), confirming that associative links enable agents to follow cross-dimensional associations rather than relying on flat retrieval. Removing the dimensional hierarchy (w/o Hierarchy) causes sim-ilar degradation as hierarchical pages provide prin-cipled entry points for navigation without which the agent lacks effective starting positions for both directed search and open-ended exploration. 

Complementary Synergy. The Proactive Rea-soning Protocol and Navigable Memory Store con-tribute to different capability dimensions: the pro-tocol enables proactive behavior while structural memory enhance retrieval quality. The full sys-tem achieves the best performance on both dimen-sions simultaneously, validating the Memory-as-Cognition paradigm in which iterative reasoning is interleaved with structured memory navigation. 

5 Discussion 

5.1 Memory Access as a Spectrum 

Rather than proposing a binary dichotomy between Memory-as-Tool and Memory-as-Cognition, we 7argue that memory access paradigms exist along a cognitive spectrum: 

No Memory → Single Search → Multi-Step Navigation → Spontaneous Recall 

At the leftmost extreme, agents operate without any persistent memory, relying solely on the cur-rent context window. Single-shot retrieval systems occupy the next position, augmenting agents with external knowledge but limiting interaction to a sin-gle query-response cycle. MemCog advances the spectrum further by enabling multi-step, reasoning-driven navigation where the agent iteratively re-fines its exploration path. The rightmost extreme, spontaneous recall fully embedded in the agent’s reasoning process without explicit tool invocation, remains a direction for future research. This spectral view clarifies MemCog’s contri-bution: we do not claim to achieve the theoreti-cal endpoint of memory-as-cognition, but rather demonstrate that meaningful advances along this spectrum yield substantial empirical gains. 

5.2 On Protocol-Level Innovation 

We situate the Proactive Reasoning Protocol within a lineage of protocol-level contributions that have shaped modern LLM-based systems: Chain-of-Thought (Wei et al., 2022) for step-by-step rea-soning, ReAct (Yao et al., 2023) for interleaving reasoning with tool use, Self-Refine (Madaan et al., 2023) for iterative self-improvement, and Mem-Cog for memory-aware proactive reasoning. Each of these works defines how an agent should behave in a specific cognitive dimension, and each derives its impact from the synergy between the protocol and accompanying structural innovations. In MemCog, the Proactive Reasoning Protocol is inseparable from the Navigable Memory Store: the protocol provides the cognitive drive to initiate ex-ploration, while the Page provides the structural affordances that sustain multi-step navigation. Our ablation results (Table 4) empirically validate this complementary relationship, as neither component alone achieves the full system’s performance. 

5.3 Scalability 

We analyze MemCog’s computational characteris-tics across three dimensions: 

Page Construction Cost. Initial Page construc-tion from conversation history is O(n) in the num-ber of dialogue turns. However, incremental up-dates after each conversation turn operate at O(1) 

amortized cost, as only newly extracted facts re-quire matching and insertion. This ensures that the system remains efficient as conversation histories grow. 

Navigation Token Overhead. Multi-step navi-gation incurs additional token consumption com-pared to single-shot retrieval. In practice, our ex-periments show that agents typically perform 2–3 navigation steps per query. This overhead is mod-est relative to the substantial quality improvements observed, particularly on complex queries requir-ing enumeration, comparison, or temporal reason-ing (Appendix D). Compared to HyperMem’s(Yue et al., 2026) three-level funnel pipeline, which pro-cesses all candidate memories through multiple fil-tering stages, MemCog’s navigation is more parsi-monious: the agent selectively reasons about which paths to explore at each step, avoiding exhaus-tive processing of irrelevant candidates and thereby achieving superior results with substantially lower token consumption. 

Page Growth Management. As conversations accumulate, the number of pages grows accord-ingly. We address potential page bloat through periodic archival of infrequently accessed pages and summary compression of highly detailed sec-tions. Additionally, the hierarchical structure itself provides natural scalability. Empirically, we ob-serve that the rate of new page creation declines by approximately 30% over the course of extended conversations, exhibiting sub-linear growth that confirms the structure’s self-regulating property. See Appendix C and Figure 3 for detailed analysis. 

6 Conclusion 

We present MemCog, a system that advances agent memory from Memory-as-Tool toward Memory-as-Cognition by integrating Navigable Memory Store with Proactive Reasoning Protocol, mak-ing memory access an integral part of the reason-ing process. Experiments validate this paradigm shift: MemCog achieves state-of-the-art on pas-sive QA benchmarks and our proposed Proactive-MemBench. Ablation studies confirm the comple-mentary synergy between the Proactive Reasoning Protocol and Navigable Memory Store, demonstrat-ing that advancing from passive retrieval toward reasoning-integrated navigation yields substantial improvements in agent memory capabilities. 87 Limitations 

MemCog has several limitations: (1) multi-step navigation incurs additional token overhead, which may be non-trivial under strict latency or cost con-straints; (2) the Proactive Reasoning Protocol re-lies on strong instruction-following capability of the backbone LLM, with weaker models exhibiting larger performance variance; (3) navigation quality is bounded by Page construction accuracy—poorly extracted facts or missing links degrade the entire system; and (4) with sparse early-stage conversa-tions, the Page is under-connected and navigation offers limited advantage.

10 A Memory-Aware Reasoning Protocol 

A.1 Protocol Definition 

The following is the complete Proactive Reason-ing Protocol provided to the agent as a system prompt. This protocol defines the behavioral norms for memory usage, including when and how to ini-tiate memory navigation.                                                                                 

> Proactive Reasoning Protocol (System Prompt)
> ## Your Memory System
> You have access to aNavigable Memory Store about the user, organized by dimensions (personal preferences, work, relationships, events, etc.).
> ### When to use memory Proactive association (most important): -When the user mentions people, places, topics, or situations that may connect to stored knowledge →navigate your memory to check for relevant context - When you sense the conversation could benefit from historical context →browse relevant dimensions - When the user’s statement might contradict or update stored information →verify against memory
> Reactive response (traditional): -When the user explicitly asks about past information →search or navigate
> ### How to use memory
> -Navigate progressively :Don’t stop at the first result. If apage has relevant links, follow them. Build a complete picture. -Integrate naturally :Use recalled information as if you naturally remember it. Don’t say "according to my records." -Be selective : Not everything you find needs to be shared. Consider whether the information is relevant, timely, and appropriate. -Think before acting :Before navigating, briefly reason about which dimension/topic is most likely relevant.

A.2 Protocol as Cognitive Activation 

The Proactive Reasoning Protocol activates the cog-nitive potential of the navigable memory structure. We provide three arguments for this claim: First, protocol-level design has partial precedent as a validated contribution. ReAct (Yao et al., 2023) defines the Think →Act →Obs format in prompts and represents a landmark contribution in agent reasoning. Proactive Reasoning Protocol similarly defines the agent’s behavioral pattern for memory usage. Second, the protocol and structure exhibit com-plementary synergy that neither achieves alone. A proactive instruction paired with flat memory yields limited improvement because returned passages of-fer no structural cues for continued exploration. A navigable memory structure without Proactive Rea-soning Protocol remains underutilized because the agent defaults to passive, query-triggered behavior. Only the combination produces the intended effect: the protocol motivates navigation initiation, while the navigable memory structure sustains multi-step exploration through its inherent navigational affor-dances. Third, this synergy is empirically verifiable through ablation experiments. Removing only Proactive Reasoning Protocol (navigable memory structure retained) substantially degrades proactive association performance while preserving passive QA capability. Removing only the navigable mem-ory structure (proactive prompt retained with flat memory) limits the depth of exploration regard-less of the agent’s motivation to explore. MemCog adopts both the proactive prompt and the navigable memory structure, yielding significant synergistic gains. 

B ProactiveMemBench 

B.1 Motivation 

Existing memory benchmarks (e.g., LoCoMo (Ma-harana et al., 2024)) primarily evaluate reactive memory retrieval, which is the ability to answer ex-plicit questions about past conversations. However, in real-world conversational AI, a more valuable yet understudied capability is proactive memory association : when a user mentions concept A in the current turn, the model should spontaneously recall and surface related concepts B, C, D from prior conversations which the user has not explicitly asked about but would benefit from being reminded of. To systematically evaluate this ability, we con-struct ProactiveMemBench , a benchmark where each test instance consists of a brief user utter-ance (i.e. a declarative sentence containing only one memory unit) paired with a ground-truth set of memory units that a memory-capable model should 11 proactively surface based on accumulated conver-sational history. 

B.2 Construction 

We identify three core design principles: 

Bottom-up construction with persona consis-tency. Rather than generating dialogues first and deriving questions post-hoc, we adopt a bottom-up approach: we first define atomic memory units and then weave them into coherent dialogues. A fixed user persona constrains all generation steps to ensure global consistency. 

Multi-type memory associations. We define five distinct types of proactive memory triggers that mirror how human memory works, which are illustrated in Table 6. 

Graph-structured association labeling. Mem-ory Units are not treated as independent ones; in-stead, we explicitly model their pairwise associa-tions, forming a graph that enables controlled place-ment of cross-session memory cues. The benchmark is constructed through a six-step LLM-driven pipeline, using Qwen3.5-397B-A17B as the generation backbone. Each step’s output is conditioned on all preceding outputs to maintain coherence.

B.5 Evaluation Metrics: Precision 

The Precision metric scores each retrieved unit on a 0–5 scale, evaluating whether the unit is well-supported by the raw retrieved content and relevant to the user’s context. The same scoring criteria are applied to both LLM-judged and human evaluation, with the prompt-defined rubric provided below. 

Precision Judge Prompt – System (English Transla-tion) 

You are a memory recall evaluation expert. Your task is: given a user message and the raw content actually retrieved by the agent from the memory store, score each “retrieved unit” output by the agent on a 0–5 quality scale. 

Scoring criteria (integer, 0–5): 0: Fabrication – the memory unit has no basis in the retrieved content and is entirely made up; or the memory unit can be directly extracted from the user message; 1: Reasonable unit name – the memory unit sounds reasonable but has no clear support in the retrieved content; 2: Reasonable path – association evidence can be found in the retrieved content, and the association path is basically logical; 3: Matches trigger condition – the memory unit is supported by raw content, and the association path explicitly matches the trigger type (temporal/entity/emotional/behavioral pattern/multi-hop); 4: High quality – the memory unit highly aligns with the retrieved content, the path is clear and natural, and strongly relevant to the user’s context; 5: Excellent quality – the memory unit 

16 is precise and compelling, the path is impeccable, and represents the most valuable memory association for the user’s context. Output must be in JSON format only, with no extra text.

B.6 Human Validation 

To verify the quality of our automatically con-structed benchmark, we conduct a human spot-check on a random sample of 125 test instances (25 per topic domain). Two annotators indepen-dently evaluate each instance along three criteria: (1) query validity —whether the trigger utterance is natural and contains only the intended trigger memory unit without leaking candidate informa-tion; (2) candidate correctness —whether each candidate in the ground-truth set is genuinely infer-able from the conversation history and absent from the query; and (3) evidence support —whether the cited source memory units fully support the associ-ation path described in the reasoning field. An instance is marked as accepted if it passes all three criteria, and rejected otherwise. Table 7 summarizes the results.

B.7 LLM-based candidate completeness check. 

Since each test instance only retains the top-3 most relevant memory units as its ground-truth candi-date set, a natural concern is whether important candidates are omitted. To verify completeness, we employ three strong LLMs (GPT-4o, Claude-3.5-Sonnet, and Gemini-1.5-Pro) as independent judges. We then compare each model’s top-3 selections against our ground-truth candidate set. A candidate is flagged as a potential omission if at least two out of three judges rank it within their top-3 but it is absent from our annotations. Results show that across all 200 instances, the three judges reach majority agreement with our ground-truth candidate set on 96% of instances (i.e., at least 2 of 3 candidates overlap). For the remain-ing 4%, manual inspection reveals that the diver-gence stems from equally plausible alternative as-sociations rather than clear omissions—confirming that our top-3 selection is well-calibrated and the ground-truth candidate sets are reliable. 

B.8 Human Evaluation Details 

We randomly recruited student volunteers from our institution to evaluate model outputs. Annotators were provided with the same scoring rubric used for our LLM-based evaluation and asked to rate outputs from all baselines on a Likert scale. Com-pensation was mutually negotiated and agreed upon prior to the task. All participants were informed that their ratings would be used for model output evaluation in this research, and they voluntarily agreed to participate. Our human evaluation in-volves minimal risk: annotators only rate model outputs on a Likert scale, no sensitive or personal information is collected, and all participants are voluntary adults who were informed of the task purpose and compensated fairly. Our institution does not require formal approval for such low-risk annotation tasks, and the study was determined ex-empt from full ethics review under our institutional guidelines. Annotators were randomly selected and their demographic characteristics have negligible influence on the scoring results, as the task solely involves rating model output quality on predefined criteria.

D Case Study: Navigation as Reasoning 

The Cross-Dimensional Navigation Interface trans-forms memory retrieval into an integral part of the reasoning process. Within a standard ReAct loop, the agent interleaves thinking and navigation: each Think step evaluates whether current information is sufficient and plans the next navigation action; each Act step invokes a navigation tool; each Obs step provides structured results that feed back into reasoning. This tight coupling fundamentally dis-tinguishes multi-step navigation from single-shot retrieval across a spectrum of query complexities. Table 8 presents cases selected from Long-MemEval (Wu et al., 2025). The corresponding navigation trace examples across five categories are detailed below. Agent Autonomy. Crucially, the agent exercises full autonomy over its navigation decisions. At each step, the agent determines which dimension to enter, whether to drill deeper into a specific page, and whether to follow cross-dimensional associative links to related content. Each Think step constitutes genuine reasoning: the agent evalu-ates whether accumulated information is sufficient, whether additional context would improve its re-sponse, and whether related entries visible in the current page’s link graph might be relevant. This autonomy means that retrieval and reasoning are no longer separate stages but rather two facets of a single, unified cognitive process . The agent does not first retrieve and then reason; it reasons 

through the act of navigation itself, with each ob-servation shaping subsequent exploration decisions. The ability to self-select the appropriate strategy— from single-step lookup to multi-step traversal— represents a form of metacognitive reasoning: be-fore initiating navigation, the agent assesses the na-ture of the query and plans an efficient exploration path. This prevents both the under-exploration that plagues single-shot systems (missing relevant context) and the over-exploration that would re-sult from always performing exhaustive multi-step traversals. 27

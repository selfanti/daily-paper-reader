## 摘要
大型语言模型（LLM）智能体依赖可复用的技能来解决复杂任务。然而，现有的技能创建方法将技能视为孤立和静态的工件，限制了其可复用性、可靠性和长期改进。我们提出了MUSE-Autoskill智能体（记忆利用技能进化），一个以技能为中心的智能体框架，允许智能体通过统一的生命周期（创建、记忆、管理、评估和优化）来持续提升其任务解决能力。我们的框架使智能体能够按需创建技能，跨任务存储和复用技能，高效组织和选择技能，并通过单元测试和运行时反馈对技能进行评估以实现持续优化。我们进一步引入了技能级记忆，为每个技能跨任务积累经验，从而实现更有效的复用和长期适应。在SkillsBench上的实验初步证明，生命周期管理的技能能够提高任务成功率、效率、复用性及跨智能体迁移能力，凸显了将技能视为长期存在、具备经验感知能力且可测试资产的重要性。

## Abstract
Large language model (LLM) agents rely on reusable skills to solve complex tasks. However, existing skill creation approaches treat skills as isolated and static artifacts, limiting their reusability, reliability, and long-term improvement. We propose MUSE-Autoskill Agent (Memory-Utilizing Skill Evolution), a skill-centric agent framework that lets agents continuously improve their task-solving capability by creating, reusing, and refining skills under a unified lifecycle (creation, memory, management, evaluation, and refinement). Our framework enables agents to create skills on demand, store and reuse them across tasks, organize and select them efficiently, and evaluate them through unit tests and runtime feedback for continuous refinement. We further introduce skill-level memory that accumulates experience for each skill across tasks, enabling more effective reuse and adaptation over time. Experiments on SkillsBench provide initial evidence that lifecycle-managed skills can improve task success, efficiency, reuse, and cross-agent transfer, highlighting the importance of treating skills as long-lived, experience-aware, and testable assets.

## 2. 方法论：核心思想、关键技术细节
### 2.1 整体架构
- 基于ReAct循环（计划-行动-观察），集成技能子系统。
- **内置基础工具**：skill_create、web_search等，其余技能均通过按需生成获得。

### 2.2 技能生命周期
- **技能封装**：每个技能为结构化目录，包含SKILL.md（接口、步骤）、可选scripts/、tests/、resources/。
- **技能创建**：智能体通过skill_create工具提供高阶规范，系统生成SKILL.md、脚本、单元测试；创建后立即执行单元测试验证，通过才注册到技能库。
- **技能评估**：基于tests/目录的单元测试，失败触发更新（update_skill）重新生成。
- **技能记忆**：引入**技能级记忆**（每个技能携带.memory.md），记录跨任务的使用经验（失败模式、输入格式技巧等），与短时记忆、长时记忆构成多级记忆体系。
- **技能管理**：技能索引基于元数据（名称、描述）；支持合并、修剪（当技能冗余或长期未用时）。

### 2.3 上下文管理
- 对话以DAG结构（节点为每次ReAct循环）存储，包含可变parent_id（活动链）和不可变历史指针。
- **两级自适应压缩**：
  - Level-1：将单个过大的节点内容压缩为摘要，保留节点边界。
  - Level-2：当Level-1不足以满足token预算时，将连续中间节点合并为单个合成摘要节点。
- 保持前K=5和后K=5个节点不压缩，确保系统提示和近期状态完整。
- 会话状态可持久化，支持跨会话恢复。

### 2.4 算法流程
1. 用户提交任务 → 智能体进入ReAct循环。
2. 计划阶段：解析任务，决定使用现有技能还是创建新技能。
3. 行动阶段：通过skill_create生成新技能（若需要），执行技能代码（在独立沙箱中运行），或直接调用已有技能。
4. 观察阶段：收集结果，更新记忆（短时、长时、技能级）。
5. 技能评估：执行单元测试，失败则触发refiner修改技能包，重新测试直至通过。
6. 上下文压缩随每次ReAct循环触发（当token总量超过阈值时）。

- **自适应上下文压缩**：两级压缩策略，在不丢失关键信息的前提下控制token预算，支持长程任务。

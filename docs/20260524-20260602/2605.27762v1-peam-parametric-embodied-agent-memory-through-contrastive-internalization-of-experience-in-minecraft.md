## 摘要
我们提出了PEAM，一个在Minecraft中的参数化具身智能体记忆框架，它将智能体记忆从推理时的检索转变为通过经验内化的参数驻留技能。PEAM将用于开放式推理的慢速深思LLM与用于反射性执行固化技能的快速参数化模块配对。快速模块是一个多模态专家混合LoRA架构，具有按类别物理隔离的适配器，使得在不发生灾难性遗忘的情况下进行参数级持续学习成为可能。我们将失败视为一等训练信号：失败-修正轨迹对通过联合的行为克隆和对比目标进行内化，因此智能体不仅学习哪些操作成功，还学习修正后的动作与失败动作有何不同。为了管理固化，PEAM引入了一个参数化价值评分来决定哪些经验应该被内化，以及一个无尺度自触发固化机制来决定何时内化，无需特定任务的手动调优阈值，从而使智能体在触发机制跨任务分布转移时无需重新调优即可自我进化。在Minecraft中的实验表明，PEAM提高了长时空任务性能，减轻了对先前固化技能的遗忘，并在检索式具身智能体和参数化记忆变体中提高了参数化与检索效率。

## Abstract
We present PEAM, a Parametric Embodied Agent Memory framework in Minecraft that transforms agent memory from inference-time retrieval into parameter-resident skills internalized through experience. PEAM pairs a slow deliberative LLM for open-ended reasoning with a fast parametric module for reflexive execution of consolidated skills. The fast module is a multimodal Mixture-of-Experts LoRA architecture with per-category physically isolated adapters, enabling parameter-level continual learning without catastrophic forgetting. We treat failure as a first-class training signal: failure--correction trajectory pairs are internalized through a joint behavioral-cloning and contrastive objective, so the agent learns not only what succeeds but also how corrected actions differ from failed ones. To govern consolidation, PEAM introduces a parameterization-worthiness score for deciding which experience should be internalized, and a scale-free self-triggered consolidation mechanism for deciding when to internalize without task-specific hand-tuned thresholds, making the agent self-evolving as the trigger transfers across task distributions without re-tuning. Experiments in Minecraft show that PEAM improves long-horizon task performance, mitigates forgetting on previously consolidated skills, and improves parametric-versus-retrieval efficiency over retrieval-based embodied agents and parametric memory variants.

### 1. 核心问题与整体含义（研究动机与背景）

- **现有问题**：当前基于LLM的具身智能体通常依赖非参数化记忆（如外部轨迹库、反思日志），每次推理时需检索并注入上下文，导致：  
  - 上下文预算消耗大（每次复用需重新注入整个轨迹）。  
  - 检索与提示构建增加决策延迟。  
  - 经验始终留在外部，无法变成智能体自身的参数化能力，重复任务时参数无变化。  
- **认知科学启发**：长期记忆由快速稀疏的情景存储和缓慢分布的参数存储组成，通过离线固化（如睡眠）将情景痕迹写入参数。现有系统实现了经验获取（技能库、检索增强），但缺乏固化环节——即决定哪些经验应成为参数化能力，以及何时固化。  
- **研究目标**：提出PEAM框架，使智能体能够将选定的经验内化为参数驻留的技能，实现参数级持续学习，同时缓解灾难性遗忘，并提升推理效率。

### 2. 方法论：核心思想、关键技术细节、公式或算法流程

- **整体架构**：双层智能体系统  
  - **慢速深思LLM（π_slow）**：负责开放式推理、代码合成、结果验证。使用Azure GPT-4o。  
  - **快速参数化模块（π_fast）**：基于Qwen3-VL-8B-Instruct的多模态MoE-LoRA架构，包含按类别物理隔离的适配器（crafting, gather, combat）。  
  - 两者通过固化管道耦合，包含“参数化价值（PV）评分”和“自触发固化（STC）”。  
- **核心思想**：将失败作为首要训练信号，通过行为克隆（BC）和对比优化的联合损失，使智能体不仅学习成功动作，还学习纠正动作与失败动作的差异。  
- **关键技术细节**：  
  - **失败-纠正轨迹对（D_cpair）**：从情景存储中提取，每对包含失败轨迹τ_f和后续成功轨迹τ_c，以及技能类别c。  
  - **联合损失函数**：  
    L_PEAM(θ_c) = L_BC(θ_c; D_succ^(c)) + λ·L_DPO(θ_c; D_cpair^(c))  
    - L_BC：标准下一token负对数似然（用于格式级建模，如代码包装器）。  
    - L_DPO：基于偏好的对比损失，以纠正轨迹为chosen，失败轨迹为rejected，β=0.1。  
  - **参数化价值（PV）评分**：决定“什么”应内化。评分公式：  
    PV(s) = w1·U_cost + w2·U_stab – w3·P_redund – w4·R_forget  
    - U_cost：节省的检索成本（未来调用频率估计 × 代码长度）。  
    - U_stab：稳定性（成功率 × 上下文方差惩罚）。  
    - P_redund：与已有技能的代码相似性（TF-IDF余弦）。  
    - R_forget：结构遗忘风险（同类别则惩罚）。权重通过网格搜索固定。  
  - **自触发固化（STC）**：决定“何时”固化。基于统计假设检验：  
    Z(s) = (p̂_recent – p̂_baseline) / √(p̂(1-p̂)(1/W + 1/B)) > z_α  
    且PV(s)处于前q分位。W=B=10, α=0.05, q=0.5，无需任务特定阈值。  
  - **物理隔离的MoE-LoRA适配器**：每个类别独立训练，更新时不干扰其他类别参数，从根本上防止跨类别遗忘。  
- **算法流程**（Algorithm 1）：任务路由→快速路径尝试（若适配器存在）→验证→失败则回退到慢速路径→将轨迹存入情景存储→提取候选→PV评分+STC触发→更新对应类别适配器。

### 3. 实验设计

- **场景与环境**：Minecraft 1.19，使用VOYAGER的Mineflayer执行框架。  
- **任务套件**：11个长时任务，覆盖craft（T1-T5）、gather（T6-T9）、combat（T10-T11），每个任务最多200步，用环境验证器判断成功。  
- **基准方法**（8种）：  
  - B6: No-memory ReAct（纯无记忆）  
  - B7: Reflexion（文本反思）  
  - B8: MrSteve（时空记忆）  
  - B3: Naive full-FT（全参数微调）  
  - B4: Single shared LoRA（单共享LoRA）  
  - B5: EWC（弹性权重固化）  
  - B1: VOYAGER（检索式技能库）  
  - B2: Optimus-1-rep.（多模态检索式）  
- **评估指标**：  
  - 任务成功率（Wilson 95%置信区间，McNemar配对检验）  
  - 遗忘程度（顺序固化craft→gather→combat后保留率）  
  - 推理效率（每步中位延迟、每任务token消耗）  
  - 触发鲁棒性（跨分布切片触发一致性）  
- **对照组设计**：消融实验覆盖三大设计选择：失败信号使用、MoE隔离、PV评分、STC vs固定调度、视觉接地权重等。

- **训练与推理硬件**：单张A100 80GB GPU（bf16精度）。推理中位延迟3.2秒。

### 5. 实验数量与充分性

- **主要实验**：11个任务×3个随机种子=33次试验，报告成功率、置信区间与p值。  
- **遗忘实验**：顺序固化三步，每步后测量保留率，对比4种方法。  
- **消融实验**：6组消融（A1-A6），分别测试BC必要性、MoE隔离效果、PV vs启发式、PV组件留一、STC vs固定调度、视觉权重。  
- **触发鲁棒性实验**：构建craft-heavy和combat-heavy分布切片，计算触发事件和Jaccard重叠。  
- **额外发现**：报告了4个方法论发现（前向偏好边缘不预测生成部署能力；量化部署失败模式；导航类无可用纠正对；轨迹重切片作为跨分布替代评价）。  
- **充分性评估**：实验覆盖主要设计空间，对比了8种强基线，消融全面，统计检验严谨（McNemar检验、置信区间）。但局限在单一环境（Minecraft）和三个技能类别，外部泛化性未验证。

### 6. 主要结论与发现

- PEAM在11个长时任务上达到69.7%成功率，比VOYAGER（54.5%）提高15.2个百分点（p=0.018）。  
- 推理效率显著提升：中位延迟从5.5s降至3.2s（-42%），每任务token消耗从约31,200降至约4,600（-85%）。  
- **遗忘抵抗**：顺序固化后PEAM保持100%初始保留，而共享LoRA下降32.4%，EWC下降43.3%，全微调下降78.5%。  
- **失败作为训练信号**：联合BC+DPO优于纯DPO（后者0/12生成格式正确代码）。  
- **PV评分改变选择**：与启发式（成功率≥0.8且检索≥15）相比，任务成功率下降8.7个百分点，Spearman ρ=-0.375。  
- **STC自触发稳定**：相同超参数下，craft-heavy和combat-heavy分布触发模式一致，无需手动调优。  
- 核心结论：具身智能体不应仅在外围积累历史，而应具备将经验内化为参数技能的通路。

### 8. 不足与局限

- **环境单一**：所有实验仅在Minecraft 1.19中，泛化到机器人操作、网页任务等未验证。  
- **技能类别有限**：仅固化craft, gather, combat三类，导航等其他类别依赖慢速LLM（且导航无可用纠正对）。  
- **适配器线性增长**：每新增类别需独立适配器，参数规模随类别线性增长，大规模部署可能受限。  
- **动作语法特定**：DPO+BC方案依赖JavaScript包装器语法，不同动作空间可能需要调整。  
- **超参数固定**：PV权重和STC超参数通过网格搜索固定，未探索自适应调整。  
- **跨分布评估方法限制**：触发鲁棒性使用轨迹重切片代替真实多分布数据，有一定合成偏差。  
- **伦理风险**：参数化记忆使内部状态更难审计，环境验证器依赖可能引入强化错误行为风险。  
- **资源消耗未详述**：训练总GPU小时数未报告，影响复现与资源估计。

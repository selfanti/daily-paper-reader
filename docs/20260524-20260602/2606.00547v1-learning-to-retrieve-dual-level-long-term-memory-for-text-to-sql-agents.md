## 摘要
交互式文本到SQL代理通过多轮交互解决数据库任务，涉及模式探索、查询执行、反馈解释和决策修订。长期记忆帮助代理重用过去的经验，但现有的检索方法仍然有限。静态方法依赖固定的相似性启发式，不会优化下游效用；而动态方法通常从稀疏的最终结果中学习，并在单一决策层面上检索记忆。当记忆有用性在交互阶段之间发生变化时，这就不足够了，因为对初始规划有用的记忆可能与局部、状态条件执行所需的记忆不同。我们提出了MERIT，一个动态的多层次记忆检索框架。MERIT维护情节级记忆用于全局策略指导，以及回合级记忆用于局部决策支持。这两个层次都使用经过强化学习优化的学习检索策略。为了在有限的中间监督下训练回合级检索，MERIT使用轻量级的过程奖励模型为局部记忆选择提供密集的代理奖励。在BIRD-Interact上的实验表明，MERIT在成功率上优于无记忆、静态检索和动态检索基线，同时减少了平均交互回合数。在Spider2-Snow上的迁移结果进一步显示了无需基准特定调优的积极跨基准迁移。这些结果表明，多层次检索改善了交互式文本到SQL代理中的经验重用。

## Abstract
Interactive text-to-SQL agents solve database tasks through multi-turn interactions involving schema exploration, query execution, feedback interpretation, and decision revision. Long-term memory helps agents reuse past experiences, but existing retrieval methods remain limited. Static methods rely on fixed similarity heuristics that do not optimize downstream utility, while dynamic methods often learn from sparse final outcomes and retrieve memories at a single decision horizon. This is insufficient when memory usefulness changes across interaction stages, since memories useful for initial planning may differ from those needed for local, state-conditioned execution. We propose MERIT, a dynamic multi-horizon memory retrieval framework. MERIT maintains episode-level memory for global strategic guidance and turn-level memory for local decision support. Both levels use learned retrieval policies optimized with reinforcement learning. To train turn-level retrieval despite limited intermediate supervision, MERIT uses a lightweight Process Reward Model to provide dense proxy rewards for local memory selection. Experiments on BIRD-Interact show that MERIT outperforms no-memory, static-retrieval, and dynamic-retrieval baselines in success rate while reducing average interaction turns. Transfer results on Spider2-Snow further show positive cross-benchmark transfer without benchmark-specific tuning. These results suggest that multi-horizon retrieval improves experience reuse in interactive text-to-SQL agents.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究动机**：交互式文本到SQL（Text-to-SQL）代理需要多轮交互（模式探索、SQL执行、反馈解释、决策修正）才能完成任务。长期记忆有助于代理重用过去经验，但现有检索方法存在局限：
  - **静态方法**（如BM25、余弦相似度）依赖固定相似性启发式，不优化下游任务效用。
  - **动态方法**（如Memento、MemRL）通常从稀疏的最终结果中学习，且仅在单一决策层面（episode级）检索，忽略了记忆有用性在不同交互阶段的变化（例如，初始规划有用的记忆可能不同于局部状态条件执行所需的记忆）。
- **核心问题**：如何让记忆检索适应不同的决策层面（全局策略 vs. 局部执行），并利用有限的中间监督信号训练局部检索策略。
- **整体含义**：提出了MERIT框架，通过双层记忆（episode级和turn级）和强化学习优化的检索策略，结合过程奖励模型（PRM）提供密集奖励，实现更有效的经验复用，提升交互式文本到SQL代理的成功率和效率。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将记忆检索分解为两个决策层面：
  - **Episode级记忆**：存储完整交互轨迹，在每次交互开始前检索，提供全局战略指导。
  - **Turn级记忆**：存储紧凑的状态-动作-观察片段，在每次交互回合前检索，提供局部状态条件的支持。
- **关键技术细节**：
  1. **双级记忆架构**：
     - Episode记忆：键为用户请求嵌入，值为原始请求、完整轨迹、结果标签、蒸馏洞察等。
     - Turn记忆：键为状态签名（原始请求 + 当前查询上下文 + 前一步动作和观察的粗粒度描述），值为抽象动作、观察片段、执行状态标签。
  2. **检索策略学习**：
     - 两级均采用两阶段流水线：第一阶段用嵌入相似度生成候选集；第二阶段用学习的选择器（基于PPO优化）决定哪些记忆注入代理提示。
     - Episode级选择器使用终端奖励（最终执行正确性）训练。
     - Turn级选择器使用PRM生成的密集奖励（对每个候选记忆-状态对评分）训练，解决中间回合监督不足的问题。
  3. **PRM（Process Reward Model）**：
     - 对状态-记忆对预测结构化评价（七个维度：状态匹配度、可操作性价值、模式泛化性、结果可靠性、对代理的清晰度、评估置信度、总体效用；每个维度0-5分；还有二元标志和文本反馈）。
     - 训练采用两阶段：SFT（模仿教师模型标注）+ GRPO（使用结构化匹配奖励优化）。
  4. **数据增强**：通过SQL级扰动（替换兼容列、修改字面量、改变排序等）和自然语言请求生成，扩增训练数据（约7000条）。
- **算法流程**（文字说明）：
  - 内存写入：每个episode结束后写入episode记忆；每个turn结束后写入turn记忆。
  - 推理时：对新请求，先进行episode级检索（候选→选择→注入初始提示）；在每个交互回合前，根据当前状态签名进行turn级检索（候选→选择→注入本地提示）。
  - 代理执行动作，获得观察，更新状态。回合结束得到最终奖励。

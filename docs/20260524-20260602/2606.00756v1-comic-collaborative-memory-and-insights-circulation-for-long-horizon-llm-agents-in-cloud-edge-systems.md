## 摘要
在边缘服务器上部署轻量级大语言模型（LLM）智能体可以降低延迟，使智能体服务更贴近用户，但资源受限的边缘模型在处理需要持久记忆、子目标跟踪和反思的长周期任务时往往力不从心。在部署后对边缘模型进行微调成本高昂且难以在异构节点间扩展，而纯本地记忆则使智能体拥有孤立的经验，并导致提示上下文不断增长。我们提出CoMIC，一种无需参数更新的云边协作记忆与洞察循环框架。CoMIC遵循“集中反思、分散执行”的设计：边缘智能体使用面向子目标的分层记忆和选择性重扩展的相关历史进行本地执行，而云端LLM评价器异步评估已完成轨迹，过滤可复用的经验，并通过语义子目标标识符聚合跨智能体指导。在涵盖符号规划与文本交互的五项长周期智能体任务中，CoMIC提升了弱边缘智能体的进度率和动作基础，并在不更新模型参数的情况下获得了依赖于任务的成功率提升。

## Abstract
Deploying lightweight Large Language Model (LLM) agents on edge servers can reduce latency and move agentic services closer to users, but resource-constrained edge models often struggle with long-horizon tasks that require persistent memory, subgoal tracking, and reflection. Fine-tuning edge models after deployment is costly and difficult to scale across heterogeneous nodes, while purely local memory leaves agents with isolated experience and growing prompt context. We propose \textsc{CoMIC}, a parameter-update-free cloud-edge framework for Collaborative Memory and Insights Circulation. \textsc{CoMIC} follows a \textit{Centralized Reflection, Decentralized Execution} design: edge agents execute locally using subgoal-oriented hierarchical memory and selective re-expansion of relevant histories, while a cloud-side LLM critic asynchronously evaluates completed trajectories, filters reusable experience, and aggregates cross-agent guidance keyed by semantic subgoal identifiers. Across five long-horizon agent tasks spanning symbolic planning and text interaction, \textsc{CoMIC} improves progress rate and action grounding for weak edge agents and yields task-dependent success-rate gains without updating model parameters.

### 6. 主要结论与发现

- **主要结论**：CoMIC在**不更新任何模型参数**的前提下，通过云边协作记忆机制显著提升了弱边缘智能体在长周期任务中的**进展率（PR）** 和**动作基础准确率（GA）**，同时**降低了上下文token消耗**约27.88%（总体平均）。成功率（SR）有任务依赖的提升（如Blocksworld从0提升至20%，平均提升6.5%）。
- **关键发现**：
  - 即使边缘模型较弱（Mistral-7b），云端对轨迹的异步反思和跨智能体聚合仍能提取有价值经验。
  - 异构场景（场景B）中，来自强智能体（GPT-4）的轨迹能生成更高质量全局指导，进一步提升弱智能体在部分任务上的子目标级表现，但整体成功率受限于弱模型自身规划能力。
  - 云端缓存命中率平均45.12%（场景A），表明轨迹重复模式存在，有效减少冗余调用。
  - 消融实验确认云端组件是增益的主要来源：无云端时所有任务SR为0或极低，有云端后出现非零成功率和进展。
- **限制性结论**：云端反思无法完全克服弱模型的根本局限，尤其在复杂任务中，弱模型难以正确分解子目标或理解指导。

### 7. 优点：方法或实验设计上的亮点

- **方法亮点**：
  - **无需参数更新**：适合实际部署中难以频繁微调的场景。
  - **异步非阻塞设计**：云端评价与边缘执行解耦，不增加实时响应延迟。

  - **分层记忆与选择性重扩展**：既保证上下文可控，又能在需要时恢复细节，平衡效率与准确性。
  - **准入机制**：基于自报告置信度及惩罚项过滤低质量反思，提升知识库可靠性。
- **实验亮点**：
  - 在同质和异构两种典型云边部署场景下系统评估。
  - 与两个强基线（STANDARD、LOCAL）对比，且骨干统一为Mistral-7b，避免模型规模干扰。

### 8. 不足与局限

- **实验覆盖局限**：仅覆盖符号规划（4个PDDL）和单一文本交互（Jericho），缺少更广泛的复杂任务类型（如多轮对话、工具使用、网页导航等），结论泛化性有待验证。
- **基础模型能力瓶颈**：最根本的局限在于弱边缘模型（Mistral-7b）自身规划能力不足，即使有云端指导，也难以实现高成功率（多个任务SR为零）。云端反思并不能完全补偿基模型缺陷。
- **云端异构对齐问题**：场景B中来自强智能体的高质量指导与弱智能体的理解能力不匹配，导致缓存命中率和ACK率降低，说明异构环境下指导的通用性有待改进。
- **上下文节省依赖比较设置**：论文中上下文节省量是与STANDARD基线对比得出的，且部分任务上下文消耗甚至略高于LOCAL基线（见消融表）。因此“节省27.88%”需结合具体比较对象理解。
- **系统开销分析不足**：虽然报告了云端延迟（约68秒），但对边缘端实际实时性的影响、网络传输与同步细节未深入讨论，可能影响实际部署。
- **未研究多智能体冲突**：多个边缘智能体同时执行时，云端聚合是否存在冲突或重复学习未被探讨。
- **置信度机制依赖LLM自我报告**：准入分数中的$s_{\text{self}}$来自LLM自身判断，可能不可靠，文中未分析其对结果的影响或与人类评价的一致性。

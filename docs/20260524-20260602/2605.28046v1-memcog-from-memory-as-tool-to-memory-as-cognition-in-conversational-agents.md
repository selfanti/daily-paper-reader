## 摘要
现有代理记忆系统普遍遵循我们称之为“记忆即工具”的范式，其中单个查询触发对扁平段落列表的单次检索，存在被动调用、推理与检索解耦、以及检索片段与代理导航需求之间的结构性不匹配等问题。我们提出了MemCog，这是一种“记忆即认知”系统，将记忆访问作为推理过程的一个组成部分。MemCog将用户知识组织为具有关联链接图的可导航记忆存储，公开了用于多步推理驱动遍历的跨维度导航接口，并采用主动推理协议，驱动代理从对话上下文中自发启动记忆探索。我们还构建了ProactiveMemBench，这是首个用于评估主动记忆触发的基准。实验表明，MemCog在被动问答基准上达到了最先进的性能（LoCoMo上92.98，LongMemEval上95.8），同时在ProactiveMemBench上大幅超过基线，展示了“记忆即认知”的优势。

## Abstract
Existing agent memory systems universally follow what we term a Memory-as-Tool paradigm where a single query triggers one-shot retrieval of flat passage lists, suffering from passive invocation, reasoning-retrieval decoupling, and structural mismatch between retrieved fragments and the agent's navigational needs. We propose MemCog, a Memory-as-Cognition system that makes memory access an integral part of the reasoning process. MemCog organizes user knowledge as Navigable Memory Store with associative link graphs, exposes Cross-Dimensional Navigation Interface for multi-step reasoning-driven traversal, and employs Proactive Reasoning Protocol that drives agents to spontaneously initiate memory exploration from conversational context. We additionally construct ProactiveMemBench, the first benchmark for evaluating proactive memory triggering. Experiments show that MemCog achieves state-of-the-art on passive QA benchmarks (92.98 on LoCoMo, 95.8 on LongMemEval) while substantially outperforming baselines on ProactiveMemBench, demonstrating the advantage of Memory-as-Cognition.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：现有对话代理的长期记忆系统普遍遵循“记忆即工具”（Memory-as-Tool）范式——通过单一查询进行一次检索，返回扁平段落列表，代理仅在检索结束后进行推理。这种范式存在三大根本性缺陷：
  - **调用瓶颈**：记忆系统仅当代理显式发出工具调用时才被激活，无法主动触发回忆。
  - **推理-检索解耦**：检索是单次调用，当首次结果不足时，代理无法动态规划进一步探索；推理能力无法介入检索管道。
  - **结构错配**：返回的top-k片段缺乏结构导航信息，代理无法从当前结果推断下一步最佳检索路径。
- **整体含义**：论文提出将记忆从“工具”转变为“认知过程”的一部分，让记忆访问与推理深度融合，从而克服上述限制。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
**Memory-as-Cognition**：将记忆访问作为推理过程不可分割的组成部分，代理在ReAct循环中结合主动记忆触发的多步导航，实现检索与推理的紧密耦合。

### 三大组件

#### (1) 可导航记忆存储（Navigable Memory Store）
- **层次化组织**：三维粒度：维度（Dimension）→ 页面（Page）→ 段落（Section）。维度提供入口，页面提供深度，跨维度关联链接提供横向连通。
- **跨维度关联链接**：四种类型：`related_to`（语义关联）、`temporal_next`（时序）、`caused_by`（因果）、`contrasts_with`（对比）。这些链接作为显式导航线索，自然激发多步遍历。
- **构建与演化**：初始从对话历史提取结构化事实→聚类成维度/页面→建立链接。增量更新：每次对话后抽取新事实→匹配现有页面或创建新页面→建立链接→冲突检测和置信度更新。

#### (2) 跨维度导航接口（Cross-Dimensional Navigation Interface）
- 替代单次检索工具，提供多粒度动作：
  - `list_dimensions()`：列出所有顶层维度及描述。
  - `browse_dimension(dim)`：返回指定维度下的页面标题和摘要。
  - `read_page(page_id)`：返回完整页面内容（包含所有段落和出向链接）。
  - `follow_link(link)`：通过超链接跳转到目标页面。
- 每个动作不仅返回请求内容，还提供结构上下文（可用链接、兄弟页面、维度位置），辅助后续导航决策。

#### (3) 主动推理协议（Proactive Reasoning Protocol）
- 在系统提示中定义代理记忆使用行为规范，包含两种模式：
  - **主动关联**：当对话上下文提示潜在连接（人物、地点、主题等）时，自发启动记忆导航。
  - **被动响应**：对用户明确询问过去信息时进行导航。
- 协议进一步指导：渐进式导航（不停止于第一个结果）、自然整合、选择性呈现（考虑相关性、时效性、敏感性、冗余性）。
- **选择性呈现**：代理在思考阶段对回忆信息进行四维判断（相关性、时效性、敏感性、冗余性），决定是否向用户展示。

### 算法流程（文字说明）
1. **离线构建**：利用LLM（GLM-5.1）从用户对话历史构建可导航记忆存储（维度、页面、段落、链接）。
2. **在线推理**：每轮对话中，代理在ReAct循环内运行：
   - 收到用户输入后，首先判断是否触发主动记忆导航（根据协议）。
   - 若触发，通过导航接口逐步探索：思考当前状态 → 选择动作（browse/list/read/follow） → 观察返回的结构化结果 → 继续推理，直到信息足够或达到最大步数（默认6步）。
   - 最后整合记忆信息生成自然回复。

## 7. 优点

- **范式创新**：首次提出从Memory-as-Tool到Memory-as-Cognition的转变，将记忆访问融入推理过程，从根本上克服被动调用和检索-推理分离。
- **多层次结构化存储**：维度-页面-段落层次+四种跨维度关联链接，为多步推理提供了丰富的导航线索。

- **新基准构建**：ProactiveMemBench填补了主动记忆触发评估的空白，采用严格的六步管道和人类验证，确保质量。

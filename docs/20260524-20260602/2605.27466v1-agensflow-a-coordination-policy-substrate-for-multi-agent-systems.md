## 摘要
基于大语言模型的多智能体系统需要许多难以先验固定的协调选择：调用哪个技能协议、哪个智能体角色应执行子任务、将哪个模型绑定到每个角色、角色之间应如何交互、何时使用检索或验证，以及何时完全跳过某个步骤。这些选择与任务机制和操作约束相互作用，因此静态流水线和一次性模型比较仅能提供设计空间的有限视角。本文介绍了AgensFlow，这是一个开源框架，它将多智能体协调视为部分可观测条件下的在线策略学习问题。该框架使协调决策可观测且可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定的流水线设计。AgensFlow在两个语料库上进行了评估：分布式系统事件任务和安全咨询任务。评估显示了三个主要结果：在协调密集的类别上，学习路由比固定流水线基线达到了更高质量的操作点；skip:X将拓扑压缩隔离为基础架构的一个有意义的部分；热启动策略图可以在保持平台质量的同时降低探索成本。总体而言，结果表明，可学习、可审计的路由可以改善静态布线下的协调密集型多智能体工作流程。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：基于大语言模型（LLM）的多智能体系统面临大量交互的协调选择：调用哪个技能协议、分配哪个角色、绑定哪个模型、采用何种拓扑、何时使用检索或验证、何时跳过步骤等。这些选择随任务类型和运行约束而变化，静态流水线或一次性模型比较无法有效探索设计空间，导致性能上限受限。
- **意义**：作者将协调决策从固定管道提升为可学习、可审计的在线策略问题，使系统能够针对不同任务特征动态调整路由，从而在协调密集型任务上取得显著质量提升，同时保持操作成本可控。

## 2. 方法论

- **核心思想**：将多智能体协调建模为部分可观测的序列决策过程，通过在线策略学习（reliability-aware UCB1）从重复轨迹中学习路由策略。学习对象是一个可审计的策略图（policy graph），键为折叠签名（folded signature），值为每个（签名，动作）对的统计量（访问次数、平均奖励、方差、失败率）。
- **关键技术细节**：
  - **状态抽象（折叠签名）**：`s_t = ϕ(x_t, h_t, b_t) = (ℓ_t, z_t, β(ĉ_t), β(û_t), β(ř_t), β(ê_t))`，其中 ℓ_t 是规则生成的机制标签（如 straightforward, evidence_heavy 等），z_t 是二进制手卫生掩码（7种信息是否填充），β 是分桶的信念估计（正确性、不确定性、矛盾风险、证据充分性）。此抽象共享不同轨迹间价值估计。
  - **动作空间**：`a_t ∈ A(s_t) = {invoke(k, m)} ∪ {skip:X} ∪ {terminate}`。其中 invoke(k,m) 表示调用某个技能协议绑定到某个模型；skip:X 表示跳过计划中尚未执行的单元；terminate 通常由评估器触发。
  - **奖励函数**：`r(t) = w_q Q(t) - w_c C(t) - w_ρ ρ(t)`，其中 Q(t) 为相对质量评分（RelativeJudge），C(t) 为标准化 token 成本，ρ(t) 为重试/失败惩罚。
  - **动作选择**：可靠性感知 UCB1 变体：`score(s,a) = mean_reward + c_s √(log(N_s+1)/N_{s,a}) - λ f(s,a)`，其中 c_s 随访问次数衰减，f(s,a) 为失败率。未访问动作被赋予无限分数以强制探索。
- **算法流程**：
  1. 预飞行检查（Layer 5）。
  2. 计算任务特征，检测机制，折叠签名（Layer 1）。
  3. 每步路由：使用 UCB1 从合法动作集中选择下一个动作（调用或跳过），执行动作，更新手卫生和信念，重计算签名。
  4. 重复直到终止（评估器标记完成、预算耗尽、无合法动作或治理停止）。
  5. 轨迹完成后，RelativeJudge 进行同类相对评分，混合奖励回填到策略图中。

## 3. 实验设计

- **数据集/场景**：
  - **语料库1**：60个分布式系统事件任务（覆盖 Paxos/Raft、逻辑时钟、gossip、CRDT、一致性哈希等）。
  - **语料库2**：60个合成安全咨询任务（覆盖8个场景类别：C1程序性、C2单文档、C3跨文档多厂商、C4合成、C5语料外模糊、C6程序衍生、C7缓解正确性、C8跨厂商对）。
- **基准（Baseline）**：固定7单元流水线（一次通过，无学习）。
- **对比方法（arms）**：
  - **baseline**：固定流水线，无学习。
  - **no-skip ablation**：8 epoch，变体池学习，强制关闭 skip:X。
  - **main**：8 epoch，skip:X 启用，冷启动。
  - **warm-start**：8 epoch，skip:X 启用，用语料库1学到的443节点策略图初始化。
- **评估指标**：混合奖励中的 RelativeJudge 评分（单法官 Claude Haiku 4.5 用于实时信号），最终轨迹使用三法官跨家族审计（Claude Haiku 4.5 + GPT-5.4 mini + Qwen3.6-flash），报告 RJ(sj) 和 RJ(3j)，以及 token 消耗。

## 6. 主要结论与发现

- **学习路由优于固定基线**：在协调密集型类别上，主实验臂相对基线提升显著（C3 +0.18，C8 +0.16，C7 +0.13），整体 RJ(3j) 从0.749提升至0.817。
- **skip:X 的价值**：无跳过臂 token 增加94%而质量未明显提升；skip 使系统能够根据任务特征压缩拓扑。
- **热启动降低探索成本**：热启动相比冷启动在平台期 token 消耗减少约10%，且平台质量保持（3法官审计 +0.012 RJ），在早期 epoch 尤其有利。
- **奖励信号需要审计**：单法官下热启动显示+0.052提升，但三法官审计后仅+0.012，表明奖励信号存在家族偏差，审计是必要组成部分。
- **跨域转移可行**：在话题不同但特征分类共享的语料库间，策略图可提供有用的先验。

## 7. 优点

- **创新性**：首次将多智能体协调正式化为在线策略学习问题，提出可审计的策略图，使技能、模型、拓扑、跳过等决策可学习、可检查。
- **设计优雅**：折叠签名抽象平衡了泛化与特异性；skip:X 将拓扑压缩引入动作空间；可靠性感知 UCB1 自然融合探索-利用与失败惩罚。
- **实验严谨**：跨域验证、消融、热转移、跨法官审计多方面验证；公开代码可复现。
- **可操作性**：框架开源，支持六层结构化设计，适合实际部署；审计性使运营者可追踪路由决策。

## 8. 不足与局限

- **拓扑覆盖有限**：仅测试线性带跳过的拓扑，未试验并行、分支、群组、层次化等更丰富的结构。
- **跨域转移限制**：只在共享特征分类的语料库间测试；若目标域特征差异大，转移效果可能下降。
- **奖励信号偏差**：实时奖励使用单一法官，存在家族偏差；尽管审计弥补，但在线学习阶段未使用多法官。
- **基准不强**：固定7单元流水线可能简单，未与更复杂的静态手工协调策略比较。
- **实验规模偏小**：每个语料库仅60任务，8 epoch，统计稳定性可能有限。
- **性能泛化性**：仅在分布式系统事件和安全咨询两个领域验证，其他领域需进一步测试。
- **无真实成本建模**：token 成本基于简单线性归一化，未考虑实际 API 定价差异或延迟。

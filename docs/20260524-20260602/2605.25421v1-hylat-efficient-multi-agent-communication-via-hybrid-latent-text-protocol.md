## 摘要
通信协议设计是基于大语言模型的多智能体系统中的核心挑战。现有的单通道方法面临固有的通信三难困境：基于文本的方法可解释但冗长，而基于潜在空间的方法高效但不透明且仅限于单向工作流。受多通道通信理论的启发，我们提出HyLaT，一种混合潜在-文本通信协议，通过潜在通道传输精细的认知信号以提高效率，同时用自然语言表达简洁的关键信号以保持可解释性和精确性。我们引入了一个两阶段训练框架，结合单智能体混合生成学习和多智能体交互协同训练，使智能体能够在多轮交互中生成和解释混合消息。实验表明，HyLaT显著降低了通信开销，同时保持了具有竞争力的任务性能，并在不同设置中展现出强大的泛化能力和鲁棒性。

## Abstract
Communication protocol design is a central challenge in large language model-based multi-agent systems. Existing single-channel approaches face an inherent communication trilemma: text-based methods are interpretable but verbose, while latent-space methods are efficient but opaque and limited to unidirectional workflows. Inspired by multi-channel communication theory, we propose HyLaT, a hybrid latent-text communication protocol that transmits elaborate cognitive signals through a latent channel for efficiency, while expressing concise critical signals in natural language to preserve interpretability and precision. We introduce a two-stage training framework combining single-agent hybrid generation learning and multi-agent interactive co-training, enabling agents to generate and interpret hybrid messages across multiple rounds of interaction. Experiments demonstrate that HyLaT reduces communication overhead significantly while maintaining competitive task performance, with strong generalization and robustness across diverse settings.

## 1. 核心问题与整体含义（研究动机和背景）

- **通信三难困境**：基于LLM的多智能体系统中，现有单通道通信协议面临可解释性、效率与透明性三者不可兼得的结构性矛盾。文本方法（如自然语言）可解释但冗长（token开销大）；潜在空间方法（如共享隐状态）高效但完全不透明，且仅支持单向、单轮工作流，无法支持多轮多智能体交互。
- **灵感来源**：多通道通信理论（Shannon, 1948; Monge & Contractor, 2003）指出不同信息类型应通过不同信道传输。智能体消息包含两类信息：① 详细的认知信号（解释、推理过程、上下文阐述）适合紧凑的潜在表征；② 简洁的关键信号（答案、承诺、最终决定）必须保持可解释。
- **目标**：设计一种混合协议，同时优化效率、可移植性和多功能性，突破三难困境。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：提出 **HyLaT（Hybrid Latent-Text Protocol）**，采用双通道通信：
  - **潜在通道**：传输密集的中间认知信号（如推理、解释），用 \(k\) 个连续向量（\(z_1,\dots,z_k\)）表示，由特殊标记 `<bot>` 和 `<eot>` 分隔。
  - **文本通道**：传输简洁的关键信号（如最终答案、结论），用自然语言序列 \(y_1,\dots,y_m\) 表示。

- **双通道响应格式**：
  \[
  o_i^{(t)} = [\langle\text{bot}\rangle\; z_1,\dots,z_k\; \langle\text{eot}\rangle] \;\|\; [y_1,\dots,y_m]
  \]
  潜在部分由模型最后一层隐藏状态自回归生成，文本部分在 `<eot>` 后生成。

- **信道专门化**：通过训练数据自然诱导——训练数据包含详细解释（对应潜在信号）和简短答案（对应文本信号），无需显式路由标注。

- **两阶段训练框架**：
  - **Stage 1：单智能体混合生成学习**
    - 从文本输入 \(x\) 学习生成混合输出（潜在 + 文本）。
    - 两次前向传播：
      1. **混合前向**：自回归生成潜在向量 \(Z\)，然后拼接目标文本 \(y\)，计算语言建模损失 \(\mathcal{L}_{\text{hybrid}}\)。
      2. **文本前向**：以教师强制方式生成完整响应 \(r=[e, y]\)（\(e\) 为详细解释），计算 \(\mathcal{L}_{\text{text}}\)。
    - **跨通道对齐损失**：在目标位置（如“is”在“My answer is A”中）对齐两个分支的隐藏状态（所有层）：
      \[
      \mathcal{L}_{\text{align}} = \frac{1}{L}\sum_{l=1}^L \|\text{sg}(h_T^l) - h_S^l\|_2
      \]
      （\(h_T\) 为文本分支隐藏状态，\(h_S\) 为混合分支隐藏状态，sg 为停止梯度）。
    - 总损失：\(\mathcal{L}_{\text{stage1}} = \alpha \mathcal{L}_{\text{hybrid}} + \beta \mathcal{L}_{\text{text}} + \gamma \mathcal{L}_{\text{align}}\)。

  - **Stage 2：多智能体交互协同训练**
    - 在包含多轮、多智能体对话的监督数据上训练，使智能体学会解读其他智能体的潜在向量并生成混合响应。
    - 输入构造：每个智能体 \(i\) 在第 \(t\) 轮累积自身上下文和其他智能体上一轮的响应。
    - 损失：对所有 \(N\) 个智能体的 Stage 1 损失取平均。

- **关键超参数**：默认 \(k=6\)（潜在向量数），\(\alpha=1,\beta=1,\gamma=20\)。

## 3. 实验设计

- **训练数据构造**：
  - **Stage 1**：从五个数据集采样带详细解释的QA对：CommonsenseQA、SocialIQA、WorldTree、PubMedQA、StrategyQA。
  - **Stage 2**：两种多智能体交互数据：
    1. **细炼（Refinement）**：用 Llama-3.2-1B 和 GPT-5 模拟多轮辩论，只保留第二轮答案正确的样本（共 3,960 条）。
    2. **分解（Decomposition）**：从 HotpotQA、2WikiMultihopQA、GSM8K-Aug-NL 采样多跳问题，分配子问题给不同智能体协作求解（各 2,000 条）。

- **评估任务**：多智能体辩论（MAD）任务，使用：
  - **域内**：CommonsenseQA、StrategyQA、SocialIQA、WorldTree、PubMedQA。
  - **域外**：MedQA、ARC-Easy、ARC-Challenge。
  - 每数据集取前300个问题，使用3个智能体、2轮通信。

- **评估指标**：
  - **性能**：每智能体平均准确率（Avg.）和多数投票准确率（Maj.）。
  - **效率**：平均每问生成的 tokens 数和运行时间。

- **对比方法**：
  - **无训练基线**：Single-Agent（无通信）。
  - **文本方法**：NL（自然语言）、AutoForm（结构化语言）、EcoLang（演化高效规则）、SDE（状态增量轨迹）。
  - **潜在空间方法**：Cipher（语义嵌入）、LatentMAS-V（KV缓存传递）。
  - **基于训练的方法**：TextFullT（纯文本通信，同等训练）、LatentFullT（纯潜在通信，仅最后轮输出文本）、HyLaT（本文）。

## 6. 主要结论与发现

1. **无训练方法无法兼顾效率与性能**：文本方法性能好但 token 开销大，潜在方法 (LatentMAS-V) 效率高但性能大幅下降（如 SocialIQA 仅 42.78% vs HyLaT 82.56%）。
2. **HyLaT 实现了效率和性能的有效平衡**：相比最有效的文本方法（EcoLang）和最有效的潜在方法（LatentFullT），token 效率分别提升 **10.6×** 和 **3.6×**，时间效率分别提升 **6.8×** 和 **2.5×**，同时保持接近或超越训练基线（TextFullT）的准确率。
3. **双通道设计互补**：纯潜在变体准确率下降且格式错误率高（6.12%），纯文本变体 token 数剧增，表明两通道缺一不可。
4. **两阶段训练不可或缺**：缺失 Stage 1 导致潜在向量不稳定；缺失 Stage 2 导致智能体无法解读其他智能体的潜在输出（格式错误率 22.27%）。
5. **兼容性与鲁棒性**：HyLaT 可与 NL 智能体混合工作，且对潜在扰动（\(\sigma\leq0.5\)）保持稳定，关键安全信息由文本通道保护。
6. **可扩展性**：效率优势随智能体数和轮数增加愈发显著；通过扩展训练数据可缓解性能下降（HyLaT-scaled）。
7. **社会模拟**：HyLaT 以更低通信成本达到与 TextFullT 相当的社会信任模拟效果。
8. **潜在编码有意义内容**：PCA 显示潜在嵌入在不同数据集上形成分离簇，表明其编码了语义信息。

## 7. 优点

- **创新性**：首次提出混合双通道通信协议解决多智能体通信三难困境，理论动机清晰。
- **方法体系完整**：两阶段训练框架自洽地解决了混合输出生成和跨智能体潜在解读问题，无需额外监督信号。
- **实验验证全面**：涵盖性能、效率、消融、鲁棒性、可扩展性、异构兼容性、社会模拟等多个维度，且在不同模型家族上验证。
- **可解释性改进**：相比纯潜在方法，HyLaT 保留文本通道使部分消息可读，并提供了注意力分析和 PCA 可视化。
- **实用性**：与现有文本智能体兼容（异构通信），且潜在扰动鲁棒性好，部署风险较低。

## 8. 不足与局限

- **模型规模局限**：所有实验仅在 1B~3B 小模型上完成，未验证 HyLaT 是否能扩展到 7B+ 或更大模型，这种优势可能随模型变大而减弱。
- **部署规模限制**：修改后的生成过程与当前主流推理框架（如 vLLM）不兼容，无法在真正大规模、长周期场景中评估。
- **文本通道多样性不足**：训练数据主要为 QA 辩论类型，文本通道的表达范围受限；固定答案模板可能限制更丰富的沟通行为。
- **潜在通道不可解释**：虽然 HyLaT 比纯潜在方法更透明，但潜在向量内容仍难以解码，如何平衡“可解码性与通信性能”是开放问题。
- **训练数据规模有限**：Stage 2 仅使用数千条多轮交互数据，且训练配置（2智能体2轮）与实际评估（3智能体2轮/更大）存在差距，导致可扩展时性能退化（可通过扩展数据缓解，但未系统验证）。
- **未考虑异步或动态拓扑**：方法假设所有智能体同步轮次通信，更复杂的通信模式（如异步、稀疏连接）未探索。

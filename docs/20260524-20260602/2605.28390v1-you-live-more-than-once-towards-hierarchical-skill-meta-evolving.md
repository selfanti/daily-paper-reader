## 摘要
测试时技能进化被视为增强已部署智能体系统的新范式。现有工作主要关注硬编码的技能进化策略或依赖底层大语言模型昂贵参数更新的参数化学习。在本文中，我们证明，在不同下游场景下，对技能进化框架本身进行测试时改进对于智能体系统的持续提升是必要的，并且轻量级算法适配是可行的。具体而言，我们提出HiSME，一种轻量级分层技能元进化解决方案，通过从智能体的任务执行轨迹中学习元技能，联合优化技能和技能进化策略。在多个多样化智能体基准上的实验表明，元进化能够产生比纯技能进化更高质量的技能库，并能为不同场景推导出多样化的元技能，从而促进未来的持续经验学习。我们的代码临时公开在https://anonymous.4open.science/r/HiSME-BD45。

## Abstract
Test-time skill evolving is regarded as a new paradigm for enhancing deployed agentic systems. Existing works mainly focus on hard-coded skill evolving strategies or parametric learning that rely on expensive parameter updates in the underlying LLMs. In this paper, we demonstrate that test-time refinement of the skill evolving framework itself is necessary for continuous improvement of the agent systems in different downstream scenarios, and lightweight algorithmic adaptation is feasible. Specifically, we propose HiSME, a lightweight hierarchical skill meta-evolving solution that jointly optimizes skills and the skill evolving strategy by learning meta-skills from agents' task execution traces. Experiments on diverse agentic benchmarks show that meta-evolving can produce a higher-quality skill library than pure skill evolving and can derive diverse meta-skills for different scenarios, thereby facilitating future continual experience learning. Our code is temporarily public at https://anonymous.4open.science/r/HiSME-BD45.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：技能进化（skill evolving）作为一种测试时学习范式，可在部署后从智能体的执行轨迹中蒸馏出可复用技能，以轻量、模块化方式改进智能体行为，近似于模型参数更新。
- **核心问题**：现有技能进化策略本身是**静态的**——它不积累证据来改进自身的提取、维护或精炼过程。这导致两个实际限制：(1) 当技能生成算法有系统性缺陷时，智能体系统会反复修复相同缺陷模式；(2) 固定进化策略难以适配不同目标领域（如数据处理任务要求通用可调用函数，安全关键任务要求严格检查清单）。
- **研究动机**：能否让技能进化框架自身也在测试时自动优化？即 **技能元进化（skill meta-evolving）**。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将技能进化建模为**多级残差优化问题**。技能表示轨迹效用的一阶信息，元技能表示轨迹效用的二阶信息。通过从任务执行轨迹中学习元技能（meta-skills），联合优化技能库和技能维护策略本身。
- **关键技术细节**：
  - **技能生成**：采用两种策略——提取器（Extractor）从单条轨迹中识别可复用片段；重构器（Refactorer）通过重叠图（overlap graph）从多条相似轨迹或技能中提取共享结构。技能类型包括可调用函数、工作流框架和领域知识。
  - **技能评估与维护**：
    - 短期效用：为每个技能维护测试包（bundle），包括单元测试和集成测试，验证技能是否忠实履行其契约。
    - 长期效用：信用分配器（credit assigner）利用真实执行轨迹评估每个技能是有帮助、有害还是中性。
    - 过滤器：基于信用阈值禁用有害且缺乏正面证据的技能。
    - 精炼器（Refiner）：根据证据状态修订技能的范围、接口或实现，并通过测试后发布。
  - **元技能生成**：
    - 为每个LLM角色（提取、重构、精炼）维护一个角色特定回放缓冲区，记录技能及其后续效用。
    - 定期从缓冲区采样，调用元进化智能体（META_EVO）总结出最多5条经验规则作为元技能，并注入到对应角色的上下文中。
- **算法流程**：通过不断执行任务、更新技能库、更新元技能，形成一个层次化的持续改进循环。所有更新在文本空间进行，不修改底层LLM参数。

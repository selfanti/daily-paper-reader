## 摘要
长时间跨度语言智能体积累的观察、推理轨迹和检索事实会超出其有限的上下文窗口，使得记忆保持成为一个基本的资源分配问题。现有记忆系统通过启发式评分、检索优化或学习压缩来改进管理，但大多将保持视为局部决策问题，并未在现实的可观测性约束下显式建模其长期后果。为填补这一空白，我们将记忆保持建模为一个具有显式预算可行性、证据效用以及包括缺失惩罚、重新获取延迟和过时信息风险在内的延迟成本的约束随机优化问题。随后，我们提出OSL-MR（面向记忆保持的可观测性安全学习），一个新框架，它强制执行在线可观测特征与离线可用监督之间的严格分离。OSL-MR结合了一个从已实现的证据监督中训练的证据学习器，以及一个混合评分启发式，该启发式既作为可部署的在线安全基线，又作为学习的结构化归纳先验。由此产生的策略直接从交互数据中学习查询条件的证据价值，同时在同一可观测性约束下保持可部署性。在LOCOMO和LongMemEval上的实验表明，OSL-MR在记忆预算紧张的情况下一致优于基于近期性的方法、生成式智能体风格评分及其他启发式基线。混合评分先验在保持召回率的同时进一步提高了精确度，敏感性分析表明其在广泛的成本配置下具有鲁棒性。

## Abstract
Long-horizon language agents accumulate observations, reasoning traces, and retrieved facts that exceed their finite context windows, making memory retention a fundamental resource-allocation problem. Existing memory systems improve management through heuristic scoring, retrieval optimization, or learned compression, but largely treat retention as a local decision problem and do not explicitly model its long-term consequences under realistic observability constraints. To fill this gap, we formulate memory retention as a constrained stochastic optimization problem with explicit budget feasibility, evidence utility, and delayed costs including miss penalties, reacquisition delays, and stale-information risk. We then propose OSL-MR (Observability-Safe Learning for Memory Retention), a novel framework that enforces a strict separation between online-observable features and offline-available supervision (OAS). OSL-MR combines an evidence learner trained from realized evidence supervision with a Mixed-Score heuristic that serves both as a deployable online-safe baseline and as a structured inductive prior for learning. The resulting policy learns query-conditioned evidence value directly from interaction data while remaining deployable under the same observability constraints. Experiments on LOCOMO and LongMemEval show that OSL-MR consistently outperforms recency-based methods, Generative Agents-style scoring, and other heuristic baselines, particularly under tight memory budgets. The Mixed-Score prior further improves precision while preserving recall, and sensitivity analysis demonstrates robustness across a wide range of cost configurations.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将记忆保留视为**多步顺序决策问题**，而非单步优化。
- 严格区分**在线可观测特征**（查询上下文、记忆元数据、交互历史等，决策时可用）和**离线可用监督OAS**（黄金证据、答案文本、覆盖信号等，仅训练/评估时可用）。部署策略只能依赖在线特征。
- 通过**混合评分（Mixed-Score）启发式**作为冷启动基线，并作为**归纳先验**提供给证据学习器；证据学习器从交互日志中学习查询条件下的证据价值，最终形成可部署的冻结策略。

### 关键技术细节

#### 3.1 可观测性分离
- **在线可观测输入**：当前查询、记忆元数据、交互上下文。
- **离线可用监督（OAS）**：黄金证据集、答案内容、覆盖信号、下游结果统计。
- **评估信号**：从系统实际结果导出，仅用于分析。

#### 3.2 约束记忆保留问题形式化
- 每步 $t$，记忆池 $M_t$，选择子集 $A_t \subseteq M_t$，满足存储预算 $\sum x_{t,i}s_{t,i} \le B_t$。
- 定义覆盖率、token加权命中/缺失/重新获取/过时指标。
- 每步奖励：
  \[
  r_t = \alpha_{\text{hit}} T^{\text{hit}}_t + \alpha_{\text{full}} \mathbb{I}_{\text{full}} - \alpha_{\text{store}} \sum x_{t,i}s_{t,i} - \alpha_{\text{miss}} T^{\text{miss}}_t - \alpha_{\text{reacq}} T^{\text{reacq}}_t - \alpha_{\text{stale}} T^{\text{stale}}_t
  \]
- 目标：最大化期望累积奖励，受预算约束。
- 该问题是组合优化、部分可观察、NP-hard，无法解析求解。

#### 3.3 混合评分（Mixed-Score）策略
- 结合时间特征、相关性、上下文、风险特征和大小惩罚项，计算 $M S_{t,i}$，作为每段记忆的效用得分。
- 同时加入连续新鲜度代理（基于记忆类型和衰减系数）。
- 用于冷启动部署，也作为学习器的特征先验。

#### 3.4 证据学习器（Evidence Learner）
- 目标：学习证据成员关系 $y^{\text{evid}}_{t,i} = 1[m_{t,i} \in E_t]$。
- 特征向量 $\phi_{t,i} = [ \psi^{\text{on}}_{t,i}; MS_{t,i} ]$。
- 使用加权二元交叉熵损失训练。
- 推理时，通过约束解码选择子集 $A_t$。

#### 3.5 行为克隆变体（BC-learner）
- 使用非可部署的教师策略（基于OAS的局部贪心搜索）生成伪标签，训练相同架构的学习器，用于对比。

### 算法流程（文字说明）
1. 初始化空数据集 $D$。
2. 对每个训练场景：
   - 初始化状态。
   - 每步：观察记忆池，为每条记忆计算在线特征 $\psi^{\text{on}}$ 和混合评分 $MS$，组合成特征向量 $\phi$，获取黄金证据标签 $y^{\text{evid}}$（OAS）。
   - 将 $(S, \phi, y^{\text{evid}})$ 加入 $D$。
   - 使用混合评分进行选择（冷启动），执行动作，进入下一状态。
3. 收集足够数据后，训练证据学习器 $f_\theta$ 最小化 BCE 损失。
4. 部署时，使用冻结的 $f_\theta$，仅依赖在线特征进行选择。

## 8. 不足与局限

- **混合评分的手动设计**：特征分组和权重是手工设定的，缺少自动学习机制。未来工作可尝试自动学习这些权重。
- **新鲜度估计粗糙**：仅基于记忆类型固定衰减系数，未考虑语义有效性变化。论文也提到需要更原则性的新鲜度估计。
- **未与记忆写入端或检索端联合优化**：框架专注于保留阶段，而记忆写入（如Mem0）和检索（如MemRL）是互补的，未纳入统一优化。
- **实验规模相对有限**：仅在两个对话基准上测试，未涉及更复杂场景（如具身agent、工具增强agent）。数据集规模可能较小，泛化性待验证。
- **计算资源细节未报告**：缺乏可重复性所需的具体训练硬件信息。
- **延迟反馈建模需进一步验证**：虽然形式化了延迟成本，但实验中的奖励计算可能依赖于即时标签（黄金证据），长期延迟成本（如多次交互后才发现记忆错误）未充分体现。
- **未考虑记忆的重写/更新**：系统假设记忆是静态的，而实际中记忆可能被修改或合并。

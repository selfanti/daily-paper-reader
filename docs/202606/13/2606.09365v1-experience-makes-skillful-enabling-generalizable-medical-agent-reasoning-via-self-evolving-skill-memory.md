## 摘要
医疗智能体系统越来越被期望支持交互式临床决策，而不仅仅是静态问答。在此类场景中，有效的智能体必须能在不断变化的病例中复用先前经验，但现有的记忆机制往往保留原始的、冗余且嘈杂的历史轨迹，难以管理。更重要的是，它们很少区分哪些记忆对未来推理真正有用。这限制了它们为长期临床推理积累紧凑可靠经验的能力。为弥补这一不足，我们提出SkeMex，一种部署后自演进框架，通过基于技能的记忆改进医疗智能体，无需更新模型权重。SkeMex将信息丰富的交互轨迹提炼为结构化技能，编码可复用的程序性知识，并将其组织成一个涵盖通用、任务特定和动作级别经验的多分支存储库。为确定哪些记忆应被重用和保留，SkeMex从环境反馈中估计上下文相关的效用，并用其指导价值感知的检索和存储库治理。闭环的“读-写-评估-治理”生命周期通过写入新技能、更新效用、促进有用记忆和移除有害条目来支持持续演进。在多种临床任务上的实验表明，SkeMex在离线和在线设置下均持续优于代表性的基于记忆的智能体。它还能跨模型骨干泛化并支持可迁移的技能记忆。所有数据和代码将公开发布。

## Abstract
Medical agent systems are increasingly expected to support interactive clinical decision making rather than only static question answering. In such settings, effective agents must reuse prior experience across evolving cases, yet existing memory mechanisms often retain raw historical traces that are redundant, noisy, and difficult to govern. More importantly, they rarely distinguish which memories are truly useful for future reasoning. This limits their ability to accumulate compact and reliable experience for long-horizon clinical reasoning. To close this gap, we propose SkeMex, a post-deployment self-evolution framework that improves medical agents through a skill-based memory without updating model weights. SkeMex distills informative interaction trajectories into structured skills that encode reusable procedural knowledge, and organizes them into a multi-branch repository spanning general, task-specific, and action-level experience. To determine which memories should be reused and retained, SkeMex estimates context-dependent utility from environment feedback and uses it to guide value-aware retrieval and repository governance. A closed-loop ``Read--Write--Assess--Govern" lifecycle further supports continual evolution by writing new skills, updating utilities, promoting useful memories, and removing harmful entries. Experiments across diverse clinical tasks show that SkeMex consistently outperforms representative memory-based agents in both offline and online settings. It also generalizes across model backbones and supports transferable skill memory. All data and code will be released publicly.

- **核心思想**：SkeMex 将智能体的交互轨迹（Trajectories）蒸馏（Distill）成结构化、可复用的“技能”（Skills），存储于一个外部记忆库中。通过一个“读-写-评估-治理”的闭环生命周期，实现记忆库的自我迭代和智能体的持续改进。

- **关键技术细节**：
    1.  **基于技能的记忆库（Skill-based Memory Bank）**：
        - 记忆单元不再是原始轨迹，而是结构化技能 `mi = (ki, ci, ui)`，其中 `ki` 是检索键，`ci` 是可复用内容，`ui` 是效用值。

            - `M_task`：任务级别模式（如“鉴别诊断的流程”）。
            - `M_act`：工具使用操作知识（如“正确调用搜索工具的格式”）。

    2.  **价值感知检索 (Value-aware Retrieval)**：
        - 任务开始时进行一次性检索，提供稳定的技能上下文。
        - 检索评分融合三个信号：
            `Score_ret(m | s0, κ0) = λ_sim * Sim(s0, m) + λ_u * U(m | κ0) + λ_h * h0(m)`
            - `Sim(s0, m)`：语义相似度。
            - `U(m | κ0)`：在特定临床类别 `κ0` 下的历史效用。
            - `h0(m)`：随时间衰减的记忆强度（基于艾宾浩斯遗忘曲线）。
        - 采用分支感知的 Top-K 选择。

    3.  **轨迹到技能的蒸馏 (Trajectory-to-Skill Distillation)**：

3.  **技能记忆的有效性**：消融研究证实了该方法所有核心组件的必要性。特别是**价值感知检索**、**效用-驱动的估值**、**多分支结构**和**完整的闭环生命周期治理**是取得优异性能的关键。

#### 7. 优点（方法或实验设计）

- **方法创新性**：
    - **后部署自进化**：无需更新模型参数，规避了灾难性遗忘和高计算成本，是可部署和可扩展的关键。
    - **技能作为记忆单元**：比原始轨迹更紧凑、可复用、安静。多分支结构则实现了不同层次经验的有效隔离。
    - **效用驱动的记忆管理**：引入了“效用”这一概念，并设计了优势估计和信用分配机制，使记忆管理从“存储”转向“价值驱动”，实现了自动化的高质量记忆筛选。
    - **闭环生命周期**：读-写-评估-治理的设计形成了一个完整的自我改进系统，逻辑清晰且完备。
- **实验设计严谨性**：
    - **域内外评估**：同时评估了域内性能和域外泛化能力，验证了框架的泛化性，而非简单的记忆过拟合。
    - **跨模型泛化**：这是证明方法通用性的关键实验，远不止于“在某个模型上好用”。
    - **详尽的消融和敏感性分析**：深入分析了每个组件和关键参数的贡献，科学地证实了设计的有效性。

#### 8. 不足与局限（实验覆盖、偏差风险、应用限制）

- **实验覆盖**：
    - **真实临床环境复杂性**：作者已明确承认，所用的基准测试无法完全模拟真实临床环境的复杂性，比如患者病史、医疗系统工作流、伦理约束等。
    - **计算开销**：虽然无需训练，但引入技能检索和更长的推理步骤，导致**推理时间**和**API成本**均有所增加（表16）。这在实际部署中是需要考虑的权衡。
    - **模型覆盖**：虽然测试了多种主流模型，但并非穷尽所有前沿模型（如GPT-5等），评估结果可能因模型而异。
- **潜在的偏差与风险**：
    - **奖励设计依赖**：SkeMex 的核心依赖于从环境反馈中估计效用。如果环境反馈（奖励）本身有偏差或质量不高，学到的技能也可能存在偏差。例如，某些“有害”或医源性错误的操作可能在某些简化环境中获得正面奖励。
    - **技能固化风险**：虽然加入了治理机制（废弃/合并），但一个长期占据高效用值的技能可能难以被更好的替代方案推翻，形成**路径依赖**。
    - **安全风险**：在高风险医疗场景中，由AI自主演进出的技能可能存在安全隐患。作者强调它应作为决策支持工具而非替代品。
- **应用限制**：
    - **初始化依赖**：初始的技能库是关键。论文中是基于训练集构建的。如果从零开始，初始阶段的学习可能比较缓慢。
    - **场景设定依赖**：该方法在处理需要多步交互的临床任务上表现优异，但在简单的、不需要工具使用的单轮问答任务上增益可能有限。

## 摘要
医疗代理系统日益被期望支持交互式临床决策，而不仅仅是静态问答。在这种场景下，有效的代理必须能够跨演化病例重用先前经验，但现有的记忆机制往往保留原始历史轨迹，这些轨迹冗余、有噪声且难以管理。更重要的是，它们很少区分哪些记忆对未来推理真正有用，这限制了它们为长期临床推理积累紧凑且可靠经验的能力。为弥补这一不足，我们提出SkeMex，一个部署后的自演化框架，通过基于技能的记忆改进医疗代理，而无需更新模型权重。SkeMex将信息丰富的交互轨迹提炼为结构化技能，编码可重用的过程性知识，并将它们组织成跨通用、任务特定和行动级别经验的多分支存储库。为确定哪些记忆应被重用和保留，SkeMex从环境反馈中估计依赖上下文的效用，并用其指导价值感知的检索和存储库治理。闭环的“读-写-评估-治理”生命周期通过写入新技能、更新效用、提升有用记忆和移除有害条目进一步支持持续演化。跨多种临床任务的实验表明，SkeMex在离线和在线场景中均持续优于有代表性的基于记忆的代理。它还能跨模型主干泛化，并支持可迁移的技能记忆。所有数据和代码将公开发布。

## Abstract
Medical agent systems are increasingly expected to support interactive clinical decision making rather than only static question answering. In such settings, effective agents must reuse prior experience across evolving cases, yet existing memory mechanisms often retain raw historical traces that are redundant, noisy, and difficult to govern. More importantly, they rarely distinguish which memories are truly useful for future reasoning. This limits their ability to accumulate compact and reliable experience for long-horizon clinical reasoning. To close this gap, we propose SkeMex, a post-deployment self-evolution framework that improves medical agents through a skill-based memory without updating model weights. SkeMex distills informative interaction trajectories into structured skills that encode reusable procedural knowledge, and organizes them into a multi-branch repository spanning general, task-specific, and action-level experience. To determine which memories should be reused and retained, SkeMex estimates context-dependent utility from environment feedback and uses it to guide value-aware retrieval and repository governance. A closed-loop ``Read--Write--Assess--Govern" lifecycle further supports continual evolution by writing new skills, updating utilities, promoting useful memories, and removing harmful entries. Experiments across diverse clinical tasks show that SkeMex consistently outperforms representative memory-based agents in both offline and online settings. It also generalizes across model backbones and supports transferable skill memory. All data and code will be released publicly.

### 论文提出的方法论：SkeMex框架

- **核心思想**：SkeMex 将代理的交互轨迹转化为结构化的、可重用的“技能”（Skills），并以此来驱动一个闭环的自我进化过程。整个框架被视为一个非参数强化学习过程，其中环境反馈提供“效用”（Utility）信号来指导记忆的检索和治理。
- **关键技术细节**：
    - **技能化记忆（Skill-based Memory）**：不再存储原始轨迹，而是将信息丰富的交互过程蒸馏为结构化的技能，作为记忆的基本单元。每个技能包含可重用的过程性知识（procedural knowledge）。
    - **多分支技能仓库（Multi-Branch Skill Repository）**：技能被组织成三个抽象级别的仓库，以进行精细化管理：
        - **通用分支（General Branch）**：存储跨任务的通用推理策略。
        - **任务级分支（Task-level Branch）**：存储特定任务流程的指导知识。
        - **行动级分支（Action-level Branch）**：存储特定工具使用的操作细节。
    - **价值感知检索（Value-Aware Retrieval）**：在推理时，系统不仅依据语义相似度，还结合从环境反馈中估计的“效用”值来选择技能。这确保了高价值、被验证有效的经验更有可能被重用。
    - **闭环“读-写-评估-治理”生命周期（Read-Write-Assess-Govern Lifecycle）**：
        1.  **Read**：根据当前任务检索高价值技能。
        2.  **Write**：将成功的、信息丰富的交互轨迹蒸馏成新的技能或更新现有技能。
        3.  **Assess**：利用环境奖励评估技能的效用，基于“优势”（Advantage）估计动态调整其价值。
        4.  **Govern**：定期对仓库进行维护，合并冗余技能、弃用低效用技能、移除有害或低质量的条目，确保仓库的紧凑性和可靠性。

- **广泛的实验验证**：实验场景多样，从标准问答到交互式临床决策，从文本到多模态，从同域到跨域，并且验证了跨模型泛化能力。实验丰富度和深度是其重要优势。

### 不足与局限

- **模拟环境与现实差距**：尽管使用了多个评估基准，但这些仍然是模拟环境。真实临床环境中的患者情况、医疗数据噪声、伦理约束、工作流复杂性等远未被现有数据集完全覆盖。从模拟到真实应用的“最后一公里”仍需验证。
- **骨干模型覆盖不全**：虽然测试了多个主流模型，但远未穷尽所有可用模型。实验结论的普适性可能依赖于所测试模型家族的某些共性。
- **额外推理开销**：技能检索和更深入的多步推理过程不可避免地增加了推理延迟和API调用成本。论文承认这一点，但对于对时间敏感的临床场景，这可能是需要权衡的因素。
- **对初始轨迹质量的依赖**：SkeMex 的成功依赖于从高质量、信息丰富的交互轨迹中蒸馏技能。如果初始的训练轨迹本身噪声很大或效果不佳，可能会影响初始技能库的质量。
- **潜在风险**：虽然提出“治理”机制来移除有害技能，但系统仍然可能学习到或保留一些有偏见的临床模式，如果在不够完善的模拟环境中长期进化，可能会放大某些不安全的决策路径。作者也谨慎地将此工具定位为“决策支持和研究工具”而非专业判断的替代品。

### 未来工作与开放问题

- **更接近真实临床场景的验证**：未来工作应将SkeMex部署在更真实的临床工作流模拟器或小规模真实病例环境中，评估其在面对患者隐私、电子病历噪声、多科室协作等复杂因素时的表现。此外，如何设计安全监护机制（如人工审核环）以防范技能仓库中潜在的有害模式，仍需深入研究。
- **知识蒸馏与轻量化**：当前技能仓库依赖于大模型（如DeepSeek-V3.2）的推理能力。未来可探索将仓库中的技能蒸馏到更小的开源模型上，实现本地化、低延迟的代理系统，便于在资源受限的临床环境中部署。
- **多模态技能表示**：目前的技能主要基于文本（工具调用、自然语言推理步骤）。对于医学图像分析、心电图解读等多模态任务，需要发展多模态技能编码方法，使得视觉特征也能被抽象并存储为可重用的过程性知识。
- **终身学习与任务遗忘权衡**：SkeMex的“治理”机制目前基于效用简单淘汰低价值技能。在长期进化中，如何设计更智能的知识重构机制（如合并相似技能、抽象出更高级的元技能）以避免知识碎片化，同时保留对罕见但重要病例的记忆，是一个开放问题。
- **跨中心、跨语种泛化**：论文中的医疗数据集主要基于英文和中文医疗内容。未来需测试技能仓库在不同语言、不同医疗体系（如欧美 vs 亚洲）下的迁移能力，确保其不受语言和文化偏见的影响。

### 个人评价与贡献总结

SkeMex 为医疗领域的大语言模型代理提供了一种新颖、务实且高效的持续学习范式。其核心贡献在于将“技能”（而非原始轨迹）作为记忆的基本单元，并引入基于效用信号的闭环治理机制，使代理在不更新模型参数的情况下实现自我进化。相比现有记忆增强方法，SkeMex在性能、鲁棒性和跨模型迁移能力上均有显著优势。尽管存在模拟环境与现实差距、额外推理开销等局限，但其设计思想——**将交互经验抽象为可评估、可重用的结构化知识**——对未来构建安全、可信任、持续进步的医疗AI系统具有重要启发意义。

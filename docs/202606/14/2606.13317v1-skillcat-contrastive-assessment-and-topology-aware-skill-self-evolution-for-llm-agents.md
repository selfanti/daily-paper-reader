## 摘要
面向LLM智能体的技能自我进化方法旨在将执行轨迹转化为可重用的技能文档，但现有流程通常每个任务仅学习一条轨迹、在检查之前合并候选技能补丁，并在推理前加载完整技能库。我们提出SkillCAT，一个无需训练的框架，将该流程分为三个阶段。对比因果提取（CCE）为每个任务采样多条轨迹，并对比同任务的成功/失败对，以识别解释结果差异的证据。评估增强进化（AAE）在源任务克隆上回放每个候选补丁，并在分层技能补丁合并前仅保留能改善或维持任务结果的补丁。拓扑感知任务执行（TTE）将进化后的技能编译为可路由的子技能拓扑，从而使推理仅加载与任务相关的能力节点。我们在常见智能体基准测试（包括SpreadsheetBench、WikiTableQuestions和DocVQA）上评估SkillCAT，并进一步测试跨模型和分布外泛化。在这些设置下，SkillCAT将基线平均得分提升了最多40.40%，证明了无需模型训练的可靠技能进化。

## Abstract
Skill self-evolution methods for LLM agents aim to turn execution trajectories into reusable skill documents, but current pipelines typically learn from one trajectory per task, merge candidate skill patches before checking them, and load the full skill corpus before inference. We propose SkillCAT, a training-free framework that separates this process into three stages. Contrastive Causal Extraction (CCE) samples multiple trajectories for each task and compares same-task success/failure pairs to identify evidence that explains outcome differences. Assessment-Augmented Evolution (AAE) replays each candidate patch on source-task clones and keeps only patches that improve or preserve task outcomes before hierarchical skill patch merging. Topology-Aware Task Execution (TTE) compiles the evolved skills into a routable sub-skill topology, so inference loads only the capability nodes relevant to the task. We evaluate SkillCAT on common agent benchmarks, including SpreadsheetBench, WikiTableQuestions, and DocVQA, and further test cross-model and out-of-distribution generalization. Across these settings, SkillCAT raises the average score over baselines by up to 40.40%, demonstrating reliable skill evolution without model training.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
将技能生命周期分离为三个可观测阶段，分别对应证据提取、补丁验证与集成、测试时技能部署，每个阶段采用针对性策略提高技能质量和推理效率。

### 关键技术细节

#### (1) 对比因果提取（Contrastive Causal Extraction, CCE）
- **目的**：克服单轨迹偏差，获得可靠因果证据。
- **做法**：
  - 对每个任务采样多条轨迹（多种子运行），形成同一任务的成功/失败对比对`(τ⁺, τ⁻)`。
  - 定位**因果分水岭**（causal watershed），即成功与失败轨迹首次出现动作分歧的时刻。
  - 围绕该点提取候选经验记录，包含局部证据、失败原因和可编辑的教训。
- **回退机制**：若某任务所有轨迹全成功或全失败，则使用单轨迹提取器`E₁`生成记录，或跳过该任务以保持严格对比证据。

#### (2) 评估增强进化（Assessment-Augmented Evolution, AAE）
- **目的**：在合并前验证每个候选补丁的有效性，避免噪声注入。
- **做法**：
  - 将每个候选补丁临时应用于原始技能，在源任务克隆上回放，记录基线结果`yⱼ`与回放结果`ŷⱼ`。
  - 基于结果转移分配评估分数`aⱼ`：
    - 失败→成功：3.0（最强证据）
    - 成功→成功：2.0（行为保持）
    - 失败→失败：1.0（未解决）
    - 成功→失败：0.0（应拒绝）
  - **阈值筛选**：仅保留`aⱼ ≥ 2.0`（θ=2.0）的补丁，剔除失败或有害补丁。
  - **分层分层合并**：按得分从低到高分批合并，高得分补丁在后期合并中占更高优先级，最终得到进化技能`S*`。

#### (3) 拓扑感知任务执行（Topology-Aware Task Execution, TTE）
- **目的**：减少推理时的上下文过载，仅加载任务相关技能内容。
- **做法**：
  - 离线将`S*`编译为可路由拓扑：一级标题构成核心技能`S°`，二级标题构成可路由节点集合`V`。
  - 每个节点保留正文，并提取标题、关键词、摘要、依赖关系作为路由元数据。
  - 在线推理时，LLM路由根据任务描述选择最多`k`个相关节点，再经确定性扩展（依赖边、基础节点），组装得到精简技能`S_c`。

### 算法流程（文字描述）
1. 对每个进化任务，使用多个随机种子运行智能体，收集轨迹和结果。
2. 若任务存在成功与失败轨迹则形成对比对，提取因果分水岭处的经验记录；否则根据回退机制处理。
3. 将所有记录转化为候选补丁，在源任务克隆上回放并计算评估分数。
4. 按阈值筛选，并按得分分层合并得到进化技能`S*`。
5. 离线编译`S*`为拓扑结构。在线推理时，根据任务描述选择相关节点，组装精简技能后执行。

## 7. 优点

- **系统性解决方案**：首次将证据提取、补丁验证、推理时上下文选择三个决策纳入统一框架，端到端提升技能质量。
- **无训练高效性**：不依赖模型微调，仅通过多轨迹采样和回放评估即可进化技能，易于扩展到新模型和任务。
- **对比因果提取的创新**：通过同任务成功/失败对比对因果分水岭定位，避免单轨迹偏差，提供更强的因果信号。
- **评估增强进化的严谨性**：基于源任务克隆回放的分数量化，确保每个补丁都通过本地验证后再合并，有效过滤有害补丁。
- **拓扑感知执行的实用性**：将技能结构化为可路由拓扑，实现动态上下文压缩，减少无关内容干扰，降低推理token成本。
- **实验设计全面**：覆盖同模型、跨模型、分布外、多模态和多种初始化条件，并通过消融和参数分析验证设计选择。

## 8. 不足与局限

- **计算成本**：虽然无需训练，但CCE需要为每个任务采样多条轨迹（5种子），AAE需要为每个候选补丁在源任务克隆上回放，总体推理次数较多，在任务规模很大时可能仍昂贵。
- **实验覆盖有限**：主要聚焦电子表格操作和文档问答，未涉及编程、游戏、对话等更广泛的智能体任务，技能自我进化的通用性需进一步验证。
- **跨模型泛化程度依赖作者模型**：35B-A3B作者的技能对gemma-4提升显著，但122B-A10B作者的技能对gpt-5.4-mini提升较少（+1.5% Vrf），表明进化技能的知识表达可能与作者模型特性相关，跨模型迁移存在天花板。
- **回退机制可能引入噪声**：当某任务所有轨迹全成功或全失败时，CCE使用单轨迹提取器，可能再次引入偏差，但论文未深入分析该情况下的表现。
- **缺少与更近期方法的比较**：如AutoSkill、EvoSkills等2026年方法仅被提及未做对比，可能受限于基线复现难度。
- **超参数敏感性**：CCE种子数、AAE阈值θ、TTE的top_k等超参数需手动设定，论文仅在某几个值上扫描，最优配置可能随任务变化。

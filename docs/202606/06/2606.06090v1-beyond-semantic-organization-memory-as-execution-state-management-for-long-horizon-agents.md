## 摘要
基于LLM的智能体日益应对具有相互依赖决策的长周期任务，其中每个动作都会重塑未来约束，且中间错误可能级联。现有的RAG和智能体记忆系统通过语义相似性组织历史，在决策时检索内容相关的条目。我们认为这种设计与执行状态依赖不匹配：它碎片化了决策轨迹，并混合了有效和错误的痕迹，阻碍了连贯的状态重建和错误隔离。我们提出MAGE（记忆作为智能体引导的探索），一种主动的执行状态管理器，将交互存储在分层状态树中。智能体从活跃的根到当前路径衍生其状态，结合子目标摘要、近期轨迹以及先前分支的提示。四个耦合操作维护该树：Grow记录新轨迹，Compress总结完成的子目标，Maintain验证摘要，Revise恢复目标边界并在新分支上继续。这种设计限制了上下文增长，同时保持了状态完整性，并将有缺陷的片段与活跃路径隔离。在MemoryArena上的实验表明，与基线相比，MAGE将平均任务成功率提高了7.8-20.4个百分点，同时将令牌消耗减少了55.1%。

## Abstract
LLM-based agents increasingly tackle long-horizon tasks with interdependent decisions, where each action reshapes future constraints and intermediate errors can cascade. Existing RAG and agent memory systems organize histories by semantic similarity, retrieving content-relevant entries at decision time. We argue that this design mismatches execution-state dependencies: it fragments decision trajectories and mixes valid and erroneous traces, hindering coherent state reconstruction and error isolation. We propose MAGE (Memory as Agent-Guided Exploration), an active execution-state manager that stores interactions in a hierarchical state tree. The agent derives its state from the active root-to-current path, combining subgoal summaries, recent traces, and hints from prior branches. Four coupled operations maintain the tree: Grow records new traces, Compress summarizes completed subgoals, Maintain validates summaries, and Revise restores a target boundary and resumes on a new branch. This design bounds context growth while preserving state integrity and isolating flawed segments from the active path. Experiments on MemoryArena show that MAGE improves the average task success rate by 7.8--20.4 pp over baselines, while reducing token consumption by 55.1%.

- **问题**：现有基于 LLM 的智能体在长周期任务（如购物、旅行规划、搜索、形式推理）中面临**相互依赖的决策**——每一步动作都会改变未来约束，而中间错误可能级联放大。传统 **RAG 和记忆系统** 通过**语义相似性**组织历史，在决策时检索内容相关的条目。  
- **核心矛盾**：这种“相似性驱动”的设计与 **执行状态依赖** 不匹配。它导致：  
  - **状态碎片化**：决策轨迹被割裂为语义片段，无法重建完整的执行状态（如遗漏前序购买记录）。  
  - **错误难以隔离**：有效与错误轨迹混合在同一个记忆池中，错误传播污染后续推理。  
- **含义**：需要将记忆从“被动的事实存档”转变为 **主动的执行状态管理器**，保留决策链上的依赖关系。

### 2. 论文提出的方法论

- **核心思想**：MAGE（Memory as Agent-Guided Exploration）将智能体历史组织为**两层层次状态树**，并通过四个耦合操作主动维护，确保活动路径上的状态完整且无污染。

- **关键技术细节**：
  - **层次状态树**：
    - **底层（Bottom Layer）**：记录每步原始动作-观察对（action-observation node）。
    - **顶层（Top Layer）**：存储已完成子目标的压缩摘要（subgoal summaries）。
  - **状态表示**：当前执行状态由三部分构成：
    - `C`：从根到当前位置的顶层摘要（压缩状态）。
    - `R`：最近未压缩的原始动作-观察序列（原始状态）。
    - `H`：来自兄弟分支的执行提示和历史诊断反馈。
  - **四个操作（Algorithm 1）**：
    - **Grow**：当智能体执行新动作时，在底层追加节点。若已有相同内容，则合并（避免重复）。
    - **Compress**：当子目标完成或原始状态超长时，将底层连续节点压缩为顶层摘要，释放上下文空间。
    - **Maintain**：每次压缩后，用 LLM 验证新摘要是否正确（检查遗漏、错误、依赖违背）。若失败，记录诊断并触发 Revise。
    - **Revise**：在检测到错误后，回溯到目标边界（通常是子目标起始），将失败摘要从活动路径移除，并在新分支继续探索。错误片段被隔离到非活动分支。
  - **流程**：Grow → Compress → Maintain →（若失败）Revise → 继续 Grow（新分支）。

### 6. 论文的主要结论与发现

- MAGE 在 MemoryArena 全部领域平均提升 **7.8–20.4 个百分点（SR）**，同时减少 **55.1% 的 token 消耗**。
- 相比语义检索系统，**路径式状态维护**更有效地支撑长周期任务：保持状态完整性、隔离错误分支、避免级联。
- 三个核心操作（Compress、Maintain、Revise）缺一不可：移除任一操作均导致性能下降（SR 降低 4.0–7.3 pp）。
- 结论强调：**执行状态结构是构建可靠高效记忆系统的关键原则**，而非语义相似性。

### 7. 优点

- **创新性**：将记忆从被动语义检索转向主动执行状态管理，认知启发的设计（层次 chunking、错误监控、选择性修正）匹配长周期需求。
- **状态完整性**：通过根到当前路径提供完整的依赖链，避免了语义碎片。
- **高效性**：边界感知压缩 + 确定性树结构大幅降低 token 消耗（辅助 LLM 调用仅发生在子目标边界）。
- **实验扎实**：多领域、多模型、消融实验齐全，案例提供直观解释。
- **可扩展性**：不依赖特定骨干模型，代码可用（附录未提供但暗示开源）。

### 8. 不足与局限

- **泛化性**：仅在 MemoryArena 上评估，未涵盖更复杂的多模态或交互式环境（如机器人操控、现实对话）。
- **依赖 LLM 判断**：Maintain 和 Revise 依赖 LLM 进行验证和诊断，可能引入幻觉或偏差，且未量化其正确性。
- **复杂性**：需要定义子目标边界（可能需人工或启发式），在模糊任务中可能难自动划分。
- **探索策略**：MAGE 假设 Revise 仅回溯到一个边界并分支，未处理需要多步回溯或环境噪声的情况。
- **模型成本**：虽然总 token 减少，但 Maintain 和 Compress 仍需额外 LLM 调用，低配模型上可能抵消好处。
- **实验覆盖**：消融仅在两个领域进行，未在 Formal Reasoning 等任务上验证；未与更多新兴记忆系统（如图检索+路径回溯）对比。

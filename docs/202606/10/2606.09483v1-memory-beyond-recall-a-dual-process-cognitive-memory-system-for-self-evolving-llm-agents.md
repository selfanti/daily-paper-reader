## 摘要
对于LLM代理而言，长期记忆不仅仅是适时检索正确的段落。当前的记忆系统将信念修正、因果耦合和跨领域抽象压缩到一个专为表面回忆优化的单一检索表面，因此在需要推理用户如何演化的隐式个性化任务中表现不佳。我们提出DCPM，它沿着认知能力层级重新组织代理记忆，从原始输入和原子事实出发，依次上升至历时信念轨迹与身份、领域模式、潜在意图及跨领域模式。该层级由继承双过程理论架构分割的两个过程驱动：一个同步的日间写入器（系统1），以双向链接的取代链记录信念修正；一个异步的夜间引擎（系统2），归纳模式与意图，并扫描被抽象为更高层核心模式的跨领域冲突。在LongMemEval、PersonaMem和PersonaMem-v2上，启用系统2在基准测试奖励隐式跨会话推理的任务中贡献最大（在PersonaMem-v2上提升高达+5.20），在跨度回忆任务中贡献最小，这与架构预测一致。

## Abstract
Long-term memory for an LLM agent is more than retrieving the right passage at the right time. Current memory systems collapse belief revision, causal coupling, and cross-domain abstraction into a single retrieval surface tuned for surface recall, and consequently struggle on implicit personalisation that requires reasoning over how a user has evolved. We propose DCPM, which reorganises agent memory along a cognitive capability hierarchy ascending from raw inputs and atomic facts, through diachronic belief trajectories and identity, to domain schemas, latent intentions and cross-domain patterns. The hierarchy is driven by two processes inheriting the architectural split of dual-process theory: a synchronous daytime writer (System1) that records belief revisions as doubly linked supersedes chains, and an asynchronous nighttime engine (System2) that induces schemas and intentions and sweeps for cross-domain collisions abstracted into higher-level core schemas. On LongMemEval, PersonaMem and PersonaMem-v2, enabling System2 contributes most where the benchmark rewards implicit cross-session inference (up to +5.20 on PersonaMem-v2) and least on span recall, matching the architectural prediction.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前 LLM 代理的长期记忆系统被简化为单纯的检索问题（如向量存储、时序知识图谱），将信念修正、因果耦合、跨领域抽象全部压缩到同一个检索表面，导致无法建模用户的动态演化（如信念如何被修订、偏好为何转移、不同生活领域行为间的潜在模式），因而在需要隐式个性化推理的任务中表现不佳。
- **整体意义**：论文主张记忆应被视为认知过程而非存储过程，提出基于双过程理论的认知能力层次，将代理记忆组织为从原始事实到高层次模式（模式、意图、核心模式）的递进结构，从而让代理真正“理解”用户演化，而非仅做表面回忆。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想：双过程认知记忆系统（DCPM）
- 基于 Kahneman 的双过程理论（系统 1 快速在线处理 vs. 系统 2 慢速离线抽象）。
- 将记忆组织为认知能力层次（从低到高）：原始输入 → 原子事实 / 身份项 → 历时信念链（取代链） → 领域内模式（schema） → 潜在意图（intention） → 跨领域核心模式（core schema）。

### 2.2 系统 1：同步日间写入器
- **触发条件**：仅在用户或应用发出显式 `add_memory` 请求时调用（非每个对话轮次）。
- **四步流程**：
  1. **原始持久化**：将原始输入写入向量数据库（零丢失）。
  2. **提取**：用提取 LLM（`M_e`）从当前轮次+近期上下文窗口中提取身份项和原子事实，附带类型、主题、显著性、时间戳。
  3. **协调（Reconcile）**：用协调 LLM（`M_r`）对每个新项与向量库中 top-k 近邻比较，决定操作：`ADD`（完全新颖）、`SUPERSEDE`（矛盾或演化版本）、`UPDATE`（补充/扩展），重复项丢弃。
  4. **批量更新与取代链**：`SUPERSEDE` 操作时，建立双向指针（`new.SUPERSEDES = old` 和 `old.SUPERSEDED_BY = new`），旧项不删除，形成双向链表。这是保持历时连续性的关键。

### 2.3 系统 2：异步夜间引擎
- **运行时机**：空闲或夜间周期，宽松延迟预算（数十秒）。
- **三阶段**：
  - **阶段 1（预处理，无 LLM）**：事实分诊 → 两阶段 DBSCAN 聚类（ε1=0.35, ε2=0.25, min_samples=3）→ 检索图上下文。
  - **阶段 2（LLM 代理归纳）**：对每个聚类调用归纳代理 `M_a`，使用四个工具（`CREATE_SCHEMA`、`CREATE_INTENTION`、`ADD_EVIDENCE`、`ADD_EDGE`），增量更新图 G。
  - **阶段 3（跨领域扫描器）**：
    - 对每个模式节点，生成行为描述（LLM 抽象），嵌入到行为-认知空间 `E_beh` 和语义空间 `E_sem`。
    - 扫描跨领域模式对，当满足条件：cos(E_beh(si), E_beh(sj)) > θ_beh 且 cos(E_sem(si), E_sem(sj)) < θ_sem 时，标记为冲突。
    - 用并查集聚合冲突簇，然后调用核心归纳 LLM（`M_c`）生成更高级的核心模式，并链接到所有成员模式。

### 2.4 读取路径
- 查询嵌入通过向量相似度命中事实/身份存储 → 沿取代链遍历（O(¯k)）→ 从图 G 中通过 `evidence_vdb_ref` 边获取相关模式/意图/核心模式节点。
- 读取全过程不调用 LLM，延迟主要由向量搜索主导。

## 7. 优点：方法或实验设计上的亮点

- **理论驱动设计**：明确基于双过程认知理论和心理理论（ToM），将认知层次显式化，区别于纯工程化的存储方案。
- **不添加 LLM 调用到读取路径**：推理时延迟与纯检索系统相同，但获得了结构化认知信息。
- **取代链机制简洁有力**：通过双向指针保留完整信念演化轨迹，而非覆盖或删除，且无需复杂图更新。
- **跨领域冲突扫描**：利用行为空间 vs. 语义空间的双空间差异发现潜在抽象模式，是一种新颖的归纳方法。
- **异步离线引擎**：将高计算开销移动到空闲时段，不影响在线交互延迟。
- **实验诊断充分**：通过消融和每类问题分析清晰分离各部件的贡献，验证了架构假设。

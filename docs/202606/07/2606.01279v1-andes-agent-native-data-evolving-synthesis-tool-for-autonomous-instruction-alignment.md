## 摘要
AI智能体越来越多地被赋予自动化AI研究本身的任务，特别是将基础大语言模型转变为对齐助手的关键后训练阶段。然而，近期评估表明，即使是顶尖的智能体也难以完成这一任务。尽管后训练的成功根本上依赖于获取高质量数据，但依赖智能体自主从开放网络中策划目标训练数据集会带来严峻挑战。在嘈杂的网络环境中执行搜索、过滤和平衡数据的长周期任务，常常超出智能体有限的上下文能力，最终导致数据集质量下降和下游训练效果欠佳。为弥补这一差距，我们提出了Andes（智能体原生数据演化合成），这是一个将数据生成重新构想为即插即用的“智能体技能”的框架。Andes并非强制智能体从零开始设计复杂的数据收集策略，而是提供了一层智能抽象。通过利用自演化的世界树路由机制和可操作诊断报告，它允许训练智能体通过交互式闭环接口动态引导数据合成。我们证明，在严格的计算约束下，为较弱的基础智能体配备Andes能改善自动对齐，在PostTrainBench上取得最先进性能并实现稳健的跨任务泛化。项目地址：https://github.com/zzy1127/ANDES。

## Abstract
AI agents are increasingly being tasked with automating AI research itself, particularly the critical post-training phase that transforms base LLMs into aligned assistants. However, recent evaluations reveal that even frontier agents struggle to perform this task. While the success of post-training fundamentally relies on acquiring high-quality data, relying on agents to autonomously curate targeted training datasets from the open web introduces severe challenges. Executing the long-horizon tasks of searching, filtering, and balancing data within noisy web environments frequently overwhelms an agent's limited context, ultimately leading to degraded dataset quality and suboptimal downstream training performance. To bridge this gap, we introduce Andes (Agent Native Data Evolving Synthesis), a framework that reimagines data generation as a plug-and-play \emph{agent skill}. Rather than forcing agents to devise complex data-gathering strategies from scratch, \textsc{Andes} provides an intelligent abstraction layer. By leveraging a self-evolving World Tree routing mechanism and actionable diagnostic reports, it allows trainer agents to dynamically steer data synthesis through an interactive, closed-loop interface. We demonstrate that under strict compute constraints, equipping foundationally weaker agents with Andes improves automated alignment, securing state-of-the-art performance on PostTrainBench and robust cross-task generalization. Our project is available at https://github.com/zzy1127/ANDES.

## 2. 方法论
### 2.1 核心思想
- **框架名称**：Andes（Agent Native Data Evolving Synthesis）。
- **设计思路**：为训练智能体提供一层智能抽象 —— 一个可调用的工具接口。智能体只需声明目标能力维度，Andes 负责场景抽象、数据生成与诊断反馈，形成**闭环交互式合成**。

### 2.2 关键技术细节
- **（1）目标驱动智能体请求（Target-Driven Agent Request）**  
  训练智能体将下游基准测试分解为 \(K\) 个宏观能力域（如数学推理、代码生成），每个域构造三元组 \((\tau_j, N_j, \phi_j)\)，其中 \(\tau_j\) 为任务描述，\(N_j\) 为样本预算，\(\phi_j\) 为格式协议（如unstructured/code/tool_call）。每个域调用一次Andes。

- **（2）自演化动态世界树路由（Self-Evolving Dynamic World Tree Routing）**  
  世界树 \(W\) 初始包含 **72个主题、394个主题变体、1182个场景**，结构为Topic → Theme → Scenario。对每个任务 \(\tau\)，按权重采样 \(B\) 个主题，然后用Router LLM将每个场景评估为 **Strong / Ambiguous / Weak**。  
  - Strong → 融合生成轨道（fusion track，注入任务约束）  
  - Weak → 通用生成轨道（generic track，无约束）  
  - Ambiguous → 随机分配（概率0.8到fusion）  
  权重更新：Strong主题权重乘 \(\gamma^+ = 1.5\)，Weak主题权重乘 \(\gamma^- = 0.8\)（下限 \(\epsilon = 0.1\)）。  
  当某主题被选择次数 \(c_t > \rho \cdot |S_t|\)（\(\rho=0.8\)，\(|S_t|\)为当前场景数）时，触发节点演化：Evolver LLM基于历史子树生成新的6个主题×6个场景（共36个新场景），重置计数与权重。

- **（3）两阶段数据合成（Two-Stage Data Synthesis）**  
  第一阶段：对每个路由配置生成问答对，融合轨道追加任务描述作为hard constraint。  
  第二阶段：Refiner LLM对原始QA生成批评和努力分 \(e_i \in [1,5]\)，过滤 \(e_i > 4\) 的低质量样本，重写成精炼数据；并对融合轨道样本执行逻辑多样性检查，生成多样性摘要。

- **（4）报告驱动的分区与配置调整（Report-Driven Feedback Loop）**  
  每次Andes调用返回精炼数据集 \(D_j\) 和合成报告 \(R_j\)（包括主题分布、有效样本数、逻辑重复模式等）。智能体基于报告应用**主动过滤**（依据重复强度丢弃部分数据），并调整下一轮的任务描述 \(\tau_{j+1}\) 和样本预算 \(N_{j+1}\)，形成闭环。

### 2.3 公式与算法流程（文字说明）
- 路由采样：\(t_i \sim \text{Categorical}(w_{t_1}, \dots, w_{t_M})\)  
- 权重更新：\(w_t \leftarrow \gamma^+ \cdot w_t\)（Strong）或 \(\max(\epsilon, \gamma^- \cdot w_t)\)（Weak）  
- 节点演化触发条件：\(c_t > \rho \cdot |S_t|\)  
- 每轮配置更新：\((\tau_{j+1}, N_{j+1}, \phi_{j+1}) = \mathcal{F}(\tau_j, \phi_j, R_j, N_j - |D_j^{\text{keep}}|)\)

## 6. 主要结论与发现
- **SOTA性能**：Andes在PostTrainBench上平均得分 **33.39%**，超过最强对比方法Opus-4.7 (xHigh) 的28.56%，提升4.8个百分点。
- **基础模型无关性**：在四个不同基础模型上均取得最佳平均性能，说明方法不依赖特定架构。
- **显著提升弱agent能力**：GLM-4.7（本身能力较弱）配备Andes后，从基线7.48%跃升至33.39%，在BFCL、GSM8K、HealthBench等任务上提升超50个百分点。
- **路由与演化有效性**：可视化（图3）显示Andes能动态聚焦目标相关主题（如数学推理），同时通过节点演化持续补充新场景，避免多样性崩溃。
- **报告驱动反馈提升质量**：消融实验表明，移除报告驱动的交互环后平均分下降约3.6%，在BFCL、HumanEval上下降更明显。
- **跨任务泛化**：在5个额外基准上，Andes在10k数据下取得整体最佳（58.9%），优于静态数据管线（55.2%）和百万级数据集（49.6%），尤其在Gaokao上提升14.3点，而其他方法均出现下降。

## 7. 优点
- **概念创新**：重新定义数据生成为“即插即用agent技能”，降低agent自主对齐的能力门槛。
- **结构精巧**：世界树路由+节点演化机制在**对齐目标**与**多样性保持**之间取得良好平衡；两阶段生成（融合/通用）+精炼+多样性检查确保数据质量。
- **闭环交互**：报告驱动反馈使agent能够持续根据诊断调整，实现渐进式数据覆盖优化，避免单次生成的局限性。
- **实验严谨**：在主benchmark上进行了大量对比，包括多种最强agent，消融实验清晰解耦了scaffold、Andes核心、报告交互的贡献。
- **开源**：项目开源，便于复现与扩展。

## 8. 不足与局限
- **依赖强内部LLM**：路由、生成、精炼等所有模块均使用GPT-4o，未探讨使用开源模型（如LLaMA）时的性能，存在成本与可替代性风险。
- **轻量级scaffold设计**：仅为一个基础的CLI-based scaffold，可能不具备通用agent框架（如LangChain）的鲁棒性；未对比其他scaffold。
- **计算预算固定**：所有实验限制在10小时单GPU，未探索更长时间或更多GPU下的性能上界，也未测试Andes在大规模场景下的扩展性。
- **世界树初始化需一定领域知识**：虽然使用公开知识构建了72个主题，但手动构建成本高，自动生成世界树的方法未研究。
- **报告驱动依赖agent理解能力**：agent需正确解析报告中的多样性摘要和有效数量，当前仅测试了GLM-4.7，若agent本身理解能力弱可能影响效果。
- **偏差风险**：内部LLM（GPT-4o）可能引入自身偏差，且融合轨道注入任务描述可能导致数据分布偏向特定模式，虽然节点演化部分缓解，但量化分析不足。
- **应用限制**：目前主要针对后训练对齐任务，对其他类型（如预训练、多模态）未讨论；且仅验证了英文场景，中文或低资源语言未涉及。

## 摘要
在actor-critic强化学习中，网络架构通常是手动设计的。自动化这一设计具有挑战性，因为每个候选架构在评估前必须经过训练，且设计空间是开放的。为了解决这些挑战，我们引入了EVOM，一个用于发现高性能actor-critic架构的智能元进化框架。我们将架构搜索视为双层优化：内层通过低保真度的近端策略优化（PPO）训练权重，而外层通过迭代优化架构程序驱动元进化。关键的是，这个外层由一个基于LLM的设计智能体驱动，该智能体纯粹作为架构设计师，完全与策略执行和环境控制解耦。实验表明，EVOM优于手动设计的基线、LLM引导的随机搜索以及最先进的LLM引导的程序化策略搜索方法MLES，在Ant-v4和HalfCheetah-v4上表现出卓越性能。消融研究验证了元进化循环和LLM设计智能体对于最终性能都是不可或缺的。

## Abstract
In actor-critic reinforcement learning, network architectures are typically manually designed. Automating this design is challenging because each candidate must be trained before evaluation, and the design space is open-ended. To address these challenges, we introduce EVOM, an agentic meta-evolution framework for discovering high-performance actor-critic architectures. We frame architecture search as a bi-level optimization: an inner loop trains weights via the low-fidelity proximal policy optimization (PPO), while an outer loop drives meta-evolution by iteratively refining architecture programs. Crucially, this outer loop is powered by an LLM-based design agent that operates purely as an architecture designer, completely decoupled from policy execution and environment control. Experiments reveal that EVOM outperforms the manually designed baseline, an LLM-guided random search, and the state-of-the-art LLM-guided programmatic policy search method MLES, delivering superior performance on Ant-v4 and HalfCheetah-v4. Ablation studies validate that both the meta-evolution loop and the LLM Design Agent are indispensable for final performance.

## 2. 论文提出的方法论

### 核心思想
- **双层优化（bi-level optimization）**：
  - **内层（inner loop）**：对给定的架构程序，使用低保真（low-fidelity）PPO 训练网络权重，得到代理适应度信号。
  - **外层（outer loop）**：维护一个由架构程序组成的种群，利用 LLM 设计智能体执行变异（mutation）和交叉（crossover）生成后代，然后根据内层提供的适应度进行选择，迭代改进种群。

### 关键技术细节
- **架构表示**：每个候选架构表示为一个可执行的 Python 程序，必须包含 `PolicyNet(obs_length, act_length)` 和 `ValueNet(obs_length)` 两个类，分别输出动作均值和标量价值。这使得搜索不受固定拓扑限制，同时可通过编译检测保证有效性。
- **LLM 设计智能体**：使用 GPT/Claude 等商用 LLM，通过三类提示模板（初始化、变异、交叉）生成架构程序。提示内容包括环境描述、固定状态-动作信息、输出要求以及当前的精英架构设计。LLM 仅作为架构生成器，不接触梯度、参数值或环境交互日志。
- **进化流程**：
  1. 初始化：LLM 生成 N 个初始架构。
  2. 每代：LLM 对精英种群应用变异（N/2 个）和交叉（N/2 个）生成后代；内层 PPO 以低预算（100k 步）训练每个后代并评估 3 个目标回合的平均回报；将后代与精英合并后按适应度排序，保留前 N 个作为新精英。
  3. 最终评估：用全预算（5M 步）重新训练最佳精英，报告最终性能。
- **无效程序惩罚**：编译失败或输出非有限值的架构被赋予 -1000 的适应度，使其在精英选择中被淘汰。

## 6. 论文的主要结论与发现

- EVOM 在两个任务上均显著优于手动基线、随机搜索和 MLES：
  - Ant-v4：EVOM 平均回报 4652.10（±746.91），手动 PPO 仅 2517.06（±366.24）。
  - HalfCheetah-v4：EVOM 平均回报 5381.07（±1357.70），手动 PPO 仅 1691.42（±495.26）。
- 随机搜索在 HalfCheetah 上虽然能达到较高历史最佳，但最终评估方差极大（±1953.45），说明独立采样不稳定。
- 消融实验表明：
  - 混合进化（变异+交叉）优于单一操作符，表明局部编辑（变异）与结构融合（交叉）互补。
  - 种群大小 N=16 效果最好，过小（N=8）或过大（N=32）均导致性能下降，可能因低预算评估噪声随种群增大而放大。
  - 不同 LLM 骨干均能驱动进化，但性能存在任务依赖性（Claude 在 Ant 上最好，DeepSeek 在 HalfCheetah 上最好）。
- 最终发现的架构通常包含残差连接、层归一化、SiLU 激活等与手动设计不同的组件，说明进化能够探索出有效的新模式。

## 7. 优点

- **方法新颖**：将 LLM 作为进化算子与双层优化结合，既利用 LLM 的生成能力又保留 RL 学习参数的优势，实现架构搜索与参数学习的解耦。
- **搜索效率高**：低预算 PPO 使种群级评估在消费级 GPU 上可行，比传统 NAS 大幅降低计算开销。
- **可解释性与可复现性**：架构以可执行程序表示，可检查、可重用，不同于黑盒策略。
- **消融设计全面**：系统分析了操作符、种群大小、LLM 骨干三个关键变量，验证了各组件必要性。
- **性能提升显著**：在标准 MuJoCo 任务上大幅超越现有方法，且稳定性较好（标准差相对较低）。

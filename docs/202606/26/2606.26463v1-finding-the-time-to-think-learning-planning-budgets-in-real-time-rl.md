## 摘要
深思熟虑需要时间。在实时环境中，时间并非免费。标准强化学习（RL）避免了这一问题，因为环境会无限期等待智能体的决策。相反，我们研究了实时RL环境，其中环境在等待智能体动作时会继续推进。基于先前的实时形式化方法，我们引入了可变延迟实时强化学习，其中智能体在每个决策点选择思考多长时间，因为环境在持续推进。对于我们使用的规划智能体，正确的延迟是状态相关的，而天真地规划如何规划会导致智能体瘫痪。我们通过训练一个轻量级的门控策略，该策略置于规划器之上，来选择依赖于状态的规划预算。在实时吃豆人、俄罗斯方块、贪吃蛇、速战六角棋和速战围棋中，我们的门控策略优于固定预算和启发式基线，并且可以迁移到环境和智能体在两个不同GPU上运行的实时设置中。

## Abstract
Deliberating takes time. In real-time settings, that time is not free. Standard reinforcement learning (RL) sidesteps this as the environment waits indefinitely for the agent's decision. Instead, we study real-time RL environments where the environment progresses while waiting for the agent's action. Building on prior real-time formalizations, we introduce variable-delay real-time RL, where the agent chooses how long to deliberate at each decision point since the environment progresses. For the planning agents we use, the right delay is state-dependent, and naively planning how long to plan can paralyze the agent. We instead approach this setting by training a lightweight gating policy on top of a planner to select state-dependent planning budgets. Across real-time Pac-Man, Tetris, Snake, Speed Hex, and Speed Go, our gating policy outperforms fixed-budget and heuristic baselines, and transfers to a real-time setup where the environment and agent run on two different GPUs.

## 1. 核心问题与整体含义

*   **研究动机**：标准强化学习（RL）和马尔可夫决策过程（MDP）假设环境会无限期等待智能体做出决策，这导致了“思考时间免费”的悖论。然而，在现实世界的实时场景中（如自动驾驶、电子竞技），环境在智能体思考时是持续演进的。一个精确但迟到的行动，其效果远不如一个及时但次优的行动。
*   **核心问题**：如何在实时RL中，让一个“思考型”（使用蒙提卡洛树搜索MCTS）的智能体，能够根据环境状态动态地决定“思考多久”，从而在行动质量与时间成本之间做出最优权衡。问题的难点在于：1）最优思考时长是状态依赖的；2）智能体在思考期间必须通过快速动作控制环境；3）关于“如何规划”本身的规划（元推理）会导致无限递归的悖论。
*   **整体含义**：本文提出了一个名为“可变延迟实时RL”的通用框架，并设计了一种实用方法——训练一个轻量级“门控策略”来为底层规划器选择状态依赖的规划预算。这项工作将元推理思想与端到端RL训练相结合，为构建更高效、更具适应性的实时决策系统提供了新思路。

## 2. 方法论

*   **核心思想**：将“规划时长”本身视为一个学习问题。智能体不是一个瞬时动作函数，而是一个随时间运行的程序。通过将动作产生过程分解为快速反应与深度思考两部分，并让一个轻量级门控策略决定何时调用深度思考及思考多久。
*   **关键技术细节**：
    1.  **可变延迟实时RL框架**：正式化了实时交互协议。代理是一个在时间中运行的程序，其产生任何动作所花费的时间都会被支付为世界的进展（环境步进）。
    2.  **半马尔可夫决策过程（SMDP）与预算选项**：将固定规划预算k打包成一个“时间扩展动作”（选项）。执行选项时，智能体在前k-1帧通过快速“反射策略”采取“承诺动作”控制环境，在第k帧应用“随时算法”（MCTS）算出的更好的动作。元层次的门控策略则在此SMDP中学习如何选择最优的k。
    3.  **轻量级门控策略**：一个小型神经网络，输入包括游戏观察、规划器中间特征和标量价值估计。它输出一个关于所有预算k（例如K={1，2,3,4}）的概率分布。单次前向传播成本远低于一次MCTS模拟，从而避免了“规划-规划”的悖论。
    4.  **训练流程**：
        *   阶段一：冻结一个预训练的AlphaZero风格规划器。
        *   阶段二：使用PPO算法训练门控策略，并采用“可变时长广义优势估计”（Variable-duration GAE）来正确处理不同k值导致的折扣差异。

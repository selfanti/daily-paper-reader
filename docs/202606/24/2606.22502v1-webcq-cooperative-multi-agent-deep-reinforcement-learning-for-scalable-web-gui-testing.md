## 摘要
多智能体强化学习（MARL）技术已显示出在GUI测试中的潜力。然而，随着现代GUI软件复杂性的增加，现有的基于MARL的方法（如MARG和Fastbot）由于底层表格型强化学习算法的固有限制而难以扩展。这限制了它们在大规模商业GUI软件中的应用，尤其是具有巨大状态空间和许多交互元素的Web应用。为填补这一空白，我们提出了WebCQ，一种新颖的基于MARL的可扩展Web GUI测试方法。WebCQ结合了QTRAN用于多智能体协调和轻量级同步机制，使其能够工作在异步Web测试场景下。它为每个UI事件提取语义和探索特征以形成动作向量。该向量与当前状态向量拼接后输入策略网络，实现在动态动作空间内基于DQN的决策。我们在八个大型商业网站上评估了WebCQ。在相同时间预算和智能体数量下，WebCQ比MARG探索了多33.3%的状态，执行了多42.2%的独特动作，同时在八个被测网站中的六个上触发了更多故障。它还展示了强大的可扩展性，在20小时实验中保持更高的动作吞吐量，并且随着智能体数量增加实现了更大的性能提升。这些结果表明，WebCQ克服了现有基于MARL方法的关键限制，为增强现代Web GUI测试提供了可扩展且有效的解决方案。

## Abstract
Multi-agent reinforcement learning (MARL)-based techniques have shown promise for GUI testing. However, as the complexity of modern GUI software increases, existing MARL-based approaches (e.g., MARG and Fastbot) struggle to scale due to the inherent limitations of their underlying tabular reinforcement learning algorithms. This limits their applicability to large-scale commercial GUI software, especially web applications with vast state spaces and many interactive elements. To fill this gap, we propose WebCQ, a novel MARL-based approach for scalable web GUI testing. WebCQ incorporates QTRAN for multi-agent coordination and a lightweight synchronization mechanism, allowing it to work under asynchronous web testing scenarios. It extracts semantic and exploration features for each UI event to form an action vector. This vector is concatenated with the current state vector and fed into the policy network, enabling DQN-based decision making within a dynamic action space. We evaluated WebCQ on eight large-scale commercial websites. Under the same time budget and agent count, WebCQ explored 33.3% more states and executed 42.2% more unique actions than MARG, while triggering more failures on six of the eight websites under test. It also demonstrated strong scalability, maintaining higher action throughput during 20-hour experiments, and achieving greater performance improvements as the number of agents increased. These results show that WebCQovercomes key limitations of existing MARL-based approaches, providing a scalable and effective solution for enhancing modern web GUI testing.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **背景**：随着现代Web GUI应用日益复杂，自动化的端到端GUI测试变得至关重要。多智能体强化学习（MARL）方法（如MARG、Fastbot）已展现出优于单智能体方法的潜力，通过并行化执行和经验共享提升探索效率。
- **核心问题**：现有MARL方法底层仍采用**表格型Q学习**（tabular RL），导致：
  - 状态-动作空间巨大时，Q-table膨胀，通信开销急剧增加；
  - 行为定义局限于UI元素定位器，无法捕获动作的**语义信息**（如“提交表单”）；
  - 智能体间同步策略低效，难以扩展到大规模商业网站。
- **动机**：需要一种能够克服表格算法局限、支持动态动作空间、并实现高效多智能体协调的可扩展Web GUI测试方法。

## 2. 论文提出的方法论
### 2.1 核心思想
- 将多智能体Web测试建模为**去中心化部分可观测马尔可夫决策过程（Dec-POMDP）**，并采用**集中训练-去中心化执行（CTDE）**范式。
- 集中训练阶段引入**QTRAN**算法，学习联合Q函数与个体Q函数的分解，实现非单调、非加性的多智能体协调。
- 每个智能体使用**DQN**进行去中心化执行，通过拼接状态向量与动作向量输入网络，应对动态动作空间。

### 2.2 关键技术细节
#### 状态向量（State Vector）
- 基于HTML标签及其DOM深度构建固定维度向量：对于预定义标签列表，计算每个标签下所有元素的平均深度（未出现则填0），外加一个额外维度表示其他标签。
- 优点：高效、固定维度，可兼容神经网络输入。

#### 动作向量（Action Vector）
- 为每个UI事件构建三维特征：
  - **文本相似度**：使用GloVe预训练模型计算按钮文本与正向关键词（如“submit”“confirm”）的余弦相似度（取最大值）。
  - **执行频率**：记录该事件被各智能体执行的总次数。
  - **探索潜力**：子状态（执行后到达的页面）中动作被执行次数的分布（10维计数向量，分别统计执行0-9次的动作数量）。
- 动作向量随探索进程动态更新，鼓励新颖、高频导向未探索区域的动作。

#### 动作选择策略
- 对当前状态下的每个候选动作，将状态向量与该动作向量拼接，输入个体Q网络得到Q值：
  \[
  Q_i(s_i, a_i') = f(\text{concat}(s_i, a_i'))
  \]
- 采用ε-贪心策略（ε线性下降：0.9→0.3，在前一半时间完成衰减）。

#### QTRAN集中训练
- 适应异步场景的改进：
  - **联合状态表示**：将各智能体的局部状态向量拼接构成联合状态 \(\boldsymbol{s} = (s_1; s_2; \dots; s_N)\)。
  - **轻量同步机制**：无需每步同步；当每个智能体都至少完成一次新transition后，收集所有最新经验进行集中训练（最后一次完成的智能体触发训练，其他智能体等待）。这避免了频繁阻塞，提升了效率。
- 损失函数由三部分组成：标准TD误差（\(L_{td}\)）、最优联合动作约束（\(L_{opt}\)）、次优动作约束（\(L_{nopt}\)），共同优化联合Q网络和个体Q网络。

#### 奖励函数（混合奖励）
- 个体奖励 \(r_i\) 累积为全局奖励 \(r = \sum r_i\)。
- 包含三项：
  - **页面相似度奖励 \(R_{sim}\)**：基于APIMiner度量找到当前页面的最相似已访问页面，根据相似度阈值分段赋分（高/中/低），并且频繁访问的页面奖励递减（\(1/N(p)\)）。
  - **执行频率奖励 \(R_{exec}\)**：新动作奖励2，重复动作给予递减奖励（\(1/N(e)\)）。
  - **时间补偿 \(R_{time}\)**：随时间线性增加（\(t/t_{total}\)），缓解长期探索奖励稀疏问题。
- 总奖励：\(r = (R_{sim} + R_{exec}) \times (1 + R_{time})\)

#### 完整算法流程（Algorithm 1）
1. 初始化N个智能体的Q网络、全局缓冲区B、同步锁等。
2. 每个智能体异步执行：等待非训练状态 → 获取当前状态嵌入 → 选择并执行动作 → 存储本地transition → 标记done信号 → 尝试加锁触发集中训练。
3. 集中训练过程：合并各智能体transition为联合transition → 计算全局奖励 → 存入全局缓冲区 → 若缓冲样本数≥mini-batch大小，则采样计算QTRAN损失并更新网络 → 重置所有done信号。

## 6. 论文的主要结论与发现
1. **性能显著优于最先进方法**：
   - 在相同预算下，WebCQ平均探索**978.8个状态**，比MARG（734.1）提升**33.3%**；执行**1220.6个独特动作**，比MARG（858.2）提升**42.2%**。
   - 在8个网站中的6个上，WebCQ检测到更多失败（平均44.4个 vs MARG 42.6个）。
   - 统计检验表明改进在大部分站点上显著（p<0.05），且效应量大（A12>0.71）。

2. **消融研究证实关键设计有效**：
   - 移除通信或移除QTRAN后，性能明显下降（状态数分别减少27.1%和21.3%），说明**多智能体通信**和**先进的MARL算法（QTRAN）** 均至关重要。
   - 仅使用DRL但采用MARG通信方案（WebCQ w/o QTRAN）提升有限，说明QTRAN的协同分解机制更优。

3. **低协调开销与长期稳定性**：
   - 在20小时实验中，WebCQ的每小时执行动作数下降平缓，而MARG迅速衰减并稳定在低位，表明WebCQ的DQN+QTRAN方案有效减少了随Q-table膨胀带来的协调开销。

4. **智能体数量增加时性能稳步提升**：
   - WebCQ在3→5→8→12个智能体时，性能持续上升；而MARG在8个以上智能体时性能下降，凸显其表格算法的可扩展性瓶颈。

5. **DRL相比表格算法的优势**：
   - 即使无通信的DRL变体（WebCQ w/o comm.）也比MARG在5/8个站点上表现更好，说明深度神经网络的学习能力更能应对大规模状态空间。

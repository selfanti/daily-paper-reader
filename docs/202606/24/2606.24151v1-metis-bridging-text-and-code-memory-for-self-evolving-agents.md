## 摘要
自我进化智能体通过从过往执行中提炼经验并在未来任务中复用这些经验，随时间不断提升性能。现有系统将这些经验表示为注入智能体上下文的自然语言文本，或表示为可调用工具的代码。然而，这两种表示之间的选择通常在设计阶段做出，而非源自经验本身的特性，导致两者之间的权衡尚未被充分理解。我们首次在相同经验集合上进行了控制性研究，将文本记忆与代码记忆分离比较。结果表明，这两种形式在构建成本、执行效率和可迁移性上表现出互补的权衡，因此单一表示都不足以胜任。基于这些发现，我们提出了Metis，一种构建在分层双表示记忆上的自我进化智能体系统。Metis将文本经验组织为执行计划、环境事实和常见陷阱，并选择性地将重复出现的计划固化为经过验证的可调用工具。这种设计结合了文本记忆的广泛适用性与代码记忆的执行效率，同时仅在重复复用证明合理时才产生工具生成成本。我们在交互式智能体挑战基准AppWorld上对Metis进行了评估。结果显示，相比于ReAct，Metis将任务准确率提升了最多20.6%，同时将执行成本降低了最多22.8%。与代表性的自我进化智能体系统相比，Metis在准确率、执行效率和记忆构建成本之间始终实现了更好的平衡。

## Abstract
Self-evolving agents improve over time by distilling experience from past executions and reusing it in future tasks. Existing systems represent such experience either as natural-language text injected into the agent context or as code exposed as callable tools. However, the choice between these representations is typically made at design time rather than derived from the characteristics of the experience itself, leaving the trade-offs between them poorly understood. We present the first controlled study that isolates text memory and code memory over an identical set of experiences. Our results show that the two forms exhibit complementary trade-offs in construction cost, execution efficiency, and transferability, such that neither representation alone is sufficient. Guided by these findings, we propose Metis, a self-evolving agent system built on a hierarchical dual-representation memory. Metis organizes textual experience into execution plans, environment facts, and common pitfalls, and selectively crystallizes recurring plans into validated callable tools. This design combines the broad applicability of text memory with the execution efficiency of code memory while incurring tool-generation cost only when justified by repeated reuse. We evaluate Metis on AppWorld, a challenging benchmark for interactive agents. The results show that Metis improves task accuracy by up to 20.6% over ReAct while reducing execution cost by up to 22.8%. Compared with representative self-evolving agent systems, Metis consistently achieves a better balance between accuracy, execution efficiency, and memory-construction cost.

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：自我进化智能体通过从过往执行中提炼经验并复用来提升性能。现有系统通常将经验表示为**自然语言文本**（注入上下文）或**可执行代码**（封装为工具）。
- **问题**：这两种表示的选择是设计时固定决定的，并未根据经验本身的特性（如稳定性、可迁移性）动态选择。因此，文本记忆虽然构建成本低且迁移性好，但执行效率低；代码记忆虽然执行效率高，但构建成本高且泛化脆弱。二者优势无法同时获取。
- **贡献**：论文首次通过控制性实验，在同一经验集和同一智能体骨干上分离文本和代码记忆，揭示了二者在构建成本、执行效率和可迁移性上的**互补权衡**。基于此，提出 **Metis** 系统，通过分层双表示记忆动态选择表示形式，实现了准确率、效率和成本的更好平衡。

## 2. 方法论：核心思想、技术细节、算法流程

### 核心思想
- **分层双表示记忆**：底层是结构化文本记忆（低承诺、低成本），上层是代码记忆（高承诺、高收益）。只有经过多次复用的稳定执行计划才被提升为代码工具，从而避免无谓的构建成本和过拟合风险。

### 关键技术细节
- **文本记忆结构**：分为三类：

  | 类型 | 内容 | 作用 |
  |------|------|------|
  | 执行计划 (Plan) | 可重用的子过程模板 | 未来可被提升为代码工具 |
  | 环境事实 (Fact) | 环境属性和约束 | 指导推理 |
  | 常见陷阱 (Pitfall) | (触发条件, 错误, 后果) 三元组 | 防止重复犯错 |

- **提升机制**：记录每个执行计划被选择的次数。当计数达到阈值 θ，触发代码生成。
  - 使用计划本体、相关查询（而非原始轨迹）作为代码生成输入，避免过拟合。
  - 生成的代码工具需通过依赖扫描和编译验证，才被加入工具库。
- **记忆管理**：通过嵌入检索 + LLM 筛选进行记忆选择。反射模块离线运行，不影响在线服务延迟。
- **算法流程**：
  1. 对新任务，记忆管理器检索并选择相关文本记忆和工具。
  2. 执行器基于选定记忆完成任务。
  3. 双路径反射：
     - 先检查被选执行的计划是否达到提升阈值，若是则触发代码生成并验证。
     - 然后对当前轨迹进行文本反射，生成或更新文本记忆（合并、替换）。
  4. 更新记忆库供后续任务使用。

## 6. 主要结论与发现
- **互补性验证**：文本记忆构建成本低（约 2.5× 更少的 LLM 回合）、迁移可靠（oracle→streaming 下降仅 5.6 点）；代码记忆执行效率高（减少 54% 令牌和 46% 步骤），但构建成本高且迁移脆弱（下降 22.3 点）。
- **Metis 优势**：
  - 在 Official 分集上准确率 60.1%（No Memory 51.8%，SkillX 54.8%），执行令牌减少 13.5%。
  - 在 Resampled 分集上准确率 66.1%（No Memory 54.8%，SkillX 65.5%），执行令牌减少 22.8%。
  - 反射成本低于最强基线 SkillX（7.9M vs 9.4M tokens），即使额外包含工具合成。
- **消融关键结果**：
  - 单表示（Text-only / Code-only）均不如完整 Metis。
  - 急切代码生成导致构建成本升 47%，准确率降 3.5 点，且工具调用率低（41% vs 56%）。
- **复用率**：94% 的 Metis 工具被实际调用，而 SkillX 仅 84.1% 的代码技能被注入（且注入了不一定被使用）。

## 7. 优点

### 方法

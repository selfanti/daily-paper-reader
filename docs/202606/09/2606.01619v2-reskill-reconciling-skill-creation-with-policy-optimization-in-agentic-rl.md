## 摘要
代理强化学习（RL）使大语言模型智能体能够从环境奖励中持续改进，然而由此产生的策略并未系统性地积累可跨任务泛化的可重用策略。模块化技能可以提供此类可重用策略，但现有的技能增强RL方法将技能创建与策略优化分离，存在采用与演变中策略相冲突的技能的风险。受Anthropic的Skill Creator启发，我们提出了ReSkill，一种将技能演化与策略学习相协调的RL循环式技能创建框架。ReSkill利用GRPO的组内结构，仅以微小额外开销自然嵌入三种机制：（1）基于断言的技能创建器，从过往经验中诊断失败并提出基于触发条件的条件式技能修订；（2）组内轨迹采样，实现对技能版本的受控比较，捕获最能支持策略持续学习的版本；（3）采用自适应折扣的汤普森采样，在策略演化过程中平衡技能版本选择的探索与利用。在多个领域中，ReSkill持续优于现有基于记忆和技能的RL方法，在未见任务上增益最大。对技能生命周期的分析表明，随着策略改进，技能被自动创建、测试、完善和剪枝，展示了协调的技能-策略共同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

### **ReSkill：在代理强化学习中调和技能创建与策略优化**

#### **1. 论文的核心问题与整体含义（研究动机和背景）**

*   **研究动机**：当前基于强化学习的智能体（Agentic RL）能从环境奖励中学习，但学习到的策略往往无法系统性地积累可跨任务复用的通用策略（例如“先搜索再验证”）。
*   **现有方法局限**：模块化技能（Modular Skills）虽能提供可复用策略，但现有方法（如Anthropic的Skill Creator）通常是在策略固定（inference-time）或与策略学习解耦（decoupled）的情况下创建的。这种分离式的“先创建技能，再训练策略”模式，容易导致技能与正在演化的策略产生冲突（Skill-Policy Conflict），因为外部创建的技能可能不适用于当前的策略行为。
*   **核心问题**：如何将技能创建过程嵌入到RL训练循环中，使得技能能够根据策略的实时演化动态地创建、评估和调整，实现技能与策略的协同进化（co-evolution）？

#### **2. 论文提出的方法论：核心思想、关键技术细节**

**核心思想**：提出 **ReSkill** 框架，将技能创建视为RL训练循环内部的一个过程。它利用GRPO（Group Relative Policy Optimization）算法的“组内”（within-group）结构，在不增加采样预算的前提下，实现对不同技能版本的在线测试和选择。

**关键技术细节**：
1.  **组内技能测试（Within-Group Skill Testing）**：
    *   利用GRPO的特性：每一轮训练，同一个任务会采样 `G` 条轨迹。
    *   ReSkill保留了这个组结构，但在组内为不同轨迹分配**不同的技能版本**（旧版本 `S_old` 或新候选版本 `S_new`）。
    *   这样，`G` 条轨迹的唯一变量就是技能版本，实现了受控对比。每条轨迹同时服务于三重目的：提供GRPO梯度用于**策略优化**、存入经验池用于**技能诊断**、更新后验用于**版本评估**。

2.  **RL循环式技能创建（RL-in-the-Loop Skill Creation）**：
    *   **经验池（Experience Reservoir）**：持续收集训练过程中的成功/失败轨迹对。
    *   **断言驱动的诊断（Assertion-Driven Diagnosis）**：通过一组动态更新的规则断言（Assertions），对经验池中的轨迹进行批量评估，生成“断言的通过率”报表。一个LLM分析器结合通过率和轨迹样本，输出更广泛的失败模式诊断，并指导技能修订。这比逐条轨迹诊断更全面、高效。
    *   **历史感知的技能修订（History-Informed Skill Revision）**：结合当前技能库、诊断结果和历史技能提案的接受/拒绝记录，一个“技能创建智能体”提出新的技能版本 `S_new`（包含新增、修改、删除等操作），并对新技能的触发率进行预验证。

3.  **RL引导的技能演化与汤普森采样（RL-Guided Skill Evolution with Thompson Sampling）**：
    *   **目标**：在策略持续变化的多个训练步中，评估哪个技能版本（ `S_old` vs `S_new`）对当前策略更有效。
    *   **自适应折扣后验更新**：
        *   `S_old` 和 `S_new` 被视为两个臂（arms），其成功率服从Beta分布。
        *   在每个训练步 `t`，一个自适应折扣因子 `w_t = 1 / (1 + n_t / M)` 被用来衰减历史计数。这里 `n_t` 是该步分配给该版本的样本数，`M` 是记忆参数（由算法基于预测似然自动优化）。`w_t` 机制确保：当策略变化时，最新的、与当前策略最相关的观察对决策影响更大，而陈旧的证据会被折扣。
    *   **汤普森采样版本分配**：
        *   基于当前折扣后的Beta后验，计算 `S_new` 优于 `S_old` 的概率 `Pr(p_new > p_old)`，并裁剪出一个探索边界 `ε_ts`。
        *   该概率作为该步分配给 `S_new` 的比例，每个rollout独立地从这个分类分布中采样技能版本。
        *   这实现了探索与利用的平衡：更有希望的版本获得更多采样，而探索边界保证了另一版本也能被充分评估。
    *   **接受/拒绝决策**：在一个评估周期结束时，比较折扣后后验的期望值 `E[p_new]` 和 `E[p_old]`。若前者更高，则接受新版本 `S_new`，否则回退到 `S_old`。

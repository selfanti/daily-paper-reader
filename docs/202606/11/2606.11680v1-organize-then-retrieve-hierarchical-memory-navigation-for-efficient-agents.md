## 摘要
大语言模型（LLM）智能体因其固有的无状态性，在处理长时任务时面临困难，需要将所有任务相关信息编码到不断增长的输入上下文中，导致推理质量下降、推理成本增加和延迟升高，因此需要高效的工作记忆机制。然而，现有方法要么依赖有损压缩，要么依赖基于相似度的检索，这些方法往往无法捕捉多步智能体任务所需的时间结构和因果依赖关系。在这项工作中，我们提出了HORMA，一种层级组织与检索记忆智能体，它将经验组织成类文件系统的层级结构，其中总结的实体与对应的原始轨迹相链接，从而在保留详细信息的同时实现高效访问。HORMA将工作记忆分解为两个阶段：结构化记忆构建和基于导航的检索。构建模块通过区分由信息缺失导致的失败和由上下文误导或过载导致的失败，迭代地优化经验的组织方式。导航模块通过一个使用强化学习训练的轻量级智能体遍历层级结构，选择最小但充分的上下文，从而降低关键执行路径上的延迟。在ALFWorld、LoCoMo和LongMemEval上，HORMA在受约束的上下文预算下提升了任务性能，同时在长对话任务中仅需基线token用量的至多22.17%。与现有方法相比，它持续实现更好的效率-性能权衡，并能有效泛化到未见过的任务。

## Abstract
Large language model (LLM) agents struggle with long-horizon tasks due to their inherent statelessness, requiring all task-relevant information to be encoded in growing input contexts. The resulting degraded reasoning quality, increased inference cost, and higher latency necessitate efficient working memory mechanisms. However, existing approaches either rely on lossy compression or similarity-based retrieval, which often fail to capture temporal structure and causal dependencies required for multi-step agentic tasks. In this work, we present HORMA, a Hierarchical Organize-and-Retrieve Memory Agent that organizes experience into a file-system-like hierarchical structure, where summarized entities are linked to the corresponding raw trajectories, enabling efficient access without losing detailed information. HORMA decomposes working memory into two stages: structured memory construction and navigation-based retrieval. The construction module iteratively refines how experiences are structured by distinguishing between failures caused by missing information and those caused by misleading or overloaded context. The navigation module retrieves task-relevant context by traversing the hierarchy using a lightweight agent trained with reinforcement learning to select minimal yet sufficient context, thereby reducing latency along the critical execution path. Across ALFWorld, LoCoMo, and LongMemEval, HORMA improves task performance under constrained context budgets while requiring at most 22.17% of the baseline token usage in long conversation tasks. Compared to existing methods, it consistently achieves better efficiency-performance trade-offs and generalizes effectively to unseen tasks.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：LLM 智能体在处理长时任务时，由于**无状态性**，所有任务相关信息必须编码在不断增长的上下文窗口中，导致：
  - 推理质量下降（信息稀释、失准）
  - 推理成本增加（token 消耗大）
  - 延迟升高（上下文长）
- **现有方法的局限**：
  - **历史囤积**：直接保留全部历史（如 ReAct），导致上下文过载、信息稀释。
  - **有损压缩**：归纳总结、上下文折叠等，不可逆地丢失下游推理所需的细粒度信息。
  - **扁平外部记忆**：基于语义相似度检索的扁平存储，缺少时间层次和因果依赖，检索结果常出现时间不一致或上下文不相关。
- **研究动机**：需要一种既保留关键细节又能高效检索的工作记忆机制，且能捕捉时序结构和因果依赖。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
### 2.1 核心思想：解耦记忆构建与检索
- 将工作记忆分解为两个专门模块：**记忆管理模块（Memory Manager）** 和 **检索模块（Retrieval Agent）**。
- 两者通过共享的**类文件系统层级工作空间**交互（由 bash 命令操作），实现可解释、可追踪的记忆操作。

### 2.2 关键技术细节
#### (1) 记忆构建模块
- **初始化**：使用具有强层级推理能力的 LLM（如 Claude Sonnet 4.5）作为记忆管理器，给定领域无关的组织原则（实体跟踪、事件抽象、关系分组）。
- **递归技能细化**：
  - 对比“使用原始历史 H” vs “使用 HORMA 管理上下文 H′”的任务结果。
  - 定义两种失败类型：
    - **外生失败（exogenous）**：H 成功但 H′ 失败 → 记忆构建丢失信息。
    - **内生失败（endogenous）**：H 失败但 H′ 成功 → 结构化记忆缓解了幻觉、中间丢失等问题。
  - 基于对比生成反馈 → 迭代更新记忆管理技能库（prompt 文本梯度下降）。
- 技能以自然语言形式存储，可跨任务复用。

#### (2) 检索模块
- **导航式检索**：检索 agent 在文件系统层级中使用 bash 工具（`ls`, `cd`, `grep`, `cat`）遍历，并新增两个终端动作：
  - `select`：将验证过的内容加入检索上下文。
  - `done`：终止检索。
- **RL 优化**：使用 **GRPO**（Group Relative Policy Optimization）训练轻量级检索 agent（如 Qwen 3.5 4B）。
- **奖励函数**：基于检索上下文 \( C_t \) 与真实证据 \( E \) 的 Jaccard 重叠度：
  \[
  J(C_t, E) = \frac{|C_t \cap E|}{|C_t \cup E|}
  \]
  鼓励精确且紧凑的上下文。
- **整体策略**：主 agent 仅基于检索到的上下文 \( C_t \) 执行动作：
  \[
  \pi(a_t | o_t, F_t, q) = \text{Retrieval}(C_t | F_t, q) \cdot \text{MainLLM}(a_t | o_t, C_t, q)
  \]

## 7. 优点
- **创新性**：首次明确解耦记忆构建与检索，避免 RL 端到端训练中的信用分配难题。
- **可解释性**：使用类文件系统结构，所有操作（ls, cd, cat 等）均可记录和审计。
- **实用性**：检索 agent 轻量（4B 参数）、训练成本低、可泛化，适用于生产环境。
- **实验完备**：覆盖交互式、对话式两种典型长时场景，分析多角度（效率、性能、错误、瓶颈）。

## 8. 不足与局限
- **记忆构建依赖大模型**：实验表明记忆管理器需要强大 LLM（如 Claude 4.5），小模型效果差，导致成本较高（若使用闭源 API）。
- **技能进化需要历史轨迹**：对比分析需要成功/失败轨迹对，初期可能缺乏数据；技能库质量依赖初始 prompt 和反馈生成质量。

- **环境限制**：所有实验均在文本模拟环境（ALFWorld、对话数据），未在真实物理环境或多步骤网页操控等更复杂场景测试。

- **记忆构建未参与端到端优化**：采用非参数技能更新，未与主 agent 联合优化，可能不是全局最优。

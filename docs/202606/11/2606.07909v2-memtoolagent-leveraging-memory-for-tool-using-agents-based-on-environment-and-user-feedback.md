## 摘要
现代大语言模型智能体可以使用外部工具帮助用户解决复杂任务。然而，对于需要从长期历史事件或先前智能体-环境交互中学习的问题，智能体需要使用记忆机制来存储和检索经验。尽管对话智能体存在复杂的记忆系统，但很少有研究通过过去的用户-智能体对话来实证性地探讨如何提升智能体的工具使用能力。我们提出MemToolAgent，一种通过记忆管理改善工具使用的框架。我们的方法包含一个记忆提取模块，将过往经验处理为结构化记忆条目，以及一个检索模块，动态选择存储记忆条目的子集。这使得无需微调大语言模型即可实现与用户偏好和反馈更一致、更个性化和准确的响应。总之，本工作有三项主要贡献：(1) 一种统一的记忆条目格式，无需微调大语言模型即可改进通用和个性化工具使用；(2) 基于反思的记忆提取，利用环境和用户反馈将错误执行提炼为批判性内容进行存储；(3) 一个基于记忆相似度分布选择使用多少过往经验的检索模块。与WorkBench、NESTFUL和PEToolBench基准上的强基线相比，MemToolAgent分别实现了29%、80%和17%的相对改进。

## Abstract
Modern large language model (LLM) agents can use external tools to help users solve complex tasks. However, for problems that require learning from long-term historical events or from previous agent-environment interactions, LLM agents are required to use memory mechanisms to store and retrieve experiences. While sophisticated memory systems exist for dialogue agents, few studies have empirically examined how to improve agents' tool-using capabilities through past user-agent conversations. We propose MemToolAgent, a framework that improves tool use through memory management. Our approach contains a memory extraction module that processes past experiences into structured memory entries, and a retrieval module that dynamically selects a subset of the stored memory entries. This enables more personalized and accurate responses aligned with user preferences and feedback without requiring LLM fine-tuning. In summary, this work has three main contributions: (1) a unified memory entry format that improves both general-purpose and personalized tool use without LLM fine-tuning, (2) a reflection-based memory extraction that uses environment and user feedback to distill wrong executions into critiques to store, and (3) a retrieval module that chooses how many past experiences to use based on the memory similarity distribution. MemToolAgent achieves 29%, 80%, and 17% relative improvements compared to strong baselines on the WorkBench, NESTFUL, and PEToolBench benchmarks, respectively.

# MemToolAgent 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大语言模型智能体（LLM agents）通过外部工具解决复杂任务，但缺乏长期记忆机制。固定上下文窗口限制了维护连贯性和主题连续性的能力，特别是当对话序列包含大量无关信息时，注意力机制效能下降，这是教育、医疗等关键场景中不希望出现的。因此，需要为智能体整合记忆机制，使得可以从过去的智能体-环境交互中学习，避免重复错误，并实现个性化。
- **整体含义**：MemToolAgent 是一个无需微调 LLM 的记忆管理框架，旨在提升工具使用的准确性和个性化。它通过结构化记忆条目存储过往经验（包括用户查询、工具调用序列、二元用户反馈以及生成的反省/批评），并动态检索最相关的记忆条目作为上下文示例来指导当前任务。在 WorkBench、NESTFUL 和 PEToolBench 三个基准上分别相对强基线提升 29%、80% 和 17%。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 利用过去交互经验（记忆条目）作为上下文示例（in-context learning），通过结构化存储和动态检索改善工具使用能力，无需对 LLM 进行微调。
- 分为两个主要模块：记忆提取（Memory Extraction）和记忆检索（Memory Retrieval）。

### 2.2 关键技术细节
- **记忆条目格式**：每个记忆条目表示为元组 `m = (q, a, f, r)`，其中：
  - `q`：用户查询（query）
  - `a`：工具调用序列 `{a1, a2, ..., an}`
  - `f`：二元用户反馈（1 成功，0 失败）
  - `r`：反射/批评（当 `f=0` 时由提取模块生成，否则为 `∅`）
- **记忆检索模块（Dynamic "n" Top-n Embedding-based Retrieval）**：
  - 并非固定检索 k 条记忆，而是动态决定检索数量 n。
  - 步骤：
    1. 计算查询与每条记忆的余弦相似度，降序排列得到数组 x。
    2. 基于滑动窗口计算相似度的一阶导数估计（first-order derivative estimate），参数 R 为半径。
    3. 使用 SciPy 的 `find_peaks` 函数寻找导数估计的峰值（代表相似度突降处），基于峰值选择 n（可配置 prominence 和 peak number 参数）。
  - 相比固定 top-k，动态方法能根据查询相关性的分布自适应选择恰当数量的记忆，避免信息过少或冗余。
- **记忆提取模块**：
  - 利用 LLM 对失败的执行进行反省（reflection/critique），作为结构化反馈存入记忆。
  - 输入包括：当前查询、可用工具、智能体响应（工具调用序列）、环境反馈、用户反馈以及已检索到的相关记忆条目。
  - 对通用工具使用：主要生成对错误步骤的批评，例如参数错误、顺序错误、条件判断错误等。
  - 对个性化工具使用：提取用户偏好模式，如“用户倾向于使用 API_X 下的工具”。
  - 如果不进行反省而直接存储原始轨迹，智能体同时理解当前任务和过去错误会更困难。反省步骤分离后，智能体只需专注于当前任务并从错误中学习。

### 2.3 算法流程（文字说明）
- **推断时**：
  - 用户提交查询 → 检索模块计算查询与记忆库中所有条目的余弦相似度，经过动态 top-n 选择 n 个最相关记忆条目 → 将这些条目（包含查询、工具调用、用户反馈和反省）作为上下文示例注入系统提示词 → LLM 生成工具调用序列 → 执行后得到结果。
- **记忆更新**（当执行后获得用户反馈为失败时）：
  - 调用记忆提取模块，基于当前查询、智能体响应、环境反馈、用户反馈和相关记忆条目，生成一条反省/批评，连同查询、工具调用、用户反馈一同存入记忆库。

## 3. 实验设计

### 3.1 数据集 / 场景
- **通用工具使用（General-purpose tool use）**：
  - **WorkBench**：模拟工作场景，包含 5 个数据库、27 个工具，任务涵盖分析、日历、CRM、邮件、多域等。使用准确率（任务结果与期望结果匹配的百分比）作为指标。
  - **NESTFUL**：评估嵌套工具调用（输出作为下一个输入的参数）的能力。基于 MathQA 和 StarCoder2-Instruct 构建，1891 个测试查询；使用 F1 函数、F1 参数、部分序列匹配、全序列匹配和胜率（Win Rate）五个指标。
- **个性化工具编排（Personalized tool orchestration）**：
  - **PEToolBench**：每个查询附带 80-100 个历史用户查询（含二元评分）。测试模型能否根据用户偏好选择正确工具。评估指标为工具准确率和参数准确率。

### 3.2 对比方法
- **通用工具使用**：对比三个基线：
  - No Memory（无记忆）
  - Extended Thinking（扩展思考，让模型更仔细推理）
  - ReAct Agent（推理-行动循环）
- **个性化工具编排**：对比两个最新的记忆架构：
  - A-MEM (Agentic Memory)
  - Mem0
  以及其他变体如 Embedding-based Retrieval_v1/v2、Hard Preference Guiding 等。

### 3.3 实验设置
- LLM 模型：WorkBench 和 NESTFUL 使用 Claude 3.7 Sonnet；PEToolBench 使用 Claude 3.5 Sonnet。
- 嵌入模型：Qwen3-Embedding-8B。
- 记忆检索超参数：R=10, prominence=10^{-5}，peak number 随任务手动调整（PEToolBench 使用第二个峰值，其余使用第一个峰值）。

## 6. 主要结论与发现

- MemToolAgent 在通用工具使用（WorkBench 整体准确率从 57.56% 提升至 85.06%）和个性化工具编排（PEToolBench 工具准确率从 0.700 提升至 0.820）上均显著优于无记忆基线和现有记忆架构。
- 动态 top-n 检索优于固定 top-k，能够根据查询相关性分布自适应选择记忆数量。
- 使用记忆提取模块生成反省/批评比直接存储原始执行轨迹更有效，尤其当经验包含错误时，帮助智能体更容易地从错误中学习。
- 在个性化场景中，简单的硬偏好引导（Hard Preference Guiding，直接禁用用户反馈为负的工具）能作为记忆提取模块的低成本替代，进一步提升工具准确率。
- MemToolAgent 增加输入 token 但输出 token 增加较少，相比 Extended Thinking 和 ReAct 更节省输出成本（输出 token 往往更贵且延迟更大）。
- 合成查询可用于缓解冷启动问题，尽管对复杂多域任务效果有限。

## 7. 优点

- **无需微调**：完全基于上下文学习，适应性强，容易集成到现有 LLM 智能体。
- **统一记忆格式**：结构化存储查询、工具调用、反馈和反省，兼顾通用和个性化场景。
- **基于反思的记忆提取**：将复杂错误轨迹提炼为简洁评论，降低 LLM 在推理时的认知负荷。
- **动态检索机制**：基于相似度导数峰值选择记忆数量，避免固定 k 的缺点，对新查询更鲁棒。
- **注重用户反馈**：利用二元反馈作为信号，可扩展至文本或评分反馈。
- **结果显著**：在三个不同性质基准上取得大幅度提升，验证了方法的广泛有效性。
- **成本分析全面**：提供了 token 使用量对比，指出输出 token 节省的优势。

## 8. 不足与局限

- **冷启动问题**：在没有足够过往经验的初始阶段，记忆库为空，性能可能下降。论文尝试合成查询但仅对简单任务有效，复杂场景下合成查询质量不足。
- **基准工具数量有限**：实验中工具数量少于 40 个，现实应用可能包含数百个工具，记忆库覆盖和检索难度会剧增，冷启动问题更严重。
- **单轮任务限制**：基准均为单轮用户查询，忽略了多轮对话中需要持续更新记忆的需求。论文提及可扩展到每用户轮后检索，但未实现验证。
- **二元反馈过于简化**：仅区分成功/失败，未利用更细粒度的反馈（如文本解释或分数），可能丢失信息；但作者声称此设计可扩展。
- **实验单次运行**：未报告多次重复实验的均值和方差，结果稳定性存疑。
- **仅使用 Claude 系列模型**：未测试其他 LLM（如 GPT-4、Llama 系列），通用性未充分验证。
- **动态 top-n 的超参数需手动调整**（R、prominence、peak number），任务间不一致，鲁棒性有待提升。
- **隐私与安全风险**：存储用户查询和偏好可能带来隐私泄露风险，论文未讨论相关机制。

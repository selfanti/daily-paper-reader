## 摘要
记忆已成为自我进化智能体的标准基础，但保留经验并不等同于学习如何通过经验进行进化。现有的记忆智能体可以存储轨迹、检索反思或积累技能，但通常缺乏整体能力来选择有用的经验、据此行动、编写可复用的知识以及维护不断增长的知识库。我们提出OPD-Evolver，一个慢-快协同进化框架，通过同策略自蒸馏来培养这样一种智能体进化器。在快循环中，OPD-Evolver与四级记忆层次结构交互，以读取、使用、写入和维护经验，实现快速的测试时进化。在慢循环中，经过结果校准的记忆归因和特权后见之明将这四种能力蒸馏到可部署的策略中。在跨领域基准测试中，OPD-Evolver超过了ReasoningBank等记忆系统高达11.5%，以及Skill0等基于训练的方法约5.8%。进一步分析表明，OPD-Evolver内化了高价值经验和记忆管理，使OPD-Evolver-9B能够挑战Qwen3.5-397B-A17B和Step-3.5-Flash等巨大的对手，指向超越记忆增强的智能体，迈向真正合格的智能体进化器。

## Abstract
Memory has become a standard substrate for self-evolving agents, yet retaining experience is not the same as learning how to evolve through it. Existing memory agents can store trajectories, retrieve reflections, or accumulate skills, but often lack the holistic competence to select useful experience, act on it, write reusable knowledge, and maintain a growing repository. We introduce OPD-Evolver, a slow-fast co-evolution framework that cultivates such an agent evolver through on-policy self-distillation. In the fast loop, OPD-Evolver interacts with a four-level memory hierarchy to read, use, write, and maintain experience for rapid test-time evolution. In the slow loop, outcome-calibrated memory attribution and privileged hindsight distill these four abilities into the deployable policy. Across multi-domain benchmarks, OPD-Evolver surpasses memory systems such as ReasoningBank by up to 11.5%, and training-based methods such as Skill0 by ~5.8%. Further analysis shows that OPD-Evolver internalizes high-value experience and memory management, enabling OPD-Evolver-9B to challenge giant counterparts such as Qwen3.5-397B-A17B and Step-3.5-Flash, pointing beyond memory-augmented agents toward genuinely qualified agent evolvers.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
采用**慢-快协同进化框架**，将在线经验交互与离线蒸馏相结合，通过同策略（on-policy）自蒸馏将四种演化能力内化到部署策略中。

### 关键技术细节

#### （1）四级记忆层次结构（Fast Loop 中的记忆组织）
| 层级 | 内容 | 目的 |
|------|------|------|
| Trajectory | 完整轨迹 | 保留情节证据 |
| Tip | 局部警告或启发式 | 存储轻量级提示 |
| Skill | 可复用过程 | 抽象出可转移技能 |
| Tool | 可执行代码模板 | 工具化经验 |

#### （2）快循环（Fast Evolution Loop）
- **检索与选择**：基于任务查询嵌入余弦相似度从每个层级检索Top-K候选记忆；然后由**选择器**（Selector）压缩为紧凑上下文。
- **执行**：策略以紧凑上下文和任务历史为条件生成动作。
- **写入**：同一策略决定向哪些层级写入多少新记忆。
- **维护**：每Q个任务进行一次仓库级维护（查找、合并、删除），通过工具调用执行。

#### （3）慢循环（Slow Evolution Loop）
- **结果校准归因**：通过比较记忆被选中时的平均奖励与未被选中的平均奖励，计算每个记忆的价值分数 V(m)。公式中利用了任务组、置信度因子等，将终端奖励反向传播到选择、使用和写入决策。
- **统一后见自蒸馏**：为每个生命周期决策（选择、执行、写入、维护）构造学生输入 z_k 和教师特权上下文 h_k。学生生成同策略输出，教师基于相同前缀计算分布，最小化token级KL散度。
  - **选择蒸馏**：教师提供每个候选记忆的价值，让学生学习筛选。
  - **执行蒸馏**：教师提供正价值记忆和成功轨迹，让学生内部化。
  - **写入蒸馏**：教师提供已产生记忆的未来价值，让学生学会写高质量经验。
  - **维护蒸馏**：教师提供仓库级诊断（价值、置信度、冗余度），让学生学习合并/删除操作。

#### 算法流程（Algorithm 1）
1. 任务流到来，对每个任务进行快循环：检索 → 选择 → 执行 → 写入。
2. 每Q个任务进行维护。
3. 收集日志后，对每个记忆估计价值。
4. 针对k∈{sel, act, write, maint}构造训练样本，执行同策略蒸馏，更新策略参数θ。
5. 最终部署学生策略。

1. **OPD-Evolver显著优于现有方法**：在10个子集上超越所有同骨干记忆基线，最大提升达11.5%（vs ReasoningBank）；超越训练方法5.8%（vs Skill0）。

6. **所有四个组件均不可或缺**：消融显示移除任意一个能力（选择、写入、维护、归因）均导致性能下降，其中移除归因或慢进化伤害最大。

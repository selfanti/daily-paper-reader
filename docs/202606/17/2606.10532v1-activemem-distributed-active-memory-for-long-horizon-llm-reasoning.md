## 摘要
记忆对于使大型语言模型（LLM）智能体能够处理长程推理任务至关重要。现有的记忆机制大多是集中式的，通常将检索到的信息和交互历史组织在单一模型上下文中。这种设计带来了一个根本性的权衡：扩展推理轨迹可能导致上下文过载，而激进的修剪内容则可能造成不可逆的信息损失。为了寻求更好的权衡，我们从人类认知系统中汲取灵感，特别是前额叶皮层（执行控制）与海马体（记忆管理）之间的功能互补性，表明这种权衡并非固有，而可能源于集中式的记忆组织。为此，我们提出了ActiveMem，一个异构框架，将智能体记忆与核心推理过程解耦。具体来说，一个高层规划器利用蒸馏后的语义要点进行推理，同时一个轻量级的分布式记忆系统并行运作，在整个任务过程中主动积累和巩固这些要点。在BrowseComp-Plus和GAIA上的实验表明，ActiveMem以显著降低的开销实现了最先进的准确性，证明了分布式主动记忆在长程推理中的有效性。

## Abstract
Memory is essential for enabling large language model (LLM) agents to handle long-horizon reasoning tasks. Existing memory mechanisms are largely centralized, typically organizing retrieved information and interaction history within a single model context. This design imposes a fundamental trade-off: scaling reasoning trajectories risks context overload, whereas aggressive content pruning may result in irreversible information loss. Seeking a better trade-off, we draw inspiration from human cognitive systems, especially the functional complementarity between the prefrontal cortex (executive control) and the hippocampus (memory management), suggesting that such a trade-off need not be inherent, but may instead stem from centralized memory organization. To this end, we propose ActiveMem, a heterogeneous framework that decouples agent memory from the core reasoning process. Specifically, a high-level Planner utilizes distilled semantic gists to execute reasoning, while a lightweight, distributed memory system operates in parallel to actively accumulate and consolidate these gists throughout the task. Experiments on BrowseComp-Plus and GAIA show that ActiveMem achieves state-of-the-art accuracy with significantly reduced overhead, demonstrating the effectiveness of distributed active memory for long-horizon reasoning.

- **整体含义**：提出**ActiveMem**设计范式——将智能体记忆与核心推理解耦，通过分布式主动记忆架构，在保持推理上下文紧凑的同时避免信息丢失，实现更优的准确率-效率折衷。

## 2. 论文提出的方法论

### 2.1 核心思想
- 采用**异构框架**，将记忆系统从推理过程中分离：高层**Planner**仅维护一个紧凑的推理上下文（原始问题x、修剪后的最近K步交互历史ht、上一步的蒸馏记忆m_{t-1}），生成检索查询；轻量级**分布式记忆系统**并行运作，负责主动积累、蒸馏和存储语义要点。

### 2.2 关键技术细节

#### 2.2.1 Planner（规划器）
- 维护状态 \( s_t = (x, h_t, m_{t-1}) \)。
- 通过`Trim`操作仅保留最近K步交互步骤：\( h_t = \text{Trim}(h_{t-1} \cup \{a_t, o_t\}) \)。
- 将状态映射为检索查询集 \( Q_t = \pi(s_t) \)，每个查询指定检索预算k_i（所需文档数）。
- 在收到蒸馏要点后更新状态，或直接提交最终答案。

#### 2.2.2 分布式记忆系统

- **Memory Shards（记忆分片）**：固定数量的持久化键值存储层。每个分片B_j以文档c为键，存储其蒸馏要点g_c和查询历史H_c：\( B_j[c] = \langle g_c, H_c \rangle \)。逻辑隔离，避免不同文档信息混杂。
- **Memorizers（记忆提取器）**：轻量级模型（4B参数），受海马体启发，给定查询q和文档c，生成查询条件化要点 \( g_c = \omega(c, q) \)。若文档与查询不相关则输出空（∅）。训练：从实际Agent轨迹中收集12,000个查询-文档对，使用gpt-oss-120b作为教师生成目标要点，用条件下一个token预测损失 \( \mathcal{L}_{\text{SFT}} = -\mathbb{E}_{(q,c,g_c)\sim\mathcal{D}} \sum_t \log p_\omega(g_{c,t}|q,c,g_{c,<t}) \) 进行微调。
- **Operator（操作器）**：控制层，负责路由、内存命中判断（根据语义相似性J(q, H_c) ∈ {SIMILAR, NEW}）和异步合并（Consolidate）。内存命中时可以复用已有要点，避免重复蒸馏；若为新查询则触发新蒸馏并将结果异步合并到分片。

#### 2.2.3 推理流程（Algorithm 1）
1. 初始化所有记忆分片，将系统提示和输入问题设为对话历史。
2. 对于每一步t：
   - Planner观察状态s_t并生成查询集Q_t，若为submit_answer则返回答案。
   - 检索文档集D_t，通过ROUTE分配每个文档到稳定分片。
   - 并行处理最多N个文档（N等于分片数）：
     - 若文档不在分片中（miss）：蒸馏新要点并写入分片。
     - 若文档在分片且查询语义相似（hit）：直接复用存储的要点。
     - 若文档在分片但查询不同（hit with new query）：蒸馏新要点并异步合并。
   - 收集所有要点G_t，序列化为观测o_t，追加到历史后修剪保留最近K步。

- **LasJ（LLM-as-a-Judge）**：使用Qwen3.5‑397B‑A17B作为评判模型衡量答案正确性。

## 6. 论文的主要结论与发现

1. **ActiveMem在所有基准上取得SOTA准确率-计算开销折衷**：BrowseComp-Plus总LasJ 0.79（最优），ACT 0.785；GAIA LasJ 0.62，PFLOPs仅187（最低）。
2. **解耦记忆与推理有效缓解集中式权衡**：通过并行Memorizers将大部分token处理转移到轻量模型（Memorizer占68.9%总PFLOPs），Planner上下文保持紧凑（仅28.3%计算）。
3. **难度越高优势越明显**：在BrowseComp-Plus Hard上ActiveMem相对最好基线提升22.2%绝对LasJ（0.36→0.44），体现长程、多约束任务中分布式记忆的价值。
4. **Memory Shards减少冗余并保留信息**：避免重复蒸馏同一文档，且不同查询下的要点可复用或合并。
5. **更强的Memorizer提升推理质量**：经过SFT训练的4B模型输出更精准要点，直接降低Planner计算消耗并提高最终准确率。
6. **记忆巩固保持上下文清洁**：异步合并防止分片过度增长，使Planner始终操作于简洁、无冗余的记忆。

## 7. 优点

- **神经科学启发的架构创新**：将前额叶-海马体功能类比映射为Planner-记忆系统解耦，设计新颖且具生物可解释性。
- **高效的分布式并行处理**：Memorizers可并行运行，Operator复用已蒸馏要点，异步合并不阻塞推理，极大提升资源利用率。
- **指标设计合理**：提出PFLOPs作为硬件无关计算开销度量，修正了混合注意力/动态稀疏注意力的计算

## 8. 缺点与局限性

- **任务域局限性**：所有实验仅在WebSearch类基准（BrowseComp-Plus和GAIA）上进行，未在数学推理、代码生成或常识问答等典型长程推理场景中验证。ActiveMem的分布式记忆架构是否泛化至非检索密集型任务尚不明确。
- **LLM-as-a-Judge评估偏差**：使用Qwen3.5-397B-A17B作为评判模型衡量答案正确性可能引入系统性偏好（如对自身输出或类似风格答案的偏向），且未进行人类标注验证或跨评判模型的一致性检验。
- **PFLOPs度量的近似性**：PFLOPs虽为硬件无关计算开销度量，但未考虑实际硬件中的内存带宽、延迟、并行效率等因素（如A100上MoE模型的通信开销、动态序列的注意力计算实际利用率等），因此ACT指标与真实部署成本之间存在差距。
- **扩展性隐含假设**：Shards数量设为固定值（实验中为8个），Memorizer仅4B参数。当任务规模极大增加（如千级文档检索）时，固定分片可能成为瓶颈，且分布式系统同步与争用开销未在文中分析。
- **训练数据来源与教师模型依赖**：Memorizer的SFT训练数据（12,000条）来自实际Agent轨迹，但使用gpt-oss-120b作为教师模型生成要点目标，存在教师模型自身的知识局限与生成偏差；且微调数据量较小，可能不足以覆盖分布外查询。
- **记忆合并策略细节欠析**：Operator中的异步合并（Consolidate）仅简要描述为“合并冗余要点”，未给出具体算法、阈值或消融实验中的合并频率控制，可复现性受影响。

## 9. 个人评价与展望

ActiveMem提出的分布式主动记忆架构在长程LLM推理领域中开辟了一条新颖且有效的技术路线。其核心贡献在于通过神经科学启发的功能解耦，从根本上避免了集中式记忆带来的上下文过载与信息丢失之间的权衡。实验结果表明，在检索密集型任务中，该方法在准确率和计算效率上均优于现有集中式方法，尤其在高难度场景下优势显著。

然而，该工作当前局限于检索主导的WebSearch类任务，且评估指标和训练设置尚存在若干可改进之处。未来方向可包括：将ActiveMem扩展到多模态长程推理（如视频问答、多步工具使用）、探索更高效的自适应记忆分片策略（如根据查询复杂度动态调整分片数）、以及将Memorizer的训练与Planner的决策进行端到端联合优化。此外，在实际部署中，分布式系统的通信开销与内存占用仍需进一步量化与优化。

总体上，ActiveMem为构建具备持久、高效记忆能力的智能体提供了一种极具潜力的范式，值得在更广泛的长程推理任务中验证与推广。

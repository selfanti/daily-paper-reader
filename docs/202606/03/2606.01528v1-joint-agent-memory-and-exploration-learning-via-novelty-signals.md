## 摘要
在开放环境中，探索是自主智能体的基础，但当前的语言模型智能体在这方面存在困难。有效探索需要记忆，但保留原始交互历史在长轨迹中计算成本高昂。虽然潜在记忆提供了压缩交互历史的解决方案，但其训练缺乏可靠的监督信号。我们提出了联合智能体记忆与探索学习（JAMEL），这是一个通过新奇驱动交互同时训练智能体记忆和探索策略的框架。我们观察到记忆和探索形成了一个相互依赖的循环：持续的探索需要记忆来区分已耗尽的行为和未见过行为，而寻求新奇的交互则为使记忆对未来探索有用提供了所需的监督。通过利用确定性且持久的信号，如GUI领域的代码覆盖率，我们为记忆模块提供了自然的、无需标注的监督。实证评估表明，我们的方法成功泛化到未见环境。其探索能力优于开源基线，并与闭源模型的探索深度相当，同时减少了token消耗。我们的代码和模型已在 https://github.com/MobileLLM/JAMEL 开源。

## Abstract
In open-ended environments, exploration is fundamental for autonomous agents, yet current language model agents struggle with this. Effective exploration requires memory, but retaining raw interaction histories is computationally expensive over long trajectories. While latent memory offers a solution to compress interaction histories, its training lacks reliable supervisory signals. We introduce \textbf{J}oint \textbf{A}gent \textbf{M}emory and \textbf{E}xploration \textbf{L}earning (\textbf{JAMEL}), a framework that trains agentic memory and exploration policy together through novelty-driven interaction. We observe that memory and exploration form a mutually dependent loop: sustained exploration requires memory to distinguish exhausted behaviors from unseen ones, while novelty-seeking interaction provides the supervision needed to make memory useful for future exploration. By utilizing deterministic and persistent novelty signals such as code coverage in the GUI domain, we provide natural, annotation-free supervision for the memory module. Empirical evaluations demonstrate that \ours successfully generalizes to unseen environments. Its exploration capability outperforms open-weight baselines and rivals the exploration depth of a closed-source model while reducing token consumption. Our code and model are open-sourced at https://github.com/MobileLLM/JAMEL.

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：在开放环境中，语言模型（LLM）智能体缺乏有效的探索能力。即使遇到意外但任务相关的信息，它们也常常无法据此采取行动。深层原因在于训练方式而非推理配置。
- **记忆的困境**：有效探索需要记忆（区分已探索与未探索行为），但保留完整的交互历史在长轨迹上计算成本过高。潜在记忆（latent memory）可以压缩历史，但缺乏可靠的逐步骤监督信号——我们不知道每个记忆状态应该编码什么，也不知道策略应如何利用它们。
- **关键洞察**：探索和记忆是相互依赖的循环。探索需要记忆避免重复，而探索过程本身可以提供记忆的监督（因为只有利用历史信息发现新行为才能获得新奇奖励）。这一观察自然构成了联合训练的基础。

## 2. 方法论

### 核心思想
- **JAMEL（Joint Agent Memory and Exploration Learning）**：通过新奇信号联合训练智能体的潜在记忆模块和探索策略。利用确定性、持久的新奇信号（如GUI领域的代码覆盖率）作为无需标注的监督信号，最大化累加新奇奖励。

### 关键技术细节

1. **问题建模**：
   - 将探索问题建模为有限视界部分可观察马尔可夫决策过程（POMDP）。每一步获得新奇奖励 \( r_t \)，其值取决于当前状态是否带来新的行为（代码路径）。探索目标为最大化累积新奇奖励。

2. **模型架构**：
   - **历史压缩器**：使用冻结的视觉语言模型 \( F_\phi \)（Qwen3-VL-2B-Instruct）。将每个历史步骤的观测-动作对序列化后输入，取EOS token的隐藏层状态作为该步骤的压缩表示（一个记忆token）。
   - **记忆状态**：\( M_t = [h_1, ..., h_{t-1}] \in \mathbb{R}^{(t-1) \times d_c} \)。
   - **线性对齐器**：可学习的线性层 \( W \in \mathbb{R}^{d_c \times d_{LM}} \) 将记忆token投影到策略模型的嵌入空间。
   - **策略解码**：投影后的记忆前缀与当前观测的嵌入拼接，输入策略模型（Qwen2.5-VL-7B-Instruct）生成动作。

3. **新奇奖励设计**：
   - 使用应用代码覆盖率作为持久的新奇信号。定义覆盖得分 \( C(t) = \text{cov}_{\text{lines}} + \text{cov}_{\text{branches}} + \text{cov}_{\text{statements}} + \text{cov}_{\text{functions}} \)。
   - 奖励 \( r_t = 1 \) 当且仅当 \( C(t) > C(t-1) \)，否则为0。覆盖基线跨 episode 共享，形成自然课程。
   - 数据收集：使用通用LLM（带全历史上下文）探索应用，记录覆盖奖励。从每个 episode 中保留直到最后一个正奖励步的前缀作为训练样本。共收集86个应用的24k样本。

4. **训练**：
   - 监督微调：最大化给定当前观测和记忆条件下动作的似然。记忆对齐器 \( W \) 与策略模型一同更新。

## 3. 实验设计

### 数据集/场景
- **ScaleWoB**：一个包含96个真实Web应用的基准，覆盖电商、社交、视频、旅行、物流、生产力等。其中86个用于训练，10个用于测试。

### 基准（Benchmark）
- 每个应用执行50步长会话，计算累积累积覆盖奖励（即触发新代码路径的步数总和）。

### 对比方法
- **ReAct-text / ReAct-vision**：基于Gemini 3.1 Flash-Lite的ReAct框架，保留完整历史（文本或文本+截图）。

- 主要指标：累积累积覆盖奖励（50步内正奖励步数）。
- 次要指标：输入token消耗量。

## 7. 优点

1. **新颖的联合训练范式**：将记忆与探索的相互依赖转化为训练信号，解决潜在记忆缺乏监督的痛点。
2. **无需标注的监督**：利用应用代码覆盖率这一确定性、持久的新奇信号，无需人工标注。
3. **架构高效**：使用冻结的压缩器+可学习的线性对齐器，将长历史压缩为少量记忆token，显著降低token消耗。
4. **泛化能力**：在未见应用中表现优异，探索深度媲美闭源模型。
5. **自然课程学习**：奖励稀疏化自动形成由易到难的课程，无需人工设计。

## 摘要
近期的人工智能智能体可以灵活调用技能以解决复杂任务，但其长期改进从根本上受到缺乏系统性的技能构建、积累与迁移的制约。特别是，在没有统一技能整合框架的情况下，智能体往往在不同任务中冗余构建相似能力，无法有效将经验转化为可复用资产，并且难以将任务特定技能泛化到新场景。为解决这一局限，我们提出SkillPyramid，一种通过复用现有技能经验实现更广泛任务泛化的技能整合框架。SkillPyramid运行于层次化的技能拓扑结构之上，进一步引入自我进化机制，使智能体能够在任务执行过程中组合、验证并融入新技能。在ALFWorld、WebShop和ScienceWorld上使用四种骨干模型的实验表明，SkillPyramid平均奖励提升了38.0%，执行步骤减少了27.7%。总体而言，我们的方法将技能集合从静态资源池转变为动态进化系统。

## Abstract
Recent AI agents can flexibly invoke skills to solve complex tasks, but their long-term improvement is fundamentally constrained by a lack of systematic skill construction, accumulation, and transfer. In particular, without a unified framework for skill consolidation, agents tend to redundantly construct similar capabilities across different tasks, are unable to effectively transform experience into reusable assets, and struggle to generalize task-specific skills to novel scenarios. To address this limitation, we propose SkillPyramid, a skill consolidation framework that reuses existing skill experience for broader task generalization. Operating on a hierarchical skill topology, SkillPyramid further introduces a self-evolution mechanism that enables agents to compose, validate, and incorporate new skills during task execution. Experiments on ALFWorld, WebShop, and ScienceWorld across four backbone models show that SkillPyramid substantially increases the average reward by 38.0% and reduces execution steps by 27.7%. Overall, our method transforms a skill collection from a static resource pool into a dynamic evolution system.

## 1. 核心问题与整体含义（研究动机和背景）

**核心问题**：当前基于大语言模型的智能体能够通过调用技能（skills）解决复杂任务，但其长期改进受到严重制约，具体表现为：
- 缺乏系统性的技能构建、积累与迁移机制；
- 不同任务中冗余构建相似能力，无法有效将经验转化为可复用资产；
- 任务特定技能难以泛化到新颖场景；
- 现有方法要么将技能作为孤立单元存储（如ReAct+Skills），要么直接生成新技能而无结构化组织。

**整体含义**：需要一种能够将技能从静态资源池转变为动态演化系统的框架，使技能能够被分解、抽象、复用和组合，从而支持智能体的持续自我进化。

## 2. 方法论

### 核心思想
SkillPyramid 通过构建一个层次化的技能金字塔（hierarchical skill pyramid），将技能按抽象程度分层组织：底层为细粒度的原子技能（atomic skills），顶层为抽象的高层模式（abstract skills）。系统通过关系分析（Relation Analyzer）和关系构建（Relation Builder）两个代理自动发现技能间的复用关系，并利用任务驱动的自我进化机制在任务执行过程中动态生成、验证和融入新技能。

### 关键技术细节

**（1）技能表示**：每个技能表示为结构化程序 \( s = (n, d, c) \)，其中 \( n \) 为名称，\( d \) 为描述，\( c \) 为技能内容（包含适用条件、执行流程、输入/输出）。

**（2）金字塔构建**：
- **Relation Analyzer**：先进行粗粒度分组（仅基于名称和描述），再对每组进行细粒度分析，输出关系构建任务 \( D_m = \{(g_k, \tau_k, e_k)\} \)，其中 \( g_k \) 为技能子集，\( \tau_k \) 为关系类型（向下原子提取或向上抽象归纳），\( e_k \) 为证据和指导。
- **Relation Builder**：根据 \( \tau_k \) 执行两种操作：
  - **Downward Atomic Extraction**：提取共同的低层可复用能力作为原子技能 \( a_k = B_{\text{atom}}(g_k) \)，并将原始技能重写以引用该原子技能，原子技能放置在源技能的下一层。
  - **Upward Abstract Induction**：归纳共同的高层解决模式作为抽象技能 \( u_k = B_{\text{abs}}(g_k) \)，抽象技能放置在源技能的上一层，并记录对下层技能的引用。

**（3）技能创建（Skill Creator）**：给定新任务指令 \( x \)，检索相关技能 \( R(x) \)（优先高层抽象技能用于框架构建，优先低层原子技能用于细节实例化），然后合成新技能 \( \hat{s} = C_{\text{create}}(x, R(x), P) \)。若已有技能完全覆盖则直接复用。

**（4）自我进化**：新技能 \( \hat{s} \) 生成后，关系分析器仅以 \( \hat{s} \) 为查询检索相关已有技能，关系构建器构建新的复用关系 \( \Delta P_t \)，然后增量更新金字塔 \( P_{t+1} = \text{Update}(P_t, \hat{s}, \Delta P_t) \)，无需重建整个金字塔。

**整体流程**：初始化 → 金字塔构建（Relation Analyzer + Relation Builder 迭代最多 T 轮） → 任务到来时 Skill Creator 生成技能 → 自我进化更新金字塔 → 持续循环。

## 6. 主要结论与发现

1. **性能提升显著**：SkillPyramid 在所有环境和模型上平均奖励比 ReAct+Skills 提升 7.9 个点（73.7 vs 65.8），比 ReAct 提升 20.3 个点；平均步骤减少 27.7%（14.6 vs 20.2）。
2. **泛化能力突出**：在 unseen 任务上，平均奖励为 84.8，高于 ReAct+Skills 的 73.9（提升 10.9 个点）。
3. **各组件均不可或缺**：消融实验显示去除任一组件（原子技能、抽象技能、自我进化）均导致性能下降，且从头生成技能（不基于金字塔）效果最差，甚至低于扁平技能基线。
4. **自我进化带来持续适应**：动态更新金字塔比静态冻结或简单追加技能更稳定，最终收敛到更高成功率。
5. **原子技能与抽象技能作用不同**：原子技能主要辅助低层操作密集型任务，抽象技能则对多层次过程组合任务更关键。
6. **对网络挖掘技能同样有效**：在 GAIA-Lite 上，经过金字塔组织的网络技能比扁平技能准确率更高、步骤更少。

## 7. 优点

- **层次化组织结构**：通过向下原子提取和向上抽象归纳，自动将技能组织成多层金字塔，既保留了可复用的底层操作能力，也提炼了高层问题解决模式，实现“1+1>2”的协同效果。
- **任务驱动进化机制**：新技能不是简单附加，而是通过与金字塔中已有技能建立复用关系增量融入，形成动态演化系统，避免了知识遗忘和冗余积累。
- **计算高效**：不需要模型训练，仅通过 API 调用即可完成；自我进化只需增量分析新技能，而非全局重建。
- **跨模型泛化性**：在 4 种不同规模/系列的骨干模型上均取得一致提升，证明框架的鲁棒性。
- **消融实验设计合理**：通过 ReAct+Skills 基线隔离了技能组织本身的贡献；通过去组件消融、从头生成等变体清晰展示了每个模块的价值。
- **分析深入**：不仅报告总体性能，还通过任务分型（原子密集型 vs 抽象密集型）解释了为何原子和抽象技能各有侧重，理论解释力强。

## 8. 不足与局限

- **初始构建成本高**：金字塔的初始构建需要对整个技能库进行一次完整分析，对于超大规模库可能开销过大，论文未探索轻量级构建策略。
- **依赖强大骨干模型**：Relation Analyzer 和 Relation Builder 需要高性能 LLM 来准确识别复用关系和合成新技能。未系统研究当骨干模型变弱时框架如何退化。

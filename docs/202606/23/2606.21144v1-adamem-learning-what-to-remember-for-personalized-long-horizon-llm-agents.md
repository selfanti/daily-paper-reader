## 摘要
大型语言模型（LLM）智能体的长期记忆系统通常试图“记住一切”，统一提取记忆以保留尽可能多的事实。然而，在生产环境中，推理成本和有限的上下文预算使得这种做法难以维持：除了将原始对话整合为记忆外，智能体还必须施加“写入控制”，高效地仅保留每个用户真正关心的信息。否则，长期个性化交互会遭遇“记忆膨胀”，无关琐事挤走有用信息，并逐渐侵蚀问答准确性。我们认为，什么值得记住是角色依赖的，并提出 **AdaMem**（自适应记忆），一种从反馈中学习为每个用户“该记住什么”的方法。AdaMem 维护一个结构化的、角色特定的记忆策略，并通过轻量级的补丁式自我反思步骤（带失败回滚）从每周问答反馈中优化该策略。为了研究这一场景，我们构建了 **AdaMem-Bench**，一个模拟数周交互并附带每周问答的基准测试。在两种提取模型和两种反馈模式下，AdaMem 相比统一记忆的 Mem0 基线，问答准确率提升高达 **+9.0%**，同时记忆体积缩小 **9%**。

## Abstract
Long-term memory systems for Large Language Model (LLM) agents typically try to \emph{remember everything}, extracting memories uniformly to retain as many facts as possible. In production, however, inference cost and finite context budgets make this untenable: beyond consolidating raw dialogue into memory, an agent must exert \emph{write control}, efficiently keeping only the information each user actually cares about. Otherwise, long-horizon personalized interactions suffer \emph{memory bloat}, where irrelevant trivia crowds out useful information and steadily erodes question-answering (QA) accuracy. We argue that what is worth remembering is role-dependent, and propose \textbf{AdaMem} (Adaptive Memory), a method that \emph{learns what to remember} for each user from feedback. AdaMem maintains a structured, role-specific Memory Policy and refines it from weekly QA feedback through a lightweight, patch-style self-reflection step with failure rollback. To study this setting, we build \textbf{AdaMem-Bench}, a benchmark that simulates weeks of interaction with week-by-week QA. Across two extraction models and two feedback modes, AdaMem improves QA accuracy by up to \textbf{+9.0\%} over the uniform Mem0 baseline while shrinking memory volume by \textbf{9\%}.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有 LLM 长期记忆系统（如 Mem0、MemoryBank）采用“记住一切”的统一提取策略，试图保留所有事实。但在实际部署中，推理成本和有限上下文预算使得这种策略不可持续——长期交互会产生“记忆膨胀”（memory bloat），无关信息（如日常闲聊、琐碎日程）逐渐挤占有用信息，导致问答准确率持续下降。
- **核心洞察**：什么信息值得记住是**角色依赖**的（role-dependent），不同用户、不同角色有截然不同的偏好（如管理者关心商业决策，健身伙伴关心锻炼计划）。因此，记忆系统不应被动地全盘记忆，而应主动**学习该记住什么**，实现偏好对齐的“写入控制”。
- **整体含义**：本文提出将长期记忆管理从“记忆一切”转向“按需记忆”，通过轻量级策略学习实现个性化、低成本、高精度的记忆存储，对工业部署具有直接实用价值。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：维护一个结构化的、角色特定的 **记忆策略（Memory Policy）**，该策略以自然语言规则形式描述每个角色应提取的信息类别。通过每周的 QA 反馈，利用“补丁式自我反思 + 失败回滚”机制动态优化策略，从而指导日常记忆提取，实现偏好对齐。
- **关键技术细节（AdaMem 闭环流程）**：
    1. **每日提取**：利用当前记忆策略作为指令，驱动 LLM 提取每日对话中的偏好相关记忆，并执行**时间锚定**（将“明天”等相对时间解析为绝对日期）后存入数据库。
    2. **每周评估**：对智能体进行每周 QA 测试，获得反馈（显式或隐式）。
    3. **策略反思与更新**：
        - 识别错误回答并诊断失败类型：**召回失败**（策略过严导致遗漏信息）或**精确性失败**（策略过松导致噪声记忆干扰）。
        - 生成策略补丁 `∆Pt`，合并到当前策略中得到候选新策略 `Pt+1'`。
        - **失败回滚机制**：在历史 QA 上验证候选策略，若性能下降则拒绝更新，回滚至原策略，确保稳定性。
    4. **策略渲染**：将更新后的策略编译为提取指令，供下一周使用。
- **算法流程**：参见论文 Algorithm 1（每周循环：每日提取 → 每周 QA → 反馈诊断 → 策略补丁 → 回滚验证 → 结果回写）。

## 3. 实验设计

- **数据集与场景**：
    - 构建了 **AdaMem-Bench**，一个个性化长期交互基准测试，包含：

## 6. 主要结论与发现

1. **整体性能**：AdaMem 在所有设置下均优于 Mem0。最高准确率提升 **+9.0%**（DeepSeek-V4-Flash + 显式反馈），记忆体积缩小 **9.1%**；记忆效率比（MER）提升约 0.039。
2. **信息类别增益**：增益主要集中在“软偏好”类别：情绪/态度（+15.9%）、日程/时间（+9.9%）、决策/结论（+9.0%）；而事实/数字这类硬事实增益很小（+1.1%），因为统一策略已能很好捕获。
3. **检索质量**：随着时间推移，Mem0 的 Recall@5 和 MRR 持续下降（至第 10 周约 50%），而 AdaMem 保持更高且更稳定的检索性能（Recall@5 平均 64.6%，第 10 周达 68.0%），有效缓解了记忆膨胀导致的检索退化。
4. **策略学习瓶颈**：模型遵循策略的能力很高（>80%），但策略推断（从隐式反馈学习偏好）是主要瓶颈：显式反馈下策略收敛至 88.3%，隐式反馈下仅 53.3%-63.3%，差距显著。

## 7. 优点

- **问题新颖**：将长期记忆管理问题重新定义为“写入控制”问题，强调角色依赖和偏好对齐，而非简单的“记住一切”。该视角贴近工业实际，具有重要实用价值。
- **方法轻量**：AdaMem 采用补丁式自我反思 + 失败回滚，不需要重新训练模型，可轻松集成到现有 RAG 记忆栈中，成本低。
- **实验设计细致**：构建了专用基准 AdaMem-Bench，包含黄金记忆标注、多角色干扰、时间锚定等现实挑战；引入显式/隐式反馈两种模式，更贴近真实应用场景。
- **分析深入**：不仅报告整体指标，还分解信息类别、分析检索动态、剖析策略学习瓶颈（政策遵循 vs. 政策推断），提供了对问题本质的深入理解。

## 8. 不足与局限

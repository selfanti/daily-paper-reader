Abstract 

While Large Language Models (LLMs) have advanced open-domain dialogue systems, main-taining long-term consistency remains a chal-lenge due to inherent limitations in long-context reasoning and the inefficiency of pro-cessing extensive raw text. Existing approaches typically rely on either unstructured memory storage, which is prone to information loss, or computationally expensive LLMs that in-cur high latency. To address these limitations, we propose G-Long , a graph-enhanced frame-work that utilizes a fine-tuned small Language Model (sLM) for structured triplet extraction and associative retrieval, significantly reducing operational costs. Furthermore, we introduce the novel attention-aware importance scoring mechanism that leverages the intrinsic cross-attention signals of a T5 summarizer to iden-tify salient memories. Extensive experiments across diverse benchmarks demonstrate that G-Long achieves state-of-the-art performance in both response generation and memory retrieval, yielding performance gains of up to 9.8% in re-sponse quality on MSC and 40.8% in retrieval recall on LME, while significantly minimizing computational overhead. 1

1 Introduction 

Recent advancements in Large Language Models (LLMs) have revolutionized open-domain dialogue systems, enabling agents to generate fluent and context-aware responses. However, establishing long-term relationships with users requires Long-term Memory to retain persona consistency and recall past events across extended sessions (Xu et al., 2022a,b; iunn Ong et al., 2025). Without this capability, agents fail to maintain contextual continuity and consequently treat each session as an isolated interaction. This leads to disjointed and impersonal user experiences.

To achieve high-quality generation in long-term dialogues, memory systems must simultaneously overcome challenges in retrieval precision and com-putational efficiency. As shown in Figure 1 (Left), existing frameworks commonly utilize large-scale LLMs for memory management to maintain an un-structured memory bank via paragraph summaries (Tan et al., 2025; Zhong et al., 2024; Lu et al., 2023; Chen et al., 2025). These conventional frameworks inherently have the following limitations. First, text-based summaries easily lose critical details over time. When complex long-term conversa-tions are compressed into plain paragraphs, the meaningful relationships between entities become blurred. For example, specific context such as “deep-sea fishing” is generalized into abstract terms like “ [fishing] ,” which leads standard dense re-trieval methods to struggle with tracing exact facts across past conversations and impairs response gen-eration. Second, unstructured verbose paragraph summaries directly into large LLMs for memory operations leads to massive token consumption and

computational overhead, making the system highly inefficient. To overcome these limitations, we advocate for a shift from unstructured text storage to a graph-based memory bank, as illustrated in Figure 1 (Right). By explicitly modeling entities and their relations into discrete triplets with precise impor-tance scores (e.g., ranking (Article, topic, Deep-sea fishing) at 0.98 ), the graph-based paradigm mitigates retrieval ambiguity and stores rich con-texts in a compact representation. Furthermore, this structural compactness allows the framework to inject highly condensed and relevant contexts into the LLM when compared with verbose text paragraphs, which can reduce prompt token over-head and inference costs. Therefore, we propose 

G-Long , a resource-efficient and graph-enhanced framework for scalable and precise long-term dia-logue memory management. Extensive evaluations across MSC, CC, LoCoMo, and LME benchmarks demonstrates that G-Long achieves the state-of-the-art performance, driven by three key contributions. • First, a persona-driven graph structure es-tablishes a fine-grained and triplet-based mem-ory bank optimized for long-term dialogues. By capturing core relational patterns aligned with conversational personas (e.g., via Person-aExt ), G-Long resolves critical problems from conversational noise and paragraph blur, yield-ing a 9.8% improvement in response quality (BLEU-2) on the MSC dataset. • Second, the associative retrieval mechanism 

integrated with an attention-aware impor-tance scoring module maximizes retrieval precision. G-Long navigates topological con-nections via structured subgraph expansion while filtering retrieved triplets based on attention-driven importance signals. This ap-proach bridges the gap between surface-level similarity and multi-hop reasoning, yielding a 40.8% improvement in retrieval performance (Recall@3) on the LME benchmark. • Finally, a resource-efficient framework de-sign minimizes overall operational costs. By offloading memory construction to a fine-tuned small Language Model (sLM) and feed-ing compact graph triplets instead of long paragraphs into the LLM, G-Long eliminates prohibitive LLM-API dependencies. This re-duces memory maintenance costs by 4.9× and token consumption by 63.0% without sacrific-ing generation quality. 

2 Related Work 

Long-term Dialogue Systems Significant efforts have been made to construct high-quality bench-marks for evaluating long-term dialogue consis-tency. Representative datasets include MSC (Xu et al., 2022a) and CC (Jang et al., 2023) for open-domain dialogue. Furthermore, LongMemEval 

(Wu et al., 2025a) and LoCoMo (Maharana et al., 2024) have been introduced to rigorously evaluate specific memory retention capabilities, while Care-Call (Bae et al., 2022a) extends the scope to the healthcare domain. While the most intuitive approach to handling long-term dialogue is directly concatenating the entire dialogue history, this incurs prohibitive com-putational costs and degrades performance due to the "lost-in-the-middle" phenomenon (Liu et al., 2023a). Early studies (Xu et al., 2022a; Bae et al., 2022b) relied on simple summarization to com-press context, which often sacrifices detail for brevity. To address this, recent approaches have shifted towards structured external memory sys-tems that leverage granular abstraction for precise retention (Zhang et al., 2023a; iunn Ong et al., 2025). For instance, MemoryBank (Zhong et al., 2024) imitates human forgetting to preserve salient information efficiently. To enhance retrieval pre-cision, FraCom (Ke et al., 2025) and RMM (Tan et al., 2025) focus on granular restructuring by de-composing histories into proposition-level units or semantic topics. Furthermore, LD-Agent (Li et al., 2025) tackles the complexity of long-term interac-tion by separating memory into independent event and persona modules to track narrative trajectories and user characteristics distinctively. Despite these advances, heavy reliance on LLMs for memory management limits scalability, and un-structured representations hinder precise retrieval and informative response generation. 

Memory management for Long-term Dialogue Agents A parallel line of work has investigated structured memory representations that convert di-alogue into discrete semantic units. The most common substrate is the (subject, relation, ob-ject) triplet (Trisedya et al., 2019). Vossen et al., 2024 and Li et al., 2024 extract open-domain rela-tions to enrich dialogue understanding, while Wu Figure 2: Overview of the G-Long framework. 

et al., 2020 and GenRe (Wang et al., 2022) target attribute-level triplets for building dynamic user profiles. PAED (Zhu et al., 2023) introduces a con-trastive learning model for generalized zero-shot persona attribute extraction, and Papaluca et al., 2023 and Deng et al., 2024 show that LLMs can construct knowledge graphs from text in zero-shot settings. These methods establish triplets as a vi-able substrate for dialogue memory, but they rely on heavy LLM calls for extraction and treat the resulting triplets as static, with no mechanism for evaluating their relative importance to retrieval. Once such structured units are obtained, the next step is to organize them into graphs. Recent work in this direction—including GraphRAG (Edge et al., 2025), HippoRAG (Gutiérrez et al., 2024), and 

LinearRAG (Zhuang et al., 2025)—focuses on synthesizing global insights or navigating complex relations within a static document corpus. (He et al., 2024; Mavromatis and Karypis, 2025; Hu et al., 2025) This static assumption creates two structural mismatches with long-term dialogue: (i) Named Entity Recognition is unreliable on conversational utterances that often contain pronouns and ellip-sis, and (ii) reflecting an evolving persona would require rebuilding the graph after every session. Beyond graph-based organization, other external memory architectures like LongMem (Wang et al., 2023) utilize specialized key–value side-networks; however, such designs are incompatible with our plug-and-play design over frozen black-box APIs. Closer to our setting, recent work has begun adapting graph-based memory to the dialogue do-main itself. Whereas FraCom (Ke et al., 2025) op-erates at the proposition level, SGMem (Wu et al., 2025b) instead constructs sentence-level graphs across turn-, round-, and session-level chunks while combining retrieved raw dialogue with LLM-generated summaries, facts, and insights. Yet both still inherit the limitations they aim to address: proposition-based decomposition frequently yields uninformative coreference fragments (e.g., "it", "he") that dilute the retrieval space, and SGMem’s strongest variants reintroduce the very LLM-API dependency that should be eliminated. 

3 Methodology 

3.1 Framework Overview 

Figure 2 illustrates the overall architecture of our proposed framework, G-Long . To handle the long-term memory of the dialogues, we adopt graph-based approaches to long-term memory bank man-agement. The proposed framework consists of four components: (1) efficient memory construction ,(2) graph-based memory bank , (3) associative memory retrieval , and (4) response generation .We describe the details of each component in the following subsections. 3.2 Efficient Memory Construction 

To transform an unstructured dialogue into a struc-tured graph representation, we utilize the triplet ex-traction module , which extracts triplets from each utterance within a conversation, and the attention-aware importance scoring module , which assigns importance scores to each extracted triplet. 

Triplet Extraction Module In this module, we employ a fine-tuned sLM to parse raw utterances into structured facts as (subject, relation, object )

triplets. To achieve this, the sLM undergoes instruc-tion tuning on a dialogue-triplet dataset (e.g., Per-sonaExt (Zhu et al., 2023)). This training process enables the model to effectively extract schema-compliant triplets and generate a consistent set of relations. Training hyperparameters and data pre-processing details are provided in Appendix B.

3.3 Graph-Based Memory Bank 

To effectively store and associate the extracted knowledge, we construct the graph-based mem-ory bank, formally denoted as G = ( V, E). Unlike simple key-value stores, our architecture adopts the dual-storage strategy that combines dense vector indexing for entity nodes and attribute-rich edge storage .

Dense Vector Indexing for Entity Nodes The set of nodes V contains unique entities, which con-sist of subjects and objects , identified in the di-alogue. For the semantic entry point search de-scribed in Section 3.4 (Anchor Node Identifica-tion), each entity node v ∈ V is embedded into a high-dimensional vector space using a pre-trained encoder (e.g., MiniLM (Wang et al., 2020) ) and stored in a vector database (e.g., ChromaDB 2). This embedding-based indexing allows the system to identify relevant entities even when the user’s query uses synonymous or paraphrased terms, to overcome the limitations of exact string matching. 

Attribute-Rich Edge Storage The set of edges E

contains directed relations between entities. In our work, a multi-directed graph is employed to accom-modate overlapping interactions and specially, G-Long represents edges as attribute-rich containers formally denoted as es→o = {r, w, τ create , τ access }

unlike standard architectures; r is the relation text extracted from the dialogue (e.g., location ), 

w ∈ (0 , 1] is the normalized importance score de-rived from Eq.(3), and τ represents the timestamps for memory creation and the latest access. By em-bedding these temporal and significant attributes directly into the graph structure, G-Long enables the retrieval mechanism to dynamically balance semantic relevance with recency during inference.

retrieval candidate expansion and (2) two-stage hybrid reranking .

4 Experiments 

4.1 Experimental Setup 4.1.1 Datasets 

To investigate the effectiveness of G-Long in long-term dialogue scenarios, we conduct experiments on the test sets of two representative multi-session datasets: the Multi-Session Chat (MSC) dataset (Xu et al., 2022a) and the Conversation Chron-icles (CC) dataset (Jang et al., 2023). To further assess the scalability of our framework across ex-ceptionally long contexts, we incorporate the Lo-CoMo dataset (Maharana et al., 2024). Addition-ally, we utilize the LongMemEval (LME) dataset (Wu et al., 2025a), a large-scale long-term conver-sational benchmark, to directly measure retrieval performance and evaluate the intrinsic accuracy of the retrieved memory. Dataset details are provided in Appendix A. 

4.1.2 Baselines 

To benchmark the performance of G-Long, we compare it against six competitive baselines cat-egorized by their memory storage and structural properties: Unstructured & Text-based Memory Frameworks ( No History , Long Context , Memo-ryBank (Zhong et al., 2024), LD-Agent (Li et al., 2025)) and Structured & Graph-based Frameworks (FraCom (Ke et al., 2025), HippoRAG (Gutiérrez et al., 2024)). To systematically evaluate the per-formance across tasks, all baselines are evaluated on both memory retrieval and dialogue generation, with the sole exception of HippoRAG, which is evaluated strictly on the retrieval task due to its design as a specialized retriever without native end-to-end generation prompts for long-term dialogue. To ensure a rigorous evaluation that isolates the efficacy of the memory mechanism itself, we stan-dardize the backbone model for all components to gpt-4o-mini . Detailed configurations and im-plementation details for G-Long and all baselines are provided in Appendix B and C, while compre-hensive results demonstrating the generalization capability across various other backbone LLMs are detailed in Appendix H. 

4.1.3 Metrics 

We employ a multi-faceted evaluation strategy com-prising both automatic and human evaluations. For response generation tasks (MSC, CC), we utilize standard n-gram metrics ( BLEU (Papineni et al., 2002), ROUGE-L (Lin, 2004)) to measure lexi-cal overlap, alongside G-Eval (Liu et al., 2023b) to measure coherence, engagingness, and fluency. For the long-term memory QA task (LME), we adopt Recall and MRR to evaluate retrieval cov-erage and Accuracy to assess answer correctness. Finally, we conduct Human Evaluation to qual-itatively validate long-term consistency. Detailed definitions and evaluation protocols are provided in Appendix D.

5 Conclusion 

In this paper, we introduced G-Long , the resource-efficient graph-enhanced long-term dialogue frame-work designed to overcome the limitations of un-structured memory representations and the high computational costs inherent in long-term dialogue systems. By leveraging the fine-tuned sLM and the attention-aware importance scoring module, G-Long constructs the structured knowledge Graph that captures essential context while filtering out noise from raw dialogue history. Crucially, G-Long eliminates prohibitive API overhead, providing a cost-effective alternative to existing competitive frameworks while achieving SOTA results across MSC, CC, and the large-scale LoCoMo dataset. Furthermore, G-Long’s superior retrieval performance on LME—outperforming general-purpose graph architectures—validates that our framework is uniquely optimized for the nuanced dynamics of long-term dialogue. Conse-quently, G-Long serves as a scalable, plug-and-play solution for maintaining consistent and engaging agents in resource-constrained environments. Limitations 

While G-Long demonstrates significant improve-ments in long-term memory retrieval and opera-tional efficiency, we acknowledge certain limita-tions inherent to our design choices. First, the conversion of unstructured dialogue into structured triplets entails a trade-off be-tween information density and semantic granular-ity. While triplets efficiently capture core facts (e.g., (User, like, Hiking) ), highly subtle emotional undertones or stylistic nuances present in raw text may be abstracted away during this discretization process. Although our experiments show that this does not hinder factual response generation, future iterations could explore hybrid storage mechanisms that link graph nodes to their original text segments to preserve richer stylistic contexts. Second, as a pipeline framework, the overall quality of the memory bank depends on the robust-ness of the Triplet Extraction Module. While our instruction-tuned sLM demonstrates high accuracy on standard benchmarks, extreme ambiguities or complex coreferences in user utterances could po-tentially introduce noise into the graph. Currently, our attention-aware importance scoring mechanism mitigates this by assigning low weights to trivial or noisy triplets, but developing a self-correcting mechanism that verifies factual consistency within the graph remains a promising direction for future work. Furthermore, while the graph-based represen-tation effectively preserves factual consistency, our qualitative error analysis reveals two inher-ent structural boundaries (detailed in Appendix J.3). First, sparse queries containing unresolved anaphora (e.g., demonstrative pronouns without ex-plicit entities) can lead to referential disconnect, causing the retriever to fetch irrelevant nodes. Sec-ond, we observed instances of semantic drift where the retriever over-prioritizes literal keyword over-laps (e.g., specific color descriptors) at the expense of global conversational intent. These findings suggest that future iterations could benefit from a hybrid approach that more tightly couples graph nodes with dynamic contextual resolution. 

Ethical Considerations 

Our experiments utilized anonymized public datasets to ensure data privacy. Beyond standard compliance, G-Long offers a distinct ethical advan-tage through its structured knowledge graph. Un-like unstructured vector stores, the explicit nature of graph triplets enables transparent interpretabil-ity and precise control, allowing users or admin-istrators to identify and delete specific memories without retraining the model. This capability di-rectly supports the "Right to be Forgotten" and pro-motes responsible data management. Regarding potential risks, our framework relies on generative models for both memory extraction (sLM) and re-sponse generation (backbone LLM). Consequently, there is a possibility that incorrect triplets may be extracted, or the final model may exhibit halluci-nations or biases inherent in pre-trained models. Therefore, we emphasize the necessity of human oversight and fairness-aware safeguards when de-ploying such systems in sensitive environments.

A Dataset Details 

To comprehensively evaluate the effectiveness of G-Long, we employ three benchmark datasets tai-lored for multi-session and long-term conversa-tional memory modeling. 

Multi-Session Chat (MSC) MSC (Xu et al., 2022a) is a large-scale multi-session dialogue benchmark. For our main experiments, following previous work (Li et al., 2025), we randomly sam-pled 500 dialogue sessions from the test split to manage computational costs while maintaining sta-tistical significance. Each sample consists of five temporally separated sessions between the same two interlocutors (avg. 43 turns), designed to eval-uate the retention of user-specific information such as preferences and past experiences. 

Conversation Chronicles (CC) CC (Jang et al., 2023) focuses on chronological consistency and evolving personal narratives. Similarly, following (Li et al., 2025), we used a random subset of 500 test samples for evaluation. Each sample comprises five sessions (avg. 58 turns) with rich temporal dependencies, requiring models to track changes in user states and relationships over time. 

LoCoMo LoCoMo (Maharana et al., 2024) is a sophisticated long-term dialogue dataset designed to evaluate memory retention over extended tem-poral horizons. Unlike MSC and CC, LoCoMo contains an exceptionally high number of sessions (approximately 27 sessions per sample), providing a highly challenging and realistic environment for testing the scalability of long-context reasoning. 

LongMemEval (LME) To measure intrinsic re-trieval accuracy, we use the LME dataset (Wu et al., 2025a). We utilized the LongMemEval_S subset (approx. 48 sessions, 10 turns each), where a post-conversation QA task targets specific facts men-tioned throughout the history. This allows for a precise assessment of memory correctness indepen-dent of generation fluency. 

B Implementation Details 

We implemented G-Long using the PyTorch frame-work and the HuggingFace Transformers library. 

Module Configurations For the Triplet Extrac-tion Module (Section 3.2), we primarily utilized 

Qwen-3-8B 3 instruction-tuned on the PersonaExt 

(Zhu et al., 2023) dataset. This corpus, consist-ing of dialogue sessions paired with structured persona triplets, provides ground-truth supervi-sion for learning precise extraction schemas. To ensure training efficiency, we employed QLoRA (Dettmers et al., 2023) with 4-bit quantization. We configured the LoRA (Hu et al., 2021) adapter with a rank of r = 32 and α = 32 , targeting all lin-ear projection layers to maximize representational power. The model was fine-tuned for 10 epochs with a learning rate of 2 × 10 −4 and a batch size of 16. For the inference phase, we leveraged the 

vLLM engine with greedy decoding (temperature 0.0) to guarantee high-throughput and deterministic generation of structured knowledge. To guide the model’s generation, we utilized a specific system instruction emphasizing structured output. For the Attention-Aware Importance Scoring Module , we leveraged a T5-small (Raffel et al., 2023) checkpoint that was fine-tuned on the SAM-Sum corpus (Gliwa et al., 2019). Since this model

We uti-lize these cross-attention maps from the last de-coder layer to quantify the importance of extracted triplets. We leveraged a T5-small checkpoint 4 fine-tuned on the Samsum dataset. 

Generation Settings To ensure reproducibility and deterministic evaluation, we standardized the generation process across G-Long and all baseline models. We utilized gpt-4o-mini as the backbone LLM for response generation and auxiliary mem-ory management tasks such as summarizing, set-ting the decoding temperature to 0.0 to minimize randomness in the outputs. 

Retrieval Settings We implemented the under-lying graph structure using the NetworkX 5 li-brary. For the dense embedding, we utilized the 

all-MiniLM-L6-v2 sentence-transformer encoder model, which was managed within a ChromaDB 

vector store for efficient querying. Regarding the hyperparameters discussed in Sec-tion 3.4, we empirically set the number of an-chor nodes to Nanc = 5 . For subgraph expan-sion, we adopted a 1-hop neighborhood strategy (L = 1 ) to prioritize retrieval efficiency while maintaining sufficient local context, as validated by the ablation analysis in Section 4.7. In the Two-Stage Hybrid Reranking phase, we first filtered the top N = 5 triplets based on semantic similarity. Subsequently, for importance-aware reranking, the weighting coefficients were set to β1 = 0 .5 (se-mantic), β2 = 0 .3 (importance), and β3 = 0 .2 (re-cency). The time-decay factor was set to λ = 10 −7

based on validation performance. Finally, the top 

K = 3 triplets were selected and concatenated with the dialogue context for generation. 

Hardware All experiments were conducted on a single NVIDIA RTX PRO 6000 Blackwell Max-Q Edition (96GB).

E Detailed Human Evaluation Protocol 

This section provides a comprehensive breakdown of our human evaluation process to ensure trans-parency and reproducibility. 

Recruitment and Compensation We recruited three independent graduate-level researchers spe-cializing in NLP, all proficient in English. An-notators were not affiliated with the model devel-opment and had no prior exposure to the system outputs. All annotators were compensated at our in-stitution’s standard hourly rate to ensure fair labor practices. 

Task Design and Blind Setup To eliminate bias, we adopted a fully blind protocol where model identities were anonymized (presented as “Model A” vs. “Model B”) and the presentation order was randomized. Annotators performed pairwise side-by-side comparisons under a forced binary choice (1 for win, 0 for loss). For each instance, they were presented with the full long-term dialogue history (Sessions 1–4) followed by candidate responses for Session 5. They were asked to judge which re-sponse better reflects the user’s established persona and provides a more fluent, engaging, and coherent follow-up. 

Evaluation Criteria Annotators judged each pair holistically along three dimensions: (1) Flu-ency : linguistic quality and naturalness; (2) En-gagingness : proactivity and empathetic tone; and (3) Coherence : logical relevance and long-term factual consistency with the established history. Sample Size and Annotation Effort We eval-uated 50 randomly sampled instances per base-line pairing. While this number may appear mod-est, each instance spans five sessions, averaging 51 turns ( ≈2,500 total turns per pairing). To make a reliable judgment, annotators must read the entire dialogue history, requiring 15–20 minutes per instance. We therefore prioritized evaluation depth—essential for genuinely assessing long-term consistency—over the breadth of shallow, snippet-level judgments. 

Statistical Significance To assess whether G-Long’s win rates differ meaningfully from chance (0.5), we conducted a one-sample t-test. The ag-gregated win rates (64% vs. LD-Agent, 52% vs. MemoryBank, 54% vs. Long Context) yielded 

p = 0.058 . Although slightly above the 0.05 threshold due to the time-constrained sample size, the consistent trend across all independent base-lines and the substantial 14-point margin against LD-Agent support G-Long’s superiority. We view this as complementary evidence to our G-Eval re-sults. 

Data Consent and Ethics Annotators were in-formed that data would be used for academic re-search only. No personally identifiable information (PII) was collected. The study involved low-risk tasks on public, anonymized datasets and followed the ethical research guidelines of our institution.

J Case Study 

J.1 Qualitative Analysis on Dialogue Generation 

Figure 5 presents a qualitative comparison between G-Long and the Long Context baseline. This com-parison highlights the efficacy of providing struc-tured knowledge (triplets) over simply feeding raw, extensive dialogue history. We examine two criti-cal aspects of long-term dialogue quality: Persona Consistency and Context-Aware Empathy .

Case 1: Persona Consistency In the "Free-lancer" scenario, G-Long retrieves distinct facts (occupation and pet) and synthesizes them into a causal chain: "Freelance flexibility → work from home → time with Bosco." Unlike the Long Con-text baseline, which only captures surface-level associations, G-Long constructs grounded narra-tives that respect the logical flow of the persona’s lifestyle rather than merely repeating historical facts. 

Case 2: Context-Aware Empathy In the "Fish-ing Article" scenario, G-Long demonstrates nu-anced empathy. While Long Context offers generic encouragement ("great accomplishment"), G-Long explicitly leverages the user’s past hobby ("en-joys fishing") to validate their ability ("passion will shine through"). This capability to bridge current emotions with verified past interests fos-ters a deeper connection than standard supportive phrases. Notably, G-Long’s comprehensive responses stem not from verbosity, but from the ability to fully articulate retrieved structured knowledge, en-suring interactions are both logically grounded and conversationally engaging. 

J.2 Empirical Evidence for Retrieval Behavior 

To address the concern regarding the empirical evi-dence for retrieval behavior, we provide a detailed qualitative analysis using the MSC Dataset (Figure 6). These results demonstrate how G-Long’s struc-tured retrieval mechanism effectively captures and utilizes long-term memory compared to traditional summarization-based baselines. When handling a complex inquiry, G-Long ac-curately captures the core entities of the dialogue and extracts independent, highly relevant facts pri-oritized by rank. This provides zero-noise infor-mation that is ready for immediate contextual use. In contrast, LDA over-compresses past conversa-tions into a single summary sentence, resulting in critical information loss where specific entities essential for answering the inquiry (e.g., "grill", "steak", "Alaska") are completely omitted. Further-more, the Memory Bank baseline returns lengthy, narrative-style summaries mixed with metadata. Although it contains some of the target informa-tion, the excessive amount of irrelevant text causes severe context overload, making it difficult for the language model to pinpoint the necessary facts. 

J.3 Error Analysis on Memory Retrieval 

To provide a transparent view of the structural boundaries of the proposed graph-based memory representation, we conducted an extended qualita-tive analysis on failure cases that occurred outside the standard benchmark evaluation. Our analysis identifies two primary failure modes when handling nuanced or non-factual conversational content, as illustrated in Figure 7. These findings suggest that while graph-based structures are highly efficient for factual retention, future work should explore tighter integration be-tween graph nodes and dynamic context resolution to mitigate such referential and lexical sensitivities.

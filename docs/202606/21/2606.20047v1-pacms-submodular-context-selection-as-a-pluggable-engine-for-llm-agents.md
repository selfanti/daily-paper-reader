## 摘要
对话式和使用工具的LLM代理在一个上下文窗口上运行，该窗口同时从多个方向填充。随着会话的进行，代理会累积用户和助手的轮次、从持久记忆存储中提取的条目，以及通常最大的部分——工具调用的逐字输出，如文件读取、搜索结果和API响应。一旦累积的上下文超过模型的token预算，框架必须决定保留什么。

主流机制是近因截断，有时配以周期性摘要。这是主题盲目的：会话早期建立的事实仅仅因为老旧而被丢弃，即使当前用户查询正是关于那个事实；反之，冗长但无关的近期内容却被保留。必须跨多轮回忆信息的代理——这是记忆的定义性案例——正是近因截断失败的地方。

现有的替代方案位于代理的组装步骤之外。检索增强生成将外部文档取入提示，但并不仲裁代理的“已存在”池化上下文。上下文压缩方法通过重写或修剪文本来减少token数量，但操作是查询盲目且有损的。两者都没有将记忆条目、对话轮次和工具输出视为一个单一的候选池，以便在组装提示时根据相关性进行选择。

## Abstract
Conversational and tool-using LLM agents operate over a context window that fills from several directions simultaneously. As a session proceeds, the agent accumulates user and assistant turns, entries drawn from a persistent memory store, and often largest of all, the verbatim outputs of tool calls such as file reads, search results, and API responses. Once the cumulative context exceeds the model's token budget, the framework must decide what to keep.   The prevailing mechanism is recency truncation, sometimes paired with periodic summarization. This is topic-blind: a fact established early in a session is discarded simply because it is old, even when the current user query is about exactly that fact; conversely, verbose but irrelevant recent material is retained. Agents that must recall information across many turns, the defining case for memory, are precisely where recency truncation fails.   Existing alternatives sit outside the agent's assembly step. Retrieval augmented generation fetches external documents into the prompt but does not arbitrate the agent's \emph{already-present} pooled context. Context-compression methods reduce token count by rewriting or pruning text, but operate query-blind and lossily. Neither treats memory entries, conversation turns, and tool outputs as a single candidate pool to be selected from by relevance at the moment the prompt is assembled.

# PACMS：作为LLM代理可插拔引擎的子模上下文选择 — 论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM代理在长期会话中会积累来自多源（对话轮次、持久记忆、工具调用输出）的上下文，这些上下文的总量常常超过模型的token预算。现有默认的解决方法是**近因截断（recency truncation）**，它只保留最新的内容，忽略早期但相关的信息，同时还可能保留近期噪声。这种“主题盲”的方式在代理需要跨多轮回忆事实时尤其失败。
- **背景**：现有替代方案如检索增强生成（RAG）处理外部文档而非代理内部的已存在上下文；上下文压缩方法（如LLMLingua、RECOMP）进行查询盲、有损的压缩。它们都没有把记忆、对话轮次和工具输出视为统一的候选池，在每次提示组装时根据当前查询进行相关性选择。
- **整体含义**：本文提出将代理上下文组装问题形式化为**预算约束下的子模选择（submodular selection）**，并实现为可插拔引擎PACMS。这为LLM代理提供了一个查询感知、原则性、可观测的上下文管理方案，能够替代近因截断，显著提升关键信息召回和下游任务性能。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将代理当前拥有的所有候选上下文（包括持久记忆、历史对话轮次、工具输出等）视为一个集合 \(C = \{c_1, \dots, c_n\}\)，给定当前查询\(q\)、token预算\(B\)和强制保留集合\(M\)，通过最大化一个**设施选址（facility-location）子模目标函数**来选择最优子集\(S \subseteq C\)，使得在预算内最大化覆盖所有查询相关候选的“覆盖范围”，同时减少冗余。
- **关键技术细节**：
  - **相关性定义**：\(\text{rel}(i, q) = \max(0, \cos(e_i, e_q))\)，其中\(e_i\)、\(e_q\)为嵌入向量。
  - **覆盖权重**：\(w_{ij} = \text{rel}(i, q) \cdot \max(0, \cos(e_i, e_j))\)，表示候选\(j\)对候选\(i\)的加权覆盖程度。
  - **目标函数**：\(F(S) = \sum_{i} \max_{j \in S} w_{ij}\)，受约束 \(\sum_{j \in S} \text{tok}(j) \leq B\) 且 \(M \subseteq S\)。这是一个**单调子模函数**。
  - **优化算法**：采用CELF懒贪心算法（成本感知），具有常数因子逼近保证。
  - **与MMR对比的关键差异**：
    - 范围：MMR的冗余惩罚只考虑已选集内部的相似度（\(\max_{i \in S}\)），而PACMS的覆盖项求和整个池（\(\sum_i\)），奖励能覆盖未被选集覆盖区域的候选。
    - 相关性加权：PACMS的覆盖权重按每个目标项的相关性\(\text{rel}(i)\)加权，MMR的惩罚是纯相似度，盲于相关性。
    - 保证：PACMS的目标是单调子模函数，有贪心近似保证；MMR是成对惩罚启发式，无类似界。
- **系统设计**：作为OpenClaw代理框架的可插拔上下文引擎（context-engine），实现`assemble()`方法，在每次模型调用时通过本地FastAPI服务调用，使用本地Ollama提供的nomic-embed-text嵌入，支持五种策略（PACMS、top-k、MMR、last-k、RAG），可软回退到近因截断。提供调试接口和可视化UI（keep/drop网格、token仪表、预算滑块）。

## 3. 实验设计

- **数据集**：使用**LongMemEval**基准测试，这是一个多会话对话历史数据集，用于测试聊天助手的长期记忆。每个问题配对多会话的“干草堆”，并标注了证据所在的轮次。涵盖六种问题类型（single-session-user、single-session-assistant、single-session-preference、temporal-reasoning、knowledge-update、multi-session）。
- **样本**：从中随机抽取**100个问题**，用于召回率和QA评估。
- **对比方法**：
  - top-𝑘：基于余弦相关性的top-k选
  - lc-mmr：LangChain的MMR（默认\(\lambda=0.5\)），预算限制通过遍历其输出排名实现
  - last-𝑘：近因截断（查询盲）
  - PACMS：本文方法
- **评测指标**：
  - **证据轮次召回率（evidence-round recall）**：\(|S \cap E| / |E|\)，E为标注的证据轮次。
  - **端到端QA准确率**：阅读理解模型从所选上下文中回答问题，由GPT-4o-mini判断对错（LongMemEval官方提示）。
- **设置**：
  - 所有选择器共享nomic-embed-text嵌入（本地Ollama）。
  - 预算设为总上下文token的百分比（20%、45%、70%）。
  - 注入冗余：通过模板生成同义改写填充轮次，冗余级别R∈{0,2,4,8}。
  - QA实验在R=2、预算45%下进行，两个读者模型：GPT-5-mini（reasoning_effort=minimal）和GPT-5.4-mini（reasoning_effort=low）。
- **结果**（参见表格）：
  - 召回率：在低预算下top-𝑘最优；随预算和冗余增加，PACMS和MMR赶上并超过top-𝑘；PACMS和MMR召回率接近；last-𝑘崩溃。
  - QA准确率：PACMS在两种读者下均领先：GPT-5-mini 52% vs top-𝑘 50%、lc-mmr 44%；GPT-5.4-mini 68% vs top-𝑘 64%、lc-mmr 56%；差距随读者能力扩大（+8到+12分优于MMR）。排名PACMS > top-𝑘 > MMR ≥ last-𝑘。

## 6. 论文的主要结论与发现

1. **PACMS在证据轮次召回率上能与MMR持平**（甚至在高预算时略优），但两者低于top-𝑘在低预算下的表现。这表明覆盖目标在高预算时优势更大。
2. **核心发现：召回率与QA准确率不一致**——尽管PACMS和MMR召回率接近，但PACMS在端到端QA上显著优于MMR（+8到+12点），甚至优于召回率更高的top-𝑘（+2到+4点）。这说明**设施选址覆盖比成对多样化或纯相关排序产生更易提取的提示**。
3. **近因截断（last-𝑘）在所有设置下表现最差**，尤其在冗余高时召回率崩坏，且更强的读者也无法弥补（QA分数低且平坦）。
4. **PACMS的排名恒定**：PACMS > top-𝑘 > MMR ≥ last-𝑘，在两种读者下一致。
5. 系统设计的可插拔性、实时可视化、可调预算等特性使得PACMS成为实用的上下文管理引擎。

## 7. 优点

- **方法创新**：将代理上下文选择严格形式化为预算约束子模最大化，具有理论保证（常数因子逼近），不同于现有启发式。

- **可交互性**：提供实时演示UI，使选择过程和预算影响透明可控，有助于理解与调试。

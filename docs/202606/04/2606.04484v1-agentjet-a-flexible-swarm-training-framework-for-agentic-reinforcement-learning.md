## 摘要
我们提出了AgentJet，一个面向大型语言模型智能体强化学习的分布式集群训练框架。与紧密耦合智能体策略采样与模型优化的集中式框架不同，AgentJet采用了解耦的多节点架构，其中集群服务节点托管可训练模型并在GPU集群上执行优化，而集群客户端节点则在任意设备上执行任意智能体。这种设计提供了集中式框架难以支持的能力：（1）异构多模型强化学习，支持训练以多个LLM为大脑的异构多智能体团队；（2）具有隔离智能体运行时的多任务“鸡尾酒”训练；（3）容错执行，防止外部环境故障中断训练过程；（4）实时代码迭代，允许通过替换集群客户端节点在训练期间编辑智能体。为了在多模型、多轮和多智能体设置中支持高效强化学习，AgentJet引入了一个带有时间线合并的上下文追踪模块，该模块合并冗余上下文并实现1.5到10倍的训练加速。最后，AgentJet引入了一个自动化研究系统，该系统以研究主题为输入，并在大规模集群上自主进行长期多日强化学习研究。通过利用集群架构，该系统在无需人工干预的情况下复现了强化学习研究人员的核心探索工作流。

## Abstract
We present AgentJet, a distributed swarm training framework for large language model (LLM) agent reinforcement learning. Unlike centralized frameworks that tightly couple agent rollouts with model optimization, AgentJet adopts a decoupled multi-node architecture in which swarm server nodes host trainable models and run optimization on GPU clusters, whereas swarm client nodes execute arbitrary agents on arbitrary devices. This design provides capabilities that are difficult to support in centralized frameworks: (1) heterogeneous multi-model reinforcement learning, enabling the training of heterogeneous multi-agent teams with multiple LLM as brains; (2) multi-task cocktail training with isolated agent runtimes; (3) fault-tolerant execution that prevents external environment failures from interrupting the training process; and (4) live code iteration, which allows agents to be edited during training by replacing swarm client nodes. To support efficient RL in multi-model, multi-turn, and multi-agent settings, AgentJet introduces a context tracking module with timeline merging, which consolidates redundant context and achieves a 1.5-10x training speedup. Finally, AgentJet introduces an automated research system that takes a research topic as input and autonomously conducts long-horizon, multi-day RL studies on large-scale clusters. By leveraging the swarm architecture, this system reproduces key exploratory workflows of RL researchers without human intervention during execution.

- **问题**：现有的大语言模型（LLM）智能体强化学习（RL）框架多采用集中式架构，将智能体策略采样（rollout）与模型优化紧密耦合。这种设计导致以下关键缺陷：
  - **运行时脆弱性**：智能体依赖的外部环境（Docker容器、浏览器、API等）容易失败，单点故障会拖垮整个训练流程，丢失未保存的进度。
  - **调试摩擦**：修改智能体代码或奖励函数需重启整个训练（包括模型加载和vLLM初始化），每次迭代耗时5-10分钟。
  - **多模型约束**：当前系统主要优化单一策略模型，而真实应用常需异构多智能体系统（如不同大小的模型分别负责执行和规划）。
  - **冗余上下文**：多轮交互中大量重复的系统提示、工具定义和观测历史导致策略梯度更新时浪费计算。
  - **环境锁定**：多任务“鸡尾酒”训练需要强运行时隔离，但单体架构难以实现。

- **整体含义**：需要一种新的训练架构，能够解耦智能体执行与模型优化，支持异构、多任务、容错和灵活的迭代调试，从而推动LLM智能体从单轮推理走向长期自主决策。

## 2. 论文提出的方法论

### 2.1 核心思想：集群（Swarm）架构

- **完全解耦**：将模型优化平面（GPU密集）与智能体执行平面（外部依赖密集）分离，通过客户端-服务器范式组织。
- **两类节点**：
  - **Swarm Server（服务节点）**：运行在GPU服务器上，托管模型权重、优化器、vLLM/SGLang推理引擎和训练样本池。每个服务节点独立训练一个模型（可同时运行多个服务节点训练不同尺寸的模型）。
  - **Swarm Client（客户端节点）**：轻量级CPU进程，可在任意设备上运行，执行任意智能体工作流、读取数据集、计算任务奖励。客户端节点可随时加入/离开集群，不影响服务节点。

### 2.2 关键技术细节

- **上下文追踪与时间线合并**：
  - 每个被捕获的LLM调用生成一条独立的时间线，包含消息块、token ID、对数概率和损失掩码。
  - **时间线合并算法**：按逆序遍历时间线列表，将匹配前缀的较短时间线吸收到较长时间线中，保留LLM生成token的损失掩码。提供两种匹配策略：token级精确匹配和text级文本匹配（默认）。后者可容忍分词漂移，实现更激进的合并（图5示例说明Qwen3聊天模板中因thinking block被移除导致token级不匹配但文本级匹配的情况）。
  - **效果**：在AppWorld多轮训练中，合并后actor更新墙钟时间从2160秒降至346秒，**加速6.25倍**（图8），且不损失训练质量。

- **集群强化学习范式**：分为设置服务节点、连接客户端、收集轨迹、优化模型、评估五个阶段。客户端通过OpenAI兼容接口向服务节点发起推理请求，服务节点自动记录上下文和轨迹。

- **样本批处理策略（C1-C5）**：
  - C1：基于完成episode总数触发。
  - C2：基于完成的任务数（每个任务有足够episode重复）触发。
  - C3：基于有效任务数（episode奖励方差非零），动态采样确保梯度信号可靠。
  - C4：所有活跃客户端同意后触发。
  - C5：任一客户端同意后触发。
  - 这些策略取代传统固定批处理的局限，支持如DAPO的动态采样控制。

- **异构多模型多智能体训练**：
  - 共享参数训练：所有可训练智能体共享同一个LLM模型（服务节点），客户端路由每个智能体的动作到对应模型。
  - 非共享参数训练：每个智能体或每组智能体拥有独立的模型（不同服务节点），通过episode-specific的BaseUrl/ApiKey对路由请求。

### 2.3 自动化研究系统（A3R）

- **架构**：领导者智能体（制定假设、设计实验、分析结果）和工作者智能体（在GPU服务器上按蓝图执行实验）。通过结构化实验蓝图通信。
- **流程**：输入研究主题 → 领导者分解并设计多阶段实验计划 → 通过云API或SSH分发蓝图到GPU服务器 → 工作者智能体执行实验并监控错误（利用tmux实现即时崩溃检测） → 领导者收集结果并分析 → 阶段性结论指导下一阶段 → 最终输出研究报告。
- **优势**：支持多天大规模并行实验；无需闭源模型（可用MiniMax-M2.7等开源模型驱动）；成本高效。

## 6. 主要结论与发现

1. **集群架构有效**：完全解耦服务端和客户端，实现了框架无关性、容错、热插拔调试、异构多模型训练和多任务混合训练。
2. **时间线合并显著加速**：在AppWorld上实现6.25倍加速（从2160秒降至346秒），且不损失训练质量。
3. **共享参数训练实用**：在Werewolf中，7B和14B模型训练后胜率大幅提升（如狼人从23%提升到47.2%，14B从40.9%提升到64.7%）。
4. **非共享参数带来行为多样性**：在Werewolf中，三个独立模型分别控制三个狼人，最终胜率66.5%，高于共享参数模型的64.7%，因为独立模型发展出不同个性和策略，使对手更难识别。
5. **多任务鸡尾酒训练可训练通用模型**：在AIME上表现与单独训练相当（平均奖励0.72 vs 0.73），但在AppWorld上差距明显（0.58 vs 0.68），说明工具密集型任务需要更集中优化。
6. **框架无关性验证**：四种智能体框架（OpenAI SDK, LangChain, AgentScope, Raw HTTP）在相同配方下训练曲线和最终评估分数高度一致（最大跨框架差距<0.04），证明训练捕获是框架无关的。
7. **自动化研究可行**：A3R在3天内自主完成19个实验，找到最小稳定批次大小并揭示响应长度对批次下限的影响（mr=12000时bs=4即可，mr=10000时需bs=16）。
8. **vibe训练降低工程量**：从自然语言描述到第一个成功训练曲线，通过AI编码助手大幅缩短时间，但需警惕奖励劫持问题（如Who is the Spy中智能体学会抄袭描述）。

## 7. 优点

- **架构创新**：完全解耦服务端-客户端，是LLM智能体RL领域的首创，提供了其他框架难以同时支持的异构、多任务、容错和运行时代码更新。
- **框架无关且支持黑盒智能体**：仅需修改Base URL和API Key，即可接入任意OpenAI兼容的智能体框架，无需代码插桩。
- **高效的上下文追踪和时间线合并**：解决多轮训练中的冗余问题，加速显著且无质量损失。
- **灵活的样本批处理策略**：C1-C5五种策略覆盖静态到动态、服务端到客户端协调，支持DAPO等需要动态采样的先进算法。
- **自动化研究系统（A3R）**：全开源、可驱动低成本模型、并行实验管理、即时崩溃检测，大幅降低RL研究人力成本。
- **实验充分且结果透明**：公开了基准测试网站（benchmark.agentjet.top）和GitHub仓库，支持复现。
- **实用场景覆盖广泛**：从游戏AI到学术翻译，从数学推理到应用程序操作，验证了通用性。

## 9. 整体总结与未来工作

- **核心贡献**：AgentJet 提出了首个完全解耦服务端-客户端的集群式LLM智能体强化学习框架，创新性地解决了现有集中式框架的运行时脆弱性、调试摩擦、多模型约束和冗余上下文等问题。其时间线合并算法、灵活采样批处理策略和自动化研究系统（A3R）均具有实用价值。
- **未来方向**：
  - 多任务鸡尾酒训练的自动混合比例优化（如基于任务不确定性或梯度冲突的自适应调节）。
  - 奖励劫持的自动化检测与防护机制（如基于副奖励或行为熵的正则化）。
  - 跨节点分布式扩展优化（如通信流水线并行、异步梯度同步）。
  - 更丰富的自动化研究能力：包括架构搜索、多目标优化、课程学习设计等。
  - 与现有框架（如Forge、AReaL）的横向基准测试标准化，便于社区评估。

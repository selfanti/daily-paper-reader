## 摘要
智能体强化学习可能诱发工具滥用，即模型过度使用外部工具，即使对于可通过内部推理解决的查询也是如此。现有方法通过统一的工具使用惩罚或硬性限制来缓解这一问题，这些方法虽然降低了工具使用频率，但可能抑制有益的工具辅助探索。我们提出EAPO，一种高效的智能体策略优化框架，用于学习选择性工具使用。EAPO在每个生成组中引入无工具轨迹，应用难度感知的奖励塑造来主要在简单查询上惩罚冗余工具调用，并使用置信度感知的令牌重加权来改进策略学习。在九个数学和知识密集型推理基准上，EAPO在Qwen2.5-3B、Qwen2.5-7B和Llama3.1-8B上持续改善了准确性与效率的权衡。与GRPO相比，EAPO平均性能分别提升10.45%、7.27%和9.69%，同时平均工具调用分别减少18.33%、18.33%和24.59%。这些结果表明，智能体可以学习何时不使用工具，而不会损害工具集成的推理能力。

## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：智能体强化学习（Agentic RL）中普遍存在**工具滥用（tool abuse）**——模型在遇到简单查询（如“1+1=？”）时仍然会调用外部工具（Python解释器、搜索引擎），尽管这些查询完全可以通过内部推理解决。这种过度依赖工具提高了计算成本、延迟和部署费用。
- **背景**：现有的缓解方法（如统一惩罚、硬性调用次数限制）虽然能降低工具频率，但可能**无差别地抑制有益的工具辅助探索**，导致在真正需要工具的复杂查询上性能下降。
- **关键矛盾**：理想的智能体应当学会**何时**使用工具（选择性使用），而非简单地**减少**工具使用。现有方法缺乏区分“必要工具使用”和“工具滥用”的能力。

## 2. 方法论：EAPO框架

### 核心思想
提出**EAPO（Efficient Agentic Policy Optimization）**，通过学习选择性工具使用来兼顾准确性和效率。核心创新是在训练过程中引入**无工具轨迹**，提供模型是否能仅靠内部推理解决查询的信号，从而区分必要与不必要的工具调用。

### 关键技术细节
- **效率感知生成（Efficiency-Aware Rollout）**：在每次策略采样中，保证至少 $n$ 个轨迹不调用任何工具（通过提示禁用工具、禁用工具执行）。定义效率系数 $\varepsilon(T_i) = \exp(-\beta (c_i - c_{\min}))$，其中 $c_i$ 为轨迹的工具调用次数，$c_{\min}$ 为同批次正确轨迹中的最少调用次数。无工具轨迹 $c_i = c_{\min}$。
- **难度感知奖励塑形（Difficulty-Aware Reward Shaping）**：定义查询难度 $d(q) = 1 - \frac{1}{M}\sum I(T_i)$（正确轨迹比例反向指标）。将调整后的奖励设为 $\tilde{R}_i = [d(q) + (1-d(q))\varepsilon(T_i)] R_i$。这样在简单查询（难度低）上对冗余工具调用施加更强惩罚，在困难查询（难度高）上保留探索自由度。
- **置信度感知令牌重加权（Confidence-Aware Token Reweighting）**：在每个工具轮次内，基于令牌的 logit 置信度（对数概率）分配学习权重。对正确轨迹中低置信度令牌（信息量大）给予更高权重 $W_{\text{pos}}$；对错误轨迹中高置信度令牌（可能过度自信）进行降权 $W_{\text{neg}}$。最终重新计算每个令牌的优势 $A_{i,u,t} = \omega_{i,u,t} A_i$。
- **整体优化目标**：最大化重加权后的策略梯度目标，结合 KL 正则化。

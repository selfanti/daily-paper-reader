## 摘要
LLM代理积累的历史记录超出其上下文窗口，这推动了关于记忆系统的研究日益增多。然而，现有大多数设计针对单一场景（多会话聊天或单条轨迹格式）进行了调优，几乎没有证据表明它们能泛化到代理在部署中遇到的异构轨迹。我们在五个场景（单轮问答、多会话聊天、代理轨迹问答、记忆压力测试以及长周期代理任务）中重新审视了八个记忆系统及一个用于搜索问题的代理框架。该框架通过工具调用自管理平面文本文件存储，取得了最佳跨任务排名，表明记忆性能的关键在于赋予代理对存储和检索的主动控制，而非固定管道后的被动存储。我们将这一见解实例化为AutoMEM，一个具有自管理工具接口的代理记忆框架，在我们评估的系统中实现了最佳的跨场景泛化能力。

## Abstract
LLM agents accumulate histories that outgrow their context windows, motivating a growing literature on memory systems. Yet most existing designs are tuned to a single scenario (multi-session chat or a single trajectory format), and there is little evidence that they generalize across the heterogeneous trajectories agents encounter in deployment. We revisit eight memory systems plus an agentic harness for search problems, on five scenarios: single-turn QA, multi-session chat, agentic-trajectory QA, memory stress tests, and long-horizon agentic tasks. The harness, which self-manages flat text-file storage via tool calls, achieves the best cross-task ranking, suggesting that memory performance hinges on giving the agent active control over storage and retrieval rather than on a passive store behind a fixed pipeline. We instantiate this insight in AutoMEM, an agentic memory harness with a self-managed tool interface that achieves the best cross-scenario generality among the systems we evaluate.

- **核心思想**：记忆性能取决于代理能否在查询时主动、灵活地控制证据的检索与聚合，而不是在构建时固定存储模式。基于这一思想，论文设计了 **AutoMEM**——一个由 LLM 驱动的代理记忆框架，通过工具调用自管理文本文件存储，实现了构建时与查询时的解耦。
- **技术细节**：
  - **动作空间**：代理每一步从四个动作中选择：`rg`（对原始记录进行正则搜索）、`read`（按 ID 获取记录）、`dump`（将短语料直接送入答案 LLM）、`index_query`（对预构建索引执行类型化查询）。

# 6. 论文的主要结论与发现

1. **无方法在所有场景占优**：每个索引方法（预构建结构化存储的方法）至少在一个基准上落后于长上下文；而代理框架 DCI-Lite 保持了最好的泛化排名。
2. **索引方法在代理轨迹 QA 上的两种失败模式**：
   - **表示级失败**：构建时模式会丢弃步骤索引和动作标签（如 HippoRAG 的实体-关系知识图谱无法编码步骤信息），导致数据在存储时就丢失。
   - **检索级失败**：即使存储保留了信息（如 PlugMem 的密集原始轨迹），被动检索（路由器根据问题嵌入选择一条路径返回压缩笔记）无法访问细粒度步骤证据。
3. **主动控制优于被动管道**：DCI-Lite 延迟模式承诺（查询时再决定如何检索）实现了最佳泛化，而 AutoMEM 通过组合代理循环和选择性索引进一步提升了性能。
4. **索引何时值得**：当且仅当工作负载满足两个条件时才值得预构建索引：① 模式能够编码查询所需的谓词（模式表现力）；② 许多问题需要跨单元聚合（结构率高）。
5. **动态代理任务受限于策略本身**：在 ALFWorld 上，所有记忆方法都无法显著提升长上下文基线；只有参数级后训练（GRPO）能做到。说明记忆系统无法超越代理内置行为的极限。
6. **AutoMEM 表现**：在 LoCoMo 上比 DCI-Lite 提升 49.6%，平均排名 3.10（对比 DCI-Lite 的 4.10），且单次通过变体在较低成本下保持竞争力。

## 摘要
大语言模型智能体日益需要跨异构任务模式运行，这些模式要求不同的执行范式。这对固定智能体系统构成挑战，并激发超越孤立组件更新的系统级元适应。现有工作虽已适配外部控制框架或训练底层推理策略，但全系统适应性仍缺乏充分描述。结构与执行之间的适应空间很少被显式定义，且外部控制框架与内部推理器之间的兼容性未得到联合优化。我们提出HarnessForge，一种用于进化大语言模型智能体系统的元自适应框架。HarnessForge将智能体系统形式化为控制框架-策略对，定义了稳定的适应空间，将控制框架级别的执行结构与策略级别的推理行为分离。然后，通过故障引导的控制框架裁剪和受控框架条件的策略对齐，实现控制框架-策略的协同进化。在跨不同领域的五个基准实验上，HarnessForge持续提升了Qwen3-4B和Qwen3-8B主干模型，优于仅控制框架和仅策略的基线，相比最强基线提升高达12.0%，并实现了有利的部署-效率权衡，表明控制框架-策略协同进化是有效的，且控制框架与推理策略之间的可执行兼容性对智能体系统适应至关重要。代码见https://github.com/mingju-c/HarnessForge。

## Abstract
LLM agents are increasingly expected to operate across heterogeneous task regimes that require distinct execution paradigms. This challenges fixed agent systems and motivates system-level meta-adaptation beyond isolated component updates. While existing works have adapted external harness or trained underlying reasoning policies, full-system adaptation remains insufficiently characterized. The adaptation space between structure and execution is rarely made explicit, and the compatibility between the external harness and the internal reasoner is not optimized jointly. We propose HarnessForge, a meta-adaptive framework for evolving LLM agent systems. HarnessForge formulates an agent system as a harness--policy pair, defining a stable adaptation space that separates harness-level execution structure from policy-level reasoning behavior. It then performs harness--policy co-evolution through fault-guided harness tailoring and harness-conditioned policy alignment. Experiments across five benchmarks from diverse domains show that HarnessForge consistently improves both Qwen3-4B and Qwen3-8B backbones, outperforming harness-only and policy-only baselines with gains of up to 12.0\% over the strongest baseline and achieving favorable rollout-efficiency tradeoffs, demonstrating that harness--policy co-evolution is effective, and that executable compatibility between the harness and reasoning policy is essential for agent-system adaptation. The code is available at https://github.com/mingju-c/HarnessForge.

## 6. 主要结论与发现

1. **Harness-Policy 协同进化有效**：HarnessForge 在所有基准上一致超越最强的单独 harness 或 policy 基线，平均提升 3.56%，最高 12.0%（TMDB）。
2. **两者模块均必要**：消融实验显示移除 harness 进化或 policy 对齐都会导致性能下降，且差距随进化轮数增大，表明互补性。
3. **兼容性关键**：交叉兼容性矩阵显示匹配对角线显著优于非匹配组合，最终 harness 与最终 policy 搭配达 77.19%，而与早期策略组合平均仅 71.93%，证实**可执行兼容性是系统适应性的核心**。
4. **部署-性能权衡优秀**：HarnessForge 在多个预算下处于帕累托前沿，比纯训练型方法更高效（复用已收集轨迹）。
5. **通用框架**：HarnessForge 不绑定特定训练方法，可结合 SFT/GRPO/RLOO 使用，SFT 在低预算下足够高效。

## 7. 优点

- **方法论创新**：首次将智能体系统元适应形式化为 harness-policy 对联合进化，明确分离结构与执行接口，提出故障定位-档案引导-裁剪-对齐的完整闭环。
- **资源高效**：策略对齐完全复用 harness 裁剪阶段的轨迹，不额外花费部署预算。
- **可审计性**：进化过程通过分阶段元智能体链实现，每个阶段输出结构化中间产物（故障报告、改进简报、候选清单），而非黑箱生成。
- **实验全面**：覆盖多种任务类型、多种基线、消融、敏感性、兼容性分析，个案可视化清楚显示修复机制。
- **框架通用性**：不依赖特定训练算法或元智能体版本，提供了可替换的接口。

## 8. 不足与局限

- **实验规模限制**：仅评估 Qwen3-4B/8B 两个中等规模模型，未在更大模型（如 70B/130B）上验证；作者指出在资源受限场景下强耦合效应更显著，但大规模模型是否同样受惠未证实。
- **部署成本**：尽管内部高效，仍需要多轮部署（每轮约 1200 个任务），在超长任务环境下成本仍高。未来可用代理评估或早停策略优化。
- **操作空间非穷尽**：harness 编辑仅限于规划/动作/记忆三个模块的结构性修改，未探索自由编程级重写或学习验证器等更广空间。
- **元智能体依赖**：当前使用 GPT-5.5 作为元智能体，虽提供可替代接口，但进化质量受限于所选模型；不同元智能体可能导致不同结果。
- **对单一基准迁移性**：TMDB 和 API-Bank 仅作为迁移测试（从 ToolHop 进化后适配 API 接口），未在这些基准上直接搜索 harness，可能低估完全搜索潜力。
- **训练方法无关性分析较浅**：虽对比了 SFT/GRPO/RLOO，但未深入探讨偏好或 RL 方法在特定任务场景下的优势/劣势。

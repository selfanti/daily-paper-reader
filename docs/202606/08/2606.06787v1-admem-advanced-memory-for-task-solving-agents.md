## 摘要
大型语言模型（LLM）作为工具使用型智能体展现出潜力，但在需要记忆、组织及复用知识的长期任务中仍存在局限。现有记忆方法旨在解决该问题，但主要关注存储事实性信息。近期关于程序性记忆的研究改善了任务复用，但往往简化为重放过去成功案例，未能处理失败案例或实现在线可扩展性。我们提出一种统一且自动化的记忆框架，通过结合短期与长期存储的双层设计，整合语义、情景和程序性记忆。基于演员、记忆和评论智能体的多智能体架构实现了自动化记忆生成、奖励标注与自适应检索。长期记忆通过基于奖励的评估、合并与剪枝进行管理，确保可扩展性与持续改进。跨多种环境的实验表明，与现有基线相比，我们的方法在长期多轮任务上提升了鲁棒性与成功率。本工作凸显了全面、自适应的记忆对于推进基于LLM的智能体的重要性。

## Abstract
Large Language Models (LLMs) show promise as tool-using agents but remain limited in long-horizon tasks that require remembering, organizing, and reusing knowledge. Prior memory approaches aim to resolve the situation, but mainly focus on storing factual information. Recent work on procedural memory improves task reuse, yet often reduces to replaying past successes without addressing failure cases or online scalability. We introduce a unified and automatic memory framework that integrates semantic, episodic, and procedural memory in a bi-level design combining short-term and long-term stores. A multi-agent architecture with actor, memory, and critic agents enables automatic memory generation, reward annotation, and adaptive retrieval. Long-term memory is managed through reward-based evaluation, merging, and pruning, ensuring scalability and continual improvement. Experiments across various environments show that our approach improves robustness and success on long multi-turn tasks compared to existing baselines. This work highlights the importance of comprehensive, adaptive memory for advancing LLM-based agents.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **统一记忆框架**：整合语义记忆（事实知识）、情景记忆（事件摘要）和程序性记忆（决策指导），采用**短期记忆+长期记忆**的双层结构。
- **多智能体架构**：由**演员智能体**（Actor Agent，负责与环境交互）、**评论智能体**（Critic Agent，负责程序性记忆生成与奖励标注）和**记忆智能体**（Memory Agent，负责长期记忆存储、检索与维护）协作完成记忆的生命周期管理。
- **基于奖励的自适应管理**：程序性记忆的检索不仅依赖上下文相似性，还结合记忆条目的有效性评分（通过轻量级概率模型估计），并通过EM算法在线更新。

### 关键技术细节
1. **短期记忆（STM）**：采用类“调用栈”结构，支持子目标（sub-goal）的添加与结束。进入子目标时，外部上下文被压缩为摘要；子目标完成后恢复高层目标，实现上下文压缩和范围管理。
2. **长期记忆（LTM）**：
   - **语义记忆**：通过LLM摘要更新事实信息，由记忆智能体管理合并与删除。
   - **情景记忆**：存储事件摘要，使用稠密检索（dense retrieval）进行相似度匹配。
   - **程序性记忆**：核心创新点。每个条目包含上下文、动作、反思（reflection）和成功率标签。检索时评分函数为：`vm · ψ(ct)^T ψ(cm)`，其中`vm`是条目有效性参数（[0,1]），ψ是归一化稠密嵌入。使用**EM算法**在线更新`vm`：假设奖励产生过程为`rt = l_t ∨ ∨_{i} l_t^{(i)}`，其中`l_t^{(i)} ~ Bernoulli(vm^{(i)} · similarity)`。初始化乐观设置`vm=1`。
3. **记忆生成流程（算法1和2）**：
   - 演员每步行动时，评论智能体缓存“半成品记忆”（行动上下文、行动、预期结果、检索到的记忆）。
   - 当评论智能体收到后续观察，判定事件完结后，生成程序性记忆条目，包含：上下文、动作、结果、成功与否、反思。
   - 同时将奖励反馈给记忆智能体，用于更新检索到的程序记忆条目的`vm`值。
4. **记忆维护**：对程序性记忆，设置阈值ε：低频条目删除，持续共同检索的条目合并；语义记忆由LLM进行信息更新。

  - 消融实验：LLM+短期规划（STP，去除长期记忆）、LLM+长期程序性记忆（LTP，去除短期规划）、LLM+AdMem完整版。其中LTP和AdMem还测试了多轮次（多epoch）持续学习效果。

## 6. 论文的主要结论与发现

- **AdMem显著提升了长期多轮任务的鲁棒性和成功率**，在AgentBoard的8个领域中有7个优于ReAct和AWM。
- **程序性记忆不能仅靠回放成功案例**：AWM的失败表明，不加区分的经验复用可能带来混淆。AdMem通过奖励驱动评估和自适应检索有效缓解了这一问题。
- **短期规划和长期记忆是互补的**：短期规划提供上下文压缩和结构化执行能力，长期记忆提供跨任务的经验迁移。两者结合优于任何单一组件。
- **长期记忆的持续改进潜力**：多轮次实验中，AdMem性能随任务重复而提升（Jericho第1轮45% → 第3轮60%），而单独使用长期记忆（LTP）提升微弱，验证了信用分配和反思机制的重要性。
- **框架设计避免了记忆干扰**：在WebShop等记忆可能有害的领域，AdMem未大幅损伤性能（仅下降1-2个百分点），说明自适应检索和规划结构能有效控制有害经验的影响。

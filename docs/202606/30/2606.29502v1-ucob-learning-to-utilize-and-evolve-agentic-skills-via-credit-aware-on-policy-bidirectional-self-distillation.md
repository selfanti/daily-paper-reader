## 摘要
技能记忆可以通过将过去经验作为文本指导重用来改进智能体强化学习，但检索到的技能并非万无一失：它们可能在一个状态下有帮助，而在另一个状态下误导同一策略。这使得常见的特权教师假设变得脆弱，即技能条件提示可以被视为无技能提示的固定教师。我们提出UCOB，一种通过信用感知的在策略双向自蒸馏来学习利用与进化智能体技能的框架。UCOB将技能条件提示和无技能提示视为同一模型的两个在策略上下文视图，在同一任务和锚定状态下比较它们的预期回报，并将更高回报的视图作为局部教师。这种局部信用信号内化有用的技能条件行为，纠正误导性的技能使用，并指导任务/状态技能记忆更新、效用感知检索以及反射自训练。在包括ALFWorld、WebShop和Search-QA的智能体任务上的实验表明，UCOB在不同模型规模下均优于无技能强化学习、技能记忆基线和自蒸馏方法，在ALFWorld和WebShop上相比最先进基线分别提升高达23.5和18.0个点。消融和分析进一步验证了其核心机制和效率。

## Abstract
Skill memories can improve agentic reinforcement learning by reusing past experience as textual guidance, but retrieved skills are not oracular: they may help in one state while misleading the same policy in another. This makes the common privileged-teacher assumption fragile, namely that a skill-conditioned prompt can be treated as a fixed teacher for the no-skill prompt. We introduce UCOB, a framework for learning to utilize and evolve agentic skills via credit-aware on-policy bidirectional self-distillation. UCOB treats skill-conditioned and no-skill prompts as two on-policy context views of the same model, compares their return-to-go within the same task and anchor state, and uses the higher-return view as the local teacher. This local credit signal internalizes useful skill-conditioned behavior, corrects misleading skill usage, and guides task/state skill memory updates, utility-aware retrieval, and reflection self-training. Experiments on agentic tasks, including ALFWorld, WebShop, and Search-QA, show that UCOB outperforms skill-free RL, skill-memory baselines, and self-distillation methods across model scales, with up to 23.5 and 18.0 point gains over SOTA baselines on ALFWorld and WebShop. Ablations and analyses further validate its core mechanisms and efficiency.

### 1. 论文的核心问题与整体含义（研究动机和背景）

论文聚焦于**语言智能体的技能记忆学习**问题。在长期交互任务中，智能体可以通过存储和检索过往经验（即“技能”）来提升探索效率和样本利用率。然而，一个关键的隐式假设被广泛使用：即**技能条件提示**（skill-conditioned prompt）可以被视为一个“特权教师”，其产生的行为总是优于无技能提示，从而将技能知识蒸馏给无技能策略。

本文通过实验诊断挑战了这一假设，发现该假设是**脆弱且不可靠的**：

- 检索到的技能可能是**不相关、过时、过于泛化或与当前状态不匹配**的，导致技能条件视图并不总是优于无技能视图。
- 即使将技能条件视图（通过在线rollout）对齐到策略分布，其“教师权威性”仍然不明确，因为同一状态下两个视图的局部价值（预期回报）可能是**混合符号**的（有时技能更好，有时无技能更好，甚至可以在同一轨迹的不同步骤翻转）。

因此，核心问题是：**当技能和无技能两个视图在同一状态下产生冲突时，应该由谁来教导谁？**  论文旨在解决这一动态判断问题，提出一个能智能地利用有用技能、纠正误导技能，并同步进化技能记忆库的统一框架。

### 2. 论文提出的方法论：核心思想、关键技术细节

论文提出**UCOB**框架，核心思想是将技能条件和无技能提示视为同一在线模型的**两个上下文视图**，通过**局部信用信号**（同一锚定状态下的预期回报比较）动态决定哪个视图成为教师，进行双向自蒸馏，从而实现技能的利用、内化与进化。

**核心技术细节：**

1.  **双重技能记忆与UCB检索**：
    - 维护**任务级**和**状态级**两种技能池，分别存储可复用的全局策略和局部决策规则。
    - 检索时使用**基于UCB（Upper Confidence Bound）**的评分函数，结合语义相似度和技能的探索-利用平衡（效用分数与使用次数），选择Top-K技能构成技能条件提示。

2.  **混合技能/无技能在线Rollout**：
    - 在一个训练批次中，一半轨迹使用**技能条件提示**（P+）与环境交互，另一半使用**无技能提示**（P0）交互。两个视图都参与在线强化学习（RL）更新。

3.  **信用感知的双向自蒸馏（CBSD）**：
    - **锚定状态分组**：根据GIGPO方法，将同一任务、同一锚定状态（anchor state，如“到达某个房间”）下的所有记录（来自两个视图）分组。
    - **教师方向选择**：在每个锚定状态组内，选择**预期回报最高**的记录作为“信用参考”（credited reference），其对应的视图成为**局部教师**。如果技能视图赢，则技能->无技能；如果无技能视图赢，则无技能->技能。
    - **蒸馏过程**：对于每个被接受的对（教师-学生），使用**Top-K Token支持匹配**，在教师分布的前K个token上计算KL散度作为蒸馏损失。同时，使用**回报间隙权重**（gap weight）和**置信度门**（confidence gate）来调节蒸馏强度，防止错误信号。
    - 通过CBSD，有用的技能诱导行为被内化到无技能视图，而误导性的技能行为则被无技能视图纠正。

4.  **技能记忆进化**：
    - **技能写入**：基于CBSD产生的信用证据，任务级技能从轨迹反思中写入；状态级技能从同一锚定状态内高低回报记录的对比中写入。
    - **效用更新**：使用指数移动平均（EMA）更新每个被使用技能的效用得分，高回报改善的技能效用增加，反之减少。
    - **技能写入器自训练**：利用技能效用计算“反思优势”，通过策略梯度损失来训练反思生成技能的行为，使高效用技能更可能被再次生成。

5.  **整体目标函数**：  
    $L = L_{RL} + \lambda_{CBSD} L_{CBSD} + \lambda_{refl} L_{refl} + \beta_{KL} L_{KL}$  
    结合在线RL损失、CBSD蒸馏损失、反射自训练损失和KL正则化。

### 6. 论文的主要结论与发现

1.  **特权教师假设不成立**：技能条件提示并非一个可靠且自纠正的教师；即使在线rollout也无法消除其局部误导性。教师方向应由同一状态下的局部价值差距决定。

2.  **UCOB有效提升智能体性能**：UCOB在三个基准上，**所有模型规模下均一致优于**无技能RL、技能记忆基线和自蒸馏方法。在Qwen3-1.7B上，相比Skill1, 在ALFWorld和WebShop上分别提升了**+23.5和+18.0个点**。

## 摘要
AI代理的自我改进已成为关键研究前沿：系统根据积累的操作经验修改自身的提示、工作流程和决策规则。最先进的Self-Harness框架[1]通过挖掘失败集群并修补代理harness，在Terminal-Bench-2.0上实现了14-21%的提升。然而，Self-Harness仅优化单一维度——提示harness，而行为原则和工作流拓扑保持不变。我们提出APEX（自适应原则提取），一个三层共进化框架，同时进化：（L1）通过失败模式修补的harness，（L2）通过成功轨迹蒸馏[2]的行为原则，以及（L3）基于结构适应度选择[6]的代理工作流拓扑。我们在Joe[13]上实现APEX，Joe是一个基于NVIDIA Nemotron构建的生产级超级AI代理，专为NVIDIA Agent Challenge 2026设计作为边缘AI代理工厂，使用18天内收集的114个真实任务轨迹管理一个15节点计算集群。APEX在单次进化运行中达到0.570的APEX健康评分（相比基线0.300提升90%），提炼出6个新颖的可重用原则，并选择了一个研究优先的工作流拓扑，得分0.900（提升20%）。我们的结果表明，多维共进化显著优于单轴harness优化，在本地qwen2.5-coder:32b实例上仅需4次LLM调用（约270秒）。

## Abstract
Self-improvement in AI agents has emerged as a key research frontier: systems that modify their own prompts, workflows, and decision rules based on accumulated operational experience. The state-of-the-art Self-Harness framework [1] achieves 14--21% improvement on Terminal-Bench-2.0 by mining failure clusters and patching the agent harness. However, Self-Harness optimises only one dimension -- the prompt harness -- leaving behavioural principles and workflow topology unchanged. We propose APEX (Adaptive Principle EXtraction), a three-layer co-evolution framework that simultaneously evolves: (L1) the harness via failure-mode patching, (L2) behavioural principles via success-trace distillation [2], and (L3) the agent workflow topology via structural fitness-based selection [6]. We implement APEX on Joe [13], a production-grade super AI Agent built on NVIDIA Nemotron and designed as an Edge AI Agent Factory for the NVIDIA Agent Challenge 2026, managing a 15-node compute fleet using 114 real task traces collected over 18 days. APEX achieves an APEX Health Score of 0.570 (+90% vs. baseline 0.300) in a single evolutionary run, distilling 6 novel reusable principles and selecting a research-first workflow topology scoring 0.900 (+20%). Our results demonstrate that multi-dimensional co-evolution substantially outperforms single-axis harness optimisation, at a cost of only 4 LLM calls (~270 s) on a local qwen2.5-coder:32b instance.

### 核心思想

### 关键技术细节

#### 2.1 Layer 1: Harness Patching（基于Self-Harness变体）
- **输入**：轨迹数据库中的失败轨迹（通过关键词`error, fail, wrong, mistake`从`lesson`字段中筛选）。
- **处理**：选取最近30条失败轨迹，提交给本地LLM（`qwen2.5-coder:32b`），提示词：“识别前3个系统性失败模式及其根本原因，并生成一条具体的禁止规则。”
- **输出**：3条harness补丁（例如：端口冲突、前端稳定性缺失、危机检测延迟），注入下一代的系统提示中。

#### 2.2 Layer 2: Principle Distillation（受EvolveR启发）
- **输入**：高质量成功轨迹（采用多因素质量评分`s(t)`筛选）。
- **质量评分公式**：
  ```
  s(t) = 0.4 × I[lesson长度>50] + 0.3 × I[动作数>30] + 0.2 × I[修改文件非空] + 0.1 × I[来源非self]
  ```
- **处理**：取前30%的轨迹（约34条），提交LLM：“提取6条可复用的行为原则。” 然后对候选原则进行**新颖性评分**（基于与现有原则的余弦相似度，惩罚重复）、**特异性评分**（长度代理）和**完整性评分**。综合得分≥0.3的原则存入`apex_principles`。
- **输出**：6条新颖原则（平均新颖性得分0.998），例如：为AI服务超时实现回退机制、在开放式查询前摄取完整知识库等。

#### 2.3 Layer 3: Workflow Topology Evolution（受AFlow启发）
- **输入**：基于规范节点词汇表（`intake, research, plan, code, review, verify, dispatch, summarize`）定义的工作流DAG种群。
- **结构适应度评分公式**：
  ```
  score(G) = 0.50 + 0.10 × I[review ∈ G] + 0.10 × I[verify ∈ G]

  ```
- **变异算子**：`add_node`（在plan前插入research）、`add_routing`（在review失败时增加自校正循环）、`insert_verify`（在code后增加验证阶段）。
- **进化过程**：每代保留前2个拓扑，各产生2个变异子代；共3代，评估10个候选拓扑。
- **输出**：最优拓扑`research_first_v1`（评分0.900），其结构为：research → plan → code → review → verify。

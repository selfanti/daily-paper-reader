## 摘要
强化学习后训练策略依赖于数据集，并揭示了一个反复出现的经验模式：容量参数在各阶段单调累积，而正则化参数则主要随着训练动态的转变而振荡。这种区别之所以重要，是因为固定的调度将所有参数都绑定在固定的轨迹上，因此无法表达正则化必须跟踪的非平稳探索-利用权衡；该原则为多阶段训练提供了可操作的设计规则。我们通过LLMZero发现了这一点，该系统利用LLM智能体通过树搜索来搜索训练轨迹，在每个检查点诊断问题并提出协调的多参数转换。在4个不同的GRPO任务上，LLMZero发现的策略相对于基础模型提升了9%到140%，相对于网格搜索提升了6%到15%，始终优于随机搜索和基于技能的智能体。该结构原则可跨任务迁移，解释了为什么发现的策略在形式上定性不同但具有相似的参数动态。

## Abstract
RL post-training strategies are dataset-dependent and reveal a recurring empirical pattern: capacity parameters accumulate monotonically across stages, while regularization parameters predominantly oscillate in response to shifting training dynamics. This distinction matters because fixed schedules commit all parameters to fixed trajectories and therefore cannot express the non-stationary exploration-exploitation tradeoffs that regularization must track; the principle provides actionable design rules for multi-stage training. We discover this through LLMZero, a system where LLM agents search over training trajectories via tree search, diagnosing pathologies at each checkpoint and proposing coordinated multi-parameter transitions. Across 4 diverse GRPO tasks, LLMZero discovers strategies that improve over the base model by 9% to 140% relative and over grid search by 6% to 15% relative, consistently outperforming random search and the skill-based agent. The structural principle transfers across tasks, providing an explanation for why discovered strategies take qualitatively different forms yet share similar parameter dynamics.

## 2. 论文提出的方法论
- **核心思想**：构建训练轨迹树，利用LLM智能体在每阶段检查点分析训练动态（文本指标+可视化曲线），诊断病理并提出协调的多参数转换，通过蒙特卡洛树搜索（MCTS）自动发现最优多阶段策略。
- **关键技术细节**：
  - **树结构**：每个节点代表一个训练阶段，从父节点检查点恢复并修改超参数。父节点可重用相同检查点以节省计算。
  - **Proposer Agent**：四步推理：诊断训练健康状况 -> 提出协调的多参数变化（考虑因果依赖） -> 决定恢复点（从某个检查点或从头训练） -> 计算epoch数。注入人类编写的指标和超参数描述以纠正LLM误解。
  - **Agentic Early Stopper**：每900秒将当前轨迹与最佳已完成的策略进行覆盖图对比，输出继续或停止决策，停止约62%的节点，节省40-60% GPU时间。
  - **Forced From-Scratch Injection**：当从头开始的比例低于0.2时，强制从不相关节点创建新分支，防止搜索陷于次优初始配置。
  - **UCT搜索**：采用改进的UCT选择节点，结合虚拟新子节点竞争，平衡深度与宽度。
- **公式/算法**（文字描述）：选择节点时计算每个子节点的UCT值，并创建虚拟新子节点与之竞争；若虚拟节点胜出且未达最大子节点数，则扩展新节点。执行训练后，将验证分数反向传播，并剪枝终端子树。最终策略为从根到最佳叶节点的路径。

## 6. 论文的主要结论与发现
1. **结构不对称原则**：在所有四个任务的最佳策略中，容量参数（响应长度、rollout数）单调累积；正则化参数（学习率、KL系数、温度）振荡。原因：容量参数是信息构造性，减少会丢弃先前阶段成果；正则化参数跟踪非平稳探索-利用权衡，单调衰减不适用。
2. **跨任务迁移性**：该原则适用于不同任务，但具体振荡模式依赖数据集（如ChemCoT出现KL尖峰，PaperSearchQA KL单调递增）。固定调度无法预测哪种模式占主导。
3. **LLMZero显著优于所有基线**：相对基础模型提升9%-140%，相对网格搜索提升6%-15%；在WildSci上还能恢复被低劣实践者配置损害的性能（53.2%→58.5%）。
4. **迭代效率高**：在3/4任务中，最佳策略在前12次迭代内发现。
5. **消融证实多阶段和早停是关键**：移除多阶段组合导致精度下降9.4pp；移除视觉推理或早停虽精度相近，但计算效率大幅降低。
6. **多维协调转换有效**：所有最佳策略中，高收益阶段同时改变3+个超参数（如提高学习率同时增加KL惩罚，防止发散），此类组合难以通过单参数搜索发现。

## 7. 优点
- **方法创新**：将LLM推理与树搜索结合用于训练策略搜索，提出检查点组合、多模态诊断、agentic早停和强制从零启动等机制。
- **可迁移设计原则**：不仅给出策略，还提取了通用规则（容量单调/正则化振荡），为社区提供了指导。
- **计算高效**：早停减少了40-60% GPU消耗，且固定工作流API成本极低（仅$48）。
- **实验严谨**：跨4个领域、多种模型规模、充分消融、策略迁移验证，结果可靠。
- **诊断质量高**：人工验证所有检测到的训练动态均正确，且多数转换改善验证分数。

## 8. 不足与局限
- **模型家族限制**：仅验证了Qwen3系列（0.6B-8B），未测试更大模型或其他架构（如Llama、DeepSeek）的泛化性。
- **数据规模限制**：所有数据集子采样至5,000训练样本，未验证生产级海量数据下的策略有效性。
- **搜索开销依然较高**：尽管有早停，LLMZero仍需数千GPU-hour，且一次只扩展一个节点；作者建议结合种群训练（PBT）作为未来方向。
- **无法补偿模型能力不足**：在分子优化等需要结构能力而非推理能力的子任务上，所有方法得分为0%，说明自适应调度无法弥补模型固有能力缺失。

- **框架依赖**：实验基于内部VeRL变体，代码迁移至公开版本需要时间；不同框架可能影响可复现性。
- **超参数对任务差异的敏感性**：策略迁移实验显示，固定迁移的SSMR策略在PaperSearchQA上甚至低于实践者基线，说明完全套用发现策略而不考虑动态调整仍有风险。

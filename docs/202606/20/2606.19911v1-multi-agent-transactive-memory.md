## 摘要
面向多样任务部署具有不同能力的LLM智能体的去中心化方式，催生了异构智能体群体间知识共享的基础设施需求。正如搜索引擎对人类生成的制品进行索引以支持人类解决问题一样，检索系统可以组织智能体生成的制品，以便在智能体群体中重用。我们将检索增强生成（该技术证明了人类撰写制品对单个智能体的价值）扩展到对支持群体智能体的智能体生成制品的检索。特别地，智能体轨迹编码了可重用的程序性知识，但这些制品通常在使用一次后被丢弃，或仅由生成智能体保留，迫使新实例化的智能体反复重新发现现有解决方案。我们提出了多智能体交互记忆（MATM），这是一种用于群体级别存储和检索智能体生成轨迹的框架，其中生产者智能体将轨迹贡献到共享仓库，消费者智能体检索它们以改善任务执行。我们聚焦于交互式环境（ALFWorld和WebArena），这些环境中的轨迹长且编码了特别丰富的程序性结构。实验表明，从MATM检索轨迹可提高下游任务性能，并在无需协调或联合训练的情况下减少交互步骤。这些结果将MATM定位为开放智能体生态系统中群体级别经验共享的设计模式。

## Abstract
The decentralized deployment of LLM agents with diverse capabilities across diverse tasks motivates infrastructure for knowledge sharing across heterogeneous agent populations. Just as search engines index human-generated artifacts to support human problem solving, retrieval systems can organize agent-generated artifacts for reuse across agent populations. We extend retrieval-augmented generation - which demonstrates the value of human-authored artifacts to individual agents - to retrieval of agent-generated artifacts supporting a population of agents. In particular, agent trajectories encode reusable procedural knowledge, yet these artifacts are typically discarded after a single use or retained only by the producing agent, forcing newly instantiated agents to repeatedly rediscover existing solutions. We propose Multi-Agent Transactive Memory (MATM), a framework for population-level storage and retrieval of agent-generated trajectories, where producer agents contribute trajectories to a shared repository and consumer agents retrieve them to improve task execution. We focus on interactive environments (ALFWorld and WebArena), where trajectories are long and encode especially rich procedural structure. Our experiments demonstrate that retrieving trajectories from MATM improves downstream task performance and reduces interaction steps without coordination or joint training. These results position MATM as a design pattern for population-level experience sharing in open agent ecosystems.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大规模部署的 LLM 智能体在执行任务时会生成丰富的交互轨迹（action-observation trajectories），但这些轨迹通常只被使用一次或仅由产生它的智能体保留，导致新实例化的智能体反复“重新发明轮子”，浪费算力且阻碍群体层面的知识积累。
- **整体含义**：如同搜索引擎索引人类生成的文档来支持人类解决问题，本文提出**为智能体群体构建共享的“事务性记忆”**——即一个跨智能体的轨迹检索与复用基础设施。这被视为从个体优化转向集体知识基础设施的关键步骤，可减少冗余探索、支持持续学习和跨智能体能力迁移。
- **背景**：现有方法如检索增强生成（RAG）仅复用人类文档；思维/轨迹重用局限于同一智能体；迁移学习或知识蒸馏需要额外训练且难以适应异构动态群体。因此需要一种**无需协调或联合训练**的群体级经验共享机制。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **Multi-Agent Transactive Memory (MATM)**：基于“事务性记忆”概念，构建一个共享仓库，智能体可同时担任**生产者**（贡献轨迹）和**消费者**（检索轨迹辅助决策）。通过检索增强推理，使消费者智能体在当前状态基础上获得相似成功轨迹的指导。

### 关键技术细节
1. **状态条件键值索引**：
   - 对每条轨迹，按窗口大小 \(l=5\) 分割成片段。每个片段定义**键**为任务描述 + 最近 \(l\) 步观察-动作历史，**值**为后续 \(l\) 步的序列（作为检索文档）。
   - 嵌入函数 \(f\) 使用 E5-Base 模型对所有键进行编码，建立稠密索引。

2. **检索过程**：
   - 消费者智能体携带一个 **RetrievalPlanner** LLM，在每个交互步决定是否需要发起检索（避免每步都检索导致上下文稀释）。
   - 检索器返回 Top-K 候选轨迹片段（本文 K=20），再经重排序选取 Top-1 注入智能体上下文。

3. **学习排序（LTRT）**：
   - 采用**学习排序**方法对候选轨迹重新排序，设计 44 维特征，包括：生产者元数据（基准分数等）、消费者元数据、第一阶段检索分数、查询特征、轨迹特征、查询-轨迹交互特征（TF-IDF 相似度、嵌入相似度等）。
   - 标签定义为**边际效用**：将轨迹片段注入消费者后任务得分减去无检索基线得分。
   - 训练三种排序模型：点式前馈网络（FFN）、对式 LambdaMART、对式 SVMRank。

### 算法流程（文字说明）
1. **预填充阶段**：从公开轨迹源（如预训练 seq2seq 模型、Qwen3-32B、GPT-OSS 20B 等）生成初始索引 \(D_0\)。
2. **增量构建**（同时生成 LTRT 训练数据）：
   - 将训练集问题划分为多个分区 \(X_p\)，每个分区内按类型均衡分配给不同生产者智能体。
   - 对每个问题，智能体先无检索执行获得基线轨迹和得分，再随机采样 \(T\) 个分支点（ALFWorld: T=2, WebArena: T=1）。
   - 在每个分支点，滚动到该前缀，检索 Top-K 片段，对选定的排序位置 \(I=\{1,5,10,15,20\}\) 进行单步 rollout，得到轨迹及得分，并与基线得分相减得到标签。
   - 成功轨迹（得分 ≥ 阈值 θ）被加入缓冲区，处理完所有问题后分段、嵌入、加入索引，更新为 \(D_p\)。
3. **推理阶段**：消费者智能体使用预训练的 LTRT 重排序器选取最优轨迹片段，注入提示中进行决策。

## 3. 实验设计

- **数据集/场景**：
  - **ALFWorld**：基于文本的家庭任务环境，6 种任务类型（Pick & Place, Clean & Place 等），含 3553 训练集和 274 测试集。
  - **WebArena**：网页导航环境，5 类网站（电商、论坛、地图、内容管理、代码协作），共 812 个任务，自建训练/测试划分（724 训练，88 测试），保持意图分布。

- **智能体种群**：
  - 生产者：35~37 个异构 LLM 模型（如 GPT-4-Turbo, Claude-3.5-Sonnet, Qwen3-32B, DeepSeek-R1 等）。
  - 消费者：34 个模型（与生产者有部分重叠，但角色可互换）。

- **对比方法**：
  - **无检索**（baseline）。
  - **单阶段稠密检索**（仅基于嵌入相似度）。
  - **三种 LTR 重排序**：FFN、LambdaMART、SVMRank。

- **评估指标**：
  - **有效性**：成功率（SR）。
  - **效率**：每回合平均步骤数（#steps）。
  - **联合指标**：Return-Paired Preference (RPP)，衡量 Pareto 优势。

- **其他消融实验**：
  - 重排序 vs 单阶段。
  - 生产者-消费者能力差距与检索收益的关系。
  - 检索范围：全量、同任务类型、跨任务类型。
  - 索引规模缩放（10%, 25%, 50%, 75%, 100%）。

## 6. 主要结论与发现

1. **MATM 检索提升效果与效率**：在 ALFWorld 上，单阶段检索使 SR 从 47.1%→55.1%，步骤减少；在 WebArena 上从 18.2%→20.5%，步骤减少。RPP 提升表明更优 Pareto 前沿。
2. **学习排序进一步改善**：SVMRank 在 ALFWorld 达最佳（SR 64.3%，RPP 0.15）；FFN 在 WebArena 最佳（步骤最低 19.91，RPP 0.04）。不同重排序器在不同环境表现不一。
3. **收益广泛分布**：生产者-消费者能力差距与检索收益相关性弱且不显著（r≈0.03-0.09），表明即使是较弱的生产者也能提供有用轨迹；重排序增强了整体收益分布。
4. **跨任务泛化**：跨任务类型检索仍优于无检索基线（ALFWorld SR 59.9% vs 47.1%），证明轨迹包含可迁移模式。全量检索优于同任务或跨任务限制。
5. **随索引规模提升**：ALFWorld 上 SR 和步骤效率单调改进；WebArena 上步骤单调下降，SR 在中等规模处有波动后恢复，表明足够大的库能提供更可靠匹配。

## 7. 优点：方法或实验设计亮点

- **新颖视角**：将经验复用从个体优化提升至**群体级基础设施**，契合开放生态系统需求。
- **灵活去中心化**：无需智能体间协调或联合训练，生产者与消费者角色可重叠，天然支持异质性和动态加入。
- **有效的检索架构**：组合状态条件索引、选择性策略（RetrievalPlanner）和轻量级学习排序，兼顾相关性与实用性。
- **特征设计富有洞察**：引入生产者元数据实现信任建模，消费者元数据实现个性化，扩展了传统检索特征集。
- **评估体系全面**：同时考量成功率、步骤效率、Pareto 优势（RPP），并分析能力间隙、缩放行为，验证了方法的鲁棒性。
- **开放资源**：提供全部轨迹数据（HuggingFace）和代码仓库，可复现性强。

## 8. 不足与局限

- **环境覆盖有限**：仅测试了 ALFWorld 和 WebArena 两个交互环境，结论在更复杂或完全不同的环境（如机器人控制、对话系统）中是否成立尚不确定。
- **跨环境泛化未验证**：重排序器在同一环境内训练和测试，未检验输出框架在新环境或跨任务泛化的能力。

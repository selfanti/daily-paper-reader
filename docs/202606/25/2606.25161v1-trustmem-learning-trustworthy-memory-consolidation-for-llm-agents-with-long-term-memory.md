## 摘要
大型语言模型智能体依赖长期记忆来支持超出有限上下文窗口的扩展交互和个性化辅助。现有记忆智能体通过生成的写入、修订和删除操作主动更新外部记忆，但这些更新可能遗漏重要信息、破坏现有记忆或引入不支持的幻觉内容。一旦存储，这些错误就会成为持久的系统状态故障，影响未来的推理和生成。在本文中，我们提出TrustMem，一个旨在提高记忆巩固可信度的框架。TrustMem依靠记忆转换验证器来评估记忆更新在覆盖性、保留性和忠实性方面的转换过程。它进一步在相同记忆状态下构建候选更新之间的偏好对，使偏好引导的强化学习能够直接优化记忆更新行为。大量实验表明，TrustMem提高了记忆效用和可靠性：它在MemoryAgentBench、HaluMem和Mem-alpha验证集上取得了最先进的结果，将HaluMem记忆提取的F1分数提高了12.14分，与每种错误类型的最强基线相比，分别减少了40.1%、79.1%和50.0%的转换级遗漏、破坏和幻觉。

## Abstract
Large language model (LLM) agents rely on long-term memory to support extended interactions and personalized assistance beyond finite context windows. Existing memory agents actively update external memory through generated write, revise, and delete operations, but these updates may omit important information, corrupt existing memory, or introduce unsupported hallucinated content. Once stored, such errors become persistent system-state failures that can affect future reasoning and generation. In this paper, we propose TrustMem, a framework designed to improve the trustworthiness of memory consolidation. TrustMem relies on a Memory Transition Verifier to evaluate the transition process of memory updates in terms of coverage, preservation, and faithfulness. It further constructs preference pairs among candidate updates under the same memory state, enabling preference-guided reinforcement learning to directly optimize memory updating behaviors. Extensive experiments demonstrate that TrustMem improves both memory utility and reliability: it achieves state-of-the-art results across MemoryAgentBench, HaluMem, and the Mem-alpha validation set, improves HaluMem memory extraction by 12.14 F1 points, and reduces transition-level omission, corruption, and hallucination by 40.1\%, 79.1\%, and 50.0\%, respectively, compared with the strongest baseline for each error type.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 智能体依靠长期记忆来支持超出有限上下文窗口的扩展交互和个性化服务，但现有的主动记忆更新机制（写入、修订、删除）存在三大缺陷：**遗漏**关键信息、**破坏**已有记忆、**引入不支持的幻觉内容**。这些错误一旦存入持久记忆，就会成为系统状态故障，反复影响后续推理和生成，而传统的终端标量奖励（如任务成功率）无法定位到具体哪一步记忆更新出了问题（**信用分配鸿沟**）。
- **研究动机**：需要一种能够监督记忆**逐步构建过程**的框架，确保每一步记忆更新都具备可信度（覆盖性、保留性、忠实性），从而提升长期记忆系统的整体可靠性。
- **整体含义**：本文提出 **TrustMem**，从“仅评估最终记忆结果”转向“细粒度监督每一步记忆过渡”，通过记忆过渡验证器和偏好排序强化学习，直接优化记忆编辑行为，实现可信的记忆巩固。

## 2. 论文提出的方法论

### 核心思想
- 将记忆巩固建模为连续的记忆更新问题：以块（chunk）为单位处理输入流，每步智能体根据当前块和前一记忆状态生成结构化的记忆编辑动作（WRITE、REVISE、PRUNE），并由确定性执行器更新记忆状态。每一步产生一个局部过渡（transition），通过**记忆过渡验证器**评估其可信度，同时构建偏好对进行排序优化。

### 关键技术细节
1. **记忆过渡验证器（Memory Transition Verifier）**：
   - 一个冻结的 LLM，对于每个过渡 \( z_t = (c_t, M_{t-1}, a_t, M_t) \)，评估三个维度：
     - **覆盖性（Coverage）**：当前块中的重要信息是否被保留。
     - **保留性（Preservation）**：已有有效记忆是否未被不合理删除或扭曲。
     - **忠实性（Faithfulness）**：新增或修改的内容是否被当前块或前记忆状态支持。
   - 聚合得到标量可信度分数 \( v_t \in [0,1] \)，作为过渡级奖励。

2. **多粒度奖励设计**：
   - **样本级奖励**：任务奖励（下游任务指标，如 F1）、效率奖励（鼓励紧凑记忆）。
   - **过渡级奖励**：动作可执行性、内容类型、验证器奖励。
   - 总奖励 \( R_{i,t} = R_{task} + \lambda_{eff}R_{eff} + \lambda_{act}R_{act,t} + \lambda_{type}R_{type,t} + \lambda_{ver}R_{ver,t} \)。

3. **Transition-Ranked GRPO**：
   - 基于 GRPO（群体相对策略优化）进行全局轨迹优化。
   - 额外使用验证器分数构建偏好对：对每个块步的 G 个候选过渡，选取验证器分数最高和最低的两个（分数差 > 阈值 δ），构成偏好对，加入 DPO 风格的排序损失。
   - 联合目标：\( L = -J_{GRPO} + \beta_{KL} L_{KL} + \lambda_{rank} L_{rank} \)，同时优化标量奖励、KL 正则化和过渡级偏好排序。

### 算法流程（文字说明）
- 输入流按块顺序处理，每个块与当前记忆状态送入策略网络 → 生成动作序列（WRITE/REVISE/PRUNE）→ 执行器更新记忆 → 记忆过渡验证器评估 → 计算多粒度奖励 → 使用 GRPO 计算优势并优化策略 → 额外从同一状态的多条候选过渡中构建偏好对，施加排名损失。

## 3. 实验设计

### 数据集与场景

- **MemoryAgentBench** —— 涵盖准确检索（Single-Doc、Multi-Doc、LME(S)）、测试时学习（TREC-C、NLU、TREC-F、Clinic、Banking77）、长程理解（InfBench-Sum），共 9 个任务。
- **HaluMem-Medium**（30,073 对话轮次，14,948 个记忆点，3,467 个 QA 对）—— 评估操作级记忆可靠性（记忆提取、更新、问答中的幻觉、遗漏、错误）。

### 对比方法

- **RL/智能体记忆方法**：MemAgent、MEM1、Mem-α、Mem-T、AtomMem（均基于不同大小 LLM，如 Qwen3-4B、Qwen3-8B、Qwen2.5-7B）。
- **系统级记忆框架**：Mem0、Memobase、Supermemory、Zep。

### 评价指标
- MemoryAgentBench 和验证集：各数据集特定评分，取宏平均。
- HaluMem：记忆提取用 Recall、Weighted Recall、Accuracy、F1；记忆更新和问答用 Correctness、Hallucination 率、Omission 率。

## 6. 论文的主要结论与发现

1. **TrustMem 在记忆效用上达到 SOTA**：在 Mem-α 验证集上平均 66.3（超过 Mem-α 4.4 分）；在 MemoryAgentBench 上平均 65.7（超过 Mem-α 6.5 分），尤其在测试时学习任务上提升显著。
2. **在操作级可靠性上大幅提升**：HaluMem 记忆提取 F1 达 69.45，比最强基线（Mem0）提高 12.14 分；在记忆更新和问答中取得最低遗漏率，同时保持低幻觉和高正确率。
3. **显著降低三种转换级错误**：与各错误类型的最强基线相比，遗漏降低 40.1%、破坏降低 79.1%、幻觉降低 50.0%。
4. **消融实验确认关键组件**：移去动作可执行性奖励或验证器奖励均导致大幅性能下降；Transition-Ranked GRPO 优于仅使用验证器分数作为标量奖励的版本。
5. **训练动态显示稳定性提升**：无排名损失的变体后期不稳定，而 TrustMem 保持高奖励并收敛到稳定平台。表明偏好排序提供了更细粒度的监督。

## 7. 优点

- **方法论创新**：将记忆构建分解为可审计的局部过渡，并设计验证器评估三个具体维度（覆盖、保留、忠实），直接针对三种常见错误类型进行监督。
- **奖励设计精细**：多粒度奖励（样本级+过渡级）兼顾下游效用与局部更新质量，避免粗粒度的信用分配问题。
- **优化策略新颖**：Transition-Ranked GRPO 结合群体相对策略优化与偏好排序，既利用标量奖励又利用验证器提供的相对排序，提供更丰富的学习信号。
- **实验全面且公平**：覆盖多种基准（记忆效用、操作可靠性、泛化能力），与多种范式（标准、RL、系统级）对比，训练数据与测试数据分离，分布外评估可信度高。
- **可视化与案例分析**：提供清晰的错误率对比图、训练动态图、具体案例表格，增强可解释性。

## 8. 不足与局限

- **模态限制**：当前实验仅针对文本输入和文本记忆，未涉及多模态（图像、视频、语音等）环境。实际应用中用户输入和记忆证据常为多模态，扩展性待验证。
- **计算开销**：需要 32 台 H100 训练 2 天，且每步需要运行验证器（虽为冻结 LLM）和多次 rollout，训练成本较高；推理时不需要验证器，但动作生成次数与 chunk 数相关。
- **验证器依赖**：验证器使用冻结 LLM（如 GPT-4o-mini 在错误评估中使用），其评估质量可能受限于 LLM 本身的能力和偏差，且验证标准需人工定义。
- **偏好对构建敏感性**：依赖阈值 δ 选择偏好对，不同设置可能影响排序质量；论文未详细分析 δ 的敏感性。
- **可推广性**：训练数据来自 Mem-α 训练池（主要为问答和分类任务），对开放域对话或个人助理场景的泛化能力未充分验证。
- **信用分配问题虽缓解但未完全消除**：验证器只评估单步过渡，长期累积效应仍需后期任务验证，但相比仅用终端奖励已有本质改进。

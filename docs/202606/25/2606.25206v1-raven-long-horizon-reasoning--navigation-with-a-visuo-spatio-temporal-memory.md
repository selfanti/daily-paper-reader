## 摘要
长期机器人部署需要紧凑且可扩展的记忆，该记忆保留细粒度的视觉语义，在空间和时间中定位观察结果，并支持高效存储与检索。在本文中，我们提出RAVEN，一种用于长期机器人问答与导航的智能体记忆系统。RAVEN将带有姿态和时间的视觉嵌入存储在向量数据库中，并在空间地图中定位检索，以回答查询并导航到目标。通过直接在视觉嵌入上操作，RAVEN避免了有损的图像到文本描述，并实现了大规模的准确语义、空间和时间检索。在多个模拟和真实世界视频问答基准测试中，RAVEN始终优于基于描述的记忆系统，并在长期任务上与前沿的VLM相当，检索成本降低10倍。最后，我们在Unitree Go1机器人上实例化RAVEN，用于长期导航以实现自然语言目标到达，并在多个大型室内环境中展示了成功部署。

## Abstract
Long-term robot deployment requires a compact and scalable memory that preserves fine-grained visual semantics, grounds observations in space and time, and enables efficient storage and retrieval. In this paper, we propose RAVEN, an agentic memory system for long-horizon robotic question answering and navigation. RAVEN stores visual embeddings with pose and time in a vector database, and grounds retrieval in a spatial map to answer queries and navigate to goals. By operating directly on visual embeddings, RAVEN avoids lossy image-to-text captioning and enables accurate semantic, spatial, and temporal retrieval at scale. Across several simulated and real-world video question-answering benchmarks, RAVEN consistently surpasses caption-based memory systems and matches frontier VLMs on long-horizon tasks at 10$\times$ lower retrieval cost. Finally, we instantiate RAVEN on a Unitree Go1 robot for the task of long-horizon navigation for natural language goal-reaching, and show successful deployment over several large indoor environments.

- **整体含义**：本文提出RAVEN（RetrievAl via Visual Embeddings for Navigation），一种基于视觉嵌入的智能体记忆系统，直接在视觉嵌入上操作，避免了有损的图像-文本描述，同时利用向量数据库的高效检索能力，实现了在长期、大规模环境下的精准语义、空间和时间检索，为机器人问答和导航提供了一种高信息密度、低成本的记忆解决方案。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：使用多模态视觉嵌入作为记忆的基础单元，存储为“视觉-空间-时间”三元组（visual embedding + pose + timestamp），通过向量数据库实现高效检索；利用检索增强生成（RAG）和工具使用的视觉语言智能体（VLM）进行迭代推理与回答生成。
- **关键技术细节**：
  - **记忆构建**：在探索阶段，机器人采集RGB帧、深度图和位姿/时间戳。使用预训练多模态编码器（如CLIP、SigLIP、Seed1.6-Embedding、QQMM-v2）将RGB帧编码为紧凑的视觉嵌入，与位姿和世界时钟时间戳一同存入向量数据库（FAISS或Milvus），构成三元组记忆。
  - **检索与推理**：在查询阶段，VLM智能体维护工作记忆，通过调用四种原始工具进行迭代检索：
    - 文本检索：对用户查询重新编码，与视觉嵌入计算余弦相似度。
    - 时间检索：按时间戳返回连续帧。
    - 位置检索：返回空间最近邻。
    - 图像检索：支持图像查询。
  - 每次检索返回前K个记忆条目（含图像、位姿、时间、相似度分数），追加到上下文。VLM评估证据是否充分，若不充分则再次检索；若充分则生成结构化输出（文本答案、空间目标或时间参考）。
  - **导航执行**：将预测的空间坐标传递给A*轨迹规划器，引导机器人导航到目标位置。
- **算法流程（文字描述）**：
  1. 探索阶段：机器人执行前沿探索，采集RGB-D数据，运行SLAM构建3D地图，同时编码所有RGB帧为视觉嵌入，连同位姿和时间存入向量数据库。
  2. 查询阶段：用户发起自然语言查询。VLM智能体启动，调用文本/时间/位置/图像检索工具从数据库中获取相关记忆。VLM评估是否足够回答，若不够则继续检索（最多3-15次）。一旦足够，输出目标位置或答案。
  3. 导航阶段：将目标坐标传给A*规划器，控制机器人沿最短路径到达目标。

## 3. 实验设计
- **数据集与场景**：
  - **NaVQA**：基于CODa机器人导航数据集的视频问答任务，包含短、中、长地平线，涵盖描述性、空间和时序问题。
  - **FindingDory**：基于Habitat模拟器的单目标和多目标导航基准，机器人需从操作记忆（拾取和放置物体）中检索时空信息。包含100个episode，每个平均约1794帧，特别关注长地平线子集（ep_91-100，平均3023帧）。
  - **RAVEN-QA**（自建）：包含273个问答对，覆盖模拟（19个场景）、真实机器人（4个场景）、人类自我中心视频（网络和自录）。问题类型包括：主导对象、次要对象、推理、信息回忆、空间理解。分为简单和困难子集。
- **对比方法**：
  - **ReMEmbR**：最先进的基于描述的记忆系统（图像→文本描述→文本嵌入）。
  - **VLM Only**：直接将所有帧（或子采样后）喂给VLM，单次前向预测。
  - **Embedder Only**：仅使用多模态嵌入模型，返回与查询嵌入最相似的图像，无语言模型。
- **评估指标**：描述准确率、位置误差（m）、时间误差（min）、任务成功率（SR）、记忆效率（α = 检索帧数/总帧数，逆数E=1/α）。

## 6. 主要结论与发现
- RAVEN始终优于基于描述的方法（ReMEmbR），尤其在需要细粒度视觉细节（如次要对象、信息回忆、推理）的硬查询上，性能差距从简单查询的~13%扩大到硬查询的30%。
- 与直接使用VLM处理所有帧（VLM Only）相比，RAVEN在长地平线任务（>3000帧）上准确率更高（+4%），且检索效率高出10倍（仅需检索7.4%的记忆 vs 100%）。
- RAVEN的记忆高度紧凑：即使视频下采样至0.12 fps（压缩率22315倍），仍能保持90%性能；存储压缩比达250倍时准确率几乎不变。
- 中等规模开源VLM（27B-32B）在所有方法上表现较差，不如纯嵌入器方法，提示离线部署时应优先考虑轻量嵌入器。
- 真实机器人部署在4个室内场景（实验室、办公室、公共区域）成功率达到92.4%-97.1%，显著优于ReMEmbR和VLM Only。

## 7. 优点
- **直接保留细粒度视觉信息**：避免描述瓶颈，支持开放词汇、精细属性、隐式推理查询。
- **训练免费**：无需微调或场景优化，即插即用。
- **高效可扩展**：向量检索亚线性复杂度，支持天级经验；压缩率极高。
- **智能体迭代推理**：VLM通过工具调用主动检索上下文，减少无关信息干扰，提高稳定性。
- **普适性**：跨模拟、真实、网络视频场景均验证有效，支持文本和图像查询模态。

## 8. 不足与局限
- 依赖预训练多模态编码器的对齐质量：如果编码器无法捕获特定特征，检索失败。
- 记忆本质上是静态的：当前系统假设环境稳定，物体位置和状态不变。处理动态场景（物体被移动、状态变化）需要更新和修剪记忆，文中未实现。
- RAVEN-QA数据集为自建，可能引入隐性偏差；虽在其他标准基准上验证，但总体评估框架仍以自身数据为主。
- 实验中未探讨大规模（远超当前数千帧）或开放世界长时间持续运行（如数周）的情况；未见与最先进长期SLAM/语义建图方法的全面比较（如基于3D神经场的方法）。
- 中等开源VLM表现差，表明对于离线/低资源场景，需要更合适的VLM或策略。
- 错误传播：如果第一次检索失败，整个推理链可能失效（尽管智能体可以多次尝试，但仍有累积误差风险）。

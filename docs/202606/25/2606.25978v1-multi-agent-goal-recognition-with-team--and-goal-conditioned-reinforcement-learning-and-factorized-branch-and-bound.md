## 摘要
多智能体目标识别要求观察者共同推断哪些智能体协同行动以及每个团队试图达成什么目标，因此假设空间随着团队划分和每个团队的目标数量呈组合增长。诸如无人机监控和协作机器人等实际应用仅暴露智能体的轨迹，这迫使观察者仅从行为中对团队-目标假设进行排序。基于分支定界的多智能体目标识别（MAGR-BB）采用共享的团队与目标条件策略作为因子化分支定界搜索内的评分模型来处理该问题。在受控的多智能体块世界基准测试中，MAGR-BB在轨迹全程中返回与穷举搜索相同的排名第一的假设，同时将假设实例化减少数个数量级，并显著降低累积识别运行时间。

## Abstract
Multi-agent goal recognition asks an observer to jointly infer which agents act together and what each team is trying to achieve, so the hypothesis space grows combinatorially with the number of team partitions and goals per team. Real applications such as drone surveillance and collaborative robotics expose only the agents' trajectory, which forces the observer to rank team-goal hypotheses from behavior alone. Multi-Agent Goal Recognition with Branch-and-Bound (MAGR-BB) addresses this setting with a shared team- and goal-conditioned policy used as the scoring model inside a factorized branch-and-bound search. On a controlled multi-agent Blocksworld benchmark, MAGR-BB returns the same top-ranked hypothesis as exhaustive search throughout the trajectory while cutting hypothesis materialization by orders of magnitude and reducing cumulative recognition runtime substantially.

### 论文总结：基于团队与目标条件强化学习和因子化分支定界的多智能体目标识别 (MAGR-BB)

#### 1. 核心问题与整体含义（研究动机和背景）

*   **核心问题**：在多智能体系统中，一个外部观察者仅从智能体的运动轨迹中，同时推断出三个未知信息：1）哪些智能体组成了一个团队（团队划分）；2）每个团队正在共同追求什么目标（团队目标）。这是一个联合推理问题。
*   **组合爆炸的挑战**：随着智能体数量的增加，可能的团队划分数量呈指数级增长（贝尔数）。每个团队又有大量的候选目标。可能的团队-目标假设总数（`假设空间`）会变得天文数字般巨大（例如，在本文基准中，为`7,154,784`个/步）。
*   **研究背景**：现有的大多数目标识别方法要么只针对单个智能体（如GRAQL），要么依赖于显式的规划模型、动作库或意图结构。缺乏一种仅从学习到的行为策略（无需领域模型）出发，就同时识别隐藏团队和目标的可扩展方法。这在无人机监控、网络安全、自动驾驶等领域具有重要应用价值。

#### 2. 方法论：核心思想与关键技术

论文提出了 **MAGR-BB (Multi-Agent Goal Recognition with Branch-and-Bound)** 框架，核心思想是将**强化学习**作为评分模型，与**分支定界**搜索树相结合，实现高效的假设筛选。

*   **核心思想 1：共享的团队与目标条件策略 (Team- and Goal-Conditioned Policy)**
    *   训练一个单一的Transformer策略网络(πθ)，该网络以当前智能体的观察、候选团队、候选目标作为输入，预测智能体执行某个动作的概率。
    *   **输入构造（反事实观测 o<sub>it</sub>(T, g)）**：对于智能体 `i`，在时间步 `τ`，输入包含：环境状态、智能体自身身份、候选团队的**团队成员掩码**、候选目标的编码。通过固定真实轨迹，替换团队与目标特征，实现对不同假设的评分。
    *   **团队自回归解码**：假设同一团队内的智能体有固定的行动顺序，当一个智能体被评估时，其上下文（`c<sub>it</sub>`）包含了本步中早于它的队友的执行动作。这建模了团队内部的协调。

*   **核心思想 2：反事实评分 (Counterfactual Scoring)**
    *   对于一个完整的假设 `h = (P, g<sub>1</sub>, ... , g<sub>K</sub>)`，其得分 `S<sub>t</sub>(h)` 是所有智能体在所有已观测时间步下，其执行动作在该策略下的对数似然之和。
    *   **关键假设：非竞争性评分 (Non-competitive Score，定义1)**：一个团队-目标假设的得分（由该团队内智能体的动作对数似然之和构成）**只依赖于该团队自身的目标**，而**不依赖于**其他团队的目标。这使得总分可以分解为各个团队的得分之和：`S<sub>t</sub>(P, g<sub>1..K</sub>) = Σ<sub>k</sub> S<sub>t</sub>(T<sub>k</sub>, g<sub>k</sub>)`。这是整个剪枝策略的基础。

*   **核心思想 3：因子化分支定界搜索 (Factorized Branch-and-Bound)**
    *   利用上述的“可加性”和“非竞争性”特性，MAGR-BB维护一个**局部团队-目标得分表** `S<sub>t</sub>(T<sub>k</sub>, g)`，而非每次对整个新观测重新计算所有7.15M个完整假设。
    *   **三层剪枝机制（算法1）**：
        1.  **评分阶段剪枝 (Scoring-stage Prune)**：使用从上一步骤继承的“过时上限”（stale upper bound）来决定是否跳过对整个分区的评分更新。如果某个分区的最佳可能总分在当前无法进入Top-10，就直接跳过。
        2.  **分区层面剪枝 (Partition-level Prune)**：对一个分区进行评分更新后，计算其理论最大可能总分（`U<sub>t</sub>(P)`，即每个团队选取得分最高的目标），如果仍低于当前Top-10的最低分，则放弃该分区。
        3.  **局部层面剪枝 (Local-level Prune/Best-first search)**：对于通过前两层剪枝的分区，使用一个最大堆来按分数由高到低生成其具体的团队-目标组合。一旦堆顶的组合分数低于当前Top-10的最低分，则停止搜索该分区剩下的所有组合。

*   **算法流程（算法1）**：
    1.  初始化得分表，每个观测步到达时，按过时上限对分区排序。
    2.  遍历分区：如果上限足够好（能进入前k），则刷新该分区的**所有**局部团队-目标得分。
    3.  如果更新后的上限依然足够好，则对该分区内各团队的目标进行**内部最佳优先搜索**，逐一生成完整假设，并与全局Top-K堆比较。
    4.  一旦某个分区的上限或内部搜索的最佳假设低于全局Top-K阈值，则将其完全剪枝。

#### 3. 实验设计

*   **数据集/场景**：使用一个**受控的多智能体Blocksworld**基准测试。在这个场景中，有2个隐藏团队，每个团队有2个智能体（共4个智能体），各自在其独立的7块积木工作区内操作。团队目标是构建长度为2-4的积木塔。
*   **Benchmark**：
    *   每个团队有1092个候选目标（长度为2、3、4的塔的组合数），共6种候选的分区（从4个智能体中选择2个组成一个团队的组合数）。因此，总假设空间为 `6 * 1092^2 = 7,154,784` 个。
    *   标准包括：**Top-1准确率**（团队划分、目标、联合）、**识别延迟**（首次正确识别所需的步数）和**计算效率**（运行时、得分表刷新次数、分区访问次数、假设发射次数）。
*   **对比方法**：主要对比了**因子化穷举法**（无剪枝）和 **MAGR-BB** 的5个变体（逐步增加剪枝机制：评分剪枝、分区B&B、局部B&B、排序B&B、完整B&B）。重点在于通过消融实验证明各剪枝模块的有效性，而非与其他完全不同类型的方法相比。

#### 6. 论文的主要结论与发现

*   **准确性**：所有六个变体（包括穷举和全剪枝的MAGR-BB）在所有测试的轨迹和噪声水平下，**都返回了完全相同的** Top-1 假设（最终团队和联合目标准确率为100%）和相同的Top-10列表。这表明MAGR-BB的剪枝是**无损的**，没有因为剪枝而丢失正确答案。
*   **效率**：与穷举法相比，**全剪枝的MAGR-BB** 将最终观测步的**完整假设构建次数**从7.15M降低到**10次**。在累计识别时间上，实现了**2.43倍到2.91倍**的加速。

## 摘要
现有基于图的多智能体系统（MAS）设计者主要通过优化预定义智能体、角色或群体上的通信拓扑来改进协作。然而，由于每个节点仍是一个闭集实体，这些方法难以泛化到需要未见过的能力组合的任务。我们提出SIGMA，一种技能-关联图框架，将智能体构建为可复用技能的任务条件化束。给定一个任务和技能库，SIGMA预测技能-智能体关联矩阵，从选定的技能中组合智能体节点嵌入，并解码出构建后智能体之上的通信拓扑。在执行过程中，技能专用的邮箱将消息路由到相关分配的能力，使得关联结构可直接操作。在三个基础LLM的六个推理和编码基准上，SIGMA达到了最佳平均性能，并分别比最强的非组合性基于拓扑的基线CARD提高了2.06、2.36和1.75分。它还对未见过的技能库表现出更强的鲁棒性，平均性能下降仅为0.96分。这些结果表明，组合性节点构建是超越通信拓扑优化的多智能体设计的一个互补且重要的维度。代码可在https://anonymous.4open.science/r/SIGMA-2338/获取。

## Abstract
Existing graph-based multi-agent system (MAS) designers mainly improve collaboration by optimizing communication topologies over predefined agents, roles, or groups. However, because each node remains a closed-set entity, these methods struggle to generalize to tasks that require unseen combinations of capabilities. We propose SIGMA, a skill-incidence graph framework that constructs agents as task-conditioned bundles of reusable skills. Given a task and a skill library, SIGMA predicts a skill-agent incidence matrix, composes agent node embeddings from selected skills, and decodes a communication topology over the constructed agents. During execution, skill-specific mailboxes route messages to the relevant assigned capabilities, making the incidence structure directly operational. Across six reasoning and coding benchmarks with three base LLMs, SIGMA achieves the best average performance and improves over CARD, the strongest non-compositional topology-based baseline, by 2.06, 2.36, and 1.75 points, respectively. It also shows stronger robustness to unseen skill libraries, with an average performance drop of only 0.96 points. These results suggest that compositional node construction is a complementary and important axis for multi-agent design beyond communication topology optimization. ## 1. 核心问题与研究动机
现有基于图的多智能体系统（MAS）设计方法主要聚焦于在**预定义的固定角色或智能体节点**上优化通信拓扑，但每个节点仍是**闭集实体**，其能力无法根据任务动态重组。因此，当任务需要**从未见过的能力组合**（如工具调用、检索、推理）时，这些方法难以泛化。论文提出**组合性节点构建**应成为MAS设计的关键维度，并首次形式化定义了**技能-关联多智能体协议（SI-MAP）**，要求满足组合性、可执行性、拓扑兼容性和可扩展性。

## 2. 方法论：SIGMA框架
- **核心思想**：将每个智能体节点构造为**任务条件化的可重用技能包**，而非固定角色标签；通过预测**技能-智能体关联矩阵**（Bq）决定“每个智能体能做什么”，再基于组合后的节点嵌入解码通信拓扑，从而分离能力组合与拓扑生成。
- **关键技术细节**：
  - **技能卡片**：每个技能定义为包含名称、描述、可执行能力（如工具、API）、输入输出契约和来源的元组，由冻结编码器嵌入。
  - **关联矩阵生成**：对每个技能-智能体对计算软概率（Pq），通过稀疏投影得到硬矩阵（Bq），并处理重复束问题。
  - **节点嵌入**：对每个智能体槽，聚合选中的技能嵌入，结合槽身份和任务嵌入，得到最终的组合节点特征（Uq）。
  - **技能感知拓扑解码**：同时考虑节点级亲和力（rij）和技能束兼容性（cij），后者评估跨智能体技能间的互补性（如检索供给API）。最终边概率通过阈值得到通信图。
  - **邮箱执行**：每个技能-智能体关联对拥有独立邮箱，接收时根据消息与技能的相似度路由到最相关的技能，使技能分配直接操作化。

- **依赖技能库质量**：噪声或不完整的技能卡片会导致错误分配，未考虑技能间冲突或副作用。
- **伪标签偏差**：基于确定性启发式构建的伪标签可能继承标注偏差，影响关联预测器泛化。
- **固定智能体数量**：假设预先确定K个槽，未实现动态增减；未来可扩展为自动预测槽数。
- **评估范围有限**：仅限于推理和代码任务，未涉及真实工具使用、长 horizon 或社会模拟场景；现实API可能有噪音和交互副作用。
- **全局最优缺失**：采用贪心稀疏投影和固定束大小预算，未求解全局最优技能-智能体分配。
- **聚合机制脆弱**：案例显示少数正确推理可能被错误多数淹没，最终投票机制可进一步优化。

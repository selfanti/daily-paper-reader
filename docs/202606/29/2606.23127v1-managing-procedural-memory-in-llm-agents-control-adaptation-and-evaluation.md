## 摘要
程序性记忆越来越多地被用于改进LLM智能体在重复性工作任务中的表现，但其产生可复用技能的能力仍鲜为人知。我们提出了AFTER基准，包含涵盖六类职业角色和22项程序性技能的382项真实企业任务，旨在评估技能如何在任务、角色和模型主干之间迁移。该基准包括局部改进、跨任务迁移、跨角色迁移和跨模型泛化的受控评估设置。实验表明，程序性记忆在工业工作流中带来持续提升：单次优化使整体性能提高3.7-6.7个百分点，而基于多样化多模型执行轨迹演化出的技能实现了73.1%的跨模型测试准确率，优于所有单模型轨迹来源。我们进一步发现，某些技能可广泛泛化至不同任务和模型，而另一些则专精于特定角色的工作流，在迁移时失去有效性。这些结果为在生产级智能体平台上构建、评估和部署程序性记忆系统提供了实践指导。

## Abstract
Procedural memory is increasingly used to improve LLM agents on recurring workplace tasks, yet its ability to produce reusable skills remains poorly understood. We introduce AFTER, a benchmark of 382 realistic enterprise tasks spanning six professional roles and 22 procedural skills, designed to evaluate how skills transfer across tasks, roles, and model backbones. The benchmark includes controlled evaluation settings for local improvement, cross-task transfer, cross-role transfer, and cross-model generalization. Experiments show that procedural memory delivers consistent gains in industrial workflows: a single refinement round improves aggregate performance by 3.7-6.7 points, while skills evolved from diverse multi-model execution traces achieve 73.1% cross-model test accuracy, outperforming all single-model trace sources. We further find that some skills generalize broadly across tasks and models, whereas others become specialized to role-specific workflows and lose effectiveness under transfer. These results provide practical guidance for building, evaluating, and deploying procedural memory systems in production agent platforms.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：LLM智能体在重复性企业工作流（如文档处理、数据库查询）中日趋常见，但如何将执行经验转化为可复用的程序性知识（procedural memory）仍不明确。现有系统往往在单一环境中优化记忆，无法衡量技能在任务、角色、模型间迁移的泛化能力。  
- **核心问题**：程序性记忆是否真正产生可迁移的技能，还是仅仅过拟合于源环境？何时能泛化，何时会特化？  
- **整体意义**：为工业部署中的程序性记忆系统提供系统化的控制、适应与评估框架，揭示技能在不同上下文中的行为规律，推动智能体从“静态提示”向“可演化能力层”转变。

## 6. 主要结论与发现

1. **静态增益**：程序性技能平均提升完全正确率（M2）2.8个百分点；较弱模型受益更大（如Gemma 4 E4B在GenAI上+14.2点）。  
2. **单轮优化**：一次精炼即可带来3.7~6.7点额外提升，且更大模型收益更稳定。  
3. **多模型轨迹是关键**：从来自多个模型的多样化轨迹演化出的技能，跨模型测试准确率高达73.1%，远超最佳单模型源（59.4%）。弱模型反而提供更好的迁移信号（可能因为不完美执行包含更丰富的探索）。  
4. **角色特异性**：技能在跨角色迁移时可能失效（如PM→DS降幅7.5点），说明演化过程中自然产生了角色特化。  
5. **效率提升**：演化后的技能可大幅减少推理token（最高节省62%），降低推理成本。  
6. **框架对比**：Hermes在多样化训练下取得最佳测试增益（+18.0点），但多数框架在窄训练下出现过拟合（训练增益高但测试下降）。

## 7. 优点（亮点）

- **基准设计独创性**：AFTER是首个同时支持角色结构、技能注释和受控迁移划分的基准，覆盖6个角色、22技能，且任务经过严格质量审查（双人复核）。  
- **实验分离清晰**：明确区分“局部改善”与“跨上下文迁移”，避免将过拟合误认为泛化。  
- **评估维度全面**：同时考虑准确性（M1/M2）、迁移性（任务、角色、模型）和经济性（token/成本）。  
- **可复现性**：基准开源（HuggingFace），EVOLUTION框架标准化了轨迹收集、版本追踪和更新循环。  
- **实践指导价值**：揭示了“多样轨迹优于单源”“弱模型提供更好迁移信号”等反直觉发现，直接指导工业部署。

## 8. 不足与局限

- **覆盖范围**：仅限科技行业角色，未涵盖医疗、法律、科研等领域；22技能排除开放式创意任务。  
- **轨迹规模**：演化时使用有限轨迹（n≤5），真实场景中轨迹池可能大得多，规模效应未知。  
- **评估仅限功能正确性**：采用pytest验证，未评估代码可读性、鲁棒性或用户偏好。  
- **模型选择**：部分前沿模型（如GPT-5.5、Claude最新版）因API限制未完全纳入，可能导致基准结果随时间过时。  
- **跨角色验证只针对三个共享技能**（pdf, xlsx, validation），结论可推广性有限。  
- **未研究检索与记忆的交互**：固定技能分配简化了问题，技能检索单独研究可作为后续工作。

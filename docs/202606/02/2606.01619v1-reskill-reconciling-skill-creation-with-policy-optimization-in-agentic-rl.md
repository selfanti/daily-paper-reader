## 摘要
智能体强化学习使大语言模型智能体能够从环境奖励中持续改进，但由此产生的策略并未系统地积累可跨任务泛化的可复用策略。模块化技能可提供此类可复用策略，然而现有的技能增强强化学习方法将技能创建与策略优化解耦，从而存在采用与进化策略相冲突的技能的风险。受Anthropic的Skill Creator启发，我们提出ReSkill，一种将技能进化与策略学习相协调的强化学习循环内技能创建框架。ReSkill利用GRPO的组结构，仅以少量额外开销自然嵌入三种机制：（1）基于断言的技能创建器，从过去经验中诊断失败并提出条件性的、基于触发的技能修订；（2）组内轨迹采样，实现对技能版本的受控比较，捕捉哪个版本最能支持策略的持续学习；（3）具有自适应折扣的汤普森采样，在策略进化过程中平衡技能版本选择的探索与利用。在多个领域中，ReSkill持续优于现有的基于记忆和技能的强化学习方法，在未见任务上提升最大。技能生命周期分析显示，随着策略改进，技能被自动创建、测试、改进和剪枝，展示了协调的技能-策略共同进化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 利用GRPO的组结构（同一任务、同一策略下采样G个轨迹）嵌入三种机制：
  - **断言驱动的技能创建器**：从过去经验中诊断失败模式，提出条件性的、基于触发条件的技能修订。
  - **组内轨迹采样**：同一GRPO组内为不同轨迹分配不同技能版本，实现受控比较，直接评估技能版本对策略学习的效果。
  - **自适应折扣的Thompson Sampling**：在技能版本选择中平衡探索与利用，并考虑策略进化导致的历史证据过时问题。

### 关键技术细节
- **组内技能测试**（Within-Group Skill Testing）：每个训练步采样任务q，通过Thompson Sampling为组内每个轨迹i分配技能版本vi（新版本或旧版本），生成轨迹并计算奖励。所有轨迹共享同一任务和策略，仅技能版本不同，实现受控比较。每个轨迹同时用于：①策略优化（GRPO梯度）；②经验储备库（驱动技能创建）；③Thompson Sampling后验更新（版本接受/拒绝）。
- **RL循环内技能创建**（RL-in-the-loop Skill Creation）：
  - **经验储备库（Experience Reservoir）**：持续收集成功-失败轨迹对，标注技能版本和奖励。
  - **条件技能加载**（Conditional Skill Loading）：每个技能由内容（何时使用、动作指导、示例）和触发条件（类型：general、beginning、action-pattern）组成，仅触发技能被加载到上下文。
  - **断言驱动的失败分级**：动态断言集（规则谓词）应用于所有轨迹，计算通过率，LLM分析器输出失败诊断并更新断言集。
  - **历史信息驱动的技能修订**：基于当前技能库、失败诊断、先前提案历史，技能创建器提出候选版本Snew（添加、修改、删除操作），验证触发覆盖率后进入测试。
- **RL引导的技能进化**（RL-Guided Skill Evolution）：
  - **版本分配**：将新旧技能视为两臂，使用Beta后验，通过Monte Carlo估计P(p_new > p_old)得到分配概率π_t(new)，并设置探索下限ε_ts。
  - **策略感知后验更新**：引入自适应折扣权重w_t = 1/(1 + n_t/M)，其中n_t是当前步该臂的观察数，M是记忆参数（通过预测似然最大化从数据中估计）。随着策略进化，较早证据被折扣。
  - **接受/拒绝决策**：评估周期结束时，若折扣后验期望E[p_new] > E[p_old]则接受新版本，否则回退旧版本，并立即开始下一进化周期。

- **一致提升**：ReSkill在所有benchmark和模型规模上优于所有基线。在ALFWorld上，4B模型总体成功率89.8%，比最强基线SKILLRL（83.9%）提升5.9%；在Search上，4B模型总体47.6%，比SKILLRL（45.1%）提升2.5%。
- **未见任务放大增益**：在ALFWorld 8B上未见任务成功率95.3%，比SKILLRL（82.6%）提升12.7%，远大于见过任务的1.2%差距；Search上未见任务提升幅度约是见过任务的2.1倍。
- **协调共进至关重要**：断开协同进化（先技能后策略或先策略后技能）导致性能大幅下降（ALFWorld下降10.4~18.5%）；缺乏测试的自动接受导致性能崩溃（ALFWorld仅62.7%，低于GRPO的75.3%）。
- **自适应折扣有效**：替换为最后一步权重或均匀权重均降低性能，表明自适应折扣对处理策略非平稳性重要。
- **技能生命周期**：训练过程中技能被自动创建、测试、精炼和剪枝，早期构建基础技能，后期转向精细化和修剪。
- **鲁棒性**：使用基础模型代替Claude创建技能仍优于所有RL基线，说明提升源于共进化机制而非创建者能力。

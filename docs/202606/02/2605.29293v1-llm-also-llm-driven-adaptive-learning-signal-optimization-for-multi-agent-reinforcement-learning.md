## 摘要
在训练过程中提供有效的引导是多智能体强化学习（MARL）的核心，但在稀疏奖励场景中仍然困难重重，因为弱监督限制了协调和策略改进，而现有方法通常需要大量的领域专业知识或手动设计工作。大型语言模型（LLM）为灵活的学习信号设计提供了一种有前景的替代方案，但现有的基于LLM的方法仍然主要是面向单智能体、一次性或对合作MARL不断演变训练动态验证不足。为解决这些限制，我们提出LLM-ALSO，一种面向MARL的迭代式LLM驱动的自适应学习信号优化框架。LLM-ALSO并非直接部署LLM生成的奖励，而是将自适应分解为迭代诊断、提议和验证：批评者LLM从稀疏回报指标和紧凑行为证据中诊断特定阶段的训练和协调失败，生成者LLM根据诊断提出候选奖励塑形配置，分支验证反馈在候选影响主训练轨迹之前对其进行优化。通过短视域验证和阶段感知自适应，LLM-ALSO仅将经过验证的更新推广到训练中，降低了不可靠的LLM生成修改的风险。在稀疏奖励合作MARL任务上的实验表明，LLM-ALSO提高了稀疏评估性能和学习效率。

## Abstract
Effective training-time guidance is central to multi-agent reinforcement learning (MARL), yet remains difficult in sparse-reward settings where weak supervision limits coordination and policy improvement, and existing methods often require substantial domain expertise or manual design effort. Large language models (LLMs) provide a promising alternative for flexible learning-signal design, yet existing LLM-based methods remain largely single-agent-oriented, one-shot, or weakly validated for the evolving training dynamics of cooperative MARL. To address these limitations, we propose LLM-ALSO, an iterative LLM-driven adaptive learning-signal optimization framework for MARL. Rather than directly deploying LLM-generated rewards, LLM-ALSO decomposes adaptation into iterative diagnosis, proposal, and validation: a Critic LLM diagnoses stage-specific learning and coordination failures from sparse-return metrics and compact behavior evidence, a Generator LLM proposes candidate reward-shaping configurations conditioned on the diagnosis, and branch-validation feedback refines candidates before they affect the main training trajectory. Through short-horizon validation and stage-aware adaptation, LLM-ALSO promotes only validated updates into training, reducing the risk of unreliable LLM-generated modifications. Experiments on sparse-reward cooperative MARL tasks show that LLM-ALSO improves sparse-evaluation performance and learning efficiency.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将训练信号设计视为 **阶段感知的自适应优化问题**：奖励塑形配置（Reward-shaping configuration）应根据训练阶段特有的学习与协调失败进行精细化调整。
- 采用 **双 LLM 引擎**（Critic LLM + Generator LLM），结合 **短期分支验证** 和 **确定性验证门控**，确保 LLM 输出仅作为候选假设，经过实证验证后才能影响训练。

### 关键技术细节
1. **结构化 PBRS 配置空间**：
   - 使用基于势能的奖励塑形（Potential-Based Reward Shaping, PBRS）：  
     \( r_{\text{shape}}^t = \beta (\gamma \Phi_{\theta}(\tilde{s}_{t+1}) - \Phi_{\theta}(\tilde{s}_t)) \)，其中 \(\tilde{s}_t = (s_t, h_t)\) 是增强的训练时状态（包含辅助缓存 \(h_t\) 如访问区域、失败收集尝试等）。
   - 势函数是带权重的组合：\(\Phi_{\theta}(\tilde{s}) = \sum_{k \in K} w_k \phi_k(\tilde{s})\)，\(w_k \ge 0, \sum w_k = 1\)。  
     在 Level-Based Foraging（LBF）任务中实例化了六个组件：collection progress, approach behavior, coverage and discovery, target allocation, joint collection readiness, late-stage stability。
   - 配置 \(\theta = (\beta, m, w)\)，其中 \(m\) 是塑形模式（如早期探索、收集准备、后期稳定等），约束候选方向。

2. **行为证据与双 LLM 角色**：
   - **Critic LLM**：从稀疏回报指标和紧凑的行为证据（覆盖率、近目标移动、失败收集次数等）诊断训练阶段的学习和协调失败，输出综合引导卡（integrated guidance card），指明失败模式、支持证据、生成约束和潜在风险。
   - **Generator LLM**：根据引导卡在结构化 PBRS 空间内提出少量候选配置（通常为 3 个，包括针对性更新、探索性替代和保守调整）。不能引入任意奖励项或策略干预。

3. **初始奖励塑形搜索阶段**：
   - 先进行一次稀疏诊断运行，收集学习曲线和行为证据。
   - 两轮搜索：第一轮提出不同候选方向并训练有限步（pilot budget），根据稀疏回报和行为变化评估；第二轮聚焦有希望的方向并修正弱点。
   - 最终选择初始配置 \(\theta_0\) 和端点检查点 \(c_0\)，作为后续固定参考和自适应主线的共同出发点。

4. **检查点验证与替换阶段**：
   - 在固定干预检查点（如 1.0M 和 1.5M 步）进行验证。
   - 每个检查点：从当前主检查点 \(c_j\) 出发，对每个候选配置训练短视（short-horizon）分支（300K 或 500K 步），同时保持一个“无变化”（no-change）控制分支（沿用当前配置）。
   - 两轮验证：第一轮分支结果反馈给双 LLM，允许第二轮生成改进候选。
   - 最终由 **确定性验证门**（非 LLM）决定：使用稀疏回报和稳定性指标，只有当更新分支明确优于无变化控制（并避免瞬时峰值等不稳定因素）时才推广该更新。若通过，其终点检查点和新配置成为下一阶段的主线；否则保留当前配置。

### 公式与算法流程（文字说明）
1. 训练时奖励：\(r_{\text{train}}^t = r_{\text{env}}^t + r_{\text{shape}}^t\)（评价时只用 \(r_{\text{env}}^t\)）。
2. 奖励塑形由 PBRS 势函数生成，势函数为六个可解释组件的加权和，权重归一化。
3. 训练流程：
   - 阶段 1（初始搜索）：稀疏诊断 → Critic 诊断 → Generator 提出候选（两轮，每轮 3 个）→ 短 pilot 训练 → 选择最佳初始配置 \((\theta_0, c_0)\)。
   - 阶段 2（检查点验证）：在固定检查点 \(C_j\) 处，Critic 基于当前训练指标和行为证据重新诊断 → Generator 提出 3 个候选 → 每个候选训练一个短分支 + 无变化控制分支 → 两轮细化 → 验证门比较分数 → 若更新分支胜出则推广，否则保留原配置。

## 6. 论文的主要结论与发现

1. **LLM-ALSO 在大部分 LBF 配置上取得了最高的平均稀疏回报**，尤其在困难任务（如 15×15-3p-4f）上提升显著；在简单任务（峰值饱和）中仍保持竞争力。
2. **自适应奖励塑形比固定塑形和单 LLM 一次性生成更有效**：通过检查点验证，能够在恰当阶段调整塑形方向，而不是固定不变。
3. **分支验证机制有效**：在 C1 处成功识别出优于无变化的更新并推广，在 C2 处因候选与无变化接近而保留原配置，验证了“选择性替换”而非“每次更新”的设计。
4. **双 LLM 交互优于单 LLM**：消融实验表明，在相同验证机制下，双 LLM 引擎能生成更符合阶段需求的候选，改善后续训练轨迹。
5. **可解释性与安全性**：结构化 PBRS 空间+分支验证限制了 LLM 输出风险，避免了未经验证的直接部署。

## 7. 优点

- **方法设计的系统性**：将 LLM 使用限制在诊断和提议角色，通过短期验证和确定性门控分离“提议”与“采纳”，增强了可靠性和可审计性。
- **阶段感知自适应**：能够根据训练进展调整塑形方向（从早期探索到后期稳定），而非固定一个信号。
- **可解释性**：势函数由可解释的组件构成，LLM 输出有结构化格式（模式、强度、权重），便于理解。
- **实验设计严谨**：使用多随机种子、相同学习器对比，评价始终基于原始稀疏奖励；提供了分支级别的详细分析，拆解了框架各组件贡献。
- **适用面广**：同时验证了价值基（QMIX）和策略梯度（MAPPO）两类算法，表明方法具有一定通用性。

## 8. 不足与局限

- **实验覆盖有限**：仅在一个基准（LBF）上测试，未涉及其他常见 MARL 任务（如 SMAC、机器人协同、交通信号控制等），泛化性待验证。
- **依赖预定义 PBRS 组件**：方法效果依赖于组件的表达能力，组件设计需要领域知识；若组件缺失关键因素，诊断和提议可能受限。
- **分支验证额外开销**：每个检查点需要训练多个短分支（总共最多 14 分支），增加了计算成本，虽然短期间接，但相比无验证方法更耗时。
- **局部最优风险**：短期分支结果可能无法完美预测长期训练行为，存在局部改进但长期恶化或反之的风险（虽通过稳定性指标缓解）。
- **消融实验不全面**：仅在 QMIX 10×10-2p-1f 上进行了双 LLM vs 单 LLM 消融，未验证其他组件（如检查点数量、验证轮数、分支长度等）。
- **未讨论 LLM 调用的成本**：文中未提及 LLM 查询次数、延迟或 API 费用，在需要频繁调用时可能不实用。
- **偏差风险**：LLM 可能对某些行为模式有偏向（如鼓励保守策略），文献中也承认“自动生成的奖励信号可能编码错误目标”，虽然通过结构化空间和验证缓解，但风险仍存在。

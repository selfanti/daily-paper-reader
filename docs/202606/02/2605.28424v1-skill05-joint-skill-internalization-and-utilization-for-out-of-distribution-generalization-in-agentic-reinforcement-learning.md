## 摘要
将显式技能赋予大型语言模型已成为实现自主智能体解决复杂任务的一种有前景的范式。智能体技能可天然分为用于广泛认知迁移的通用技能和用于动态执行的专用技能。然而，现有的基于技能的强化学习方法通常强制在完全外化（导致高昂的上下文开销）与完全内化（存在过拟合和知识冲突风险）之间做出僵化选择。为解决这一困境，我们提出Skill0.5，一种新颖的智能体强化学习框架，通过将通用技能内化与专用技能利用相结合，明确区分技能处理方式。在动态、难度感知路由器的驱动下，Skill0.5将任务分流到不同的掌握层级以应用定制化优化策略：通过特权蒸馏内化通用技能，为困难任务构建认知基础；同时对简单任务进行诊断性探测，以惩罚捷径行为并强制专用技能利用。在ALFWorld和WebShop上的实验表明，Skill0.5在分布内和分布外场景中均优于基于记忆和基于技能的强化学习基线，带来性能提升。

## Abstract
Equipping large language models with explicit skills has emerged as a promising paradigm for enabling autonomous agents to solve complex tasks. Agent skills can be inherently divided into general skills for broad cognitive transfer and task-specific skills for dynamic execution. However, existing skill-based reinforcement learning (RL) methods typically force a rigid choice between full externalization, which incurs prohibitive context overhead, and full internalization, which risks overfitting and knowledge conflicts. To address this dilemma, we propose Skill0.5, a novel agentic RL framework that explicitly differentiates skill treatments by combining general skill internalization with task-specific skill utilization. Driven by a dynamic, difficulty-aware router, Skill0.5 streams tasks into distinct mastery tiers to apply tailored optimization strategies: it internalizes general skills via privileged distillation to build a cognitive foundation for hard tasks, while using diagnostic probing on easy tasks to penalize shortcuts and enforce specific skill utilization. Experiments on ALFWorld and WebShop demonstrate that Skill0.5 outperforms both memory-based and skill-based RL baselines, yielding performance improvements across both in-distribution and out-of-distribution scenarios.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型（LLM）作为自主智能体时，常依赖外部技能库（Skill Bank）来指导任务执行。技能分为**通用技能**（如元推理、错误恢复，跨领域适用但上下文冗长）和**特定技能**（细粒度执行规则，动态更新且易受检索噪声影响）。
- **核心矛盾**：现有技能型强化学习方法存在两个极端：
  - **全外化**（如 SkillRL）：训练和推理时将所有技能注入上下文，导致上下文过长，损害指令遵循能力，尤其在长程任务中。
  - **全内化**（如 SKILL0）：将技能完全融入模型参数，但受限于模型容量，且可能引发知识冲突，导致执行幻觉。
- **问题**：在真实部署场景中，技能库动态扩展，智能体常面临未见过的分布外（OOD）技能，上述两种范式均存在严重局限。因此，需要一种**差异化技能处理**策略：将通用技能内化为认知基础，而将特定技能作为可插拔的模块动态利用。

## 2. 论文提出的方法论

### 核心思想：差异化技能处理 + 难度感知路由 + 分层优化

- **Skill0.5** 是一个统一的智能体强化学习框架，联合优化解耦的通用技能和特定技能，基于实时任务掌握度进行动态路由。
- **关键组件**：
  - **难度感知路由器**：在每个训练步，对每个任务采样 G 条轨迹，计算经验通过率 pi。通过滑动窗口平均动态阈值 ηt 将任务分为三个层级：
    - **Hard** (pi = 0)：任务完全失败
    - **Medium** (0 < pi ≤ ηt)：有部分成功但待提升
    - **Easy** (pi > ηt)：高成功率
  - **分层优化策略**：
    - **Hard 任务**：特权蒸馏内化。使用特权提示（包含通用技能 SG）重采样成功轨迹，通过 token 级 Jensen-Shannon Divergence (JSD) 损失，让学生策略（仅用标准提示，即只有特定技能）模仿教师策略的推理步骤。公式：
      \[
      \mathcal{L}_{\text{hard}} = \frac{1}{|\mathcal{T}|} \sum_{\tau \in \mathcal{T}} \frac{1}{|\tau|} \sum_{k=1}^{|\tau|} \text{JSD}\big( \text{sg}[\pi_\theta^t(k)], \pi_\theta^s(k) \big)
      \]
    - **Medium 任务**：标准 GRPO 优化。直接使用 Phase-1 轨迹，按组内归一化优势进行策略梯度更新。
    - **Easy 任务**：反捷径利用。引入无技能诊断提示（cnone_t = ∅），采样诊断轨迹计算无技能通过率 pnone_i，定义利用增益 ui = pi - pnone_i。构建任务级利用优势 Au_i = (ui - ū_t) / σ_u，与轨迹级优势 A^{(g)}_i 组合得到复合优势 Â^{(g)}_i = A^{(g)}_i + Au_i，抑制捷径学习。
  - **总体目标**：\(\mathcal{L} = \mathcal{L}_{\text{hard}} + \mathcal{L}_{\text{medium}} + \mathcal{L}_{\text{easy}}\)，对每个任务仅应用其对应层级的损失。

### 算法流程（两阶段）
1. **Phase-1 (难度感知路由)**：对每个任务，用标准提示（仅含特定技能）采样轨迹，计算经验通过率，根据路由方程分配层级。
2. **Phase-2 (分层优化)**：按层级分别执行蒸馏、GRPO 或反捷径 GRPO，更新策略参数。

## 6. 论文的主要结论与发现

1. **整体性能**：Skill0.5 在两个基准的 ID 和 OOD 设置下均取得最佳平均排名，尤其在 OOD 场景提升显著（ALFWorld OOD 比 SkillRL 高 +13.2%，WebShop OOD 高 +3.9%）。
2. **内化是前提，利用是关键**：消融表明，仅内化（Internalize-Only）在 OOD 上仍有限制；仅利用（Utilize-Only）因缺乏基础推理能力而整体崩溃。两者联合才能实现最优泛化。
3. **动态路由有效**：训练初期 Hard 任务占主导，蒸馏梯度打破探索死锁；后期 Easy 任务增多，反捷径利用防止过拟合，保持 OOD 稳步提升。
4. **基线失败机制**：
   - SkillRL 因上下文过长导致 OOD 特定技能被 ID 习惯覆盖（上下文干扰）。
   - SKILL0 内化后参数冲突，即使读到正确技能仍执行错误动作。
   - SLIM 过早退休通用技能导致行为崩溃。
5. **验证了差异化技能处理的必要性**：通用技能应永久内化，特定技能应保持外部可插拔。

## 摘要
多模态智能体通过工具使用在复杂推理任务上取得了显著进展，但仍受限于两个问题：静态预定义的工具库无法泛化到未见场景，不加区分地调用工具会带来冗余成本和噪声引发的错误。我们提出MetaForge，一种多模态智能体框架，它能够学习何时调用工具以及如何按需进化其工具集。MetaForge将智能体行为分解为四个耦合阶段：判断（判断是否值得使用工具）、检索（选择合适工具）、适配（在任务上下文中确定工具参数）和锻造（在线合成新技能并将其回收至工具库以供重用），形成闭环的判断-检索-适配-锻造-回收循环。统一的编排策略使智能体能够在直接回答、重用现有工具或锻造新工具之间做出选择。我们通过强化学习联合优化调用必要性、检索准确性、执行效果和锻造技能的可重用性，并引入明确的调用成本惩罚以抑制冗余调用。在12个基准测试中，MetaForge在准确性、效率和泛化能力上持续超越16个基线模型，验证了从静态工具库到按需自我进化的范式转变。

## Abstract
Multimodal agents have achieved notable progress on complex reasoning tasks through tool use, yet remain limited by two issues: statically predefined tool inventories fail to generalize to unseen scenarios, and indiscriminate tool invocation incurs redundant cost and noise-induced errors. We propose MetaForge, a multimodal agent framework that learns when to invoke tools and how to evolve its toolset on demand. MetaForge factorizes agentic behavior into four coupled stages: Decide (judging whether tool use is warranted), Retrieve (selecting suitable tools), Adapt (grounding tool parameters in task context), and Forge (synthesizing new skills online and recycling them into the tool library for reuse), forming a closed judge-retrieve-adapt-forge-recycle loop. A unified orchestration policy enables the agent to choose among answering directly, reusing existing tools, or forging new ones. We jointly optimize invocation necessity, retrieval accuracy, execution effectiveness, and forged-skill reusability via reinforcement learning, with an explicit invocation-cost penalty discouraging redundant calls. Across 12 benchmarks, MetaForge consistently surpasses 16 baselines in accuracy, efficiency, and generalization, validating a paradigm shift from static tool inventories to on-demand self-evolution.

- **整体含义**：提出一种**自我进化**的框架，让智能体能够**动态地、按需地**学习何时使用工具、如何适配工具，以及在工具不足时**自行锻造新工具**并纳入工具库，从而实现从“使用固定工具的助手”到“自主编排并扩展能力的智能体”的范式转变。

## 2. 论文提出的方法论

### 核心思想：闭环能力编排（Decide–Retrieve–Adapt–Forge–Recycle）

- **四个耦合阶段**：
    1. **Decide（判断）**：智能体首先决定当前任务是否需要外部工具，若不需要则直接回答。
    2. **Retrieve（检索）**：若需要工具，从动态工具库中检索最合适的工具。
    3. **Adapt（适配）**：根据当前任务上下文（如具体图像、问题）确定工具的参数，确保调用成功。
    4. **Forge（锻造）**：当现有工具无法满足需求时，通过规划与实现两阶段在线生成新的可执行技能（Skill），并验证注册进工具库。
- **闭环回收**：成功锻造的工具被注册后，可在后续任务中被检索和重用，形成“锻造-重用”的生命周期。

### 关键技术细节

- **统一工具接口**：每种工具（包括锻造工具）具有统一的形式：`(name, description, schema, type)`，便于智能体理解和调用。
- **在线工具锻造**：
    - **规划阶段**：根据任务需求 `d` 生成结构化计划 `z`，包含工具名称、描述、参数接口和所需脚本。
    - **实现阶段**：为每个脚本生成代码，最终形成三元组 `(Markdown 规格, JSON schema, 可执行脚本)`。
    - **验证门控**：通过编译/执行检查（`v_exec`）和语义匹配检查（`v_sem`）后，工具才注册到工具库。
- **多轮 GRPO 复合奖励训练**：
    - 采用 Group Relative Policy Optimization，每组采样 `M=8` 条轨迹。
    - 复合奖励 `R(τ) = r_ra + r_dec + r_forge + r_fmt`：
        - **Retrieve/Adapt 奖励**：答案是否正确（二元信号）。
        - **Decide 奖励**：在正确轨迹中，鼓励更短的调用延迟（归一化后作为奖励），惩罚不必要的工具调用。
        - **Forge 奖励**：在正确轨迹中，鼓励成功锻造工具（`Forge(τ)`）和成功重用已注册锻造工具（`ReuseForged(τ)`）。
        - **Format 奖励**：奖励符合格式规范的响应，否则惩罚。
- **容量约束的工具池管理**：
    - **语义去重**：合并功能相似的新锻造工具，保留完整性和通用性最好的一个。
    - **使用量保留**：超过容量上限时，优先保留历史使用次数高（上四分位数+1.5倍IQR）的旧工具，剩余容量按功能组大小排序填充新工具。

### 算法流程文字说明

1. 输入：图像 `I` 和文本提示 `T`，历史 `h_t`，当前工具池 `T_t`。
2. 策略 `π_θ` 在每一轮从 `h_t` 和 `T_t` 中选择动作 `a_t`（调用现有工具、创建新工具或直接回答）。
3. 若调用工具，获得观察 `o_t`，更新历史 `h_{t+1}`；若创建工具且通过验证，则 `T_{t+1} = T_t ∪ {新工具}`。
4. 直到输出最终答案或达到最大轮数 `N=10`。
5. 训练时，采样多条轨迹并计算复合奖励，利用 GRPO 优化策略参数 `θ`。

## 6. 论文的主要结论与发现

- **准确率全面领先**：在 IID 和 OOD 设置下，MetaForge 均取得最高平均准确率（IID 88.14%，OOD 88.98%），显著超过所有基线（最佳基线 ZwZ-8B 为 83.14% / 84.41%）。
- **自适应工具编排有效**：相比固定工具策略的基线，MetaForge 能根据任务自动决定是否调用工具、调用哪个工具、如何适配参数，并能在失败时切换到合适工具。
- **在线工具锻造显著提升 OOD 性能**：在工具不足时（如 WebSight），MetaForge 主动锻造新工具且成功率（FSR 64.7%）远高于其他方法（最高 24.5%）。
- **效率与准确率 Pareto 优化**：通过 Decide 奖励，MetaForge 在保持高准确率的同时，平均交互轮数仅 2.3 轮，延迟和 token 消耗均处于较低水平，实现最优权衡。
- **稳定且集中的工具选择**：TUE（工具使用熵）仅 1.12~1.30，表明模型能精准选择最相关的少数工具，而非盲目尝试。
- **错误模式集中于执行错误**：MetaForge 的调用失败主要源于外部环境（如脚本超时、图像索引问题），而非模型自身的工具理解缺陷（幻觉或参数错配）。

## 7. 优点

- **创新性**：首次提出闭环的“判断-检索-适配-锻造-回收”自进化框架，将工具使用从静态预设转变为动态按需扩展。
- **技术完整性**：统一工具接口、两阶段锻造验证、容量约束管理、多轮 GRPO 复合奖励，各模块协同工作，形成完整系统。
- **实验严谨性**：在 12 个数据集、16 个基线、多种设置下进行主实验、消融、敏感性分析、骨干泛化和案例研究，证据充分。
- **实用价值**：在准确率和效率之间取得 Pareto 最优，且不受工具库变更影响，实际部署时能自适应新工具。
- **可重复性高**：使用开源模型和框架（Qwen3-VL、VerlTool），训练配置和超参数公开，便于复现。

## 8. 不足与局限

- **工具锻造形式受限**：当前锻造的技能仅支持可执行脚本（Python），适用于数据处理、代码分析等程序化任务，无法处理需外部模型或资源（如视频合成、3D 渲染）的任务。
- **评估范围有限**：仅覆盖图像-文本场景，未验证在音频、视频、实体交互或更复杂的时间推理任务中的效果。虽然显示较强泛化，但向更丰富模态的迁移能力未知。
- **效率目标单一**：优化目标仅聚焦推理延迟，未考虑工具调用的**货币成本**和**上下文窗口占用**。未来需在多维度成本（费用、token、时间）之间平衡。
- **工具池容量瓶颈**：虽然设计了去重和保留机制，但最大容量 K=50 是否足够覆盖大量任务仍需验证。若任务种类急剧增加，管理策略的效率可能下降。

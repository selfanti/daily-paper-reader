## 摘要
最近的自进化智能体已经证明，技能可以通过执行被发现、改进和积累。然而，现有的技能进化框架通常假设固定的工具层，并独立评估每个技能，限制了它们修复工具级故障或推理技能间交互的能力。我们提出SkillSmith，一种协同感知的技能-工具协同进化框架。SkillSmith引入了一个统一的提议空间，其中反思产生原子包，这些原子包联合修改技能和工具，允许在技能进化识别出可重用能力缺口时对工具进行包装、编辑、组合、拆分或退役。为了指导这种联合搜索，SkillSmith维护了一个受Lotka-Volterra动力学启发的生态效用模型，其中从执行轨迹中估计的交互矩阵捕获了技能间的互补性和冲突，并为检索、变异优先级和退役提供压力信号。此外，SkillSmith记录反模式，包括失败特征、因果归因和补救措施，以加速诊断并否决重复已知错误的提议。在三个基准测试（包括WildClawBench）和五个Qwen3.5模型规模上的实验表明，SkillSmith持续优于强基线，且随着任务复杂性和多技能共激活的增加，增益会放大。

## Abstract
Recent self-evolving agents have shown that skills can be discovered, refined, and accumulated through execution. However, existing skill-evolution frameworks typically assume a fixed tool layer and evaluate each skill independently, limiting their ability to repair tool-level failures or reason about interactions among skills. We propose SkillSmith, a synergy-aware skill-tool co-evolution framework. SkillSmith introduces a unified proposal space in which reflection produces atomic bundles that jointly modify skills and tools, allowing tools to be wrapped, edited, composed, split, or retired when skill evolution identifies a reusable capability gap. To guide this joint search, SkillSmith maintains an ecological utility model inspired by Lotka-Volterra dynamics, where an interaction matrix estimated from execution traces captures pairwise complementarity and conflict among skills and provides pressure signals for retrieval, mutation prioritization, and retirement. Furthermore, SkillSmith records anti-patterns, including failure signatures, causal attributions, and remedies, to accelerate diagnosis and veto proposals that repeat known mistakes. Experiments on three benchmarks, including WildClawBench, and five Qwen3.5 model scales show that SkillSmith consistently outperforms strong baselines, with gains that amplify as task complexity and multi-skill co-activation increase.

## 1. 核心问题与整体含义（研究动机和背景）

当前，基于大语言模型（LLM）的智能体系统广泛采用工具使用和技能（Skill）库来增强能力。技能库是一种非参数化的外部策略层，包含工作流指令、可执行脚本、元数据等。近年来，自进化智能体能通过执行发现、改进和积累技能。然而，现有技能进化框架（如EvoSkill、SkillClaw等）存在三个核心局限：
- **工具层固定**：无法修改或进化工具，当故障源于工具缺陷时，只能在技能层打补丁，导致技能臃肿且易出错。

## 6. 主要结论与发现

1. **性能提升显著**：SkillSmith在所有模型规模下一致优于Base Agent和EvoSkill。在397B上，OfficeQA准确率80.1%（+18.3%），SealQA 49.5%（+18.9%）。增益随模型规模增大而放大。
2. **持续进化能力**：在WildClawBench上，SkillSmith在6天内持续改进，而SkillClaw在2-4天即达到平台期，工具层瓶颈是主因。
3. **消融验证**：移除工具进化（Skill-only）导致最大性能下降（WCB -6.8%），库膨胀、回归率高。无约束工具编辑（FreeTool）导致工具错误率飙升（14.7%）。移除生态模型导致库膨胀（21技能+7工具），移除反模式记忆使回归率从2.1%升至7.6%。
4. **缩放效应**：增益在任务更复杂、技能共激活率更高的场景下更大。分解分析显示，工具复杂度、交互密度、上下文大小、动作空间和技能协同性每个维度的高层级都对应更大的相对增益。
5. **进化稳定性**：在100轮进化中，SkillSmith在两次扰动后能快速恢复（从低于25%恢复至近70%），而EvoSkill因工具固定而停滞在30%左右。生态控制确保库大小始终低于28组件，防止检索噪声和上下文溢出。
6. **反模式防遗忘**：通过否决检查，避免重复提出已知失败的配置，节省预算并减少回归。
7. **成本效率**：尽管每轮开销高于技能-only方法（~20%），但收敛速度更快（迭代次数减少38%），总调用次数和Token数反而更低，实现更高的准确率。

## 7. 优点

- **开创性范式**：首次提出技能与工具联合进化的框架，突破传统固定工具层的限制。

- **反模式记忆实用**：不仅存储成功经验，也存储失败经验，通过检索加速诊断和否决预防，避免重蹈覆辙。

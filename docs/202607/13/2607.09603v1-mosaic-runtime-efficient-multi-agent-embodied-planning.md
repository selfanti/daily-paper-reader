## 摘要
基于大语言模型的多智能体具身规划由于执行延迟过高而仍然不实用。我们发现失败动作是主要的瓶颈，源于两个核心挑战：部分可观测性下的不准确状态跟踪，以及产生冗余或冲突动作的低效协调。我们提出了Mosaic，一个运行时高效的多智能体规划框架，解决了这两个挑战。Mosaic通过以智能体为中心的语义记忆来维护准确而轻量的状态跟踪，该记忆以相对坐标存储物体，支持几何变换和协调。通过整数线性规划在每一步规划中分配动作，确保物理可行性和智能体间协调约束，从而保证高效协调。在AI2-THOR和搜索救援基准测试中，Mosaic实现了27-32%更快的执行速度、30-33%更少的LLM调用、25-31%更少的步骤数以及4-10%更高的成功率。这些结果表明，高效的记忆和约束引导的协调对于可扩展、低延迟的多智能体规划至关重要。

## Abstract
LLM-based multi-agent embodied planning remains impractical due to prohibitively high execution latency. We identify failed actions as the dominant bottleneck, stemming from two core challenges: inaccurate state tracking under partial observability and inefficient coordination that produces redundant or conflicting actions. We introduce Mosaic, a runtime-efficient multi-agent planning framework that addresses both challenges. Mosaic maintains accurate yet lightweight state tracking through agent-centric semantic memory that stores objects in relative coordinates, enabling geometric transformations and coordination. It ensures efficient coordination through Integer Linear Programming that allocates actions at every planning step, enforcing physical feasibility and inter-agent coordination constraints. Across AI2-THOR and search-and-rescue benchmarks, Mosaic achieves 27-32% faster execution, 30-33% fewer LLM calls, 25-31% fewer steps, and 4-10% points higher success rates. These results demonstrate that efficient memory and constraint-guided coordination are critical for scalable, low-latency multi-agent planning.

### 2. 方法论：核心思想、关键技术细节
- **整体框架**：MOSAIC 兼容集中式规划架构，包含 LLM-Planner（任务分解）、LLM-Actor（生成动作候选）、LLM-Verifier（验证子任务完成）三个模块，并引入两个核心设计。
- **设计一：智能体中心语义记忆（Agent-centric Semantic Memory, ASM）**
  - 以**相对坐标**存储任务相关物体和目的地（距离和方向），并随智能体移动而更新。
  - 支持**几何变换**：当物体离开视野时，通过运动历史推断位置；不同智能体可通过坐标系变换共享物体位置信息。
  - 轻量级：仅维护结构化文本描述（如“前方 2 步”），避免全局地图或密集三维重建。
- **设计二：基于约束优化的动作选择（Action Selection via Constraint Optimization）**
  - 在每一步，LLM-Actor 为每个智能体生成 **K 个候选动作**（K=3 为最佳折中）。
  - **整数线性规划（ILP）** 从候选集中为每个智能体选出唯一一个动作，强制满足**动作级可行性约束**：
    - 资格约束（物理可行性，如不能同时持有两个物体）
    - 每智能体单一动作约束（允许 no-op）
    - 多智能体联合动作约束（子任务需多智能体同时参与时确保人数充足）
    - 资源独占约束（同一物体或工具不可被多个智能体同时操作）
    - 碰撞与干扰避免（互斥动作不能同时选择）
  - **成本函数**进一步优化效率，惩罚时空低效模式（循环、重复失败、振荡、回溯、停滞）和负载不平衡。
- **算法流程**（以集中式为例）：
  1. LLM-Planner 根据指令 I 和 ASM 分解出开放子任务。
  2. 每步调用 LLM-Actor 生成各智能体候选动作。
  3. ILP 根据约束和成本函数选择全局一致的动作组合。
  4. 并行执行动作，更新历史 H 和 ASM。
  5. LLM-Verifier 检查子任务完成，更新开放/关闭子任务。
  6. 重复直至所有子任务完成。

### 6. 主要结论与发现
- MOSAIC 在 AI2-THOR 和 SAR 上**同时提升效率和效果**：

### 8. 不足与局限
- **依赖较准确定位**：ASM 假设合理的位姿估计，虽进行了噪声分析，但未考虑极端传感器漂移或执行器随机故障。真实机器人中的校准问题需与控制层集成。
- **约束的领域特定**：手工定义的可行性约束可能无法穷举所有动态社会约束或意外交互，扩展到新环境需手动调整。
- **行动覆盖依赖 LLM 候选质量**：若 LLM-Actor 未能生成关键候选动作（如尝试新策略），ILP 无法弥补。尽管通过 ASM 和迭代反馈改善，仍存在局限性。
- **未评估真实机器人或分布式 LLM 调用场景**：实验均在模拟器中进行，通信延迟（如 API 调用）、执行不确定性等真实因素未考虑。
- **计算资源有限**：虽然 CPU 上运行 ILP 速度快，但在大规模智能体（>8）时可能需要分层或分区裁剪；文章未对更大规模进行实证。
- **实验场景规模有限**：AI2-THOR 房间大小固定，SAR 网格规模较小；在更复杂的大规模 3D 环境（如具身 Town）中的表现未知。

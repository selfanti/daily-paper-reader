## 摘要
工具增强的视觉语言模型（VLM）能够通过调用外部工具解决多模态、多步骤任务，但在实践中仍显脆弱。现有工作存在两个常见不足。监督微调（SFT）主要基于成功轨迹构建，对工具失败后的恢复几乎没有提供信号，而稀疏的轨迹级别强化学习奖励则对哪一步失败以及如何修复提供了有限的指导。我们引入了ReGRPO（反思增强的组相对策略优化），这是一个在工具使用智能体中学习反思引导修正的框架。ReGRPO从一个结构化的反思数据引擎开始：我们执行接近失败的动作来收集基于失败观察的数据，然后构建反思思维三元组（错误类型、证据、修复计划），并与修正后的动作配对，用于热启动SFT。接着，我们使用组相对优势在局部轨迹内联合优化反思令牌和修正动作，并引入反思成本项以减少不必要的反思。在GTA和GAIA上的实验表明，在相同的骨干网络和工具套件下，ReGRPO持续优于强开源基线，并在所对比的开源控制器中取得了最佳结果。代码和RoT数据可在https://github.com/showlab/ReGRPO获取。

## Abstract
Tool-augmented vision-language models (VLMs) can solve multimodal, multi-step tasks by calling external tools, yet they remain fragile in practice. Existing works have two common gaps. Supervised fine-tuning (SFT) is built mostly on successful trajectories and offers little signal for recovery after tool failures, while sparse trajectory-level RL rewards provide limited guidance on which step failed and how to repair it. We introduce ReGRPO (Reflection-augmented Group Relative Policy Optimization), a framework that learns reflection-guided correction in tool-using agents. ReGRPO starts with a structured reflective data engine: we execute near-miss actions to collect grounded failure observations, then build Reflection-of-Thought triplets (ErrorType, Evidence, FixPlan) paired with corrected actions for warm-start SFT. We then optimize reflection tokens and corrective actions jointly within local trajectories using group-relative advantages, and include a reflection-cost term to reduce unnecessary reflection. Experiments on GTA and GAIA show that, under the same backbone and tool suite, ReGRPO consistently outperforms strong open-source baselines and achieves the best results among the compared open-source controllers. Code and RoT data are available at https://github.com/showlab/ReGRPO.

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **ReGRPO** 将“反思（reflection）”建模为可学习的潜变量，并融入策略优化。其核心是：在局部轨迹内联合优化反思令牌和修正动作，通过组相对优势提供步骤级监督，并引入反思成本惩罚不必要的反思。

### 关键技术细节

1. **结构化反思数据引擎**
   - 从MM-Traj成功轨迹出发，通过“近失动作”（near-miss actions）扰动工具选择或参数（如偏移边界框、调用错误工具），在沙箱中执行以获取真实的失败观测。
   - 使用教师VLM（GPT-4o）生成结构化的 **反思思维三元组（RoT Triplet）**：
     - **ErrorType**（错误类别）：如 `ToolMismatch`、`ArgInvalid`、`GroundingDrift`、`InfoInsufficient`
     - **Evidence**（证据）：必须引用具体观测内容（如“OCR输出为空”）
     - **FixPlan**（修复计划）：自然语言描述如何修正
   - 配对原始正确动作（`a*`）形成“失败动作→观测→反思→修正动作”的训练样本，用于SFT热启动。

2. **反思增强的GRPO训练**
   - **动态局部轨迹构造**：给定状态 `x_i`，采样K条候选局部轨迹。若初始动作 `a(0)_i` 成功则终止；若观测指示硬失败（如执行错误、空结果），则策略生成反思 `z_i` 和修正动作 `a(1)_i`。
   - **复合奖励函数**：`R(τ) = λ_exec * 1{success} - η * C(τ) + λ_val * V(x_i, τ)`
     - `1{success}`：环境回馈的成功信号
     - `C(τ)`：反思成本（若触发反思则按反思令牌长度惩罚，否则为0）
     - `V(·)`：可选训练时验证器（由RoT元数据确定性计算，不调GPT-4o）
   - **组相对优势**：组内平均奖励作为基线 `R_bar`，优势 `A(k) = R(τ_k) - R_bar`。
   - **优化目标**：`L_ReGRPO = - (1/K) * Σ A(k) * log πθ(τ_k|x_i) + β * KL(πθ || π_ref)`
   - 反思令牌的梯度直接通过轨迹似然得分反向传播，实现端到端优化。

3. **零验证器推理阶段**
   - 单路径执行，无外部验证器调用。确定性触发门 `g_i = 1{ToolError ∨ EmptyObs ∨ (i>1 ∧ u_i < κ_i)}`，其中：
     - `u_i` 为动作令牌对数概率的均值指数化（置信度）
     - `κ_i` 为在线自适应阈值（历史置信度均值）
   - 若触发，最多执行一个局部反思-修正块（`a(0)→o(0)→z→a(1)→o(1)`），之后继续。

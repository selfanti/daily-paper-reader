## 摘要
在长程任务中，决策相关的状态往往分散在日益增长的轨迹中，而行动智能体必须提取这些状态并采取行动。随着轨迹增长，任务需求、环境事实、先前的尝试、诊断以及未完成的子目标可能被深埋在上下文窗口中或被推至窗口之外，从而无法在需要时影响决策。我们将这种失败模式称为“行为状态衰减”。我们将记忆视为一种主动干预机制而非被动检索。一个独立的记忆智能体与未经修改的行动智能体并行运行，根据最近的轨迹更新结构化记忆库，并决定是注入基于记忆的提示还是保持沉默。该模块可与前沿行动智能体及现有智能体框架即插即用。在Terminal-Bench 2.0和τ^2-Bench上，它提升了较弱和较强行动智能体的pass@1指标，在Terminal-Bench上提高了8.3个百分点，在τ^2-Bench上提高了6.8个百分点。消融实验表明，选择性干预优于被动记忆库暴露、始终注入、仅顾问引导以及一般检索。作为向开放权重记忆策略迈出的早期一步，我们使用SFT和GRPO在SETA上训练了Qwen3.5-27B，提高了验证奖励，并实现了向Terminal-Bench的部分迁移。

## Abstract
In long-horizon tasks, decision-relevant state is often scattered across an expanding trajectory, while the action agent must surface it and act. As trajectories grow, task requirements, environment facts, prior attempts, diagnoses, and open subgoals can be buried in the context window or pushed beyond it, failing to influence decisions when needed. We call this failure mode "behavioral state decay". We study memory as an active intervention mechanism rather than passive retrieval. A separate memory agent runs alongside an unmodified action agent, updating a structured memory bank from the recent trajectory and deciding whether to inject a memory-grounded reminder or remain silent. The module is plug-and-play with frontier action agents and existing agent harnesses. Across Terminal-Bench 2.0 and $τ^2$-Bench, it improves pass@1 for both weaker and stronger action agents, with gains of +8.3 pp on Terminal-Bench and +6.8 pp on $τ^2$-Bench. Ablations show that selective intervention outperforms passive bank exposure, always-on injection, advisor-only guidance, and general retrieval. As an early step toward open-weight memory policies, we train Qwen3.5-27B on SETA using SFT and GRPO, improving validation reward and achieving partial transfer to Terminal-Bench.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心架构**：两阶段记忆智能体（Memory Agent）与行动智能体（Action Agent）并行运行。
  - **阶段1（记忆管理）**：记忆智能体定期（固定间隔步数）观察最近轨迹窗口（k=8消息）和当前记忆库，通过预定义工具调用（`memory_save_knowledge`, `memory_save_procedural`, `memory_update_status`, `memory_delete`）更新结构化记忆库。
  - **阶段2（选择性干预）**：基于更新后的记忆库，决定是否生成一条**基于记忆的简洁提醒**（`<context_for_action>`）注入到下一轮行动智能体的调用中，或选择沉默（`<no_intervention/>`）。
- **记忆库结构**：三元组 `Bt = (st, Kt, Pt)`
  - `st`：私有状态字段（仅记忆智能体使用，不暴露给行动智能体），跟踪进度、未解决问题与风险。
  - `Kt`：知识记忆，存储稳定事实（任务要求、环境属性、路径、验证过的事实等）。
  - `Pt`：过程记忆，存储尝试与结果（失败命令、成功修复、已排除的假设等），避免重复错误或复用有效证据。
- **干预触发**：固定间隔触发（每步执行），但干预决策是二元的（注入或沉默），避免过度干涉。

## 6. 论文的主要结论与发现

- **行为状态衰减是长时域智能体的核心失败模式**，单纯的长上下文无法保证信息对决策的有效控制。
- **主动记忆干预显著提升长时域任务成功率**：
  - Terminal-Bench：Sonnet 4.5提升8.3pp（37.6%→45.9%），Opus 4.6提升2.4pp（43.5%→45.9%）。
  - τ²-Bench：Sonnet 4.5提升6.8pp（55.0%→61.8%），Opus 4.6提升2.5pp（66.2%→68.7%）。
  - 较弱智能体获益更大，但较强智能体仍能获得非零增益。
- **选择性干预优于被动暴露、始终注入、仅顾问引导和通用检索**：消融实验中完整两阶段系统在宏平均上最高，尤其是保持沉默的决定对域平衡结果至关重要。
- **记忆干预策略可学习**：通过SFT+GRPO训练的Qwen3.5-27B记忆智能体提升了SETA验证奖励，并部分迁移到Terminal-Bench（+3.5 pp）。
- **定性分析**：成功的干预通常是在行动智能体即将忽略关键状态时，注入特定、有事实依据且及时的提醒；失败的干预主要是校准错误（过度自信、重复已知信息、不必要的担忧）。

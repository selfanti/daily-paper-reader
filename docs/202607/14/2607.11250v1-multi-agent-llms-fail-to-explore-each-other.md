## 摘要
探索对于多智能体系统的可靠自主性至关重要，但当前尚不清楚大语言模型智能体在相互交互时能否进行有效探索。我们发现现代大语言模型智能体无法做到这一点，常常表现出短视和两极化的交互模式，导致协调欠佳并增加遗憾值。我们将这一挑战形式化为多智能体探索问题，将其建模为部分可观察随机博弈问题，其中智能体必须探测同伴以推断其能力并识别有效的交互策略。为解决这一问题，我们引入了多智能体上下文探索（MACE），这是一个轻量级框架，通过结构化同伴选择明确促进探索。在上下文多样性和参数多样性两种设置下，MACE显著改善了探索行为及下游任务表现。我们还从理论上证明了探索的价值随智能体多样性增加而增加。总体而言，我们的结果揭示了当前大语言模型智能体的根本局限性，并强调了明确引导探索对于可靠多智能体自主性的重要性。代码将在 https://github.com/deeplearning-wisc/mace 发布。

## Abstract
Exploration is essential for reliable autonomy in multi-agent systems, yet it remains unclear whether large language model (LLM) agents can explore effectively when interacting with one another. We show that modern LLM agents fail to do so, often exhibiting myopic and polarized interaction patterns that lead to suboptimal coordination and increased regret. We formalize this challenge as the Multi-Agent Exploration problem, modeling it as a partially observable stochastic game (POSG) problem in which agents must probe peers to infer their capabilities and identify effective interaction strategies. To address this, we introduce Multi- Agent Contextual Exploration (MACE), a lightweight framework that explicitly promotes exploration through structured peer selection. Across both contextual and parametric diversity settings, MACE substantially improves exploration behavior and downstream task performance. We further show theoretically that the value of exploration increases with agent diversity. Overall, our results highlight a fundamental limitation of current LLM agents and underscore the importance of explicitly guided exploration for reliable multi-agent autonomy. Code will be released in https://github.com/deeplearning-wisc/mace

## 2. 方法论
### 核心思想
- 将多智能体交互建模为**部分可观测随机博弈（POSG）**，但由于精确求解是NEXP-hard，论文提出**MACE（Multi-Agent Contextual Exploration）** 框架，将联合问题分解为每个智能体独立的**上下文多臂赌机（contextual bandit）** 问题，通过显式算法引导探索，避免过早承诺。

### 关键技术细节
- **上下文特征设计**：对智能体`i`与同伴`a`在轮次`t`的每次潜在交互，构造9维关系特征向量 `x_{i,a,t}`：
  - **响应多样性**（unigram/bigram/trigram）：度量同伴回答与智能体自身回答的词汇差异。
  - **同伴独特性**（unigram/bigram/trigram）：度量该同伴回答在全体智能体中的偏离程度。
  - **历史绩效**：同伴先前获得的平均奖励。
  - **交互轮次**：归一化的当前交互轮次。
  - **偏置项**：常数1。
- **LinUCB选择规则**：对每个同伴维护岭回归估计 `θ̂_{i,a}` 和设计矩阵 `A_{i,a}`。每轮选择最大化下式的同伴：
  ```
  a_{i,t} = argmax_a [ θ̂_{i,a}^T x_{i,a,t} + α √(x_{i,a,t}^T A_{i,a}^{-1} x_{i,a,t}) ]
  ```
  其中第一项是估计期望奖励，第二项是乐观不确定性奖励（探索激励）。
- **奖励定义**：混合奖励 `r_i^{(t)} = 0.5 * [ (s_i^{(t)} - s_i^{(t-1)}) + s_i^{(t)} ]`，其中 `s` 是任务分数（F1或准确率），既奖励改进也奖励最终高质量。
- **统计更新**：选择同伴后更新对应 `A` 和 `b`，并重新计算参数。

### 算法流程（文字说明）
1. 初始化所有`A=λI`、`b=0`。
2. 对每个任务样本：
   - 第0轮：各智能体独立生成初始回答。
   - 对每轮交互（1..R）：
     - 对每个智能体，计算所有候选同伴的上下文特征，用LinUCB规则选择同伴。
     - 智能体读取所选同伴的回答，生成更新回答。
     - 计算奖励，更新该同伴的`A`和`b`。
3. 学习阶段结束后可冻结参数用于无更新开发阶段。

## 3. 实验设计
### 设置
- **上下文多样性**：使用**HotpotQA（distractor模式）**，每样本含10个段落（2个相关+8个干扰），分配给10个Qwen2.5-7B-Instruct智能体各一个段落。智能体需探索谁拥有相关证据。交互轮数R=5。
- **参数多样性**：4个不同模型：GPT-5、Qwen2.5-7B、Llama3.1-8B、Mistral-7B。任务为**Math500（L4-5）** 和**GPQA（Diamond split）**。交互轮数R=3。

### 对比基线
- **In-Context Exploration**：相同上下文信息以自然语言提示给智能体，让其自主平衡探索与利用。
- **Random**：每轮均匀随机选择同伴。
- **Pre-defined**：固定选择右侧邻居（类似常见多智能体辩论固定结构）。

### 评估阶段
- **Trial-and-Error 阶段**：所有方法相互交互并积累观测。
- **Exploitation 阶段**：MACE冻结线性参数，In-Context保留累加统计，Random/Pre-defined不变。目的是测试学到的策略的泛化性。

## 6. 主要结论与发现
1. **LLM智能体探索严重不足**：在简单双人臂赌机任务中，LLM（Qwen2.5-7B、GPT-4、GPT-5）均表现出过早承诺，选择呈双峰分布，而非逐渐累积证据。
2. **In-Context Exploration 甚至不如随机**：在上下文多样性设置中，用提示鼓励探索的智能体比均匀随机选择更差，说明仅靠提示无法解决探索问题。
3. **MACE显著提升探索和任务性能**：在两种异质性设置下，MACE在所有交互轮次中均优于所有基线，且在冻结参数后仍保持优势，说明学到可泛化的交互策略。
4. **探索价值随智能体多样性增加**：理论证明MACE实现 `O(√T log T)` 遗憾，而非探索策略在多样性δ下产生 `Ω(δT)` 线性遗憾。实验与理论一致。
5. **MACE参数可迁移**：在HotpotQA上学习的策略直接应用于2WikiMultihopQA时仍优于基线，证明学到的是通用结构而非任务特异性。

## 7. 优点
- **轻量级**：MACE仅维护线性回归统计，无需复杂协调协议或大量交互数据。
- **算法化探索**：通过LinUCB显式量化不确定性，自动平衡探索与开发，避免依赖LLM脆弱的内在探索能力。

- **实验设计全面**：涵盖两种异质性、迁移学习、超参数敏感性和消融，结果可靠。

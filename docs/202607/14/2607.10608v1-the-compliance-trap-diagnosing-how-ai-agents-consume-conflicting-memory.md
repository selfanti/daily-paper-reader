## 摘要
记忆正成为长周期AI代理的核心组件，允许代理在操作网页浏览器、软件工具及其他交互环境时重用过去经验。现有工作大多将记忆视为供应问题，询问写入何种经验、如何存储以及检索哪条记录用于下一任务。然而，我们仍缺乏对模型如何在多步动作轨迹中消耗检索到的记忆的清晰描述。这一消耗过程之所以重要，是因为它不仅决定了应检索哪些记忆，还决定了需要何种模型和控制策略来安全使用它们。为诊断这一过程，我们提出“入口-传播-恢复”（E-P-R）框架，一个在轨迹层面询问记忆首次改变动作的位置、该变化是否向前传播以及代理在偏离正确路径后能否恢复的框架。我们在WebArena和我们构建的用于隔离这些阶段的受控基准MemTrapBench上实例化E-P-R。我们发现，主要失败往往始于入口：即使在首次暴露的决策点上记忆是任务错误的，代理也会采纳冲突记忆。反复暴露随后放大这一早期错误，而偏离后的恢复能力较弱。这些效应共同造成了一个“合规陷阱”：跨模型而言，冲突记忆诱发相似的合规率，但一旦代理合规，其成功率骤降至低点。因此，更强的代理遭受更大的绝对损害，因为每次合规事件都抹除了更多基线能力。这些结果表明，记忆增强代理的评估不应仅依据检索质量或最终成功率，还应依据其在轨迹中如何消耗记忆。

## Abstract
Memory is becoming a core component of long-horizon AI agents, allowing agents to reuse past experience when operating web browsers, software tools, and other interactive environments. Existing work mostly treats memory as a supply problem, asking what experience to write, how to store it, and which entry to retrieve for the next task. Yet we still lack a clear account of how models consume retrieved memory across a multi-step action trajectory. This consumption process matters because it determines not only what memories should be retrieved, but also what models and control policies are needed to use them safely. To diagnose this process, we propose Entry--Propagation--Recovery (E-P-R), a trajectory-level framework that asks where memory first changes an action, whether that change carries forward, and whether the agent can recover after leaving a correct path. We instantiate E-P-R on WebArena and on MemTrapBench, a controlled benchmark we build to isolate these phases. We find that the main failure often begins at entry: agents adopt conflicting memory at the first exposed decision point even when it is task-wrong. Repeated exposure then amplifies this early error, while recovery after divergence is weak. Together, these effects create a compliance trap: across models, conflicting memory induces similar compliance rates, but once agents comply, their success rates collapse to a low floor. Stronger agents therefore suffer larger absolute damage because each compliance event erases more baseline capability. These results suggest that memory-augmented agents should be evaluated not only by retrieval quality or final success rate, but by how they consume memory throughout the trajectory.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有AI代理的记忆系统主要关注“供应端”（如何写入、存储、检索经验），但忽略了“消费端”——即模型在长周期多步轨迹中如何实际使用检索到的记忆。特别是，当检索到**冲突记忆**（任务错误的指导）时，代理会如何决策？错误是否会沿轨迹传播并放大？代理能否纠正？

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：从**消费视角**研究记忆，提出 **E-P-R框架**，将记忆消费分解为三个诊断阶段：
  - **入口（Entry）**：记忆是否在首次暴露的决策点改变代理的动作？
  - **传播（Propagation）**：这种改变是否在持续或重复暴露下继续影响后续动作？
  - **恢复（Recovery）**：在记忆导致偏离后，代理能否回到正确轨迹？

- **关键技术细节**：
  - 使用**配对干预**：每个（任务，模型）组合运行一个无记忆轨迹和一个有记忆轨迹（相同环境种子），通过成功率差值（Δ）评估影响。
  - 三种注入调度：**早期**（只第一步可见）、**持续**（每一步可见）、**后期**（第三步后才可见），分别对应入口、传播、恢复的探测。
  - 定义**建议合规率（RCR）**：LLM判断代理在第一步的动作是否合理执行了记忆中的DO建议。
  - 定义**每次合规的损害（DPC）**：在合规子集上，配对的无记忆成功率减去有记忆成功率，量化合规带来的条件性成本。
  - 记忆类型：**有益记忆**（正确指导）、**冲突记忆**（看似合理但错误）、**跨任务控制记忆**（同网站无关任务）。

## 6. 论文的主要结论与发现

1. **记忆自动提升性能并不成立**：在完整WebArena上，在线检索的冲突记忆导致成功率下降或无改善（表1），无论是否质量门控。
2. **冲突记忆早期进入，持续暴露放大危害**：图3显示持续冲突记忆对所有模型造成显著损害（Qwen3.5-27B为-20.8pp，Gemini-3-Flash为-14.9pp），后期注入效果微弱。
3. **“合规陷阱”**：表2显示，不同模型的RCR（合规率）相似（63%-72%），但合规后成功率骤降至低点（17%-31%），强模型因基线更高，绝对损失更大（DPC达-25.5pp）。
4. **入口是主要失败起点**：轨迹级分析（图4）显示首次分歧几乎都发生在前几步，且冲突记忆下的恢复率极低（7%-15%），有益记忆恢复率仅27%-42%。
5. **落地门控是关键瓶颈**：在MemTrapBench的Grounding家族中，有益记忆带来最大增益（+57.9~+73.7pp），冲突记忆因缺乏页面元素而无法造成损害。
6. **陷阱依赖于足够长的轨迹**：短周期MiniWoB++（中位数2步）无显著冲突损害；WebArena（≈7步）出现；MemTrapBench（15步）效应最大。
7. **失败条件触发记忆有效**：重试策略（第一次失败后才注入有益记忆）优于始终注入，接近调度预言机上限（表7），表明记忆应基于需求而非始终使用。

## 7. 优点：方法或实验设计上的亮点

- **视角创新**：将记忆研究从“供应”转向“消费”，填补了代理如何实际使用记忆的空白。
- **诊断框架清晰**：E-P-R的三个阶段（入口/传播/恢复）直观且可操作，通过注入调度和配对干预直接测量。

- **发现具有实际指导意义**：指明记忆系统的评估应关注消费过程，并建议“失败后再注入”的简单策略就能显著改进，具有实用价值。

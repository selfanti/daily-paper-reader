## 摘要
大型语言模型（LLM）智能体积累异构上下文，包括系统指令、计划、用户轮次、检索文档、工具输出和中间推理，其键值（KV）缓存可能成为主要的内存瓶颈。现有的淘汰策略通常对每个令牌应用相同的基于注意力或基于最近使用的规则，忽略了智能体编排器已可用的语义结构。我们提出了MemDecay，一种无需训练、区域感知的KV缓存淘汰策略。MemDecay为令牌分配区域特定的基础优先级和衰减率，在令牌获得注意力时刷新保留分数，并在固定缓存预算下淘汰得分最低的页面，同时允许关键区域固定。我们还提供了一种根据测量的注意力生命周期校准衰减率的方法。我们使用Qwen2.5-1.5B和3B模型在约450和1,700个令牌上下文中评估了MemDecay。在所有设置中，不同区域的注意力生命周期相差一个数量级：系统令牌的半衰期范围为148到189个解码步骤，而草稿板令牌为14到16个步骤。固定系统区域事实在所有设置中均保持全缓存准确性，而任何基线在24个事实中最多保留13个。随着上下文增长，区域感知保留仍然有效，而基于最近使用的保留则会崩溃。然而，累积注意力保留在未固定内容上表现更好，消融实验将注意力得分归一化识别为当前公式的主要限制。这些结果确立了语义提示结构作为KV缓存管理的稳健信号，同时阐明了应如何与基于注意力的重要性相结合。

## Abstract
Large language model (LLM) agents accumulate heterogeneous context, including system instructions, plans, user turns, retrieved documents, tool outputs, and intermediate reasoning, whose key-value (KV) cache can become a major memory bottleneck. Existing eviction policies generally apply the same attention- or recency-based rule to every token, ignoring semantic structure already available to the agent orchestrator.   We introduce MemDecay, a training-free, region-aware KV-cache eviction policy. MemDecay assigns tokens region-specific base priorities and decay rates, refreshes retention scores when tokens receive attention, and evicts the lowest-scoring pages under a fixed cache budget while allowing critical regions to be pinned. We also provide a procedure for calibrating decay rates from measured attention lifetimes.   We evaluate MemDecay at approximately 450 and 1,700 token contexts using Qwen2.5-1.5B and 3B. Across all settings, attention lifetimes differ by an order of magnitude across regions: system-token half-lives range from 148 to 189 decoding steps, compared with 14 to 16 for scratchpad tokens. Pinning preserves system-region facts at full-cache accuracy in every setting, while no baseline preserves more than 13 of 24. Region-aware retention remains effective as context grows, whereas recency-based retention collapses. Accumulated-attention retention performs better on unpinned content, however, and ablations identify attention-score normalization as the main limitation of the current formulation. These results establish semantic prompt structure as a robust signal for KV-cache management while clarifying how it should be combined with attention-based importance.

## 一、论文的核心问题与整体含义

- **研究背景**：大语言模型（LLM）被越来越多地部署为长期交互的智能体，其上下文积累系统指令、计划、用户轮次、检索文档、工具输出和中间推理等多阶段信息。KV缓存随序列长度线性增长，成为内存和带宽的主要瓶颈，制约长程推理与服务。
- **核心问题**：现有的KV缓存淘汰策略（如基于注意力、基于最近使用）对所有令牌应用相同的保留规则，忽略了智能体编排器已可用的语义结构，导致重要信息（如系统指令）可能被过早丢弃，而短暂信息（如草稿板推理）可能被不必要保留。
- **整体含义**：提出一种利用语义区域标签来指导缓存淘汰的框架，使保留策略与智能体工作负载的异构性对齐，在固定缓存预算下实现更高效、更可控的推理。

## 二、论文提出的方法论

### 2.1 核心思想
- MemDecay是一种**无需训练**的KV缓存淘汰策略，核心是“区域感知的优先级与衰减”：为每个语义区域（如系统、计划、用户、工具输入/输出、检索、草稿板）分配**基础优先级**和**衰减率**，并结合**注意力重要性**动态调整保留分数。
- 系统区域可**固定**（永不驱逐），计划区域可选固定。
- 令牌在获得注意力时**刷新**衰减时钟（重置年龄），从而让持续使用的令牌保持活跃。

### 2.2 关键技术细节

1. **区域标签与配置**：
   - 每个区域r配置基础优先级\( b_r \)和衰减率\( \lambda_r \)（默认设置见表I，后通过校准拟合更新）。
   - 固定区域（系统）始终保留。

2. **注意力重要性跟踪**：
   - 使用**指数加权移动平均（EWMA）** 更新每个令牌的注意力重要性\( a_i \)：
     \[
     a_i \leftarrow \rho a_i + (1 - \rho) \bar{A}_i
     \]
     其中\( \bar{A}_i \)是观测步内跨头、跨层、跨窗口的平均注意力质量，\( \rho = 0.9 \)。
   - 注意力仅在每隔k个解码步的**观测步**采样，以减少计算开销（实验中k=1）。

3. **衰减时钟与刷新**：
   - 衰减时钟\( \Delta t_i(t) = t - t_{\text{ref}}^i \)，其中\( t_{\text{ref}}^i \)为令牌插入时间或最近一次注意力超过阈值\( \tau \)的观测步。
   - 阈值采用缩放形式：\( \tau = 2 / |K_t| \)（随上下文长度自适应）。

4. **保留分数公式**：
   \[
   s_i(t) = b_{r(i)} \exp(-\lambda_{r(i)} \Delta t_i(t)) + \alpha a_i(t)
   \]
   - 第一项：衰减后的结构先验；第二项：注意力重要性权重（\(\alpha=0.5\)）。
   - 无独立的新近性项，避免冗余计时。

5. **页面级驱逐**：
   - 生产系统通常按固定大小页面（16令牌）管理KV缓存。页面得分为其内所有令牌得分的平均值。
   - 当总令牌数超过预算B时，从非固定页面中按升序（最低分）逐出页面，直至满足预算。
   - 算法流程（算法1）：更新注意力EWMA → 刷新时钟 → 计算保留分数 → 页面评分 → 若超预算则构建最小堆逐出。

6. **校准方法**：
   - 从智能体轨迹中测量每个区域在不同年龄\(\Delta t\)下的平均注意力质量\( \bar{A}_r(\Delta t) \)。
   - 拟合指数衰减模型：\(\log \bar{A}_r(\Delta t) \approx \log c_r - \hat{\lambda}_r \Delta t\)，得到衰减率\(\lambda_r \leftarrow \hat{\lambda}_r\)。
   - 关联半衰期：\(t_{1/2}^r = \ln 2 / \hat{\lambda}_r\)。

### 2.3 算法流程（文字说明）
- 在每个观测步t：
  1. 对所有缓存令牌更新注意力EWMA。
  2. 根据注意力阈值刷新每个令牌的衰减时钟。
  3. 用公式(4)计算非固定令牌的保留分数。
  4. 计算每个页面的平均得分。
  5. 若总令牌数超过预算B，则构建非固定页面的最小堆，逐出得分最低的页面直至满足预算。

## 三、实验设计

## 六、论文的主要结论与发现

1. **区域注意力生命周期差异显著**：在所有设置中，系统令牌的半衰期（148-189步）比草稿板令牌（14-16步）长约一个数量级，且置信区间不重叠。检索区域的半衰期最长（移动区域中），推翻默认快速衰减的假设。
2. **固定功能有效**：通过固定系统区域，MemDecay在所有预算和设置下均达到全缓存上限的系统探针召回率（例如短层24/24，长层21/24），而任何基线最多保留13/24。
3. **区域感知在长上下文中保持鲁棒**：Streaming（基于最近使用）在短层尚可，但在长层几乎崩溃（0.01-0.13召回），而MemDecay的区域结构先验在长层仍有效（非系统召回率0.18/0.28/0.22等，优于Streaming）。
4. **累积注意力（H2O）在非固定内容上更强**：尤其在3B大型号上，H2O在非系统召回上显著优于MemDecay（0.60 vs 0.22）。消融实验表明，当前注意力项因幅度归一化不足（EWMA加上页面平均后太小）无法保护旧令牌，这是主要限制。
5. **校准比定性默认更好**：实验表明检索区域的慢衰减优于表I中设置的快速衰减，说明需要每工作负载校准。

## 七、优点

1. **无需训练**：完全基于推理时可获得的信号（注意力、区域标签），无需额外训练或学习模块。
2. **语义感知**：利用已有编排器提供的区域标签，将缓存管理与智能体工作负载结构对齐，符合认知上的合理性。
3. **可解释性**：每个保留分数由清晰的先验衰减、注意力、刷新机制组成，易于理解和调试。
4. **页面级兼容**：与PagedAttention等生产页面缓存系统直接兼容，仅需页面平均分数。
5. **校准方法实用**：提供明确的测量协议，可从轨迹数据拟合衰减率，避免手动调参。
6. **实验设计细致**：通过变体控制插入顺序解耦区域与位置；使用cluster bootstrap置信区间和成对符号比较；精度影响验证；留一交叉验证确保校准无泄漏。

## 八、不足与局限

1. **不可逆性**：基础策略无法恢复已驱逐的令牌；若后续查询使其再次相关，信息永久丢失（文中称为“saliency shifts”）。用户区域事实被驱逐导致丢失是一个实例。
2. **跨区域信息流问题**：如果智能体将工具输出内容复制到草稿板，两份副本都快速衰减，信息可能同时消失。校准和来源跟踪可部分缓解但未解决。
3. **标签质量依赖**：假设编排器提供标签；若无，需分类器，但其鲁棒性未评估。
4. **精细化不足**：每个令牌使用单一分数，未区分头/层重要性差异；正交的头自适应预算分配可组合，但未实施。
5. **工作负载依赖**：区域生命周期随工作负载变化，校准需要代表部署的样本数据；文献中结论存在矛盾，说明了这一点。
6. **实验尺度有限**：仅测试1.5B和3B模型，上下文约1.7k token，远小于生产级（4k-32k+）。硬件不一致（M3 Pro vs T4），虽然精度影响已验证。统计置信度基于8个场景的bootstrap，近似性需谨慎。
7. **位置混淆**：系统、计划、草稿板为固定插入位置，其生命周期估计仍与位置相关。
8. **主要性能缺口**：在非固定内容上（尤其是旧事实），H2O显著优于MemDecay；即使调整注意力权重也无法解决，指向需要归一化和更长记忆的机制设计。

## 摘要
自动化仓储仓库必须持续分配和执行拾取与交付任务，同时避免拥堵。在多对多多智能体拾取与交付（MAPD）中，请求指定的是库存单元而非固定端点，要求控制器在路径规划前选择智能体、源点和目的地。现有的图引导方法主要在目标固定后影响路径选择，而端点实例化未受近期交通信息的影响。我们引入了群体图记忆（SGM），一种有界、衰减的记忆层，记录仓库节点和有向边上的近期执行信号，用于对可行端点和路径偏好进行排序，而不改变碰撞约束或规划器的有效性。在五种布局、三种负载水平和每种条件下25个种子的配对请求流实验中，SGM在所有15种地图-负载条件下均优于两种重构的多对多分配基线，配对吞吐量提升20.5%-36.7%。这些结果表明，近期执行记忆可以通过塑造哪些可行目标进入规划器，而不仅仅是智能体如何到达已固定目标，来提高仓库吞吐量。

## Abstract
Automated fulfillment warehouses must continuously assign and execute pickup-and-delivery work while avoiding congestion. In many-to-many Multi-Agent Pickup and Delivery (MAPD), a request specifies a stock-keeping unit rather than fixed endpoints, requiring the controller to select an agent, source, and destination before path planning. Existing graph-guidance methods primarily influence routing after goals are fixed, leaving endpoint instantiation uninformed by recent traffic. We introduce Stigmergic Graph Memory (SGM), a bounded, decaying memory layer that records recent execution signals on warehouse nodes and directed edges to rank feasible endpoints and route preferences without altering collision constraints or planner validity. Across paired request streams on five layouts, three load levels, and 25 seeds per condition, SGM outperforms two reconstructed many-to-many allocation baselines in all 15 map-load conditions, with paired throughput gains of 20.5-36.7%. These results show that recent execution memory can improve warehouse throughput by shaping which feasible goals enter the planner, not only how agents travel to already fixed goals.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：自动化仓储仓库中，移动机器人车队需要持续执行拾取与交付任务，同时避免拥堵。该问题被建模为多智能体拾取与交付（MAPD）问题。在多对多MAPD中，每个请求指定的是库存单元（SKU）而非固定端点，因此控制器需要在路径规划前同时选择智能体、源点（取货位置）和目的地（交付位置），构成四维分配问题。
- **现有方法不足**：现有图引导方法（如Highways、交通流成本）主要在目标固定后影响路径选择，而端点实例化（即选择具体的源点、目的地）未受近期交通信息指导。M2M（2026年工作）利用库存位置灵活性，通过静态分配目标（如旅行距离、库存分布成本）实现任务分配，但其评估在路径规划前进行，不直接考虑拥堵、等待时间和共享资源竞争，导致局部看似低廉的分配可能重复引导机器人进入同一拥挤走廊或服务点，降低整体系统效率。
- **整体含义**：本文提出一种轻量级的内存层——Stigmergic Graph Memory (SGM)，通过记录近期执行信号（节点和边上的等待、阻塞、通过等事件）来指导端点选择和路径偏好，从而将端点实例化塑造为拥塞控制决策点，提高长期吞吐量。

## 2. 论文提出的方法论

- **核心思想**：借鉴群体协调中的“stigmergy”（环境痕迹间接协调）概念，在仓库图（节点和边）上维护带衰减的数值记忆通道，每个通道对应不同类型的执行事件（等待、阻塞、端点压力、完成等）。这些记忆信号在每次控制循环中按指数衰减更新，并转化为端点排序得分和路径偏好成本。
- **关键技术与细节**：
  - **记忆更新公式**：对于每个通道c和图元素x，记忆值 \( M_{t+1}^{c}(x) = \rho_c M_t^{c}(x) + \Delta_t^{c}(x) \)，其中 \(\rho_c \in (0,1)\) 是通道特定的保留因子（如等待/延迟0.85，阻塞/端点压力0.90，完成0.95等），\(\Delta_t^{c}(x)\) 是当前时间步的事件沉积。
  - **端点引导**：对每个可行候选（agent, task, source, destination），总代价 \( C = C_0 + \lambda_p P_{\text{path}} \)，其中 \( C_0 \) 是基础分配代价（如M2M-wSKU的旅行距离+库存分布项），\( P_{\text{path}} \) 是沿最短路径代理上的有向边惩罚之和（当前有向边代价减去1），\(\lambda_p\) 是权重（实验中设为1.75）。候选源点和目的地短列表上限各64个，每请求最多32个源-目的对。
  - **路径引导**：对合法有向移动 (u,v)，代价 \( \tilde{c}_t(u,v) = 1 + \min\{\kappa, \lambda_r R_t(u,v)\} \)，其中 \( \kappa = 0.1 \) 为上限，\(\lambda_r = 0.2\)，\( R_t(u,v) \) 是综合了节点拥堵、阻塞压力、延迟、反向流、成功正向流和规划占用折扣的信号。
  - **规划器**：基于滚动视野冲突消解（RHCR）和优先搜索（PBS）作为MAPF后端。控制循环包括：更新执行记忆→短列表候选→端点引导（排序）→路径引导（派生边成本）→RHCR/PBS规划→执行一个时间步→更新状态。
  - **可行性保持**：SGM只改变候选的评分和合法有向边的代价，不修改冲突谓词、图结构或端点可行性，因此任何被接受的联合计划都是无冲突且任务可行的。

- **控制器变体**：
  - **SGM**：持续使用端点记忆打分和记忆派生路径代价，保留任务队列（在路径修复时保留已分配但未开始的任务）。
  - **ASGM**（自适应变体）：在弱拥塞证据下使用M2M-wSKU风格分配和单位代价路由；只有当等待、积压、阻塞等持续拥塞指标出现时，才激活记忆引导。加权路径需通过审计。

## 3. 实验设计

- **场景与数据集**：使用5种仓库布局（Restricted, Open-top, Open, Maze A, Maze B），每个布局为50×27网格，含1350个单元格、93个指定停车位。三种车队水平：28、56、84个机器人（分别占停车容量的30.1%、60.2%、90.3%）。每轮运行3600个时间步（步长1秒）。所有方法使用相同的配对请求流（种子从10到34共25个）。
- **基准方法**：
  - M2M：基于旅行距离估计的分配（重构自Schneider等人2026的官方代码库）。
  - M2M-wSKU：添加库存分布项，基础旅行权重1.0，SKU权重0.25。
  - 两种方法均使用RHCR/PBS规划器，每时间步重建排队任务。
  - 对照方法：SGM、ASGM。
- **评估指标**：主要指标为完成的任务总数C(3600)（吞吐量=任务数/时间）。辅助指标：最终窗口活动、服务时间、阻塞移动、分配代价、规划器时间、等待事件等。
- **消融实验**：端点仅（只保留端点记忆打分，路径代价为单位）、路径仅（只保留路径记忆代价，无端点记忆）、无队列保持（保留两个接口但每周期重建任务队列）。路由控制：M2M-wSKU+近期路径代价（使用SGM边成本）、M2M-wSKU+静态高速。敏感性分析（路径记权重、记忆衰减率、端点对上限、路径记忆权重）。匹配候选上限控制。
- **统计方法**：每地图-舰队条件使用25个配对种子，对最终完成任务数进行Wilcoxon符号秩检验，并用Holm过程校正30个比较的族级错误率。

## 6. 论文的主要结论与发现

- SGM在所有15种地图-舰队条件下均优于M2M和M2M-wSKU，配对吞吐量提升20.5%-36.7%（95%置信区间）。
- 端点引导是吞吐量提升的主要驱动力：端点仅消融保留了SGM吞吐量的99.3%-100.5%，而路径仅仅提升4.3%或更差。
- 完整的SGM控制器在保持吞吐量同时降低了规划时间（9.1%）、等待事件（0.8%）、阻塞移动（11.5%）和阻塞重规划（10.6%），使执行更高效。
- ASGM在高舰队水平下提供最佳计算归一化吞吐量，是计算资源受限时的可选方案。
- 路由控制实验表明，仅添加近期路径代价或静态高速到M2M-wSKU几乎不改变吞吐量（变化-0.1%和-1.4%），而SGM大幅提升（+29.2%），证明端点记忆是关键。
- 在中等规模传输研究中，SGM依然提升7.3%±3.3%，表明可泛化到更大布局。

## 摘要
部分可观测强化学习需要随时间决定保留、检索和遗忘什么。我们引入了一种神经符号元策略，该策略学习在每个决策点应用哪个符号记忆启发式，同时保持执行符号化。我们的设置使用RoomKG中的时间知识图谱记忆，其中隐藏状态和观察被表示为资源描述框架（RDF）图，并且记忆通过时间RDF三元组注释增强。该模型将记忆内容的知识图谱编码与用于问答、探索和遗忘的价值头相结合，产生一个既自适应又可检查的控制器。这通过基于RDF的表示、注释兼容的图语义以及基于图的符号操作直接为工作提供了语义网基础。在长期记忆容量为512的训练/测试房间分割中，限定符感知的StarE-GNN配置在比较的符号、神经和神经符号系统中实现了最佳的持有性能，同时保持了记忆管理决策的步骤级可追溯性。

## Abstract
Partially observable reinforcement learning requires deciding what to retain, retrieve, and forget over time. We introduce a neuro-symbolic meta-policy that learns which symbolic memory heuristic to apply at each decision point while keeping execution symbolic. Our setting uses temporal knowledge-graph memory in RoomKG, where hidden state and observations are represented as Resource Description Framework (RDF) graphs and memory is augmented with temporal RDF triple annotations. The model combines knowledge-graph encoding of memory contents with value heads for question answering, exploration, and forgetting, yielding a controller that is both adaptive and inspectable. This gives the work a direct Semantic Web grounding through RDF-based representation, annotation-compatible graph semantics, and graph-based symbolic operations over explicit memory state. On train/test room splits at long-term memory capacity of 512, the qualifier-aware StarE-GNN configuration achieves the best held-out performance among the compared symbolic, neural, and neuro-symbolic systems while preserving step-level traceability of memory-management decisions.

## 2. 论文提出的方法论：核心思想、关键技术细节与流程

### 2.1 核心思想
将记忆管理建模为一个**元决策问题**。智能体不直接学习如何操作记忆（这由符号程序完成），而是学习**在哪个时机调用哪个符号启发式**。具体来说，元策略输出一个三维的动作：
- **问答（QA）启发式**：如何从记忆中检索答案？
- **探索（Explore）启发式**：如何利用记忆规划下一步行动？
- **遗忘（Forget）启发式**：当长时记忆容量满时，应移除哪条信息？

### 2.2 关键技术细节

- **记忆表示**：
    - 记忆由RDF三元组构成，每个三元组附加三个元数据（qualifiers）：`time_added`、`last_accessed`、`num_recalled`。
    - 区分**短期记忆（M_short）**（来自当前观察）和**长期记忆（M_long）**（容量固定为K=512）。

- **记忆编码器（Memory Encoder）**：
    - 将当前的全部记忆（Mt）编码成一个固定长度的表示。论文比较了三种编码器：
        1.  **GCN**：只考虑图拓扑结构，忽略关系和限定符。
        2.  **R-GCN**：考虑关系类型，但忽略限定符。
        3.  **StarE-GNN**：同时考虑关系类型和限定符（qualifier-aware），信息最丰富。
    - 通过**注意力池化**（Attention Pooling）将变长的节点嵌入聚合为固定长度的、用于不同头的上下文向量 `m(p)`。

- **启发式选择头（Heuristic-Selection Heads）**：
    - 三个独立的值函数头，分别对应QA、Explore、Forget：
        - **QA头**: `Q_qa([m_qa, phi(qt)], h_qa)`，输入记忆上下文和当前问题，输出3个候选QA启发式的值。
        - **Explore头**: `Q_e(m_e, h_explore)`，输出3个候选Explore启发式的值。
        - **Forget头**: `Q_f(m_f, h_forget)`，输出3个候选Forget启发式的值。
    - **符号启发式库**：
        - **QA/Explore**: 根据元数据排序，包括**MRA**（最新添加）、**MRU**（最近使用）、**MFU**（最常使用）。
        - **Forget**: 包括**FIFO**（先入先出）、**LRU**（最近最少使用）、**LFU**（最不常使用）。

- **学习与执行流程（算法1）**：
    1.  **观察**：获得原始观察 `ot`。
    2.  **构建短期记忆**：将`ot`转换为带时间戳的暂存项。
    3.  **编码**：将所有记忆 `Mt` 通过编码器（如StarE-GNN）编码。
    4.  **评分**：三个值头分别对各自的候选启发式进行评分。
    5.  **选择**：使用 ε-贪心策略，为每个头选择价值最高的启发式（形成联合符号动作 `ht`）。
    6.  **符号执行**：执行选定的QA、Explore、Forget启发式，得到环境动作（如移动+回答），并更新记忆。
    7.  **学习**：使用DQN框架进行离线学习，通过奖励信号（仅由答案正确性给出）反向传播更新编码器和三个值头的参数。

- **训练技巧**：
    - **不同折扣因子**：QA头是立即奖励任务，γ设为0；Explore和Forget头影响长远回报，γ设为0.95。

## 8. 不足与局限

## 摘要
最近基于LLM的数学推理代理已开始着手解决研究级别的问题，并在多个案例中为开放问题的解决做出了贡献。然而，有效扩展和编排这类代理仍然具有挑战性，因为难以协调并行证明搜索，同时保持中间主张的有序和可靠。在本文中，我们提出了Danus，这是一个以共享事实图谱作为全局内存管理机制的研究级数学推理编排系统。Danus由一个执行规划和协调的主代理、多个并行执行证明搜索的工作代理，以及一个在将提议的数学主张纳入事实图谱之前对其进行验证的无状态验证器组成。每个经过验证的事实与其证明和逻辑依赖关系一起存储，使得系统能够逐步构建长论证，同时保持共享证明状态的有序性。主代理定期总结不断演化的证明状态，将工作代理重新导向有前景的方向，并通过进度报告支持与人类数学家的交互。我们通过六个研究级案例研究（涉及代数几何、奇点理论和组合学）评估了Danus，展示了事实图谱内存机制如何使Danus构建长而详细的数学证明。我们的结果表明，基于事实图谱的编排为扩展数学推理代理以解决长期研究问题提供了一条有效途径。Danus是开源的，地址为https://github.com/frenzymath/Danus。

## Abstract
Recent LLM-based mathematical reasoning agents have begun to tackle research-level problems and, in several cases, have contributed to the resolution of open problems. However, scaling and orchestrating such agents effectively remains challenging, due to the difficulty of coordinating parallel proof search while keeping intermediate claims organized and reliable. In this paper, we propose Danus, an orchestration system for research-level mathematical reasoning centered on a shared fact graph as a global memory-management mechanism. Danus consists of a main agent that performs planning and coordination, multiple worker agents that carry out proof search in parallel, and a stateless verifier that checks proposed mathematical claims before they are admitted into the fact graph. Each verified fact is stored together with its proof and logical dependencies, allowing the system to build long arguments incrementally while keeping the shared proof state organized. The main agent periodically summarizes the evolving proof state, redirects workers across promising directions, and supports interaction with human mathematicians through progress reports. We evaluate Danus through six research-level case studies in algebraic geometry, singularity theory, and combinatorics, illustrating how the fact-graph memory mechanism enables Danus to construct long, detailed mathematical proofs. Our results suggest that fact-graph-based orchestration provides an effective route toward scaling mathematical reasoning agents for long-horizon research problems. Danus is open source at https://github.com/frenzymath/Danus.

## 2. 方法论

### 核心思想
Danus 基于**生成-验证-修复**循环，但通过**事实图谱**解耦并行工作。事实图谱是一个有向无环图（DAG），其中节点是经过验证的数学陈述（事实），边表示逻辑依赖关系。该图谱是整个系统的唯一真理来源。

### 关键组件
- **主代理（Main Agent）**：负责全局规划和协调。定期读取工人日志、记忆和事实图谱，汇总状态，咨询GPT-5.5-pro等高级模型获取战略指导，重新分配工人方向。主代理基于Claude Code + Claude Opus 4.8构建，擅长阅读大规模日志和代码。
- **工作代理（Worker Agents）**：继承自Rethlas的生成代理，每个专注于一个具体子任务（如证明引理、构建反例）。工人与验证器进行**提交-验证-修复**循环，通过验证的事实被写入事实图谱。实际运行3-9个工人，分为“high”和“xhigh”两种推理努力级别。
- **无状态验证器（Verifier）**：作为独立服务运行，每次提交都启动全新实例。验证器可读取事实图谱以检查依赖，通过精细设计的技能（如逐步检查步骤、确认引用存在且适用）保证高正确率。论文宣称几乎无假阳性。
- **事实图谱**：存储每个事实与其证明及依赖，支持撤销（revocation）：如果某一事实后来被发现错误，则递归删除其所有依赖节点。

### 工作流程
1. 数学家以自然语言向主代理提出问题。
2. 主代理形成初始计划，可咨询GPT-5.5-pro，并向工人分配方向。
3. 工人并行探索（包括构造性和反驳性路径），每个工人反复提交声明+证明给验证器。
4. 通过验证的事实被加入事实图谱，工人可引用图谱中的事实。
5. 主代理每1-2小时检查一次状态，生成摘要并重新分配工人。
6. 迭代直到目标定理或其反驳作为已验证事实出现在图谱中。
7. 主代理将事实图谱转换为论文，并再次通过验证器整体检查，修复错误。

### 关键设计点
- 角色分离：主代理不直接进行数学推导，工人不涉及规划，验证器仅负责正确性。
- 上下文管理：工人每次只提交一个事实，上下文保持短小；记忆系统分本地和全局，避免重复探索。
- 写作阶段：完成证明后，Danus 将事实图谱转化为可读论文，但需经过验证器重审，因为压缩和线性化可能引入新错误。

## 6. 主要结论与发现

1. **事实图谱编排有效**：通过事实图谱作为全局内存，Danus能够并行构建含数百个验证事实的长证明（最长达54层依赖），而不会出现上下文混淆。
2. **超越单个模型**：与直接使用GPT-5.5-pro或Rethlas相比，Danus显著提升了性能，表明编排系统比底层模型更重要。
3. **自主创造能力**：Danus在多个案例中独立找到专家未预料的解题路径，甚至自动从其他数学领域借用工具（如实代数几何）。

5. **论文写作仍需验证**：事实图谱到论文的转换会引入新错误，但通过整体验证可修复大多数问题。

## 摘要
智能体系统在分支、重试和副本之间积累冲突观测结果，然而许多实际的内存层仍然在难以检查或纠正的覆盖规则背后隐藏分歧。我们提出StateFuse，一种基于标准OpSet/CRDT合并的冲突感知复制内存契约。StateFuse并没有引入新的连接代数；它定义了一个面向智能体的语义层，具有不可变历史、显式冲突对象、精确和语义修正句柄（claim_id/claim_ref）、确定性谓词契约以及投影时分辨率（不能重写复制状态）。我们在匹配的解析器和验证策略下，将StateFuse与扁平多值、原始日志、溯源风格和折叠基线进行了评估。在一个包含282个问题的官方冲突承载MemoryAgentBench切片上，比较的方法在答案准确性上相当，但冲突保留表面使矛盾可见，而折叠表面则不然。在一个具有统一验证的受控智能体循环中，保留模糊性比早期折叠能够实现更安全的弃权和修正。一个修正句柄消融进一步表明，当无法获得精确的先验标识符时，语义句柄很重要。最终结论是狭窄的：StateFuse最适宜作为用于矛盾显现、弃权和可审计修正的更安全的公共内存契约，而不是作为通用的准确性增益。

## Abstract
Agent systems accumulate conflicting observations across branches, retries, and replicas, yet many practical memory layers still collapse disagreement behind overwrite rules that are difficult to inspect or correct. We present StateFuse, a conflict-aware replicated memory contract built on standard OpSet/CRDT merge. StateFuse does not introduce a new join algebra; it defines an agent-facing semantics layer with immutable history, explicit conflict objects, exact and semantic correction handles (claim_id / claim_ref), deterministic predicate contracts, and projection-time resolution that cannot rewrite replicated state.   We evaluate StateFuse against flat multi-value, raw-log, provenance-style, and collapsed baselines under matched resolver and verification policies. On a 282-question official conflict-bearing MemoryAgentBench slice, the compared methods tie on answer accuracy, but conflict-preserving surfaces keep contradictions visible while collapsed surfaces do not. In a controlled agent loop with uniform verification, preserving ambiguity enables safer abstention and correction than early collapse. A correction-handle ablation further shows that semantic handles matter when exact prior identifiers are unavailable.   The resulting claim is narrow: StateFuse is best supported as a safer public memory contract for contradiction surfacing, abstention, and auditable correction, not as a universal accuracy gain.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：多智能体系统在分支执行、重试和副本间会产生矛盾的观测结果。现有的大多数内存层采用隐式的覆盖规则（如最后写入胜利）来处理冲突，导致分歧被隐藏，难以被审查或修正。这种设计会引发一系列问题：分支分歧被静默覆盖、过时观测在验证后仍存活、下游策略基于已被折叠的不安全内存表面做出错误决策。
- **研究动机**：分布式系统领域已经提供了成熟的收敛基础（如 CRDT 和 OpSet），但多智能体系统的内存层缺少一个面向智能体的显式契约，用于规定如何保存矛盾、如何暴露出冲突、如何精准定位修正，以及下游策略在什么范围内可以做出决定。
- **整体含义**：本文不追求提出新的 CRDT 合并代数，而是聚焦于在标准不可变合并之上构建一个更安全的**公共内存契约**，该契约能够显式保留矛盾、支持可审计修正，并限制投影时的决议权限。其价值不在于通用准确率提升，而在于为矛盾显现、弃权和可审计修正提供更强有力的保障。

## 2. 论文提出的方法论

### 2.1 核心思想
- 基于 OpSet/CRDT 的集合合并（`merge(O1, O2) = O1 ∪ O2`），提供确定性收敛。
- 定义面向智能体的语义层，包含五个核心组件：不可变历史、显式冲突对象、双重修正句柄（精确/语义）、确定性谓词契约、投影边界决议。
- 强调**投影时决议**：决议器（resolver）只能在查询时从候选值中选择或弃权，不能修改底层的复制状态。

### 2.2 关键技术细节
- **操作类型**：
  - `EvidenceAdded(evidence)`：证据记录。
  - `ClaimAdded(claim)`：声明，由 `(namespace, subject, predicate)` 组成的键、值、置信度、时间戳、证据链接、来源和两个标识符构成。
  - `ClaimRetracted(target_claim_id?, target_claim_ref?, reason, supersedes_claim_id?, supersedes_claim_ref?)`：针对一个声明实例或语义句柄的作废。
  - `DecisionAdded(decision)`：决策元数据，不影响真相状态。
- **双重句柄**：
  - `claim_id`：每个声明唯一的精确标识符，用于本地精确修正。
  - `claim_ref`：由键和标准化值派生的确定性语义句柄，用于跨副本或未知 ID 的修正。
- **谓词契约**：每个谓词可注册确定性的 `normalize(value)`、`equal(left,right)` 函数，用于判断是否功能型（functional，只允许一个值）或多值。函数必须确定性且副本不变。
- **材料化（Materialization）**：
  从操作集生成可查询状态：索引证据和声明、计算 `claim_ref`、收集作废（精确ID和语义句柄）、移除不活跃声明、按键分组、对功能型键且存在多个不同活跃值的键生成 `ConflictSet` 对象。
- **投影（Projection）**：
  `build_view(state, constraints, resolver)` 返回选中的声明、未解决的冲突集、显式冲突、可审计解释。决议器不能在材料化后修改基内存。
- **修正语义**：
  - 精确修正：作废特定 `claim_id` 的声明。
  - 语义修正：作废所有具有该 `claim_ref` 的声明（包括尚未到达的）。
  - 未到达目标不作废：语义句柄的作废在其目标声明出现后仍然生效（抑制）。
  - 仅活跃声明参与冲突检测。
- **紧凑化与认证同步**：投影等价紧凑化保证当前决议器可见语义不变；认证同步在合并前可隔离无效操作。

### 2.3 公式与算法（文字说明）
- 合并：`O1 ∪ O2`（OpSet 标准合并）。
- 活跃声明集：`A(O) = {c ∈ C(O) | id(c) ∉ Rid(O) ∧ ref(c) ∉ Rref(O)}`，其中 Rid、Rref 分别来自精确和语义作废。
- 材料化函数 `M(O)` 返回证据索引、活跃声明、决策、冲突集。
- 投影函数 `V(M, resolver, constraints)` 不改变 M。
- 复杂性：材料化 O(n log n)，冲突检测每键 O(m²)，投影线性。

## 6. 论文的主要结论与发现

- **核心结论**：StateFuse **不是通用的准确率提升方法**。在官方基准上，所有方法（包括折叠基线）的最终准确率均达到 97.5%，没有显著差异。
- **关键发现**：
  - 冲突保留表面（StateFuse、扁平多值）在所有 282 个冲突任务上都显式暴露了矛盾，而折叠和原始日志表面则完全不暴露。
  - 在保守弃权策略下，所有非折叠表面达到相同安全点（64.9% 准确率，100% 矛盾召回，2.1% 错误置信）；折叠表面错误置信率更高。
  - 在下游智能体循环中，保守非折叠表面均达到 100% 事后验证成功率，0% 错误置信；折叠表面仅 60% 成功，40% 错误置信。
  - 语义修正句柄（claim_ref）在精确 ID 不可用时能恢复所有修正目标，且不会导致被修正值复活；精确句柄在有语义目标的任务中恢复率为 0%，复活率为 23.1%。
- **总结**：StateFuse 的最大优势在于提供更安全的公共内存契约，使矛盾可见、支持安全弃权和可审计修正，而非提升答案准确性。

## 7. 优点

- **方法创新**：
  - 显式冲突对象（ConflictSet）作为契约边界，而不是隐式的内部状态。
  - 双重修正句柄设计（exact + semantic）解决了跨副本修正的精准性和通用性。
  - 投影时决议 + 不可变基内存的设计保证了历史可审计性，避免了决议器越权修改。
  - 确定性谓词契约保证了跨副本的行为一致性。
- **实验设计**：
  - 采用“匹配信息”评估策略，为所有基线提供相同的下游策略预算，比先前工作更公平。
  - 明确区分“保守”和“标准”变体，揭示了弃权策略带来的安全收益。
  - 消融实验直接验证了语义句柄的独特价值，避免了句柄名称替换的混淆。
- **理论保障**：提供了收敛性、确定性、作废语义等命题的草图证明，增强了方法的可信度。

## 8. 不足与局限

- **实验覆盖不足**：
  - 仅使用 MemoryAgentBench 的一个冲突承载子集（282 题），缺乏大规模自然痕迹（如 Wiki 编辑冲突、Git 合并历史）或更广泛的多智能体任务（如 ReAct、Reflexion 场景）。
  - 未与现有工业级内存系统（如 MemGPT、AgentGit）进行集成或端到端对比。
- **偏差风险**：
  - 所有实验均为确定性，使用同一套解析器族，可能不利于 StateFuse 的显式结构优势（例如在复杂 LLM 决议器下，显式冲突可能带来更好的准确率，但本文未测试）。
  - 智能体循环任务仍然合成，不反映真实噪声和长尾问题。
- **应用限制**：
  - 方法依赖于谓词契约的确定性（normalize/equal），不稳定的函数可能导致不可预测结果。
  - 紧凑化与认证同步仅为界限验证，未做形式化证明或对抗性测试。
  - 不支持拜占庭容错、抗垃圾信息或副本成员身份认证。
  - 操作集大小增长时，材料化时间为 O(n log n)，对于超大规模日志可能成为瓶颈（论文承认需要未来工作）。
- **结论保守性**：作者自评该方法“不是通用准确率提升”，这虽然诚实但可能限制其影响力。读者需要理解其目标场景（需要冲突可视化和修正的系统），而非追求 SOTA 准确率。

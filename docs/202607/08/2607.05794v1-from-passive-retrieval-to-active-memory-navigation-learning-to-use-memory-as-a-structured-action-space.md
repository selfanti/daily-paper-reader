## 摘要
长期用户记忆对于个性化对话代理至关重要，然而许多记忆系统仍然通过被动检索接口暴露记忆，使模型成为预选证据的消费者。我们提出NapMem框架，这是一种学习将长期用户记忆用作结构化行动空间而非被动检索上下文的框架。NapMem将用户历史组织成链接的多粒度记忆金字塔，其中原始对话、类型化记忆记录、主题轨迹和用户配置文件通过溯源关系连接，并通过记忆工具暴露这些层级。代理被训练根据查询和中间证据选择记忆，使其能够在回答前检查不同的记忆粒度。在PersonaMem-v2、LongMemEval和LoCoMo上的实验表明，使用记忆工具强化学习训练的NapMem代理在多种记忆密集型任务中具有竞争力，而在非记忆任务上的评估表明，学习到的策略基本保留了一般推理和工具使用能力。附加分析考察了存储、推理成本、工具使用行为，以及在导航、记忆粒度和强化学习训练上的消融实验。我们的结果表明，长期用户记忆受益于将结构化存储与学习到的适当粒度记忆使用策略相结合。

## Abstract
Long-term user memory is essential for personalized conversational agents, yet many memory systems still expose memory through passive retrieval interfaces, making the model a consumer of pre-selected evidence. We introduce NapMem, a framework for learning to use long-term user memory as a structured action space rather than passively retrieved context. NapMem organizes user history into a linked multi-granularity memory pyramid, where raw conversations, typed memory records, topic tracks, and user profiles are connected through provenance relations, and exposes these levels through memory tools. The agent is trained to select memory according to the query and intermediate evidence, allowing it to inspect different memory granularities before answering. Experiments on PersonaMem-v2, LongMemEval, and LoCoMo show that a NapMem agent trained with memory-tool reinforcement learning is competitive across diverse memory-intensive tasks, while evaluations on non-memory tasks suggest that the learned policy largely preserves general reasoning and tool-use abilities. Additional analyses examine storage, inference cost, tool-use behavior, and ablations over navigation, memory granularity, and RL training. Our results suggest that long-term user memory benefits from coupling structured storage with a learned policy for using memory at the appropriate granularity.

## 论文详细总结

### 1. 核心问题与整体含义（研究动机与背景）
- **问题**：现有对话系统中的长期用户记忆（long-term user memory）多采用**被动检索接口**（passive retrieval），即系统预先检索并拼接上下文后交给模型，模型只能消费预选证据，无法自主决定是否需要更多信息、从哪个粒度查阅、何时停止。
- **动机**：不同查询对记忆粒度（如原始对话、事实记录、主题轨迹、用户画像）的需求不同，固定的记忆表示或检索策略难以泛化。需要让代理主动导航记忆空间，将记忆从“被动检索的上下文”转变为**结构化的行动空间**（structured action space），由代理通过工具调用自主选择何时、如何访问记忆。

### 2. 方法论：核心思想、关键技术细节
- **核心思想**：NapMem（Navigate over Pyramid Memory）将用户历史组织为**多粒度记忆金字塔**，并暴露为一系列记忆工具（memory tools）；代理通过**强化学习（GRPO）** 训练出根据查询和中间证据选择合适粒度的导航策略。
- **关键技术细节**：
  - **记忆金字塔四层**：
    - **原始对话层（Raw Conversations）**：消息级，完整保真，支持精确查找。
    - **记忆记录层（Memory Records）**：类型化（fact/event/instruction/preference），紧凑结构化，通过混合检索获取，并维护溯源链接到原始消息。
    - **主题轨迹层（Topic Tracks）**：跨会话聚合相关记录形成的叙事性摘要，支持演化追踪。
    - **用户画像层（User Profile）**：全局稳定的属性、偏好、交互模式，长度受限。
  - **记忆工具**（共5个）：`search_records` / `search_conversation`（混合检索），`get_records` / `get_conversation`（精确ID查找），`read_file`（读取主题轨迹或用户画像文件）。代理每次可调用一个工具或直接回答，最多4步。
  - **学习策略**：使用**GRPO（Group Relative Policy Optimization）** 优化，奖励函数基于三项二元指标：格式有效性（F）、答案正确性（C）、是否至少调用一次记忆工具（U）。正确回答并使用记忆得+1，不正确且没用记忆得-1，其他组合设定中间惩罚。轨迹级优势均匀应用于所有输出token，联合优化工具调用和答案生成。

### 3. 实验设计：数据集/场景、基准、对比方法
- **记忆密集型任务（3个基准）**：
  - **LoCoMo**（长会话记忆问答，1,315测试题，open-ended，F1 + LLM Judge准确率）
  - **LongMemEval**（长程交互记忆，100题，open-ended，F1 + L-J）
  - **PersonaMem-v2**（隐式偏好与个性理解，911题，多项选择，准确率）
- **非记忆任务（3个基准）**：
  - **GPQA-Diamond**（科学推理，多项选择，准确率）
  - **BFCL-v3**（函数调用，工具/执行/任务准确率）
  - **V*Bench**（视觉工具问答，答案/工具/执行准确率）
- **对比方法**：
  - 5个记忆系统基线：Mem0、Zep、MemOS、MemoryOS、AgeMem（AgeMem是RL训练的代理记忆基线）。

  - NapMem-9B w/ RL（默认），NapMem-9B w/o RL（仅金字塔+工具无RL）。

### 6. 主要结论与发现
- NapMem-9B w/ RL在三个记忆密集型基准上平均得分62.74，**超过了NapMem-397B（59.85）**，表明训练策略比单纯扩大模型更有效。
- RL训练能**减少不必要的记忆调用**（GPQA-D上从34.51%降至6.90%），同时保留甚至提升非记忆任务性能。
- 消融实验表明：主动导航、完整记忆粒度、RL训练三者均贡献显著，移除任何一项都会导致性能下降。
- 存储效率上，NapMem比多数基线更轻量（总4.83 GiB vs Mem0的10.44 GiB），推理延迟和生成长度也低于被动检索方法。
- 不同模型规模表现不同的导航偏好：大模型更倾向先读取文件（宏观），小模型更倾向先搜索记录（微观）。

### 7. 优点（方法与实验设计亮点）
- **创新性**：将记忆使用重新定义为**结构化行动空间上的主动导航**，而非被动检索，视角新颖且符合认知直觉。
- **方法论完整性**：金字塔设计兼顾证据保真与抽象效率，工具接口支持上下双向导航（顶层→底层或反方向）。

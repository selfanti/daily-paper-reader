## 摘要
大语言模型（LLM）代理通常通过修改提示、模型或手动编写的工作流来改进，而模型周围的执行框架被视为固定的基础设施。我们认为该框架本身是一个可学习的控制层。我们将框架操作形式化为一个有限时域的框架MDP，其中轻量级控制器选择结构化的执行动作，而LLM执行器保持冻结。控制器通过优势加权回归，仅使用终端任务评分奖励，从离线轨迹中训练。我们还将最终任务质量与事后框架成熟度评分分开，该评分衡量框架是否遵循可靠执行模式，而不仅仅是最终答案是否正确。这种分离提供了框架学习的有限缓冲区视图：最终质量提升需要离线缓冲区中的高回报支持，而过程行为可以在其与优势加权动作一致时发生改变。在六个受控领域和两个公开基准适配器上，学习控制器持续改进验证行为，并选择性提升最终任务质量，其中在适配的tau-bench零售、适配的AgentBench DB-Bench以及使用校准的结构化验证器进行编码时提升最大。与行为克隆和强制检查的消融实验表明，这些提升不能通过模仿或简单添加检查来解释。这些结果将框架控制识别为冻结LLM代理的可学习层，同时表明离线支持限制了何时更好的过程控制能转化为更好的最终答案。

## Abstract
Large language model (LLM) agents are usually improved by changing prompts, models, or hand-written workflows, while the execution harness around the model is treated as fixed infrastructure. We argue that this harness is itself a learnable control layer. We formalize harness operation as a finite-horizon Harness MDP, where a lightweight controller selects structural execution actions while the LLM executor remains frozen. The controller is trained from offline rollouts using advantage-weighted regression with only terminal task-rubric rewards. We also separate final task quality from a post-hoc Harness Maturity Score, which measures whether the harness follows reliable execution patterns rather than only whether the final answer is correct. This separation gives a finite-buffer view of harness learning: final-quality gains require high-return support in the offline buffer, while process behavior can shift whenever it aligns with advantage-weighted actions. Across six controlled domains and two public-benchmark adapters, the learned controller consistently improves verification behavior and selectively improves final task quality, with the largest gains on adapted tau-bench retail, adapted AgentBench DB-Bench, and coding with a calibrated structural verifier. Ablations against behavior cloning and Forced CHECK show that the gains are not explained by imitation or by simply adding checks. These results identify harness control as a learnable layer for frozen LLM agents, while showing that offline support limits when better process control becomes better final answers.

- **问题**：当前的 LLM 代理改进主要聚焦于修改提示（prompt）、微调模型或设计手写工作流，而代理周围用于控制执行流程的“框架”（harness）被视为固定基础设施。作者认为，框架本身实际上是一个可学习的控制层，它决定了模型何时观察环境、检索证据、调用工具、起草答案、检查、修订或终止，其重要性不亚于模型本身的能力。
- **背景**：现有系统通常使用静态规则（如“总是检查”）或固定执行图，难以适应不同任务难度和轨迹阶段的差异。常见失败模式并非模型知识缺失，而是控制决策序列不佳。因此，本文研究**能否直接从离线轨迹中学习框架的控制策略**，以提升代理的执行质量和最终任务表现。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将框架操作形式化为一个**有限时域的 Harness MDP**（框架马尔可夫决策过程），其中状态（state）包含轨迹进度、草稿状态、证据覆盖、工具输出、验证器反馈、错误次数、剩余预算等；动作空间为 7 个结构化动作：OBSERVE、RETRIEVE、CALL-TOOL、DRAFT、CHECK、REVISE、SUBMIT。一个轻量级控制器（单隐藏层 MLP，64 单元）根据当前状态选择下一个框架动作，而**冻结的 LLM 执行器**仅在动作调用时生成内容，但不更新参数。
- **学习算法**：采用**离线优势加权回归（Advantage-Weighted Regression, AW）**，从有限的离线轨迹缓冲区中训练。每条轨迹获得一个终端任务评分奖励 G，计算优势 A(s,a)，损失函数为：
  \[
  L(\theta) = -\mathbb{E}_{(s,a)\sim\mathcal{D}}\left[\exp(A(s,a)/\beta) \log \pi_\theta(a|s)\right]
  \]
  其中 β=0.2，权重剪裁在 [0.1,10.0] 之间。这等价于加权行为克隆，增加高回报轨迹中动作的似然。
- **过程与结果分离**：引入**Harness Maturity Score (HMS)**，基于七个过程事件（如检查后提交、证据后声明、测试后提交、失败后修订等）的加权归一化分数，但**只用于评估，不作为训练奖励**。理论部分（定理1-2，命题1-2）证明：最终质量改进受限于缓冲区中的最高回报轨迹支持（buffer slack），而过程改进则取决于优势权重与过程统计量之间的协方差，即过程改进可能发生在最终质量不变甚至下降的情况下。
- **训练细节**：Adam 优化器，学习率 1e-3，batch size 256，训练 20 个 epoch，熵正则化 0.01，三个独立种子。超参数在所有领域固定。

## 8. 不足与局限

- **离线缓冲区的根本限制**：最终质量改进依赖缓冲区中的高回报轨迹支持，如果缓冲区本身缺乏优质样本，AW 无法创造出超越现有轨迹的行为。这限制了方法的实际应用，尤其在新领域或冷启动场景。
- **受控领域规模小且人工标注**：每个领域仅 100 个任务，人工标注可能引入主观偏差，且任务多样性有限。结论的外部有效性需要更大规模、更自然的任务集验证。

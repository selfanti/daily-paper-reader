## 摘要
LLM智能体在长程交互中常出现性能退化：随着轨迹增长，它们会重访已探索状态、重复失败动作，并丢失先前有效的策略。测试时训练（TTT）提供了一种使模型权重适应动态任务状态的方法，但现有的LLM TTT方法主要针对固定输入进行一次适配。我们研究多轮智能体回合中的连续TTT，其中每次更新都会改变后续训练文本的生成策略。这形成了一个自训练循环：当新轨迹信息出现时有助于提升，但当智能体陷入困境并反复训练相似文本时可能放大漂移。我们发现更新文本的重复性区分了这两种状态，并提出了智能体测试时训练（aTTT），这是一种基于词元级别的重加权方法，对先前更新中重复出现的n-gram中的词元赋予更低损失权重，而对新词元保持完全权重。为了在实时回合中执行此类更新，我们利用vLLM的运行时LoRA API构建了一并发服务系统，将额外开销限制在无TTT成本的1.9倍。aTTT在ALFWorld上最高提升5.0个百分点，在SWE-bench Lite上提升4.9个百分点。性能提升集中在模型已具备任务能力但会在长程轨迹中漂移的场景，这表明aTTT主要保护现有能力而非传授新技能。

## Abstract
LLM agents often degrade over long episodes: as trajectories grow, they revisit explored states, repeat failed actions, and lose strategies that previously worked. Test-time training (TTT) offers a way to adapt model weights to the evolving task state, but existing LLM TTT methods largely adapt once to a fixed input. We study continuous TTT in multi-turn agent episodes, where each update changes the policy that generates later training text. This creates a self-training loop that helps when new trajectory information appears, but can amplify drift when the agent gets stuck and repeatedly trains on similar text. We find that update-text repetition distinguishes these regimes and introduce Agentic Test-Time Training (aTTT), a token-level reweighting method that downweights the loss on tokens appearing in repeated $n$-grams from prior updates while leaving novel tokens fully weighted. To run such updates inside live episodes, we build a concurrent serving system using vLLM's runtime LoRA API, limiting overhead to 1.9$\times$ the no-TTT cost. aTTT improves success by up to 5.0 points on ALFWorld and 4.9 points on SWE-bench Lite. The gains concentrate where models already have task competence but drift over long trajectories, suggesting that aTTT mainly preserves existing competence rather than teaching new abilities.

## 2. 方法论

### 核心思想
通过 token 级重加权，降低重复 n-gram 的损失贡献，保留新颖 token 的完全权重，从而在利用新轨迹信息的同时避免强化已失败的重复行为。

### 关键技术细节
- **更新信号**：三种轨迹文本来源
  - **Self**：模型自身的推理和动作 token（最近一步）。
  - **Env**：环境观测文本。
  - **Summary**：通过额外 LLM 调用压缩轨迹生成的摘要。
- **更新规则**：每 K 个环境步（默认 K=5）更新一次 LoRA 适配器，适配器在回合内持久化，回合间重置。
- **重复性度量**：定义更新文本与历史更新文本的 Jaccard 相似度作为重复率，对应的新颖度 ν=1−ρ。
- **Token 级重加权**：
  - 对于当前更新文本的每个 token 位置 j，计算其在过去所有更新文本中的最大 n-gram 出现次数 f_k(j)。
  - 权重 w_{k,j} = max( w_min , 1/(1+f_k(j)) )，其中 w_min 为地板值（默认 0.05）。
  - 损失函数为加权交叉熵：L_aTTT = (1/L) Σ w_{k,j} CE(x_{k,j}, ˆx_{k,j}; θ)。
- **系统实现**：基于 vLLM 的运行时 LoRA API，主模型冻结，每个回合持私有 LoRA 适配器；训练 GPU 异步计算更新并热切换（<100ms），16 个回合并发，开销仅为无 TTT 的 1.9 倍。

### 算法流程（文字描述）
每回合开始前初始化 token 历史为空。在每个训练点（每 K 步），从轨迹中选取更新文本 → 对每个 token 计算其与历史 token 的 n-gram 重叠次数 → 计算权重 → 计算加权损失 → 梯度更新 LoRA → 将当前更新文本加入历史。

## 3. 实验设计

### 数据集与 Benchmark
- **ALFWorld**：140 个基于文本的家居指令任务，每回合 50 步预算，即时环境反馈。
- **SWE-bench Lite**：150 个真实软件工程问题，使用 mini-swe-agent 脚手架，每问题 150 次调用的预算，验证延迟。

### 对比方法
- **ReAct**：无 TTT 基线。
- **qTTT**：静态预启动适应（一次适配后冻结）。
- **No filter**：在线 TTT 但使用普通交叉熵损失（无重复控制）。
- **Sequence filter**：基于序列级新颖性阈值（τ=0.5）跳过整个更新。
- **aTTT**：本文方法，使用 token 级重复重加权。
- 额外基线：相同 Summary 文本的上下文注入（in-context）、解码时重复惩罚（1.1 系数）。

### 模型
Qwen3.5-4B、Qwen3.5-9B、Qwen3.5-27B、Gemma-3-12B。

### 评估指标
成功率（ALFWorld） / 分辨率（SWE-bench），均报告 3 个种子的均值±标准误。

## 6. 主要结论与发现

- **aTTT 显著提升性能**：ALFWorld 上最高提升 5.0 个百分点（Qwen3.5-9B, Self 信号），SWE-bench Lite 上最高提升 4.9 个百分点（Qwen3.5-27B, Summary 信号）。
- **增益集中在特定场景**：模型已有一定成功率但长轨迹漂移明显时效果最佳（如 Qwen3.5-27B 在 ALFWorld 基线高且短，增益小；在 SWE-bench 长轨迹中增益大）。弱模型（4B）或已稳定模型增益有限。
- **重复性是关键诊断信号**：失败轨迹的更新文本重复率早于成功轨迹显著上升。aTTT 通过降低重复 token 损失，避免了无过滤在线 TTT 的大幅 KL 漂移。
- **优于序列过滤器和随机丢弃**：序列级过滤丢失部分新信息；随机丢弃任意更新无法复制增益。aTTT 的 token 级重加权更精细。
- **权值更新优于相同信息的上下文注入**：注入摘要到 prompt 因上下文长度竞争失效（49.3% vs 无 TTT 50.7%），而 TTT 权重更新达 54.3%。
- **主要稳定已有能力而非教授新技能**：增益来自保护已有策略不被重复自训练破坏，而非提升模型在完全未知领域的表现。

## 7. 优点

- **问题识别新颖**：首次系统研究多轮智能体连续 TTT 中的“重复自训练循环”，并用更新文本重复性作为可操作信号。
- **方法设计巧妙**：从序列级决策（跳过整个更新）到 token 级重加权，避免了全部丢弃的损失，保留了新观测和对象名称等关键 token。
- **系统工程实用**：基于 vLLM 的 LoRA 运行时实现并发，1.9 倍开销可接受，具备实际部署潜力。
- **实验全面且客观**：覆盖多模型、多信号源、多消融，并且进行了大量的诊断和对比（KL 漂移、动作多样性、任务类型、案例研究），验证了假设。
- **开箱即用**：aTTT 不需要外部知识或额外训练数据，仅利用在线轨迹，适应性强。

## 8. 不足与局限

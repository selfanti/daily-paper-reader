## 摘要
长寿AI智能体需要跨交互的连续性，但连续性不能通过简单地扩展提示窗口来获得。智能体必须保留有用的先前经验，选择性检索，区分个人背景与外部证据，并在底层情况发生变化时修正记忆。我们提出了一种架构化的记忆基板，沿着两个正交轴组织：一个表示轴，涵盖结构化记录、向量表示和图关系；一个时间轴，涵盖短期痕迹、中期抽象和长期语义承诺。其关键设计约束是同步结构化-向量-图记忆：结构化记录决定资格，向量表示支持召回，图关系在门控上下文投影之前裁决支持、矛盾和取代。其核心主张是，可靠的个性化是一个记忆设计问题：有用的记忆是结构化的、选择性暴露的、持续巩固的，并且带有认知标签，而不是存储为未区分的对话历史。除了框架之外，我们将MRMS实现为一个轻量级原型，实现结构化记录、向量检索、时间策略和基于图的修正。该原型通过预生成记忆选择、修正、边界执行和证据归因，在具有明确证据要求的受控长期交互场景中，锻炼核心基板机制。

## Abstract
Long-lived AI agents require continuity across interactions, but continuity cannot be obtained by simply extending the prompt window. An agent must preserve useful prior experience, retrieve it selectively, distinguish personal context from external evidence, and revise memory when the underlying situation changes. We propose an architectural memory substrate organized along two orthogonal axes: a representational axis spanning structured records, vector representations, and graph relations; and a temporal axis spanning short-term traces, medium-term abstractions, and long-term semantic commitments. Its key design constraint is synchronized structured-vector-graph memory: structured records govern eligibility, vector representations support recall, and graph relations adjudicate support, contradiction, and supersession before gated context projection. Its central claim is that reliable personalization is a memory design problem: useful memory is structured, selectively exposed, continuously consolidated, and epistemically labeled rather than stored as undifferentiated conversation history. Beyond the framework, we instantiate MRMS as a lightweight prototype implementing structured records, vector retrieval, temporal policies, and graph-based revision. The prototype exercises the core substrate mechanisms through pre-generation memory selection, revision, boundary enforcement, and evidence attribution under controlled long-lived interaction scenarios with explicit evidence requirements.

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：现有的 LLM 驱动的 AI 智能体缺乏跨长时交互的连续性。仅通过增大上下文窗口无法有效管理智能体的经验：长上下文成本高、注意力不均匀，且对话历史无法区分持久偏好与临时指令、验证事实与推测、个人语境与外部证据。
- **背景与动机**：长生命周期智能体需要“稳定性-可塑性”平衡——既能吸收新证据，又不会每次交互都重写用户画像。记忆错误（如过时偏好、被取代事实）会被反复带入未来推理，危害远大于临时性错误。因此，可靠的个性化本质上是**记忆设计问题**，而非简单的检索扩增（RAG）或长上下文模型所能解决。
- **整体含义**：提出 MRMS（Multi-Resolution Memory Substrate），将记忆从被动的存储提升为预生成的控制层，通过结构化-向量-图同步机制，控制哪些记忆能影响当前行为、如何暴露其来源以及如何处理矛盾与更新。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 沿两个正交轴组织记忆：
  - **表示轴**：结构化记录（structured records）、向量表示（vector representations）、图关系（graph relations）。
  - **时间轴**：短期痕迹（short-term traces）、中期抽象（medium-term abstractions）、长期语义承诺（long-term semantic commitments）。
- **关键设计约束**：**同步结构化-向量-图记忆**（synchronized structured-vector-graph memory）：
  - 结构化记录：控制资格、状态、范围、来源。
  - 向量索引：支持语义近似检索。
  - 图关系：裁决支持、矛盾、替代，并在生成前完成门控上下文投影。

### 2.2 技术细节

## 3. 实验设计

### 3.1 数据集/场景
- 使用**诊断性基准**（diagnostic benchmark），包含 800 个确定性生成的合成任务，分布在 8 个任务族：
  - 延迟回忆（delayed recall）、边界控制（near-miss boundary, source boundary）、修订（revision）、过时抑制（stale suppression）、证据归因（evidence attribution）、矛盾处理（contradiction）、时间承诺（temporal commitment）。
- 平均每个场景包含 17 条记忆记录（最少 13，最多 21）。共 9,503 条个人记忆，4,095 条外部指导记录。

### 3.2 Benchmark 结构
- 基准不依赖下游 LLM 答案，而是测量**预生成阶段的记忆选择**：是否正确选中并仅选中所需的记忆标识符。
- 评价指标：总体准确率、修订成功率、过时抑制、边界控制、归因、弃权准确率。

### 3.3 对比方法（消融条件）
1. **近期上下文**（最近上下文窗口）
2. **仅向量检索**（用户范围内的近邻检索，忽略来源/状态/时间）
3. **结构化+向量**（加用户、任务、来源的门控）
4. **结构化+向量+时间**（加活跃状态过滤、层感知排序、过时抑制、弃权）
5. **完整 MRMS**（加图展开和证据处理）

### 3.4 核心结果（表3）
- 完整 MRMS 总体准确率 98.8%，修订、过时抑制、边界控制、归因、弃权项均达 95~100%。
- 消融显示：仅向量检索仅 21.2%，结构化+向量+时间达 75%，完整 MRMS 提升至 98.8%。
- 上下文包平均仅选择 1.11 个记忆对象（候选 1.8 个），保持紧凑。

## 6. 主要结论与发现

- 可靠个性化是记忆设计问题：记忆必须结构化、选择性暴露、持续巩固、带有认知标签。
- 同步结构化-向量-图记忆是核心：结构化控制授权，向量支持语义检索，图处理支持/矛盾/取代关系。
- 时间分层是稳定性机制：快速捕获低层，保守提升高层，防止瞬时记忆变成永久信念。
- 预生成阶段的门控比后生成修正更关键：过时或错误的记忆一旦进入上下文，即使生成模型再强也难以纠正。
- 消融实验强有力支持了各组件（结构化门控、时间策略、图展开）的必要性：完整 MRMS 准确率 98.8%，而缺少任何一个都会显著下降（如仅向量 21.2%）。

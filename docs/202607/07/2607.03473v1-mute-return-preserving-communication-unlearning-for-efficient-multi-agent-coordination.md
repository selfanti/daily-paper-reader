## 摘要
智能体间通信对于在部分可观测环境下协调多智能体强化学习（MARL）智能体以在协作游戏中有效执行至关重要；然而，现实世界的带宽限制要求稀疏交互。先前的方法主要通过优化信息论替代指标来解决这一权衡。我们认为这些统计代理从根本上与真实目标不一致：一条消息可能信息量很大，但与任务的联合回报无关。在这项工作中，我们提出了面向目标效率的消息遗忘（MUTE）框架，该框架将通信减少视为一个价值引导的机器遗忘问题。MUTE使用基于注意力的估计器严格量化反事实消息价值，并系统性地从无通信限制训练的 policy 中遗忘低价值消息的传输。这是通过一个双目标机制实现的，该机制在保留原始联合 policy 回报的同时强制通信稀疏性。我们推导了这种稀疏化导致的性能差距的理论上界，保证了可控的回报退化。我们还在各种复杂多智能体环境中对MUTE进行了实证评估，在保持与最先进基线相当性能的同时，实现了80%到90%的带宽减少。

## Abstract
Inter-agent communication is critical for coordinating Multi-Agent Reinforcement Learning (MARL) agents under partial observability to perform effectively in cooperative games; however, real-world bandwidth constraints demand sparse interactions. Prior approaches primarily address this trade-off by optimizing information-theoretic surrogates. We argue that these statistical proxies are fundamentally misaligned with the true objective: a message can be highly informative yet irrelevant to the joint return of the task. In this work, we propose Message Unlearning for Targeted Efficiency (MUTE), a framework that views communication reduction as a value-guided machine unlearning problem. MUTE rigorously quantifies the Counterfactual Message Value using an attention-based estimator, and systematically unlearns the transmission of low-value messages from a policy trained without any communication constraints. This is achieved through a dual-objective mechanism that enforces communication sparsity while preserving the return of the original joint policy. We derive a theoretical upper bound on the performance gap induced by this sparsification, guaranteeing controlled return degradation. We also empirically evaluate MUTE on various complex multi-agent environments, achieving 80% to 90% bandwidth reduction while maintaining performance comparable to state-of-the-art baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：在部分可观测环境下，多智能体强化学习（MARL）依赖智能体间通信实现有效协调，但现实世界带宽有限，需要压缩通信。
- **现有方法缺陷**：主流方法通过优化信息论代理（如互信息最小化或最大化）来压缩通信，但这些统计指标与任务真实目标（联合回报）不一致。例如，一条消息可能具有高互信息却导致回报下降，而低互信息消息可能对协调至关重要。
- **本文动机**：直接优化通信对联合回报的贡献，将通信缩减视为**值引导的机器遗忘问题**，从已训练好的全网通信策略中有选择地遗忘低价值消息，保留回报关键的通信。

## 2. 方法论

### 核心思想
- 首先训练一个无通信限制的专家策略（全通信策略），然后识别并遗忘其中冗余的低价值消息，同时通过行为锚定保持回报。
- 提出 **反事实消息价值 (Counterfactual Message Value, CMV)**，衡量屏蔽某条消息后联合动作值函数的变化量。
- 设计**消息价值估计器 (Message Value Estimator, MVE)**，基于注意力机制一次性预测所有消息的CMV，避免指数级计算。
- 通过**双目标损失**实现稀疏化：稀疏损失（对低价值消息施加 L1 惩罚） + 行为锚定损失（使当前策略的个体 Q 值逼近冻结的专家策略 Q 值）。

### 关键技术细节
- **CMV 定义**：$ \Delta Q_i(\tau, m) = Q^*_{tot}(\tau, a) - Q^*_{tot}(\tau, \tilde{a})$，其中 $\tilde{a}$ 是屏蔽 $m_i$ 后重新采样接收者动作的联合动作。
- **MVE**：输入所有消息集合，输出每个消息的预测CMV，用 MSE 损失训练：$L_{MVE}(\phi) = \frac{1}{N} \sum_i (V_\phi(m)_i - y_i)^2$。
- **稀疏损失**：$L_{sparse}(\psi) = \sum_{m \in \mathcal{M}_{red}} \|m\|_1$，其中 $\mathcal{M}_{red}$ 是 CMV 低于自适应阈值 $\lambda \mu_{MVE}$ 的消息集合。
- **行为锚定损失**：$L_{anchor}(\theta, \psi) = \sum_i \|Q^*_i - Q_i\|_2^2$，冻结专家策略的个体 Q 值，防止策略崩溃。
- **最终损失**：$L = L_{sparse} + \beta L_{anchor}$。
- **理论保证**：证明了性能下降上界 $J(\pi^*) - J(\pi) \leq \frac{1}{1-\gamma} \mathbb{E}[2\epsilon + |M_{red}|(\lambda+\delta) + L\sum_{m_i\in M_{red}}\|m_i\|_2]$，其中 $\epsilon$ 是动作值近似误差，$\delta$ 是 MVE 估计误差。

### 算法流程（三阶段训练）
1. **预训练**（2M 步）：标准 MARL 训练全通信策略。
2. **MVE 训练**（0.5M 步）：使用离线数据训练消息价值估计器。
3. **通信遗忘**（1.5M 步）：冻结专家策略，优化双目标损失，逐渐压制低价值消息。

## 6. 主要结论与发现

- MUTE 在几乎所有场景上**显著降低通信带宽**（80%–99%），同时保持 **与全通信基线（MASIA）相当的胜率**。
  - 例：Hallway 上通信率降至 1%，胜率 100%；MMM2 上通信率 <5%，胜率 94%。
- 在**消息丢弃测试**中，MUTE 的 AUC 在所有基准上最高（表2），表明保留的消息真正是回报关键的。
- **行为锚定**是防止策略崩溃的关键，**MVE** 确保选择性地遗忘冗余消息而非随机丢弃。
- 理论性能上界经实证验证（见 Appendix I）：实际动作值误差 $\|Q^*_{tot}-Q_{tot}\|_\infty$ 在 0.15–0.48 之间，MVE 损失稳定，保证了遗忘的安全性。

## 7. 优点

- **创新性方法**：将通信缩减重新定义为**值引导的机器遗忘**问题，避免传统信息论代理与回报的错配。
- **理论严谨**：推导了性能下降的上界，并验证了实际误差满足理论假设。
- **计算高效**：MVE 一次性估计所有消息价值，避免 Shapley 值方法的指数复杂度；推理 FLOPs 与 MASIA 相当。
- **实验充分**：覆盖多种复杂环境，包含充分的消融和敏感性分析，结果可靠。
- **实用性强**：在 SMAC 等标准测试中达到 90%+ 带宽缩减而性能不降，有实际应用潜力。

## 8. 不足与局限

- **CMV 的局限性**：CMV 是屏蔽单条消息的局部效应，未显式枚举所有消息子集，可能无法完全捕捉高阶交互（如冗余信息或干扰效应）。论文通过行为锚定损失部分缓解，但理论上仍存在遗漏。
- **SMACv2 场景**：在 Terran 10v10 上，MUTE（2M 步时）性能低于其他基线（4M 步时），因为 SMACv2 的随机性降低了通信的相对重要性，非通信方法也可达到较高胜率。
- **多智能体可扩展性**：Appendix K 显示agent数量增加时计算时间上升（20 agents 时 30ms/timestep），但性能略有下降，可能受环境难度影响而非方法瓶颈。需在更大规模场景进一步验证。
- **超参数敏感性**：λ 和 β 对性能-通信权衡敏感，默认值在多数场景表现好，但未见跨环境自动调参分析。
- **未报告 GPU 数量**：虽给出单卡推理时间，但训练阶段是否使用多卡未说明，可能影响复现。
- **应用限制**：方法依赖先训练全通信专家策略，若初始环境不允许全通信（如带宽永久受限），则需调整策略。

## Abstract 

While Large Language Model (LLM) agents demonstrate proficiency in static benchmarks, their deployment in real-world scenarios is hin-dered by the dynamic nature of user queries, tool sets, and interaction dynamics. To address this generalization gap, we formalize OpenA-gent (Tool-Use Agent in Open-World), a prob-lem setting characterized by distributional shifts across query, action, observation, and domain dimensions. To systematically diagnose its im-pact, we construct a controlled sandbox envi-ronment where we define fine-grained environ-mental shifts across a four-tier hierarchy, Percep-tion , Interaction , Reasoning , and Internalization ,and conduct a comprehensive series of experi-ments. Our analysis yields a series of key in-sights, demonstrating that agents trained via both Supervised Fine-Tuning(SFT) and Reinforcement Learning suffer from varying degrees of perfor-mance degradation when confronting open envi-ronmental shifts. Building on these insights, we propose Perturbation-Augmented Fine-Tuning, a disturbance-based intervention strategy for SFT that lays the foundation for enhancing agent ro-bustness and utility in realistic environments. Our code will be released at: https://github. com/LAMDA-NeSy/OpenAgent .

3.2. The OpenAgent Setting: Distributional Shifts 

Unlike distributional shifts in passive models, where each in-put is perturbed independently, tool-use agents face cascad-ing shifts along the agent-environment loop. A perturbation at step t may alter observation ot, which enters the history 

ht′ and conditions all subsequent decisions πθ (at′ | ht′ )

for t′ > t , causing errors to compound along the trajectory. As illustrated in Figure 2, OpenAgent covers query shifts (q → q′), action-space shifts including perturbed existing tools and newly introduced tools, observation shifts such as 

None or Error , and joint domain shifts where all interaction elements change together. We decompose this interactive shift space into four dimensions. 

Intent Shift in Query Space ( ∆Q). The intent distribution 

P (q) shifts across both semantic directives and linguistic surface forms. Unlike passive input perturbations, query misinterpretation in tool-use agents propagates through the trajectory: an initial misparse may seed a misaligned tool chain whose errors compound across subsequent steps. 

Structural Shift in Action Space ( ∆A). The tool space be-comes non-stationary ( Atest ̸ = Atrain ), spanning multiple levels of change: surface drift , which renames identifiers while preserving functionality; semantic conflict , which re-tains familiar names but alters documented behavior; and 

structural reconfiguration , which merges, splits, or inverts tool dependencies. The agent must distinguish cosmetic changes from fundamental reconfigurations and ground ac-tions in the current tool semantics. 

Dynamics Shift in Observation Space ( ∆O). The feed-back channel becomes non-stationary: test-time observa-tions may include return formats, error states, null values, or corrective instructions that are absent or rare in training. Unlike static outputs, these observations directly update the agent’s belief state and reshape the subsequent action space. Crucially, the agent must discriminate informative 

anomalies that require policy adjustment from terminal fail-ures that require task refusal, a distinction absent in passive settings where each input simply calls for a prediction. 

Compositional Domain Shift ( ∆D). This shift jointly changes Q, A, and O into a new domain while preserv-ing a latent problem-solving structure G. Despite distinct 3Can Agents Generalize to the Open World? Unveiling the Fragility of Static Training in Tool Use 

surface semantics and interfaces, the agent must transfer the underlying reasoning topology by re-grounding its nodes and dependencies in the target domain, rather than relying on domain-specific lexical or procedural patterns. 

## 4. Evaluation Tasks 

4.1. Experimental Setup Controlled Sandbox Environment. We utilize a syn-thetic sandbox environment to enable controlled probing of agent capabilities. Unlike real-world APIs which introduce uncontrollable transient noise, a sandbox allows us to pre-cisely isolate environmental variables and establish a pure closed environment baseline to measure open environment shifts. We synthesized a diverse set of tool-use tasks focused on simple geographical POI queries and calculation prob-lems to test reasoning boundaries, the full specifications of which are detailed in the Appendix A. 

Datasets and Protocols. The dataset comprises 6,050 training samples and 880 evaluation samples. To ensure rigorous evaluation, we enforced a strict separation between the datasets where no variation patterns from the test set appear during the training phase. This guarantees that the reported performance reflects genuine robustness rather than information leakage or pattern memorization. 

Training Implementation. We employ Qwen2.5-7B-Instruct (Qwen et al., 2024) as the backbone model. The SFT stage uses full-parameter fine-tuning, while the RL stage adopts the GRPO framework with sparse answers and format rewards. To further ensure a fair comparison with SFT, the RL environment provides only basic error codes upon failure without detailed guidance. We trained both models under comparable computational budgets until they achieved convergence in the closed environment as illus-trated in Figure 1. The Training details are provided in Appendix B. 

Metrics. We employ four metrics to assess performance across correctness, efficiency, robustness, and adaptability: • Tool Error Rate (TER): Measures robustness via the ratio of invalid invocations to total calls ( T ER =

Nerror /N total ). A lower TER indicates precise adher-ence. • Active Exploration Score (AES): Quantifies adaptability 

by penalizing “cognitive inertia” when tools fail. 

AES = e−(nold −1) (1) where nold is the count of repeated invalid attempts. A score of 1.0 signifies immediate adaptation. • Average Tool Chain Length (ATCL): Gauges efficiency 

by averaging tool invocations on solvable tasks. Lower values reflect concise reasoning topologies. • Refusal Rate (RR): Evaluates boundary awareness in un-solvable scenarios by measuring the ratio of correctly iden-tified unsolvable cases ( RR = Nref usal /N unsolvable ). A higher RR indicates superior active boundary detection. 

4.2. Tier 1: Perception Generalization 

In the context of agentic interaction, perception acts as the cognitive gateway grounding raw observations into action-able representations. To successfully initiate a task, an agent must traverse three representational alignments: decipher-ing unstructured user intents, aligning with rigid interface protocols, and comprehending functional tool definitions. We systematically evaluate the generalization boundaries of these capabilities through the following three dimensions. 

Instruction Robustness ( ∆Q). Real-world user instruc-tions are rarely canonical. We evaluate whether the agent can extract invariant intents from non-standard inputs by introducing two distinct types of variations. • Instruction Blurring: We simulate scenarios where user intent is obfuscated by syntactic variance (e.g., collo-quialisms, rhetorical questions) or semantic ambiguity (e.g., implicit references, omitted entities). This tests the model’s ability to reconstruct exact intents without relying on explicit keyword matching. • Information Redundancy: We test the robustness of the model’s attention mechanism against information over-load. By embedding the core instruction within irrelevant context or verbose background details, we evaluate the agent’s capability to filter out red herrings and lock onto functional constraints. 

Schema Adaptability ( ∆A). Agents often overfit to spe-cific API formats seen during training. In this experiment, we introduce protocol shifts by modifying the reserved keywords in the tool invocation schema. For instance, we alter the parameter dictionary key from the standard 

arguments to variants like args or parameters . This verifies whether the model comprehends the abstract con-cept of parameter passing or merely performs rigid template completion based on memorized structural tokens. 

Semantic Grounding ( ∆A). Current agents often rely on Symbolic Anchoring with rigid dependence on memorized identifiers rather than true understanding of functional defi-nitions. To decouple these factors, we investigate whether models can adapt to dynamic documentation shifts through a Drift-Trap-Erasure trilogy: • Symbolic Drift: We test tolerance to perturbations via two sub-variants: Noise Injection (appending non-semantic suffixes) and Synonymous Rewriting (using LLMs to paraphrase). This verifies generalization across semantically equivalent but symbolically distinct labels. • Semantic Trap: We engineer a conflict between sym-4Can Agents Generalize to the Open World? 4.3. Tier 2: Interaction Generalization 

Interaction generalization evaluates whether an agent can use environmental feedback to refine subsequent actions. In open environments, observations may indicate format changes, execution failures, or new operational constraints rather than merely confirming previous tool calls. We ex-amine three capabilities: Format Adaptability for parsing diverse observation structures, Error Correction for recover-ing from explicit or silent failures, and Dynamic Alignment 

for adjusting actions according to feedback-provided rules. 

Format Adaptability ( ∆O). Agents typically overfit to standard JSON formats for parsing observations. This ex-periment tests the fundamental capability of the agent to parse non-standard feedback. We introduce Format Shift by altering the structure of tool return values from structured JSON to unstructured formats such as Markdown tables or raw text logs. This evaluates whether the agent extracts key information based on semantic content rather than relying on rigid syntactic parsing templates. 

Error Correction ( ∆O). Open environments may expose agents to execution failures that interrupt the planned tra-jectory. We evaluate whether the agent can diagnose such failures and recover by revising its invocation plan or select-ing alternative tools. We consider two types of failure: • Error Return: The primary tool explicitly returns an error code or traceback message after invocation. This tests whether the agent can interpret explicit failure signals and switch to a redundant tool to resolve the blockage. • Null Return: The primary tool returns a null value or an empty string without any warning. This tests whether the agent can detect an implicit anomaly where the return is syntactically valid but semantically uninformative, and infer that corrective action is required. 

Dynamic Alignment ( ∆O). Real-world environments are non-stationary and may provide corrective instructions within feedback to steer the agent away from obsolete as-sumptions. We evaluate whether the agent can perform feedback-conditioned policy adjustment through two active guidance mechanisms: • Value Redirection: The environment notifies the agent that the valid parameter logic has shifted (e.g., “ID 1024 is deprecated, use ID 2048”). We verify if the agent can grasp this dynamic constraint and reissue the tool call with corrected values. • Tool Redirection: The environment signals that a spe-cific tool schema is obsolete and explicitly suggests an alternative one. This tests whether the agent can abandon its pre-planned trajectory and strictly adhere to the new operational manual provided in the feedback. 

4.4. Tier 3: Reasoning Generalization 

Beyond perception and interaction, Reasoning determines whether an agent can compose individual tool calls into a valid execution plan. Instead of triggering APIs sequentially according to memorized trajectories, a robust agent should infer the current rules governing tool behavior and construct an execution graph that satisfies the task dependencies. We evaluate this ability through two complementary dimensions: 

Rule Reasoning , which tests whether the agent can adapt to changed functional definitions, and Path Planning , which tests whether it can reorganize tool dependencies under altered execution structures. 

Rule Reasoning ( ∆A). Models may memorize training-time rules instead of performing deduction from the cur-rent tool description. We introduce a Calculation Shift by modifying the functional semantics of selected tools while keeping their interfaces unchanged. For example, a tool originally defined to return a value multiplied by 0.5 is redefined in the test setting to use a factor of 0.1. This ex-periment verifies whether the agent can derive the correct result from the updated rule description, or instead reverts to the memorized rule from the training distribution. 

Path Planning ( ∆A). This dimension evaluates whether the agent can construct valid execution graphs under changed tool dependencies. We test whether it genuinely reasons over the current causal structure among tools, rather than overfitting to fixed tool chains observed during training, through two complementary sub-experiments: • Trace Simplification: We test whether the agent can improve execution efficiency when a shortcut becomes available. Specifically, we introduce a composite tool that functionally replaces a multi-step tool chain used during training. The agent should recognize the new affordance and select the efficient composite tool, rather than rigidly following the previously memorized multi-step trajectory. • Logic Inversion: We test robustness to causal reversal in the tool dependency graph. At test time, we invert the documented information-acquisition order, for example by making the output of tool B a prerequisite for invok-ing tool A. This verifies whether the agent can resolve preconditions from the current dependency graph, or in-5Can Agents Generalize to the Open World? Unveiling the Fragility of Static Training in Tool Use

4.5. Tier 4: Internalization Generalization 

Internalization generalization evaluates whether an agent has learned task-level principles that go beyond local tool invocation patterns. At this level, a robust agent should not only execute valid tool-use trajectories, but also recognize when a task is infeasible under the current constraints and transfer abstract problem-solving structures across domains. We examine this through two capabilities: Active Refusal ,which tests solvability boundary recognition, and Domain Transfer , which tests whether learned reasoning topologies can be reused in semantically different environments. 

Active Refusal ( ∆Q ∪ ∆O). A robust agent should deter-mine whether a task is solvable given the available informa-tion and executable tools, rather than forcing a plausible but unsupported answer. We evaluate this boundary-awareness ability through two insufficient-condition scenarios: • Missing Condition: We remove necessary conditions from the user query through a human-verified process, making the task theoretically unsolvable due to informa-tion deficiency. This tests whether the agent can detect the logical incompleteness of the request and refuse to an-swer, instead of fabricating missing information to force completion. • Fatal Error: We simulate an unrecoverable failure where a necessary tool returns a fatal error and no alternative tool is available. This tests whether the agent can recognize the resulting operational deadlock and conclude that the task is currently unexecutable, rather than repeatedly retrying or hallucinating a successful result. 

Domain Transfer (∆D). This dimension evaluates whether the agent can transfer internalized problem-solving structure across domains. We introduce a Holistic Domain Shift , where user queries, tool definitions, and environmen-tal observations change jointly. For example, we migrate the testing environment from an address-query scenario to a medical-registration scenario. Although the surface se-mantics and interfaces change substantially, the underlying causal structure is preserved as a latent structural invariance. This experiment tests whether the agent can re-ground the same abstract reasoning topology, such as a search-select-confirm chain, in a novel target domain rather than overfit-ting to domain-specific lexical or procedural cues. 

## 5. Key Observations 

Synthesizing the results across the four tiers of generaliza-tion, we distill observations regarding the behavioral diver-gence between SFT and RL in open-world agentic tasks. 

> Tier 1: Tool Perception

RL achieves semantic grounding through interaction while SFT relies on brittle symbolic anchoring that degrades as fitting increases. As illustrated in Figure 4, we observe a consistent degra-dation pattern where robustness declines as training steps increase. Specifically, in the Semantic Trap and Identity Era-sure experiments, the SFT model displays a sharp increase in TER accompanied by a significant drop in accuracy. The dashed blue line indicates that SFT frequently invokes the wrong tools when surface-level names are perturbed. Con-versely, while RL also experiences some performance decay due to overfitting, it maintains a consistently lower error rate and higher accuracy retention compared to SFT. The RL agent demonstrates superior resistance to adversarial descriptions and effectively ignores misleading name cues. This disparity stems from SFT’s reliance on symbolic an-choring , where the model acts as a superficial pattern matcher that collapses once surface tokens are decoupled from their semantics. In contrast, the penalty-driven na-ture of RL necessitates semantic grounding , compelling the agent to scrutinize functional descriptions rather than memorizing rigid name-context associations. 

> Tier 2: Interaction with Guidance

RL leverages explicit guidance for dynamic policy adaptation, while SFT exhibits trajectory inertia and often hallucinates outcomes under corrective feedback. As illustrated in Figure 5, we observe a distinct behavioral divergence dependent on feedback clarity. Under ambigu-ous signals like null returns, both models exhibit perfor-mance degradation. Crucially, this failure mode follows a clear trend where the inability to adapt intensifies as train-ing progresses. As the models become more fitted to the training distribution, their capacity to actively explore under uncertainty progressively declines. A significant disparity emerges under explicit tool redirection. Qualitative case studies reveal that SFT tends to interpret observations as successful confirmations, even when the feedback explicitly redirects the agent. Consequently, it ignores the guidance and hallucinates successful observations to force the continu-ation of its pre-planned trajectory. In contrast, RL maintains high adaptability by parsing the instruction and migrating to the new tool immediately, and then continues reasoning. We hypothesize that this behavioral divergence stems from differences in training paradigms. SFT is optimized on clean demonstrations where observations consistently validate ac-tions. Tier 3: Reasoning Generalization 

RL demonstrates generalization in local adaptation, yet both paradigms suffer substantial degradation under global dependency inversion. As illustrated in Figure 6, RL exhibits superior generaliza-tion in local optimization tasks. In Trace Simplification ,the RL model achieves the lowest ATCL, indicating it suc-cessfully abandons the memorized training-time sequence to utilize efficient composite tools. Similarly, in Rule Rea-soning , RL outperforms SFT by dynamically adhering to updated calculation logic, whereas SFT shows a stronger tendency to regress to memorized training constants. However, a critical shared failure emerges in Logic Inver-sion . When the causal dependency between tools is reversed (e.g., shifting from A → B to B → A), both SFT and RL suffer precipitous performance drops. Qualitative anal-ysis in Appendix E reveals that despite the documentation explicitly stating the new prerequisite order, both models fre-quently hallucinate the existence of the training-set topology. This suggests that while RL generalizes well to local rule shifts, both paradigms overfit to the temporal order of action sequences, treating the dependency graph as a fixed action order rather than a dynamically specified logical structure. 

Tier 4: Boundary Awareness 

Both paradigms struggle with boundary awareness in unsolvable states, but exhibit distinct failure patterns: SFT often fails to perceive fatal feedback, whereas RL perceives the failure but still favors forced completion. As illustrated in Figure 7, both models struggle to identify unsolvable queries, leading to low refusal success rates when tools are unavailable. However, a fine-grained analysis of non-refusal cases under Fatal Error scenarios reveals that SFT and RL fail through qualitatively distinct mechanisms. Case studies in Appendix E.4.1 show that upon receiving an error, SFT directly outputs an unsupported answer without acknowledging the error at all, effectively treating the error observation as if it were a normal return value. ## 6. Perturbation-Augmented Fine-Tuning 

Our analysis reveals that SFT’s generalization decay stems from a distributional mismatch: standard SFT optimizes 

L = −E(h,a )∼D clean [log πθ (a|h)] , where all training his-tories h are drawn exclusively from successful execu-tions. This confines the learned policy to a narrow man-ifold Hclean that excludes error states, ambiguous feedback, and symbolic variations. To bridge this gap, we propose 

Perturbation-Augmented Fine-Tuning (PAFT) , a data intervention strategy that augments Dclean with controlled trajectory-level perturbations: 

DPAFT = (1 − α) Dclean ∪ α · G (Dclean ) (2) where G applies perturbation operators that expand the sup-port of h into regions containing anomalous observations, unsolvable states, and symbolic variants. 6.2. Experimental Results 

We evaluate PAFT on the most challenging tasks from each tier: Semantic Trap (Tier-1), Tool Redirection (Tier-2), Logic Inversion (Tier-3), and Fatal Error (Tier-4). For Tiers 1–3, we report the Accuracy Delta ( ∆) relative to the closed-set baseline; for Tier-4, we report the absolute RR. We set α = 0 .3 as the default perturbation ratio. 

Performance. As shown in the upper portion of Table 1, PAFT yields consistent improvements across all training stages and tiers. For early-stage models, PAFT transforms severe degradation into positive deltas across Tiers 1–3, indicating that perturbation-augmented training can preemp-tively prevent overfitting to clean trajectories. 

Ablation Study. The lower portion of Table 1 reveals two key findings. 

(1) Perturbation ratio. Increasing α from 0.2 to 0.3 yields clear gains across all tiers, while further increasing to 0.4 provides marginal or no additional benefit. We attribute this diminishing-return pattern to a trade-off between distri-butional coverage and signal quality: at low α, the model is exposed to too few perturbed trajectories to learn robust recovery behaviors; as α grows, the augmented data progres-sively covers more failure regions, but beyond a saturation point, an excessive proportion of perturbed samples dilutes the clean supervision signal and introduces conflicting gradi-ents between normal task completion and anomaly handling, leading to no further improvement or even slight regression. The result suggests that α=0 .3 achieves a near-optimal bal-ance between robustness and task performance. 

(2) Component contributions. EFP improves Tier-2 inter-action adaptability, as the injected observation anomalies teach the model to parse and react to unexpected feedback rather than executing blindly along pre-planned trajecto-ries. Notably, EFP contributes minimally to Tier-4 refusal, indicating that error adaptation and unsolvable-state recog-nition capture distinct capabilities. SBP almost exclusively restores Tier-4 refusal capability by introducing fatal error samples that break the implicit solvability assumption, while leaving other tiers largely unaffected, which suggests that, in our setting, boundary blindness is strongly tied to data coverage problem where the model never learns to refuse simply because Dclean contains no refusal demonstrations. SRP primarily enhances Tier-1 perception robustness and Tier-3 reasoning generalization, as symbolic perturbations force the model to attend to functional descriptions rather than surface-level identifiers, mitigating both symbolic an-choring and the associated downstream reasoning errors. This targeted effectiveness, where each component is neces-sary yet individually insufficient, confirms that PAFT is a structured, diagnosis-driven intervention rather than generic data augmentation.

In this paper, we formalize the OpenAgent setting and lever-age a controlled sandbox to diagnose the open-world adapt-ability of LLM agents. Our systemic analysis reveals that both SFT and RL paradigms exhibit vulnerabilities when facing non-stationary environmental shifts. ## Impact Statement 

This work studies robustness and failure modes of tool-use agents under open-world shifts. Its primary positive impact is to improve the reliability and safety of deployed agents by identifying conditions under which models ignore feedback, misuse tools, or hallucinate answers. Potential risks include misuse of stronger tool-use agents in automated decision-making systems. We mitigate this by emphasizing refusal behavior, boundary awareness, and controlled evaluation rather than unrestricted deployment.

A.1. Database 

We built our experimental database using public Point of Interest (POI) data from Beijing. Each POI entry has five key attributes: Phone Number, Address Name, Adname (Administrative Name), Coordinates (Latitude/Longitude), and Category. Specifically, all Address Names and Phone Numbers are replaced with unique, randomly generated alphanumeric strings and non-repeating number sequences, respectively. A.3. Tool Call Module 

We set up the different tool call modules for RL training and evaluation testing in two different ways: • For RL training, we use the Verl Tool framework. The core idea is to run the code sandbox as a service on the server and open an external port. This port receives tool names and corresponding parameters, then sends back the tool call results. During each RL training run, we use the terminator <tool call >to detect tool call commands. Specifically, when the model outputs this keyword, it stops generating text and triggers the tool detector. Consequently, the model must generalize its reasoning capabilities to unseen entities rather than recalling specific POI attributes seen during training. Regarding task complexity distribution, the dataset employs a stepped logical chain design covering a range from one to six steps; two-step tasks serve as the core backbone (accounting for 28.4%), while nearly 40% of the samples consist of high-difficulty reasoning tasks involving three or more steps, ensuring depth in the model’s learning of long-range tool invocation paths. In terms of tool and scenario distribution, the dataset exhibits a high degree of domain focus and balance: the core tool caller id lookup represents 24.4% of invocations, while ticket price calculation (adult/child cost) and geographic measurement (coords/distance) tools work in close coordination to form the foundation of task execution. This distribution is further corroborated by the source scenarios, where price-related (34.1%) and distance-related (28.4%) contexts dominate, providing the model with abundant material for numerical reasoning and logical alignment. Overall, this dataset covers basic attribute queries (e.g., POI names, phone numbers, and administrative divisions) and utilizes complex step distributions to force the model into semantic grounding rather than simple symbolic matching during extreme testing, such as anonymization or logical inversion. With its scientific tool sampling proportions and diverse business scenarios, this dataset serves as an ideal benchmark for training agents equipped with efficient path planning and robust generalization capabilities. ## B. Experimental Details 

We adopt Qwen2.5-7B-Instruct as the backbone model and perform full-parameter fine-tuning based on the LLaMA-Factory framework. The AdamW optimizer is employed, with the learning rate and batch size set to 3e-5 and 2, respectively. The training process runs for a total of 800 steps, with a model checkpoint saved every 200 steps. For the RL paradigm, training leverages the GRPO tool calling algorithm implemented on the Verl-Tool platform. Notably, we use the normal GRPO to keep comparison fairly. ## F. Real-World API Validation 

To provide direct empirical evidence, we replace the sandbox tool calculate distance by coords with a real-world driving-distance API provided by Amap 1, substituting both training and test data with real API traces, and retrain both SFT and RL models under identical protocols. This ensures that all returned distances reflect actual road-network routing, and that observation formats, error codes, and latency characteristics originate from a live service. Figure 10 presents the full results. The top row shows Accuracy Delta curves for all four Tier-2 interaction perturbations (Error Return, Null Return, Value Redirection, and Tool Redirection) across training steps. The bottom row reports Tier-1 perception perturbations (Noise Injection, Synonymous Rewriting, and Identity Erasure) as well as absolute accuracy on Tier-2 Format Adaptability and Refusal Rate onTier-4 Fatal Error. Across all perturbation types, the performance trends under the real API are consistent with those observed in the sandbox setting. Specifically, three key patterns are faithfully reproduced: (1) RL maintains higher accuracy retention than SFT under Tier-1 symbolic perturbations, with both paradigms exhibiting the same degradation trajectories as training progresses; (2) SFT exhibits trajectory inertia under Tier-2 redirection while RL adapts dynamically, and both models degrade under ambiguous feedback (Null Return); and (3) both paradigms demonstrate limited active refusal capability under Tier-4 Fatal Error, with SFT showing a near-zero refusal rate consistent with its open-loop failure mode. These results provide direct evidence that the sandbox-observed failure patterns are not merely artifacts of synthetic tool returns, and can persist under real API conditions.

## 摘要
尽管自主智能体的技能优化已受到关注，但现有方法依赖复杂流程。这留下了一个基本问题：什么构成了技能优化的最小可行流程，其中每个组件都有理论或经验必要性作为依据？我们通过零阶(Zero-Order, ZO)优化形式化技能优化，将经典对应方法(中心差分、信任域)映射到近期文献。注意到与经典ZO中盲目的数值扰动不同，技能轨迹作为可解释的调试反馈。基于Claude Code理念和PAC学习理论，我们建立了收敛与泛化的三个原则：基于文件系统的轨迹探索、共识属性挖掘、独立验证门控。消除冗余后，我们提出SkillOpt-Lite。它加速收敛并超越完整版SkillOpt：在GPT-5.5上提升LiveMath 8.8个百分点，在GPT-5.4-nano上提升25.4个百分点，使nano模型超越经SkillOpt优化的标准GPT-5.4。最终，我们将框架集成到VSCode Copilot等生产级编码智能体中，使开发者能够通过一行提示进化智能体技能。由于我们的框架将所有智能体组件视为标准可编辑代码，这一最小流程自然泛化到完整工具优化(HarnessOpt)。在SpreadsheetBench上，HarnessOpt使GPT-5.4-nano达到0.7758的准确率，超越运行标准流程的较大模型GPT-5.5(0.7620)。代码已开源：https://github.com/EvolvingLMMs-Lab/SkillOpt-Lite。

## Abstract
While skill optimization for autonomous agents has gained traction, existing methods rely on complex pipelines. This leaves a fundamental question unaddressed: What constitutes a minimal viable pipeline for skill optimization, where every component is justified by theory or empirical necessity? We formalize skill optimization via Zeroth-Order (ZO) optimization, mapping classical counterparts (central difference, trust regions) to recent literature. Noting that unlike blind numerical perturbations in classical ZO, skill trajectories serve as interpretable debugging feedback. Grounded in Claude Code philosophy and PAC learning, we establish three principles for convergence and generalization: file-system-based trajectory exploration, consensus attribute mining, and independent validation gating. Eliminating redundancies, we propose SkillOpt-Lite. It accelerates convergence and outperforms full SkillOpt: improving LiveMath by +8.8 points on GPT-5.5 and +25.4 points on GPT-5.4-nano, allowing the nano model to surpass standard GPT-5.4 optimized by SkillOpt. Finally, we integrate our framework into production coding agents like VSCode Copilot, enabling developers to evolve agent skills via one line of vibe. Because our framework treats all agent components simply as standard editable code, this minimal pipeline naturally generalizes to full harness optimization (HarnessOpt). On SpreadsheetBench, HarnessOpt enables GPT-5.4-nano to achieve 0.7758 accuracy, outperforming the larger GPT-5.5 running standard pipelines (0.7620). Code is available at https://github.com/EvolvingLMMs-Lab/SkillOpt-Lite.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：现有智能体技能优化方法（如 SkillOpt、Reflexion、Trace2Skill 等）取得了显著经验收益，但普遍存在管线复杂、组件冗余的问题。作者质疑：什么是技能优化**最小可行管线**（minimal viable pipeline），其中每个组件都必须有理论或经验必要性？
- **背景**：实际部署中，智能体能力不仅取决于基础大语言模型（LLM），还取决于操作框架（harness）和领域启发式（技能）。基础模型在推理时冻结，因此智能体工程常归结为优化技能文档——微小的文本变化能引发下游性能的非线性差异。为此，自动化技能优化已从开环自我反思演变为复杂算法管线（如 mini-batch 树合并、文本学习率调度、拒绝编辑缓冲区等）。本文尝试消除冗余，构建极简且高效的方案。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想

- **理论形式化**：将技能优化形式化为**零阶优化**（Zeroth-Order Optimization, ZO）问题。技能文档 \( s \) 是文本搜索空间 \( S_{\text{text}} \) 中的点，期望奖励 \( f(s) = \mathbb{E}_{z \sim \mathcal{D}}[R(H(M, z, s))] \)。由于离散和非可微，只能用零阶 Oracle 估计。现有方法（如单轨迹反思、对比分析、故障隔离编辑、编辑预算、拒绝缓冲区）分别对应零阶优化中的 1 点梯度估计、中心差分、坐标下降、信任域半径和控制变量。
- **关键概念分歧**：经典 ZO 依赖盲目数值扰动，而智能体技能优化本质是**语言介导的程序编译**——每次推出（rollout）产生可读的执行轨迹，作为可解释调试反馈。因此技能优化应从黑盒函数查询转变为随机编码问题。
- **三个设计原则**：
  1. **轨迹共识挖掘**（Consensus Mining）：基于 PAC 学习的期望平均稳定性，需避免对单次异常过拟合，应提取跨任务不变结构，作为 βexp 稳定算子。
  2. **独立验证门控**（Independent Validation Gating）：引入独立验证集可完全移除 βexp 项，保证泛化；验证集必须严格独立于训练集。
  3. **技能优化的“苦涩教训”**（Bitter Lesson）：随着基础模型扩展，为减慢更新速度而设计的复杂拓扑变得适得其反；赋予智能体原始文件系统工具直接检查日志文件，往往优于精心设计的基线。

### 关键技术细节

- **SkillOpt-Lite 管线**（核心贡献）：
  - **轨迹分阶段存储**：每批推出后，每个轨迹独立存储为磁盘上的文本文件（而非聚合为 mini-batch）。
  - **轨迹探索**：优化器模型使用文件系统工具（如 `list`, `read`）遍历目录，在固定 token 预算内选择高杠杆文件。
  - **共识挖掘与最小编辑**：读取选中的轨迹文件，发现跨任务的共同失败模式，生成紧凑的代码补丁（diff），遵循最小更新原则（类似信任域约束）。
  - **验证门控**：应用补丁得到候选技能库 \( \tilde{S} \)，在独立验证集上评估；若优于当前基线则接受，若优于历史最佳则覆盖生产的 `best_skill.md`。拒绝的更新记录到日志。
- **HarnessOpt 扩展**（自然泛化）：
  - 因为 SkillOpt-Lite 将所有智能体组件视为标准文件，只需在 IDE 中放宽文件路径限制，就能将相同三支柱结构应用到运行时代码库（harness）的修改。
  - 包含第 0 轮引导（用户审批配置提案）、自动连续进化阶段、三个循环不变性：允许列表约束、冒烟与验证门控、可逆进化（git 回滚、环境变量切换）。
- **命令行接口**：提供 VS Code 扩展，一行斜杠命令触发优化：
  ```/skillopt-loop rounds=10 batchsize=40 target=gpt5.4-nano [custom_requirements]```
  ```/harnessopt-loop rounds=2 batchsize=40 target=gpt5.4-nano skill=best_skill_nano.md```

### 公式与算法流程（文字说明）

- 形式化零阶 Oracle：\( f(s + \mu u) \)，其中 \( \mu \) 为步长类似物，\( u \) 为随机搜索方向。
- 稳定性系数 βexp 由 PAC 学习定义：\( \epsilon(S) \leq \hat{\epsilon}_D(S) + O(\beta_{\text{exp}} + \sqrt{\ln(1/\delta)/N}) \)。
- 验证集模型选择界：\( \epsilon(S_{\text{val}}) \leq \hat{\epsilon}_{\text{val}}(S_{\text{val}}) + O(\sqrt{\ln(1/\delta)/m}) \)，其中 βexp 被移除。

## 6. 主要结论与发现

- **最小可行管线有效**：SkillOpt-Lite 通过删除 mini-batch 池化、缓慢更新阻尼、拒绝缓冲区等复杂模块，在几乎所有任务上加速收敛并取得更高或相当的性能。尤其是在推理密集型任务（LiveMath, Spreadsheet）上优势明显。
- **零阶优化视角有价值**：将技能优化映射为 ZO 优化，揭示了现有方法如何对应经典 ZO 组件，且指出技能轨迹作为程序编译反馈这一关键区别。
- **“一切皆文件”哲学优越**：赋予智能体直接操作文件系统工具的权限，替代精心设计的数学拓扑，能实现更快更好的优化。
- **HarnessOpt 实现小模型逆袭**：联合优化 harness 与 skill，使 GPT-5.4-nano 在 SpreadsheetBench 上超越标准 harness 下 GPT-5.5 + SkillOpt 的结果，展示了环境协同设计的潜力。
- **验证门控至关重要**：单批无验证的修改（图 2 中的 Copilot）可能在某些任务退步，独立验证集有效防止退化。

## 7. 优点

- **理论驱动**：将 ZO 优化、PAC 学习稳定性与系统设计结合，为去除冗余组件提供了理论依据（βexp 移除、独立验证界）。
- **极简设计**：SkillOpt-Lite 仅保留文件系统探索、共识挖掘、验证门控三个组件，极大降低实现和调试复杂度，可直接嵌入开发环境（VS Code 扩展）。
- **通用性**：框架不限于技能优化，自然推广到 harness 优化（HarnessOpt），且有望扩展到模型训练阶段（多模态预训练数据缩放等）。
- **实用性**：一行 slash 命令启动优化，适合开发者日常使用；开源代码降低复现门槛。
- **实验结果有说服力**：在多个模型尺度上一致性提升，HarnessOpt 展示了显著的模型能力逆袭（小模型超越大模型），突出了环境工程的重要性。

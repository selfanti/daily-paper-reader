## 摘要
推测执行通过使用更小、更便宜的模型在环境空闲时预测并预启动下一步，从而加速LLM智能体。然而，现有的推测器是无状态的，会丢弃任务间的所有信息，阻止预测质量随着经验提升。我们为推测器配备了三种在线记忆系统，从过去的智能体轨迹中学习：对比转移表，跟踪动作序列统计；情景记忆，检索上下文相似的片段；以及困惑追踪器，抑制重复错误。我们在六个基准测试上评估了这种方法，涵盖三种推测类型：动作预测、观察预测和链式预测。记忆增强推测在动作预测上相对准确性提升19-39％，在具有重复动作空间的观察预测任务上提升高达2.5倍。这些增益随着记忆积累而持续增长，并泛化到不同成本的推测器模型。所有推测都是无损的，因为在空闲时间运行，零增加墙钟成本，并且智能体的轨迹与非推测执行相同。

## Abstract
Speculative execution accelerates LLM agents by using a smaller, cheaper model to predict and pre-launch the next step while the environment is idle. However, existing speculators are stateless and discard all information between tasks, preventing prediction quality from improving with experience. We equip the speculator with three online memory systems that learn from past agent trajectories: a contrastive transition table tracking action-sequence statistics, an episodic memory retrieving contextually similar segments, and a confusion tracker suppressing recurring errors. We evaluate this approach on six benchmarks spanning three speculation types: action prediction, observation prediction, and chained prediction. Memory-augmented speculation yields a 19--39\% relative accuracy improvement on action prediction and up to a $2.5\times$ increase on observation prediction tasks with repetitive action spaces. These gains grow continuously as memory accumulates and generalize across speculator models of varying cost. All speculation is lossless because it runs during idle time at zero added wall-clock cost, and the actor's trajectory is identical to non-speculative execution.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）智能体在与外部环境交互时，每一步都包含 LLM 推理延迟和环境执行延迟。为了加速，已有工作提出**推测执行**（speculative execution）：使用一个更小、更便宜的推测模型在环境空闲时预测智能体的下一步动作并预启动。然而，现有的推测器是**无状态**的，每次任务独立运行，丢弃所有历史信息，导致预测质量不随经验提升。例如，一个处理了 500 次客户交互的推测器与从未处理过的推测器做出相同的预测。
- **整体含义**：本文旨在通过为推测器配备**结构化记忆**，使其能够从过去的智能体轨迹中在线学习，从而持续提高预测准确率，实现该猜测过程**无损**（预测错误直接丢弃，不影响智能体的最终行为和轨迹）且零额外墙钟成本，最终加速整个智能体系统。

## 2. 方法论

- **核心思想**：在原有推测执行框架的基础上，为推测器增加三种互补的在线记忆系统，这些记忆从已完成的智能体轨迹中在线更新，并作为结构化的上下文注入推测器的提示中。
- **三种记忆系统**：
  - **对比转移表（Contrastive Transition Table）**：记录从当前动作到下一动作的转移统计，包括成功/失败次数和参数签名。提供转移置信度和成功率，帮助推测器优先选择高概率、高成功率的后续动作。
  - **情景记忆（Episodic Memory）**：存储完整任务片段（包括上下文、动作列表、观察、结果等）。在猜测时，用当前上下文嵌入检索最相似的过去片段作为示例。同时记录**猜测失败片段**，让推测器学习自己的历史错误模式。
  - **困惑追踪器（Confusion Tracker）**：记录猜测验证后（预测动作类型 vs. 实际动作类型）的重复错误模式。当某模式出现 ≥3 次时，作为硬约束加入提示，明确指示推测器避免该动作类型。
- **推测类型**：根据延迟出现在智能体-环境循环的不同位置，定义了三种类型：动作预测（Type 1）、观察预测（Type 2）、链式预测（Type 3，预测观察+下一步动作）。
- **无损性**：预启动工作仅限于无副作用的操作（LLM 调用、只读 API 查询、页面抓取等），写操作被排除。猜测错误时丢弃预启动结果，智能体轨迹与非推测执行完全相同。
- **运行时机**：所有记忆更新在任务完成后在线进行；猜测 API 调用在环境空闲时间运行，墙钟成本完全隐藏。

## 3. 实验设计

- **数据集/场景**：覆盖六个基准测试，按推测类型分组：
  - Type 1（动作预测）：WebArena（网页导航，812任务）、VisualWebArena (VWA)（视觉网页导航，910任务）
  - Type 2（观察预测）：ALFWorld（家庭任务，134 episode）、PDDL（规划任务，60任务）
  - Type 3（链式预测）：τ²-bench（客户服务，164任务）、HotpotQA（多跳问答，300任务）
- **Benchmark 指标**：Type 1 使用只读动作完全匹配准确率；Type 2 使用观察完全匹配准确率；Type 3 使用动作完全匹配准确率（决定是否可预启动）。
- **对比方法**：共7种渐进设置：
  1. Stateless（无记忆）
  2. Confusion（仅困惑追踪器）
  3. Table（+转移表）
  4. Episodic（仅情景记忆，替换表）
  5. Table+Episodic（表+情景）
  6. Episodic+Miss（情景+猜测失败片段）
  7. Full（全部组合）
- **模型**：演员模型：GPT-5.4（ALFWorld, PDDL, τ²-bench, HotpotQA）或 GPT-5-mini（WebArena, VWA）；主要推测器：GPT-4.1-mini；还测试了 GPT-5.4-mini 和 GPT-5.4-nano。使用 \(k=1,2,3\) 个并行推测器，采用 best-of-k 策略。

- **所有基准上记忆增强推测均优于无状态基线**：动作预测（Type 1）准确率相对提升 19-39%（绝对提升 3.7-4.9pp）；观察预测（Type 2）提升高达 2.5 倍（绝对提升 16.2-23.7pp）；链式预测（Type 3）提升 7.0-7.2pp。

- **记忆改善跨模型通用**：在 GPT-4.1-mini、GPT-5.4-mini、GPT-5.4-nano 上，记忆带来的提升幅度相似（Stateless→best 的增量在相同基准上基本一致）。
- **最佳记忆配置因领域而异**：Type 2 任务（ALFWorld, PDDL）中 Full 或 Episodic+Miss 最佳；Type 1 中 Episodic 或 Episodic+Miss 最佳；Type 3 中 Full 或 Table+Episodic 最佳。简单拼接所有记忆不一定最优，有时会引入冗余/冲突。
- **在线验证**：在 HotpotQA 上，实时加速达 1.05 倍（节省 892s/19024s）；ALFWorld 上在环境延迟 7s 时加速 1.119 倍。节省时间受推测器自身延迟限制（中位数 2.7s），与环境延迟窗口大小密切相关。

## 7. 优点

- **无损零开销**：推测运行在空闲时间，预测错误不影响智能体轨迹；所有记忆更新不产生额外推理延迟（规则启发式 + 快速嵌入检索）。
- **记忆设计互补性强**：对比转移表捕捉总体统计模式，情景记忆提供上下文相关示例，困惑追踪器提供即时硬约束。三者组合效果显著。
- **广泛的实验覆盖**：覆盖网页、体现、规划、客户服务、问答等多种任务类型，验证了方法的通用性。
- **深入的分析**：除主结果外，还给出了步骤深度、成本效益、排序鲁棒性、环境延迟敏感性等细致分析，增加了结论的可信度。
- **模型无关性**：在不同能力/成本的推测器模型上效果一致，说明方法的普适性。

## 8. 不足与局限

- **主要误差来源**：网页导航基准（WebArena, VWA）中元素索引预测是瓶颈。元素索引是页面特定的、动态变化的，推测器无法访问目标页面的 DOM，导致准确率有限（绝对命中率仅 20-40%）。
- **记忆组合干扰**：全记忆设置在某些任务上不如简单设置（如 VWA 上 Table+Episodic 低于纯 Episodic），表示冗余或冲突信息可能误导推测器。作者指出未来可用轻量分类器选择性路由记忆。
- **推测器方差**：在 Type 3 基准上，不同排序间的全记忆准确率波动 1.14-1.23pp（表7），表明对任务顺序有一定敏感性。
- **Replay 评估局限**：回放评估可能无法完全捕捉在线环境中的真实反馈循环（正确猜测加速后可能改变后续上下文）。作者虽用 live 验证部分弥补，但未在全部基准上做在线实验。
- **未报告训练算力**：所有实验基于 API 调用，没有本地训练，因此无法评估可复现性所需的硬件资源。
- **应用限制**：方法依赖只读操作的可预启动性；写操作不能预启动。此外，记忆存储可能包含用户隐私信息，需要数据保留和匿名化策略。

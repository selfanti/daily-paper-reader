## 摘要
LLM智能体的实际任务绩效既受其模型周围的装备影响，也受冻结模型本身的影响：包括提示、注入知识、运行时控制和配置。在部署中，装备通常是唯一可用的杠杆，因此自动改进它是提高性能而不触及权重的自然方式。困难之处不在于生成变更，而在于知道哪些变更真正有帮助。自生成反馈是有噪声的，表面上的增益可能是测量伪影或仅仅过度拟合了其调优任务的编辑。我们提出了一种自进化智能体装备框架，它将提出变更与归因变更分开：语言模型诊断失败并提出补丁，而所有采样、测量和显著性检验由确定性代码负责，因此每个被归因的改进在构造上都是可信的。补丁填充了一个门控、分类的质-量多样性档案（GSME），其键是编辑所解决的病理（位置 x 原因），而不是它修复的任务，这是一种反过拟合的归纳偏差；泛化能力在进化后仅在密封测试上评分。在七个领域中使用冻结的开放权重模型，装备经过训练选择并在密封测试上评分一次；其归因增益为+9到+15.5个百分点，并保留了训练增益的86-147%，证据表明它们泛化而非过拟合。获胜补丁追踪模型的主要病理，而非其大小或家族：改变模型可能会改变病理和补丁，而相同的病理-补丁匹配在两个模型家族中重复出现。转移的是诊断-归因循环，而非任何特定装备。

## Abstract
An LLM agent's real-task performance is shaped as much by the harness around its model as by the frozen model itself: its prompts, injected knowledge, runtime control, and configuration. In deployment the harness is often the only lever available, so improving it automatically is the natural way to raise performance without touching the weights. The hard part is not generating changes but knowing which one truly helped. Self-generated feedback is noisy, and an apparent gain can be a measurement artifact or an edit that merely overfits the tasks it was tuned on. We present a self-evolving agent-harness framework that separates proposing changes from crediting them: a language model diagnoses failures and proposes patches, while all sampling, measurement, and significance testing are owned by deterministic code, so every credited improvement is trustworthy by construction. Patches populate a gated, categorical quality-diversity archive (GSME) keyed on the (WHERE x WHY) pathology an edit addresses rather than the tasks it fixes, an anti-overfitting inductive bias; generalization is measured on a sealed test scored only after evolution. Across seven domains with a frozen open-weight model, the harness is train-selected and scored once on a sealed test; its credited gains there are +9 to +15.5pp and retain 86-147% of the training gain, evidence they generalize rather than overfit. The winning patch tracks the model's dominant pathology, not its size or family: changing the model can change the pathology and the patch, while the same pathology-to-patch match recurs across two model families. What transfers is the diagnose-and-credit loop, not any specific harness.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：LLM 智能体的实际任务性能不仅取决于冻结的模型权重，还很大程度上取决于其外部**装备（harness）**——包括系统提示、注入知识、运行时控制回路、工具定义和配置。在部署中，模型权重通常是固定的（租赁、闭源或再训练成本过高），因此装备是唯一可调整的杠杆。
- **核心问题**：如何让 LLM 智能体**自动改进自身的装备**？困难不在于生成变更（LLM 可以无限生成），而在于**判断哪些变更真正有帮助**——自生成反馈噪声大（单次评估波动、沙箱不稳定、验证器超时），训练集上的增益可能只是过拟合。
- **研究动机**：现有工作（2026年的自进化编码装备研究）通常用多轮峰值通过率来归因增益，缺少统计显著性验证和保持测试集隔离，导致无法区分真实改进与噪声/过拟合。论文旨在建立一个**可信、可审计、可泛化**的自进化框架。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
**分离提出（proposing）与归因（crediting）**：
- **语言模型（evolver）** 负责语义步骤：读取失败轨迹、诊断病理（where × why）、设计并编写补丁。
- **确定性代码** 负责所有定量判断：采样、测量、显著性检验、归档、回滚。语言模型从不执行算术或统计计算，从而确保每个被归因的改进**在构造上可信**。

### 关键技术细节

#### ２.1 系统架构（图1）
- **任务智能体（Task Agent）**：固定权重模型 M，运行训练任务。
- **进化器（Evolver）**：更强的模型 D（本文用 Claude Opus 4.8），诊断失败轨迹、生成补丁。
- **三个门控**：有效性门（环境健康检查，重试基础设施失败）、激活门（补丁必须实际触发）、配对-2σ 显著性门（per-task 配对差异的 z 检验，z ≥ 1.96）。
- **密封测试**：进化过程中从不看到测试集，进化结束后仅评分一次，防止任何信号泄露。

#### ２.2 Gated Semantic MAP-Elites (GSME) 档案
- **描述符**：每个补丁被放入分类档案，索引是 (where × why) **病理**，而非修复的任务集。where 是四个杠杆（prompt, knowledge, runtime, config）；why 是 LLM 诊断的失败病理（如思维逃逸、过早终结等）。这种以病理为键的设计是一种**反过拟合归纳偏差**。
- **档案结构**：每个细胞只有一个门控精英（gated elite），跨细胞重组允许堆叠补丁。选择采用质量偏向选择（扩展当前全局最佳），而非均匀照亮所有细胞，以节省预算。
- **进化树**：基于 git，每个候选是一个分支+提交，记录完整谱系和激活记录，保证可审计和去重。

- **三层成本递增评估**：
  1. **探针（probe）**：约15-30个锚定任务，K=1，快速筛除明显差的候选。
  2. **估计器（estimator）**：完整训练集的分层子集，K=3，用于轮次间导航。
  3. **完整集（full set）**：训练集全量 K=3，产生冷启动基线和最终归因数字（从不用于子集标题）。

## 7. 优点：方法或实验设计上的亮点

1. **分离提出与归因**：从根本上解决了自反馈噪声和虚假增益问题，使每个归因改进在构造上可信。

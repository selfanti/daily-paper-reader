## 摘要
将推理与工具使用交织在一起的语言智能体会随着推理链变长而急剧退化，即使每一步都很简单。我们将此归因于上下文稀释：智能体的调查状态（已确认的内容、怀疑的内容以及仍需完成的任务）仅隐式地存在于不断增长的上下文窗口中，早期的发现被后来的检索掩埋。我们提出SLEUTH，通过结构化的认知工作记忆使这种状态显式化并可操作：智能体维护基于来源的已确认事实、按证据排序的活跃假设以及直接驱动其下一步行动的开放问题。在五个多跳基准测试和五个既定基线上，SLEUTH的优势随难度增加而增长，从HotpotQA上的+5分到4跳链上的+11分，且无需多轮迭代即可超越Reflexion。通过分析剩余差距所在，我们识别出证据充分性问题：智能体常常找到答案但未能承诺，将预算浪费在不必要的验证上。一个轻量级承诺触发器解决了这个问题，但仅在智能体已维护结构化状态时有效：对非结构化智能体应用相同的触发器没有任何改进，从而将组织化的认知状态隔离为实现有效承诺的必要条件。最后，在较弱模型上强制执行协议遵循，在最难问题上可获得高达+19分的提升，表明组织推理的方式，而非原始模型能力，是缩放多跳推理的活跃成分。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：语言智能体（如 ReAct）在需要多跳推理（multi-hop reasoning）时，随着推理链长度增加（如从2跳到4跳），性能急剧下降，即使每个子步骤本身很简单。作者将这种退化归因于**上下文稀释（context dilution）**：智能体的“调查状态”（已确认了什么、怀疑什么、还需要什么）仅隐式存在于不断增长的上下文窗口中，早期发现被后续检索结果淹没，导致信息丢失和决策瘫痪。
- **研究意义**：探索如何通过显式组织推理状态来扩展多跳推理能力，而非依赖更大的模型或更多的测试时计算。这为语言智能体设计提供了新的设计维度——推理组织方式可能比原始模型能力更重要。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：受人类侦探、诊断医生的启发，智能体应维护结构化的**认知工作记忆（Epistemic Working Memory）**，将推理状态分解为三个显式组件：已确认事实（Confirmed Facts）、活跃假设（Active Hypotheses）、开放问题（Open Questions）。每个组件在每一轮都被输出和更新，从而实现显式的推理轨迹。
- **SLEUTH 框架**（Structured Layered Evidence-Updated Tracking of Hypotheses）：
  - **工作记忆结构**（每轮 t）：
    - **Confirmed Facts (Ft)**：从工具输出中提取的已验证语句，每个事实带有来源引用（如 `[F1] Von Miller won Super Bowl 50 MVP [1]`），单调累积，永不撤回。
    - **Active Hypotheses (Ht)**：候选答案，按证据支持排序，每个假设与支持/矛盾的事实及置信度（高/中/低）关联。
    - **Open Questions (Qt)**：尚未解决的不确定性，每个问题附带建议的下一步动作和优先级。最高优先级问题直接决定下一个工具调用。
  - **更新协议**：每轮工具调用后，执行四步：（1）提取新事实；（2）更新假设的置信度；（3）更新问题列表（标记已解决，生成新问题）；（4）从最高优先级问题选择下一个动作。当没有开放问题且存在高置信度假设时，调用 `finish()`。
  - **承诺机制（Nudge）**：解决“证据充分性问题”——智能体找到答案但过度验证，耗尽预算而不承诺。当轮数 ≥ α*T（T为总预算，α=0.7）且已确认事实数 ≥ τ（τ=2）时，触发一条合成消息，强制从当前事实和假设综合答案，不再搜索。
- **实现**：完全通过提示工程（system prompt）实现，无需微调或架构修改。所有工具（search、lookup、finish）与基线保持一致，仅系统提示不同。对于指令遵循能力较弱的模型，通过提示适配和运行时检查（拒绝不合规响应并重试）来强制协议遵守。

## 6. 论文的主要结论与发现

- **SLEUTH 在所有基准上取得最高 EM，优势随难度增加**：HotpotQA +4.7，2/3/4跳 MuSiQue 分别 +11.9/+10.4/+10.9，2Wiki +2.8，且超过 Reflexion 而不能多轮次。
- **两个失效模式被分离**：信息丢失（被上下文稀释主导）被结构化事实追踪解决；信息瘫痪（证据充分但无法承诺）被承诺机制解决。后者是 4 跳上超时的主要原因，而 nudge 只在有结构化状态时才有效。
- **承诺机制（nudge）本身不足**：将相同 nudge 应用于 ReAct 无改进（+0.2），说明组织化的认知状态是必要条件。
- **协议遵守比模型能力更重要**：对较弱模型 GLM-5 强制遵守工作记忆结构，在最难题上恢复 +19 分，甚至超过较强模型 Sonnet 运行非结构化 ReAct（48.9% vs 42.0%）。
- **证据充分性问题**：智能体找到答案但不承诺，92% 的超时案例中包含黄金答案。这是首次在语言智能体研究中被明确识别和解决。

## 7. 优点

- **方法轻量且通用**：完全通过提示实现，无需训练、搜索或多轮次，可立即应用于任何指令遵循模型。
- **创新性强**：将认知工作记忆的心理学概念系统化为语言智能体的具体结构，并证明其必要性。
- **分析深入**：不仅报告整体性能，还通过错误分解、条件准确率、超时率、收敛曲线等揭示内在机制。
- **充分的消融与隔离**：从组件、承诺时机、模型、协议遵守四个层面逐一验证，结论可靠。
- **跨模型泛化**：在 Claude Sonnet 和 GLM-5 上均取得一致增益。
- **可解释性**：每轮输出完整的认知状态，使推理过程透明，便于调试。

## 8. 不足与局限

- **实验覆盖**：限于封闭域（给定段落池）。未在开放域、时间敏感或矛盾来源的场景下验证。承诺阈值 α=0.7 可能在任务结构差异大的场景下需要重新调整。
- **单调事实策略的潜在风险**：即使某些事实在上下文中具有误导性，SLEUTH 也不会撤回，可能导致错误固化。论文中列举了一个例子（Samuel Brannan 在旧金山 vs. 萨克拉门托）。作者提出了一些扩展思路但未实现。
- **令牌开销**：在容易问题上 token 使用量增加 42%（价格加权 1.50x），虽然难问题上较少（29%/1.33x），但仍比 ReAct 高。不过这与硬问题上的性能增益相比是值得的。
- **依赖足够预算**：推荐预算至少为所需推理深度的 2.5 倍以保证 nudge 有效，在极紧预算下可能不如其他方法灵活（但作者展示了在10轮预算下仍有 +8.6 优势）。
- **需要指令遵循能力较好**：在弱模型上需额外运行时强制才能保持一致性，虽然提示适应+检查可以解决，但增加了工程复杂度。
- **未与基于训练的方法对比**：如 RAG-Star、Search-R1 等需要微调的方法，这些方法可能提供互补增益，但 SLEUTH 的设计目标是无需训练的即用方案。

## 摘要
大型语言模型（LLM）智能体越来越依赖外部记忆系统来跨任务积累经验。然而，几乎所有现有方法，从图结构记忆到反思性洞察存储，都通过固定的、手工设计的启发式规则访问记忆。我们认为这种静态的记忆观是智能体学习的核心瓶颈，因为最优的记忆行为从根本上依赖于上下文。任务早期阶段，由于记忆稀疏，最小化检索更有益；重复的目标类型受益于计划重用而非通用的最近邻查找；陷入困境的智能体受益于使用替代查询重新检索；在长任务流中，记忆本身必须被整合和修剪以保持有用。我们提出了记忆作为受控过程（MemCon），一个将记忆操作建模为马尔可夫决策过程的框架，并学习一个在线策略，自适应地决定何时、检索什么以及检索多少，何时注入精简计划，以及何时整合或遗忘。MemCon是后端无关的：它封装任何现有的记忆实现，从逐任务二元反馈中学习，无需预训练也无需额外的LLM调用，并使用一个轻量级的表格型上下文赌博机与UCB探索，在几十个任务内收敛。在6个基准测试、3个智能体框架和3个LLM骨干网络上，MemCon在任务成功率上持续优于多个记忆基线，最高提升15.2个百分点，同时将令牌消耗减少5%-20%。

## Abstract
Large Language Model (LLM) agents increasingly rely on external memory systems to accumulate experience across tasks. Yet nearly all existing approaches, from graph-structured memories to reflective insight stores, access memory through fixed, hand-designed heuristics. We argue that this static view of memory is a core bottleneck for agentic learning because optimal memory behavior is fundamentally context-dependent. The early stages of the tasks, benefit from minimal retrieval because memory is sparse; recurring goal types benefit from plan reuse rather than generic nearest-neighbor lookup; stuck agents benefit from re-retrieval with alternative queries; and across long task streams, the memory store itself must be consolidated and pruned to remain useful. We present Memory as a Controlled Process (MemCon), a framework that models memory operations as a Markov Decision Process and learns an online policy that adaptively decides when, what, and how much to retrieve, when to inject a distilled plan, and when to consolidate or forget. MemCon is backend-agnostic: it wraps any existing memory implementation, learns from task-by-task binary feedback with no pretraining and no additional LLM calls, and uses a lightweight tabular contextual bandit with UCB exploration that converges within tens of tasks. Across 6 benchmarks, 3 agent frameworks, and 3 LLM backbones, MemCon consistently outperforms multiple memory baselines by up to 15.2 points in task success while reducing token consumption by 5--20%.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：现有LLM智能体依赖外部记忆系统，但几乎所有方法（如图结构记忆、反思洞察存储）都采用**固定的、手工设计的启发式规则**进行记忆访问（如固定top-k检索、固定查询模板），无法适应任务上下文的变化。  
- **关键瓶颈**：最优记忆行为是上下文相关的——早期任务应减少检索，重复目标需计划复用，卡壳时应重新检索，长时间任务流需整合和遗忘。固定规则在这些场景下都会失配：检索过多则膨胀上下文并增加成本，检索过少则丢失关键信息。  
- **总体意义**：提出MemCon框架，将记忆管理从静态流水线转变为**可学习的决策过程**，使智能体能够自适应调整记忆操作，从而显著提升任务成功率并降低令牌消耗。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
将记忆操作建模为一个**马尔可夫决策过程（Memory MDP）**，在线学习一个轻量级策略，动态决定何时、检索什么、检索多少，以及何时注入计划、重新检索、合并或遗忘。

### 2.2 关键技术细节
- **Memory MDP 定义**：
  - **状态 `s`**：融合任务进度（目标类型、步数阶段、是否卡壳、已访问位置）和记忆状态（记忆大小、计划是否可用、学习阶段）。离散化为哈希键 `ϕ(s)`（约几百个状态）。
  - **动作 `a`**：包含 `{RETRIEVE, PLAN INJECT, RE-RETRIEVE, CONSOLIDATE, FORGET, NOOP}`，每个动作带有参数（top-k, insight_k, hop）。共9个具体动作。
  - **奖励 `r`**：任务结束后根据成功/失败和效率计算：`r = r_succ * 1[success] + λ * max(0, 1 - Ti/T_max) - r_fail * 1[failure]`。
  - **转移**：由LLM代理驱动的环境转换，控制器仅接收终局奖励，因此整个MDP被视为**上下文赌博机（contextual bandit）**。

- **在线策略学习**：
  - **UCB选择**：`a_t = argmax_a [ Q(ϕ(s_t), a) + c * sqrt( ln N(ϕ(s_t)) / N_a(ϕ(s_t)) ) ]`。
  - **热启动先验**：为各动作赋值合理初始Q值（如RETRIEVE +0.5，NOOP -0.2），加速早期探索。
  - **反向折扣信用分配**：任务结束后，按时间远近用折扣因子γ更新每个状态-动作对的Q值：`Q(ϕ_j, a_j) ← Q(ϕ_j, a_j) + α [ γ^{|ep| - j - 1} * r_i - Q(ϕ_j, a_j) ]`。
  - **零额外LLM调用**：策略查询是毫秒级表格查找，无需二次调用LLM。

- **后端无关包装器**：MemCon作为一个薄封装层，拦截任何记忆后端的`retrieve`和`store`接口，决定如何调用参数。支持向量存储、技能库、图记忆等所有主流后端。

- **增强操作**：
  - **通用计划注入（PLAN INJECT）**：成功任务的动作序列被泛化（具体名称替换为占位符），存储在JSON索引中，当同类任务出现时预置到检索结果前。
  - **目标分解**：对需要相同原语重复多次的复合任务（如“放两个手机在桌上”），注入“先完成对象1的全部步骤，再重复对象2”的提示。

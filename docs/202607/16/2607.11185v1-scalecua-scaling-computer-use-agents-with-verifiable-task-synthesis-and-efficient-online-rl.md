## 摘要
计算机使用代理（CUA）正成为通过视觉感知和GUI执行自动化复杂数字工作流的强大接口。带可验证奖励的在线强化学习（RLVR）已成为扩展其能力的关键方向。然而，这一范式受到可验证数据稀缺和在线RL效率低下的瓶颈制约。为突破这些障碍，我们提出ScaleCUA，一个统一的框架，通过可验证任务合成和高效训练来扩展CUA的在线RL。在数据层面，我们设计了VeriGen，一个端到端框架，通过迭代的Docker交互和多智能体反馈循环生成可验证的RL任务。通过共享的Docker交互探针扩展到100多个并发智能体工作器，该流程产生了24000多个可验证任务和近3000个高质量RL任务。为最大化样本效率，我们提出Frontier Sampling，追踪每个任务的能力并将展开分配到当前学习前沿。在训练方面，我们进一步设计了Visual Context Segmentation，一个在最近视觉上下文上的滑动窗口，平衡展开和训练引擎的压力，相比逐步分解实现了2.83倍的训练加速。通过这些方法，ScaleCUA在OSWorld上达到68.7%，在ScienceBoard上达到54.0%，在开源计算机使用代理中树立了新的最先进性能。代码、模型和数据集可在https://github.com/THUDM/SCALE-CUA获取。

## Abstract
Computer use agents (CUAs) are emerging as a powerful interface for automating complex digital workflows through visual perception and GUI execution. Online reinforcement learning with verifiable rewards (RLVR) has emerged as a key direction for scaling their capabilities. However, this paradigm is bottlenecked by verifiable data scarcity and online RL inefficiency. To break these barriers, we introduce ScaleCUA, a unified framework that scales online RL for CUAs via verifiable task synthesis and efficient training. At the data level, we design VeriGen, an end-to-end framework for generating verifiable RL tasks through iterative docker interactions and a multi-agent feedback loop. Scaled to 100+ concurrent agent workers via a shared docker interaction probe, this pipeline produces 24K+ verifiable tasks and nearly 3K high-quality RL tasks. To maximize sample efficiency, we propose Frontier Sampling, which tracks per-task capability and allocates rollouts to the current learning frontier. On the training side, we further design Visual Context Segmentation, a sliding window over recent visual context that balances rollout and training-engine pressure, yielding a 2.83x training speedup over step-wise decomposition. Together, ScaleCUA achieves 68.7% on OSWorld and 54.0% on ScienceBoard, establishing new state-of-the-art performance among open-source computer use agents. Code, models, and datasets are available at https://github.com/THUDM/SCALE-CUA.

## 2. 论文提出的方法论

### 核心思想
- 通过自动化生成大量带有确定性评判函数（judge）的可验证GUI任务，解决数据稀缺问题。
- 通过智能任务采样和高效的多轮训练策略，最大化样本效率和训练速度。

### 关键技术细节
- **VeriGen（可验证任务生成）**：
  - 端到端框架，接受环境知识文档和模型轨迹作为输入。
  - 使用**三个LLM智能体**（proposer, judger, checker）协同工作：
    - Proposer 生成初始任务和验证器。
    - Judger 进行静态语义审查（判断指令是否清晰、验证器逻辑是否合理）。
    - Checker 在真实Docker环境中执行“干运行”，动态验证任务是否可完成以及验证器是否正确评估结果。
  - 通过**共享Docker交互探针**（agent workers通过探针获取截图和可访问树，不直接进入容器），支持**100+并发worker**和100+Docker环境。
  - **轨迹引导任务合成**：根据模型展开轨迹的成功/失败，将失败任务拆分为更简单的子任务，将成功任务组合为更复杂的多目标任务，进一步产出**近3000个高质量RL任务**（从24000+候选任务中筛选）。
- **Frontier Sampling（前沿采样）**：
  - 维护每个任务的**指数移动平均（EMA）成功率** \( \hat{p}_i \)。
  - 对每个批次的采样，使用**高斯核分布**计算权重 \( w_i = \exp\left(-\frac{(\hat{p}_i - \mu)^2}{2\sigma^2}\right) \)，其中 \( \mu = 0.5 \), \( \sigma = 0.25 \)。
  - 保留 \( \gamma = 0.2 \) 的均匀采样用于探索。动态将展开集中在当前学习前沿的任务上。
- **Visual Context Segmentation（视觉上下文分割）**：
  - **滑动窗口策略**：保留最近的K张截图作为视觉上下文（实验中 \( K = 5 \)），同时保持文本连续性。
  - 维护**token ID流**：训练时仅对最新的assistant response进行梯度更新，旧截图被丢弃并重新分词，避免序列过长。
  - 将 \( O(nT) \) 的逐步训练项压缩为 \( O(nT/K) \) 个段，实现 **2.83倍端到端训练加速**（actor更新从485秒降至154秒，参考策略更新从241秒降至88秒）。

## 7. 优点

- **方法创新性强**：VeriGen实现了端到端自动生成可验证GUI任务（含确定性判断函数），解决了数据稀缺的根本问题；Frontier Sampling利用EMA和Gaussian核动态分配采样，优于均匀/课程学习；Visual Context Segmentation巧妙平衡推理与训练压力。

## 8. 不足与局限

- **episode长度限制**：每任务最多50步，可能低估超长工作流（如多步骤系统管理）的难度，实验中已覆盖大部分任务，但仍有遗漏。
- **环境通用性**：仅在Ubuntu桌面进行评估，未验证在Windows/macOS及其他操作系统上的表现，限制了跨平台泛化结论。
- **模型规模范围**：仅验证了8B~9B参数量的视觉-语言模型，更大（如72B+）或更小（如1B）的模型行为未知，方法的缩放规律尚未刻画。

- **训练成本**：虽然加速了2.83倍，但整体仍需64块H800 GPU和1000次迭代，资源开销仍较大，对小型实验室可能不友好。

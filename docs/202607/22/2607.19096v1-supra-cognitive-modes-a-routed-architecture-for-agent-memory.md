## 摘要
智能体记忆工作负载混合了直接事实查找、关系链与当前状态推理，以及对长历史进行广泛综合。我们描述了超认知模式（SCM），一种将显式或自动选择的每查询模式映射到基于共享摄取基底的检索和综合负载的架构。一个冻结的语义分类器和运行时门控将查询分派到融合的词法与密集查找、图或迭代多跳处理，以及分层长形式综合中。该基底结合了多粒度嵌入、提取的三元组、事实版本元数据以及可选的异步增强。我们在三个基准上描述了部署配置：长期会话记忆（LoCoMo；n = 1,986）、MemoryAgentBench（MAB；n = 3,671）和LongMemEval（n = 500）。参考运行在LoCoMo事实类类别上记录为84.87%，在对抗性弃权上为68.61%，在MAB上两次重复平均为61.49%，在LongMemEval上为86.00%。基于仓库的复现产生了相似的聚合分数，并支持任务和模式条件的失败分析。原始基线输出、LoCoMo和LongMemEval的对齐端到端计时以及完整的令牌账本不可用；存储的行也省略了一些最终的运行时决策。结果描述了一个实现的路由配置及其诊断失败模式，而源代码检查验证了每查询控制接口和共享基底设计。因果路由效应、效率提升和统计显著性仍在现有证据之外。

## Abstract
Agent-memory workloads mix direct factual lookup, relation-chain and current-state reasoning, and broad synthesis over long histories. We describe Supra Cognitive Modes (SCM), an architecture that maps explicit or automatically selected per-query modes to retrieval and synthesis payloads over one shared ingest substrate. A frozen semantic classifier and runtime gates dispatch queries among fused lexical and dense lookup, graph or iterative multi-hop handling, and stratified long-form synthesis. The substrate combines multi-granularity embeddings, extracted triples, fact-version metadata, and optional asynchronous enrichments. We characterize the deployed configuration on three benchmarks: Long-term Conversational Memory (LoCoMo; n = 1,986), MemoryAgentBench (MAB; n = 3,671), and LongMemEval (n = 500). The reference run records 84.87% on LoCoMo factoid categories and 68.61% on adversarial abstention, 61.49% on MAB across two repetitions, and 86.00% on LongMemEval. A repository-backed reproduction produces similar aggregate scores and supports task- and mode-conditioned failure analysis. Raw baseline outputs, aligned end-to-end timing for LoCoMo and LongMemEval, and complete token ledgers are unavailable; stored rows also omit some final runtime decisions. The results characterize one implemented routed configuration and its diagnostic failure patterns, while source inspection verifies the per-query control interface and shared-substrate design. Causal routing effects, efficiency gains, and statistical significance remain outside the available evidence.

### 1. 核心问题与整体含义

-   **研究动机**：当前语言模型智能体需要处理异构的记忆查询工作负载，包括直接事实查找、多跳关系链推理和长历史总结等。不同的查询类型对检索和综合策略的要求截然不同，任何一种单一的、固定的策略都无法在所有场景下都取得最佳效果。这构成了一个“记忆三难困境”：准确性、延迟和成本三者难以兼得。
-   **整体含义**：论文提出了一种新的视角，即不寻找一种通用的最优策略，而是设计一种**按需可切换**的架构，让每次查询都能选择最合适的处理路径。这为构建更灵活、更高效的智能体记忆系统提供了新的思路。

### 2. 方法论

-   **核心思想**：**超认知模式（Supra Cognitive Modes, SCM）** 将记忆系统的操作点作为一个“每查询控制”参数。系统通过一个**冻结的语义分类器**或应用层提供的显式标签，将当前查询分配给一个初始的“模式”（mode）。然后，结合运行时门控逻辑，将这个模式映射到具体的“负载”（payload，即检索和综合的配置参数），并最终决定执行哪个“程序系列”（procedure family）。
-   **关键技术细节**：
    1.  **模式与路由**：系统包含一个冻结的文本分类器，可输出四种语义标签：`SINGLE_FACT_LOOKUP`（单事实查找）、`LONG_FORM_SYNTHESIS`（长文本综合）、`TIME_ANCHORED_LOOKUP`（时间锚定查找）和 `LATEST_VERSION_RESOLUTION`（最新版本解析）。分类器在1,065个合成问题上训练，在568个真实问题上达到90%以上的准确率。
    2.  **运行时门控**：除了初始分类，系统还有运行时门控，可以根据查询的多跳模式、检索结果的形状（如是否包含示例语料）来动态改变初始策略。
    3.  **程序系列**：包含三种主要的程序系列：
        -   **直接查找**：结合BM25和稠密检索，使用倒数秩融合（RRF）合并结果，最后用一个LLM（Claude Sonnet）生成答案。
        -   **图/迭代多跳处理**：处理关系链和状态查询。系统利用提取的三元组（主语、谓语、宾语）构建图，通过规划器和步行器进行图遍历。当图遍历失败时，可回退到迭代稠密检索。
        -   **长文本综合**：采用分层策略，先对语料库进行时间排序采样，然后进行“要点-扩展”的多轮生成过程。
    4.  **共享异步基底**：所有程序系列都共享一个统一的存储基底。这个基底包含多粒度嵌入、提取的关系三元组、事实版本元数据等。这是通过一个必需的Tier-1摄入流程创建的，随后可选的异步补充任务会为基础添加更高层次的信号（如文档结构、预期要点等）。

### 3. 实验设计

-   **数据集**：使用了三个公开的智能体记忆基准测试：
    -   **Long-term Conversational Memory (LoCoMo)** : 评估长期会话记忆，包含1,986个问题。
    -   **MemoryAgentBench (MAB)** : 一个综合性的记忆代理任务套件，包含3,671个问题。
    -   **LongMemEval** : 评估纵向个人记忆，包含500个问题。
-   **对比方法**：
    -   **Mem0 v2 OSS**：一个开源的长短期记忆系统，被用作基线之一。
    -   **Production Comparator**：一个未公开细节的“生产比较器”，被用作更强的基线。
-   **评估指标**：各基准测试使用各自的标准评估方法，包括基于LLM的评判、精确子串匹配和词法评分器等。

-   在描述的配置下，SCM在三个基准测试的**聚合得分上均高于**对比系统（Mem0和Production Comparator）。

### 7. 优点

2.  **强大的诊断能力**：由于系统记录了每个查询的模式标签和检索结果，可以方便地进行基于路由和任务类型的失败分析（如附录D），这为未来系统改进提供了明确的工程方向。

4.  **第三方可复现性**：提供了详细的工件来源、参数设置和复现步骤（附录A），并且其本地复现与作者参考结果一致性很高，增强了结果的可信度。

### 8. 不足与局限

1.  **实验覆盖的完整性不足**：
    -   **缺乏关键消融实验**：论文没有进行与“固定策略”（non-routed）的对比实验，无法证明路由架构本身优于任何单一策略。
    -   **基线不可审计**：原始基线输出缺失，导致其声称的性能优势无法被独立验证，也排除了对基线系统本身进行行级修正（如修正LoCoMo的计分错误）的可能性。
2.  **证据与结论的鸿沟**：论文承认关于**路由的因果效应、效率提升、统计显著性**等问题均超出了现有证据范围。这导致结论非常谨慎，无法支撑其方法的优越性。

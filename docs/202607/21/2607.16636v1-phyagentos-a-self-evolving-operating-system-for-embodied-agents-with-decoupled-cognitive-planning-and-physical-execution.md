## 摘要
视觉-语言-动作模型、世界模型和智能体规划器各自推动了物理智能的发展，但它们的组合缺乏通用的执行抽象、共享状态、语义验证以及跨异构具身的持久经验。我们提出PhyAgentOS，一个运行时基础，将调度、验证、记忆、基准测试和安全作为系统级服务提供。其以会话为中心的运行时会话（而非动作）作为调度、兼容性预检、监督执行、证据收集和验收的最小单元。为了将认知与物理执行解耦，认知-物理边界采用文件系统：状态即文件协议将跨层状态具象化为带有YAML的Markdown格式，产生可检查、可版本化的记录，且智能体层与运行时层之间无需代码依赖。这些视图形成一个统一的认知状态空间，对齐意图、能力、环境、执行和经验。会话验证器通过基于证据的成功、失败或重新规划判定，区分执行终止与语义任务完成。验证结果通过认知记忆整合为可复用知识和纠正经验，从而在不重新训练的情况下闭合试错循环。基准测试复用部署会话和验证路径，使结果可追溯至真实执行。分层安全约束策略驱动和智能体驱动的执行：预检、动作桥接、安全防护、心跳监控和目标本地约束。验证是渐进式的：游戏测试认知规划，模拟增加动力学和控制，真实机器人增加硬件噪声，而认知层保持不变。PhyAgentOS在Optimus-67、StarDojo和DST-Dojo上进行了基准测试，在19个以上模拟和物理具身上得到验证，并在LIBERO、Calvin和RoboCasa365上跨多个VLA模型取得了提升。

## Abstract
Vision-language-action models, world models, and agentic planners each advance physical intelligence, yet their composition lacks a common execution abstraction, shared state, semantic verification, and persistent experience across heterogeneous embodiments. We present PhyAgentOS, a runtime foundation delivering scheduling, verification, memory, benchmarking, and safety as system-level services. Its Session-Centered Runtime treats a session, not an action, as the minimum unit of scheduling, compatibility preflight, supervised execution, evidence collection, and acceptance. To decouple cognition from physical execution, the cognition-physics boundary is a file system: the State-as-a-File protocol materializes cross-layer state as Markdown with YAML, yielding inspectable, versionable records without code dependencies between Agent and Runtime layers. These views form a unified cognitive state space aligning intent, capabilities, environment, execution, and experience. The SessionVerifier distinguishes execution termination from semantic task completion via evidence-grounded verdicts of success, failure, or replan. Verified outcomes are consolidated through epistemic memory into reusable knowledge and corrective lessons, closing a trial-and-error loop without retraining. Benchmarking reuses the deployment session and verification path, so results trace to real execution. Layered safety constrains both policy-driven and agent-driven execution: preflight, action bridges, SafetyGuard, heartbeat monitoring, and target-local constraints. Validation is progressive: games test cognitive planning, simulation adds dynamics and control, real robots add hardware noise, with the cognitive layer held constant. PhyAgentOS is benchmarked on Optimus-67, StarDojo, and DST-Dojo, validated on 19+ simulated and physical embodiments, and gains on LIBERO, Calvin, and RoboCasa365 across multiple VLA models.

### 论文核心总结：PhyAgentOS

#### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前具身人工智能（Embodied AI）领域存在一个结构性的“验证鸿沟”（verification gap）。视觉-语言-动作模型（VLA）、世界模型（World Model）和智能体系统（Agentic System）各自取得了进步，但它们组合使用时缺乏一个统一的操作系统层来提供：
    - **统一的执行抽象**：各组件在异构的接口和抽象层次上运行。
    - **共享状态表示**：认知层的意图（如“抓杯子”）与物理层的执行状态（如“轨迹完成，但没抓到”）无法对齐。
    - **语义验证**：执行完成（action finished）被错误地等同于任务成功（task completed），无法区分执行终止和语义完成。
    - **持久化经验**：一次会话中获得的经验（如失败原因、成功策略）无法跨会话、跨任务、跨具身迁移和复用。
- **整体含义**：为了解决上述问题，论文提出需要一个位于现有模型和传统中间件（如ROS）之上的、面向具身智能体的“操作系统”。该系统并非要取代VLA或世界模型，而是要提供一个运行时基础，将调度、验证、记忆、安全等功能作为系统级服务，从而构建一个可闭环、可演化、可验证的通用具身智能执行框架。

#### 2. 论文提出的方法论：核心思想、关键技术

- **核心思想**：将“认知-物理执行”的边界抽象为一个文件系统协议，即 **State-as-a-File（状态即文件）**。所有跨进程的数据（目标、技能、会话、观察、教训、知识）都以人类可读的Markdown+YAML文件形式具象化。
- **关键技术细节**：
    1.  **以会话为中心（Session-Centered Runtime）**：将“会话”（Session）作为系统调度、预检、执行、验证的最小单元，而非单个动作。这为复杂任务提供了有界的生命周期。
    2.  **解耦的认知与执行（Decoupled Cognitive Planning and Physical Execution）**：
        - **Agent层**（认知）：负责理解意图、分解任务、选择技能和目标，输出一个结构化的“会话合同”（Session Contract）。
        - **Runtime层**（物理）：负责执行会话，包括预检、监控、与目标（仿真器/真实机器人）交互。Agent不直接发送硬件指令，两者通过文件协议解耦。
    3.  **SessionVerifier（会话验证器）**：在任务执行后，**区分执行终止与语义任务完成**。它基于任务目标、初始/最终环境状态和执行证据，输出 `success`、`failure` 或 `replan` 的语义判定，取代了简单的成功/失败布尔值。
    4.  **Epistemic Memory（认知记忆）**：分为多级记忆（工作记忆、情景记忆、语义记忆、程序性记忆），将验证后的结果整合为可复用的知识（`KNOWLEDGE.md`）和纠正性教训（`LESSONS.md`）。这驱动了一个**不重新训练神经网络**的自我演化（self-evolution）闭环：执行 → 验证 → 诊断 → 修改 → 再验证 → 固化经验。
    5.  **分层安全架构（Layered Safety Architecture）**：包含兼容性预检、动作桥接、SafetyGuard（安全卫士）、心跳监控和具身本地约束，形成递进式防护。

## 摘要
两个智能体如何从零开始发明一种共享语言？在刘易斯信号博弈中，发送者和接收者必须仅依靠交互历史来协调一种编码。我们研究了五种记忆架构在不同信道配置下与LLM智能体的表现，发现记忆架构比信道容量更重要。拥有持久私有笔记本的智能体能从剩余信道容量中受益，并避免了无状态智能体中观察到的高容量崩溃，实现了最可靠的协调（容量=25时，0.867±0.023）。无状态智能体在中等容量时达到峰值，随后随着词汇量超出滚动上下文窗口的追踪能力而退化。笔记本将习得的约定外部化，使智能体无需每轮重新推导编码。受信息瓶颈启发的论证预测最优容量等于物体数量。然而，瓶颈（容量=8）被证明是一个脆弱点，而剩余容量通常更好。我们表明，仅信道容量无法预测协调；记忆架构决定了智能体是否将交互历史转化为稳定约定，而理解信号如何变成语言需要这两个维度。

## Abstract
How do two agents invent a shared language from scratch? In a Lewis signaling game, a sender and receiver must coordinate on a code using only their interaction history. We study five memory architectures across varying channel configurations with LLM agents and find that memory architecture matters more than channel capacity. Agents with a persistent private notebook benefit from surplus channel capacity and avoid the high-capacity collapse seen in stateless agents, achieving the most reliable coordination ($0.867 \pm 0.023$ at capacity = 25). Stateless agents peak at moderate capacity and then degrade as the vocabulary grows beyond what a rolling context window can track The notebook externalizes learned conventions, freeing agents from having to re-derive codes each round. An information bottleneck-inspired argument predicts an optimal capacity equal to the number of objects. Instead, the bottleneck (capacity = 8) proves to be a fragility point, and surplus capacity is generally better. We show that channel capacity alone cannot predict coordination; memory architecture determines whether agents turn interaction history into stable conventions, and both dimensions are needed to understand how signals become language.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：两个LLM智能体如何从零开始发明一种共享的语言？具体地，在刘易斯（Lewis）信号博弈中，发送者与接收者仅依赖交互历史进行协调，最终形成稳定的信号-意义映射——即一种新语言。本研究旨在探究**记忆架构**（memory architecture）与**信道容量**（channel capacity）这两个因素如何影响语言涌现的成功率与结构。
- **研究背景**：传统上，语言涌现研究使用梯度可训练的神经网络智能体，且信道容量被视为关键变量（Resnick et al., 2020），信息瓶颈理论预测当信道容量等于物体数量时存在最优。然而，由大语言模型（LLM）驱动的智能体具有先验语义和强大的上下文学习能力，其学习机制不依赖于梯度更新，而是通过**上下文推理**。因此，研究重点应从模型架构转向**记忆架构**——智能体跨回合保留何种信息、以何种形式保留。
- **整体含义**：理解LLM智能体能否以及如何通过交互形成稳定约定，对多智能体协作、人工语言进化以及AI系统的通信协议设计有重要意义。本研究表明，**记忆架构比信道容量更能决定语言涌现的成功与否**，打破了传统仅关注容量的视角。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：在标准的刘易斯信号博弈框架下，让两个LLM智能体（同为 `gpt-5.4-mini`）扮演发送者和接收者，进行200轮交互。每轮发送者观察到4个候选物体（从8个物体池中抽取）并指定目标，然后发送一个固定长度的符号消息；接收者根据消息从4个候选中猜测目标。轮次结束后，双方都获得反馈（正确/错误及真实目标）。通过改变**记忆架构**和**信道容量**，观察协调成功率（Accuracy）和语言结构（TopSim、最佳互信息、碰撞率）的变化。
- **关键技术细节**：
  - **记忆架构**：在统一的20轮滚动窗口（rolling window of last 20 interactions）基础上，添加不同的持久存储方式。共5种条件：
    1. **memory only**：只有滚动窗口，无持久存储。
    2. **env board**：共享公共约定表（环境编译的），双方可读不可编辑。
    3. **scratchpad**：每轮由智能体自主重写的自由格式笔记本（≤150词），仅最新版本保留。
    4. **codebook**：固定容量（10个槽位）的槽列表，通过每轮一次结构化操作（append / edit / none）进行编辑，条目持续存在直到被覆盖。
    5. **codebook meta**：在codebook基础上增加一条持久性元备注（meta-note），描述语言级规则。
  - **信道容量**：`capacity = |V|^L`，其中|V|是词汇表大小（{2,3,4,5}），L是消息长度（{2,3}），因此容量取值为{4,8,9,16,25,27,64,125}。
  - **评估指标**：
    - **Accuracy**：在50轮窗口或晚期（151-200轮）计算接收者猜对的比例，随机基线0.25。
    - **TopSim**：测量信号空间与意义空间的斯皮尔曼等级相关，用于衡量组合性（compositionality）。
    - **Best MI**：评估消息的每个位置与物体特征的最大互信息，反映位置槽结构。
    - **Collision rate**：不同物体映射到同一消息的比例，衡量歧义性。
  - **游戏细节**：物体池包含8个对象，每个有3个二进制特征（颜色、形状、大小）。发送者输出严格JSON格式（含tokens和rationale），接收者输出猜测索引。

## 6. 论文的主要结论与发现

1. **记忆架构比信道容量更关键**：拥有持久笔记本（scratchpad）的智能体在容量25时达到最高协调成功率（0.867±0.023），且受益于剩余容量；而无持久存储的智能体（memory only）在中等容量达到峰值后崩溃（容量64时碰撞率1.0）。
2. **容量8是脆弱点而非最优**：信息瓶颈启发下常预测容量等于物体数量（8）时是组合性最优，但实验发现该点结果高度双模态（要么成功≥0.66，要么失败≤0.56），协调方差最大，且显著低于容量25。无剩余信号空间导致早期碰撞无法修复。
3. **剩余容量稳定约定**：当容量大于8（如25、64、125），智能体有冗余信号可覆盖错误碰撞，性能更可靠。
4. **窗口大小无关紧要（高容量下）**：Study 3表明，memory only在高容量（64）时，即使将滚动窗口从20扩展到40轮也无改善；而scratchpad仅需10轮即可达0.94准确率。因此崩溃是**巩固问题**而非历史长度问题。
5. **笔记本作用机制**：scratchpad将学习到的约定外部化，避免每轮重新推导；但也会引入**约定漂移**（convention drift）——发送者可能重新分配已有信号给新物体，导致后期准确率下降。
6. **组合性仅偶发出现**：多数运行产生的是全局歧义但局部可解的代码（碰撞率高但接收者利用4选1上下文），真正的位置组合性只出现在少数高容量scratchpad运行中。
7. **信息理论缺失变量**：传统信息瓶颈分析假设智能体为最优编码器，但LLM智能体非最优，记忆架构决定了它们接近理论边界的程度。

## 8. 不足与局限

- **模型单一**：仅使用`gpt-5.4-mini`，结论可能对模型专有特性敏感（如特定tokenizer、训练数据中的语言先验）。需要复现于开源模型（如Llama、Mistral）以验证泛化性。

- **窗口大小研究单种子**：Study 3全部基于seed=7，结果可能不具备代表性。

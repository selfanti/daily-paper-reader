## 摘要
虽然记忆系统对于智能体架构至关重要，但普遍存在的架构碎片化限制了系统性研究。现有实现通常将记忆生命周期的不同阶段耦合在一起，将评估逻辑与特定数据集纠缠在一起，并且对异构记忆类型的支持有限。我们引入了MemTools，这是一个互操作性研究框架，将记忆系统组件与其底层部署环境解耦。MemTools通过声明式数据契约标准化记忆生命周期，使得不同系统中的组件可互换组装。它正交地将基准数据集与执行协议分离，以促进受控评估。此外，MemTools提供了一个统一的计算接口，用于在共享运行时内协调符号、神经和多模态记忆表示。对跨系统组件集成、评估协议重新配置和异构记忆协调的实证评估表明，MemTools能够系统地隔离和分析记忆设计变量。这些发现表明，MemTools为推进智能体记忆的原则性研究提供了一个实用且可扩展的基础设施。

## Abstract
While memory systems are essential for agent architectures, pervasive architectural fragmentation restricts systematic research. Existing implementations typically couple different stages of the memory lifecycle, entangle evaluation logic with specific datasets, and provide limited support for the management of heterogeneous memory types. We introduce MemTools, an interoperability research framework that decouples memory system components from their underlying deployment environments. MemTools standardizes the memory lifecycle through declarative data contracts, enabling the interchangeable assembly of components across different systems. It orthogonally separates benchmark datasets from execution protocols to facilitate controlled assessments. Furthermore, MemTools provides a unified computational interface for coordinating symbolic, neural, and multimodal memory representations within a shared runtime. Empirical evaluations on cross-system component integration, evaluation protocol reconfiguration, and heterogeneous memory coordination demonstrate that MemTools enables systematic isolation and analysis of memory design variables. These findings suggest that MemTools provides a practical and extensible infrastructure for advancing principled research on agent memory.

### 1. 核心问题与整体含义（研究动机和背景）

*   **核心问题**：当前大型语言模型（LLM）智能体的记忆系统存在严重的“架构碎片化”问题。不同系统的实现高度耦合，具体体现在三个层面：
    1.  **组件耦合**：记忆生命周期（形成、存储、检索、演化）的各阶段被封装在封闭代码库中，缺乏标准化的数据接口，导致跨系统的组件无法复用和互换。
    2.  **评估逻辑绑定**：记忆流水线的评估逻辑与特定的基准数据集深度绑定，导致性能差异可能源于评估协议（如记忆操作的时序）而非记忆算法本身，引入了评估偏差。
    3.  **表征碎片化**：对符号、神经参数、多模态等异构记忆形式缺乏统一的协调接口，使得在单一实验环境中组合使用它们变得困难。
*   **整体含义**：这种碎片化阻碍了系统性研究，使得研究人员难以隔离和分析单个设计变量（如检索策略、记忆形成时机），从而无法公平地比较不同记忆架构的优劣。论文提出MemTools旨在提供一个标准化的基础设施，推动智能体记忆领域从事后叙述（post-hoc）转向系统性、可控的实验。

### 2. 论文提出的方法论

*   **核心思想**：通过**解耦**与**标准化**打破现有架构的耦合关系。具体来说，将记忆系统的组件、评估协议和基准数据集视为相互独立的模块，并通过声明式数据契约（Declarative Data Contracts）来定义它们之间的交互界面。
*   **关键技术细节**：
    1.  **标准化生命周期接口**：
        *   将记忆生命周期分解为`Backend`（后端）、`Formation`（形成）、`Retrieval`（检索）、`Evolution`（演化）和`Utilization`（利用）等独立阶段。
        *   定义一个标准化的`MemoryEntry`数据结构作为各级组件间数据流动的载体。
        *   每个组件通过`requires_keys`和`provides_keys`属性明确声明其依赖和产出。
        *   **自动匹配引擎（Matching Engine）**：该机制在初始化时自动验证所有组件的输入输出字段是否兼容（即`provides_keys ⊇ requires_keys`），并枚举所有有效的流水线组合，无需手动集成代码。
    2.  **统一评估范式**：
        *   将**基准数据集**（如ALFWorld, MemGallery）与**评估协议**（如离线声明式、在线过程式）正交分离。
        *   基准数据集仅作为数据提供者，而评估协议定义了记忆操作（如形成、检索）的时序和逻辑（例如，是先集中学习再查询，还是在执行任务过程中边积累边查询）。
        *   这种设计使得同一数据集可以用于验证不同评估协议下的记忆能力，反之亦然。数据契约机制同样用于检查数据集和协议是否兼容。
    3.  **异构记忆管理**：
        *   将符号、神经、多模态等不同类型的记忆封装为独立的处理流水线，每条流水线都遵循标准化的生命周期接口。
        *   设计一个`MultiSystem`协调层，作为统一的计算接口。当智能体发起查询时，该层可以并行地向所有异构记忆子系统发送请求，并在利用阶段通过链式适配（Chained Adaptation）过程将检索结果整合后应用到LLM上。

## 摘要
有效的记忆对LLM智能体至关重要，但构建有效记忆仍具挑战。记忆构建策略决定随着交互积累，哪些信息应提取、存储、更新、压缩或丢弃。启发式记忆方法依赖主观、特定任务的规则，可能偏离下游目标并限制跨任务适应性。相比之下，基于强化学习的方法从任务反馈中学习，但主要使用结果或模块级别的奖励。这些粗略信号指示任务成功与否，却无法识别哪些中间记忆内容支持最终答案，造成细粒度信用分配瓶颈。然而，构建此类过程反馈异常困难，因为中间记忆决策缺乏唯一真实目标，而适当的信用随智能体不确定的推理轨迹变化，因此无法预先指定。我们提出AttriMem，一种归因引导的过程反馈框架，用于通过强化学习学习记忆构建策略。AttriMem用源自最终答案的token级别贡献的局部奖励增强全局结果奖励。在长程对话问答实验表明，AttriMem优于基于检索、启发式和强化学习的基线，可跨基准和答案模型泛化，并稳定强化学习优化。

## Abstract
Effective memory is crucial for LLM agents, yet constructing it effectively remains challenging. A memory-construction policy decides what information to extract, store, update, compress, or discard as interactions accumulate. Heuristic memory methods rely on subjective, task-specific rules, which can misalign with downstream objectives and limit cross-task adaptability. RL-based methods, by contrast, learn from task feedback but mainly use outcome- or module-level rewards. These coarse signals indicate task success but cannot identify which intermediate memory contents support the final answer, creating a fine-grained credit-assignment bottleneck. However, constructing such process feedback is prohibitively difficult because intermediate memory decisions lack unique ground-truth targets, while the appropriate credit varies with the agent's uncertain reasoning trajectory and therefore cannot be specified in advance. We propose AttriMem, an attribution-guided process-feedback framework for learning memory-construction policies with RL. AttriMem augments the global outcome reward with local rewards derived from token-level contributions to the final answer. Experiments on long-horizon dialogue question answering show that AttriMem outperforms retrieval-based, heuristic, and RL-based baselines, generalizes across benchmarks and answer models, stabilizes RL optimization.

## 一、核心问题与研究动机

- **研究背景**：LLM 智能体在长程交互任务中需要有效记忆历史信息，但现有记忆构建方法存在根本缺陷。启发式记忆方法依赖人工规则，主观性强且易与下游目标错位，跨任务泛化能力差；强化学习方法虽然能从任务反馈中学习，但仅提供粗粒度的结果奖励（如答案正确性），无法区分中间记忆内容的具体贡献，造成“细粒度信用分配瓶颈”。
- **核心挑战**：中间记忆决策缺乏唯一真实目标，其信用取决于智能体不确定的推理轨迹，因此无法预先指定过程真实标签。现有方法难以提供直接且细粒度的过程反馈。

## 二、方法论：AttriMem 框架

- **核心思想**：利用归因分析将最终答案作为信用分配目标，追溯其支持的证据到中间记忆过程，并转换为局部过程奖励，与全局结果奖励共同优化记忆构建策略。
- **关键技术细节**：
  1. **记忆构建策略**：智能体在每轮会话中输出记忆操作（如提取、更新、压缩），生成文本记忆内容 ∆M_t。
  2. **固定检索-回答接口**：完成所有会话后，冻结的检索器从记忆库中检索相关记录，答案模型生成最终答案。
  3. **归因计算**：采用 ContextCite 方法，对最终答案固定不变，随机遮罩记忆内容的词元子集（二进制包含向量），评估遮罩后答案模型对答案的对数概率变化，通过 LASSO 回归拟合稀疏线性代理，得到每个词元的归因分数 ˆw_j。
  4. **过程奖励转换**：将每个记忆操作对应的词元归因分数聚合（除以 √l_t 以消除长度偏置），得到该操作的局部过程奖励 r_t。最终优势函数为结果优势与过程优势的加权和（λ 控制权重）。
  5. **优化方法**：基于 GRPO（Group Relative Policy Optimization），在组内标准化优势，最大化 KL 约束下的期望奖励。

- **公式/算法流程（文字说明）**：
  - 采样 G 个记忆构建轨迹，每个轨迹包含 T 步记忆操作。
  - 固定检索-回答接口为每个轨迹生成答案并计算结果奖励（如答案准确率）。
  - 对每个轨迹，对最终答案进行归因，计算每个记忆操作 ∆M_t 中所有词元的归因分数平均，得到过程奖励。
  - 组内标准化结果奖励和过程奖励，分别得到结果优势 ˆA_out 和过程优势 ˆA_proc。
  - 每个时间步的联合优势 ˆA_i,t = ˆA_out_i + λ ˆA_proc_i,t。
  - 使用 GRPO 更新策略，包含重要性采样比例裁剪和 KL 正则化项。

## 六、主要结论与发现

1. **归因引导的过程反馈优于粗粒度奖励**：词元级归因奖励（RL_Tok）在所有基准上均显著优于结果-only 和动作级归因奖励，提升幅度为 2–5 个百分点准确率。
2. **与 SFT 互补**：RL 进一步优化 SFT 初始化的记忆策略，SFT+RL_Tok 取得最佳结果。
3. **跨域与跨模型泛化**：仅在 LongMemEval 训练，在 LoCoMo 和 PerLTQA 上仍优于现有最强非 RL 方法；不同答案模型下性能保持稳定。
4. **中间记忆质量提升**：AttriMem 生成的中间记忆更忠实、覆盖问答关键信息、操作更合理（胜率 60–78%）。
5. **训练稳定性**：词元级奖励提供更稳定、持续提升的优化信号，避免了结果奖励的陡峭骤升和平台期。
6. **性能-效率权衡**：AttriMem 位于帕累托前沿，在相似 token 消耗下取得最高准确率。

## 七、优点

- **细粒度信用分配**：首次将归因分析引入记忆学习的过程反馈，实现词元级别的局部奖励，克服了结果奖励和动作级奖励的局限。
- **框架通用**：归因方法可替换（如未来更优的长上下文归因方法），与不同的策略优化算法（GRPO）兼容。
- **实验设计严谨**：覆盖多基准、多基线、多消融、跨模型和跨域验证，充分展示有效性和泛化性。
- **效率可控**：归因开销通过并行批处理和逐步增加遮罩数来控制，增加一次前向传播的延迟。

## 八、不足与局限

- **计算开销**：训练时需要额外进行归因计算（32 次遮罩评估），虽然可并行但仍增加训练成本（作者认为可接受）。
- **依赖归因质量**：归因失误可能导致错误信用分配，尤其在长上下文下。论文指出该风险有限（最终答案通常依赖少数记录），但未测试弱归因情形下的鲁棒性。
- **仅适用于文本可分解记忆**：当前框架要求记忆输出为可分词文本，不直接适用于向量、图结构记忆，需要额外对齐机制。
- **实验覆盖**：虽然包含多个基准，但所有实验均基于同一类型（长程对话问答），未验证在更广泛任务（如工具使用、多轮决策）上的有效性。
- **未对比更大模型**：仅使用 4B 参数模型，更大模型上的效果未知；且归因计算可能随模型增大而更昂贵。

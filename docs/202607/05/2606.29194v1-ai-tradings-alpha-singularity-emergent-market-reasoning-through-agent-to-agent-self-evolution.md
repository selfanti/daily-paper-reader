## 摘要
自动化阿尔法挖掘固定评分函数，并在其上变化搜索算法。针对固定评分器收敛的搜索会过拟合评分器无法惩罚的任何内容，这是样本外泛化差距的主要原因。我们将评分函数视为与阿尔法因子一起的搜索人工产物，并研究何种条件下这种联合搜索是可行的。密封联合搜索（SJS）是一个框架：对自主发现系统中的信息流施加一组结构条件，防止联合搜索崩溃为自我确认，同时保持评估器密封。条件涵盖角色分解、类型化角色间通信、来源密封读取、版本化存储以及底层局部促进。Agora对SJS进行了实证测试：五个LLM智能体类通过三个通道通信，演化出八个技能库，阿尔法库基于AlphaGen算子构建。三位评估器撰写报告汇总为一份简报，传递分歧而非投票。我们在CSI 1000上运行Agora 100轮，并在2026年91天保留集上评估，该保留集对所有LLM输入密封。Agora实现保留集夏普比率+1.87；最佳基线在有利种子下为+1.334，交叉种子均值为-0.755。将Agora的两个指标预加载到冻结库消融实验中仅恢复+2.25夏普差距中的+0.40，而添加不进行库演化的PPO则加剧了差距。这两个指标是涌现的而非设计的。注意事项：单种子运行，空头侧信号集中，旨在多空策略。

## Abstract
Automated alpha mining holds the scoring function fixed and varies the search algorithm over it. A search that converges against a fixed scorer overfits whatever the scorer cannot penalize, a primary cause of the out-of-sample generalization gap. We treat the scoring function as a search artifact alongside the alpha factors and study what conditions make this joint search admissible. Sealed Joint Search (SJS) is a framework: a set of structural conditions on information flow in an autonomous-discovery system that prevent joint search from collapsing into self-confirmation while keeping the evaluator sealed. Conditions cover role decomposition, typed inter-role communication, provenance-sealed reads, versioned stores, and substrate-local promotion. Agora tests SJS empirically: five LLM agent classes communicate via three channels, evolving eight skill libraries, with alpha libraries built on AlphaGen operators. Three evaluators write reports aggregated into one brief, carrying forward disagreement instead of voting. We run Agora for 100 rounds on CSI 1000 and evaluate on a 91-day 2026 holdout sealed from all LLM inputs. Agora achieves holdout Sharpe +1.87; best baseline +1.334 at favorable seed and -0.755 cross-seed mean. Pre-loading Agora's two metrics into a frozen-library ablation recovers only +0.40 of the +2.25 Sharpe gap, and adding PPO without library evolution worsens the gap. The two metrics emerge rather than being designed. Caveats: single-seed run, short-side concentrated signal, intended for long-short.

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：自动化alpha挖掘（如遗传编程、RL搜索）通常固定评分函数（如rank IC），搜索算法会快速过拟合该评分器的盲点，导致样本外泛化差距。这一现象在多测试场景下尤为严重。
- **背景**：现有自演化系统要么保持固定标准（继承盲点），要么频繁查询外部评估器（增加泄漏风险），未解决联合搜索的坍塌问题。
- **目标**：提出一种方法论，使评分函数本身成为可搜索的人工产物，同时通过信息流拓扑约束防止自我确认，提升泛化能力。

### 2. 论文提出的方法论
- **核心思想**：将搜索过程视为信息拓扑，而非单纯算法。提出**密封联合搜索（Sealed Joint Search, SJS）** 框架，包含五条结构条件（F1–F5）和一条边界条件（外部评估器运行前固定且对搜索全程不可见）。
  - **F1（分解提议）**：提议者与评估者为分离角色，无共享状态。
  - **F2（类型化通信）**：角色间通过带类型的通道通信，禁止自由形式聊天。
  - **F3（来源密封底层）**：每条记录标记数据段，角色按类型和来源被过滤读取。
  - **F4（持久版本化技能库）**：每个角色拥有独立库，状态机分为 builtin / trial / accepted / rejected。
  - **F5（底层局部促进规则）**：促进决策基于内部结果（如training Sharpe的相关性），非自我评判且结果驱动。
- **具体实现（Agora系统）**：
  - 5个角色（研究报告、alpha挖掘器、评估挖掘器、两组评估面板各3个实例）共9个独立LLM客户端（Claude Sonnet 4-6）。
  - 3类通道：A（定向建议）、B（跨轮简报）、C（LLM Wiki共享内存）。
  - 8个技能库：operator, RL-network, RL-algorithm, reward, metric, topic, rubric, meta-rubric（metric库仅4个内置，是唯一观测到演化的库）。
  - 推荐规则：计算指标得分与训练Sharpe的相关系数，|ρ|>0.3提升，<0.05降级。
  - 每轮流程：研究建议 → alpha挖掘 → alpha评估 → 指标提议 → 测试段重评分 → 因子指标评估 → 子系统提交更新。

### 6. 论文的主要结论与发现
- **核心结果**：Agora在91天保留段实现组合夏普+1.87，优于所有基线（最佳单种子B2 +1.334，但均值-0.755）。
- **指标演化**：metric库仅两个指标（monotonicity score v1 和 excess drawdown penalty v1）从trial提升为accepted，均为标准量化构造，非人工设计，通过agent间封闭循环涌现。
- **消融揭示**：
  - 冻结库（B6）降至-0.379，联合演化贡献上界+2.25 Sharpe。
  - 预加载两个指标仅恢复+0.40（<20%）。
  - 单独添加PPO中继导致Sharpe为-1.18（有害），说明不对称升级搜索过程而不共演评分器是反效果。
  - 更多轮次而不演化（B6+aug 20轮）恶化至-1.96，排除轮次数解释。
- **滚动窗口**：7/7窗口长期多空夏普为正（中位数+1.87），2026H1恰为分布中位数。
- **关键局限性**：Agora仅单种子；B2种子方差极大；holdout仅91天；F1分解未直接消融；演化仅发生在一个库中。

### 7. 优点
- **方法论创新**：提出信息拓扑视角，将评分函数从固定超参数变为搜索空间，并给出密封条件，逻辑严谨。
- **系统实现细致**：Agora的通道类型、角色隔离、来源密封、检漏审计等设计严格遵循SJS，防止数据泄漏。
- **实验设计全面**：包括7个基线、3个消融、滚动窗口、成本敏感性、跨种子分析（部分），并提供了Newey-West显著性检验，透明度高。
- **开源承诺**：代码、日志、预训练状态均将开源，利于复现和后续研究。
- **涌现发现**：关键指标是系统自发生成而非人类设计，验证了自演化有效性。

### 8. 不足与局限
- **单种子运行**：Agora完整100轮仅执行一次，系统方差未表征，可能受幸运种子影响（尽管滚动窗口提供了部分稳健性）。
- **最强基线B2的高方差**：单种子对比可能误导，跨种子均值差距支持Agora但降低了统计显著性。
- **holdout窗口短**：91个交易日使得夏普置信区间极宽（Agora 95% CI [-2.12, +6.10]），显著性测试除两个基线外均不显著。
- **未直接消融F1（角色分解）**：缺少单智能体+演化对照组，无法判断多角色分解的必要性。
- **演化局限**：仅metric库发生演化，其他七个库（包括operator、reward）保持内置，可能因prompt或探索策略不足。
- **测试段轻微泄漏**：评估挖掘器接收四个类别反馈（“远高于/高于/接近/远低于”），每轮约2比特信息，累积可能提供弱信号。
- **应用限制**：信号集中于空头侧，适合多空策略；长期多头表现弱于基准；仅在中国A股数据集验证，未在美股等推广。

## 摘要
当可靠工具在持续会话中悄然变化时，LLM代理的工具选择会发生什么？我们从认知心理学中借用定势转换来研究代理适应隐藏可靠性变化的能力。我们的基准测试构建了具有冗余的工具技能库，其中许多工具解决相同任务但隐藏可靠性不同。在我们的评估框架中，一个分支计划在隐藏边界处切换可靠工具组，并将每次切换与无切换控制配对。我们发现，代理默认在每个边界后的几轮内形成一个小型重复模式，每次可靠性变化后调用份额集中在几个离散值上。我们为每个代理轨迹评分定势转换准确率：在每个切换后窗口内路由到目标工具组的联合概率。我们在开源代理框架中测试开放权重的LLM，并发现同一套例程中存在性质不同的失败模式。我们还发现，集合框架（工具集将替代方案呈现为竞争性或互补性）会改变路由动态。

## Abstract
What happens to an LLM agent's tool choice when the reliable tool silently changes within an ongoing session? We borrow set-shifting from cognitive psychology to study how well agents adapt to hidden reliability shifts. Our benchmark mounts tool-skill libraries with redundancies, where many tools solve the same task but differ in hidden reliability. In our evaluation framework, a branched schedule shifts the reliable tool group at hidden boundaries and pairs every shift with a no-shift control. We find that agents, by default, settle on a small recurring routine within a few turns of each boundary, with call shares concentrating on a few discrete values after each reliability shift. We score the set-shifting accuracy for each agent trajectory: the joint probability of routing to the target tool group in every post-shift window. We test open-weight LLMs in an open-source agentic harness and find qualitatively distinct failure modes across the same set of routines. We also find that set framing, how the toolset presents the alternatives as competing or complementary, shifts the routing dynamics.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：在实际应用中，LLM agent 在持续会话中可能遇到“可靠工具无声变化”的情况（例如后端切换导致原本可靠的工具失效），但现有评估方法主要关注单轮工具选择，缺乏对 agent 在动态环境中适应能力（尤其是隐藏可靠性转移）的测试。
- **核心问题**：当功能等价的工具组中隐秘地切换“可靠”组时，agent 能否及时更新其路由选择？其默认行为是否存在类似人类“定势效应”（Einstellung）或“持续错误”（perseveration）的僵化现象？
- **整体含义**：借鉴认知心理学中的**威斯康星卡片分类测试（WCST）** 和**逆转学习**范式，将 agent 的工具选择转化为纵向行为测试，揭示 agent 在变化环境中的默认行为模式与故障模式，为设计更鲁棒的 agent 系统提供实验依据。

## 2. 论文提出的方法论

### 核心思想
- 构建**冗余工具-技能集**（redundant tool-skill sets）：每个任务家族包含多个功能等价的工具组，组间隐藏可靠性不同。在一个持续会话中，通过隐藏边界（silent boundary）切换可靠工具组，并配有无切换对照（no-shift control），以评估 agent 对变化的适应能力。

### 关键技术细节
- **工具与小组**：每个工具是带名称、描述和参数模式的函数。工具小组是一组功能等价的工具（同一架构和操作，但名称和描述不同）。技能是 Markdown 格式的过程说明，与工具小组绑定。
- **调度树结构**：采用分支调度树，每个边界有三个子分支：**反转**（回到之前用过的组）、**新颖**（切换到之前未用的组）、**无切换对照**。每个节点随机化显示顺序和组内顺序以消除位置偏差。
- **评估指标**：
  - **定势转换准确率** \(\Phi = \prod_{k=1}^K s_k\)，其中 \(s_k\) 是第 \(k\) 个后切换窗口中调用目标工具组的份额。\(\Phi\) 衡量从根到终点的所有窗口都路由正确的联合概率。
  - **路线类（route class）**：每个回合根据接触的工具组分为 \(R1\)（仅目标）、\(R2\)（目标+另一组）、\(R3\)（目标+其余两组）、\(R^*\)（无目标），用于分析失败模式。
  - **任务完成率 \(F\)**：回合中使用可靠工具组、必需支持工具、且未使用禁用工具的比率。
- **确定性验证器**：将每个工具调用映射为成功或失败（二进制反馈，无诊断信息），模拟 WCST 和逆转学习中的反馈。

### 公式与算法流程（文字说明）
- 对于端点路径上的 \(K\) 个边界，对每个后切换窗口 \(P_k\) 计算份额 \(s_k\)（来自目标工具组的调用数/总调用数），则 \(\Phi = s_1 \times s_2 \times \cdots \times s_K\)。\(\Phi\) 通过自助法（bootstrap）计算 95% 置信区间。

## 3. 实验设计

### 数据集/场景
- 三个领域：
  - **调度与协调**：3 个工具组 × 5 个工具
  - **DevOps 事故分类**：4 个工具组 × 5 个工具
  - **多云存储**：5 个工具组 × 2 个工具
- 每个领域构建冗余工具-技能集，工具组间通过隐藏可靠性区分，用户提示保持中性，无词法线索。

### Benchmark
- 论文提出的基准称为 **WCST-Tool-Bench**（代码开源于 GitHub: zwycl/wcst-tool-bench），包含带冗余的工具库、隐藏可靠性切换调度、分支树结构、确定性验证器。

### 对比方法
- **代理框架**：Hermes Agent（Nous Research 2026），开源且支持持续会话。
- **模型**：两个开放权重 LLM——mimo-v2.5（小米）和 deepseek-v4-pro（DeepSeek），通过 OpenRouter 服务，温度 0.7，top-p 0.95，中等推理努力。
- **干预条件**：
  - **政策提示**（instructional）：常规（无策略）、自适应（跟踪成功工具并切换）、博学（变化工具避免固定）。
  - **集合框架**（structural）：互补（各组为同一服务不同部分）、竞争（各组为可互换的提供商）。

### 实验数量
- 主要分支实验（调度域）：每个端点 16 个前缀（trajectories），共 9 个端点（表1，共 16×9=144 轨迹）。
- 政策提示实验：deepseek 三个条件（常规 n=16，自适应 n=8，博学 n=8）；mimo 仅测少量（n=3）。
- 集合框架实验：线性域（DevOps 和多云），各 8 个复制对/细胞，共 4 个细胞（2 模型×2 领域），总计 8×4=32 对。
- 消融实验：通过分支树内的兄弟对比（配对置换检验）检验深度、路径影响。

## 6. 主要结论与发现

1. **每个先验使用角色产生不同承诺模式**（RQ1）：
   - mimo-v2.5：从强到弱为 CA（反转）> CB（已强化）> CC（新颖），倾向于混合之前用过的组。
   - deepseek-v4-pro：从强到弱为 CB > CA ≈ CC，但方差高，倾向于早期选择的单组（前缀锁定）。
2. **深度和路径影响超出最近切换**（RQ2）：
   - mimo 受历史影响：同一目标通过不同父路径到达时 Φ 不同。
   - deepseek 被首次承诺主导：深度和路径影响小，双峰分布持续。
3. **政策提示可重定向锁定**（RQ3）：
   - 自适应提示大幅提高 deepseek 的 Φ（≥0.86，σ≤0.06），使代理积极切换到新目标。
   - 博学提示增加工具多样性，但未改善正确调用。
   - mimo 对政策提示不敏感（n=3 暗示）。
4. **集合框架影响路径占用**（RQ4）：
   - 互补框架导致代理固定使用所有组（混合地板）。
   - 竞争框架导致代理要么承诺单一可靠组，要么保持在混合地板（双峰）。
   - 方向在 4 个细胞中一致，符号检验显著。
5. **默认行为特征**：两个模型都在几次切换后迅速锁定为小例程，调用份额集中在 {0, 0.33, 0.5, 1} 等离散值。失败模式：deepseek 表现为前缀级锁定，mimo 表现为部分持续。

## 7. 优点

- **创新性**：首次将认知心理学定势转换范式系统应用于 LLM agent 评估，设计冗余工具库和隐藏可靠性切换调度。
- **方法论亮点**：
  - 提出 \(\Phi\) 指标量化沿路径的多窗口适应性，综合衡量 agent 的持续调整能力。
  - 路线类分析暴露失败模式（R1-R3, R*），区分持续错误与混合策略。
  - 分支树调度允许配对对照，隔离父状态影响。
- **干预双重通道**：同时测试指令性（政策提示）和结构性（集合框架）干预，提供两种改进 agent 适应性的方向。
- **开源可复现**：代码和基准已公开。
- **实验控制**：严格保持提示中性，消除词法线索；确定性验证器提供二进制反馈；工具顺序随机化。

## 8. 不足与局限

- **模型覆盖有限**：仅评估两个开放权重模型，未包括封闭前沿模型（如 GPT-4, Claude），结论可能不泛化。成本是主要原因。
- **样本量较小**：每细胞 n=16 或 n=8，尤其集合框架中每个细胞单独检验不显著，只能依靠总体方向。mimo 政策提示试验仅 n=3，证据不足。
- **轨迹长度有限**：最大 90 轮，分支深度仅 2 层，可能遗漏长期恢复动态或更深层的影响。
- **集合框架极端化**：仅构造“完全互补”与“完全竞争”，未探索中间连续区间的阈值效应。
- **量化框架度量缺失**：未给出定量指标衡量“互补”/“竞争”程度，仅依赖人工构造。
- **政策提示的模型特异性**：自适应提示对 deepseek 有效，但对 mimo 几乎无效，提示结果可能高度依赖模型架构或预训练。
- **潜在偏差**：使用 OpenRouter 服务可能引入 API 层变化（如缓存、模型版本），但论文未讨论。所有实验使用相同设置，影响较小。
